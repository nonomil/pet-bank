## 🌱 项目缘起

每年暑假，都是家长和孩子「拉锯战」的高发期——**「暑假成长目标」与「孩子的即时正反馈」之间，天然存在一条鸿沟**。

家长希望孩子多读书、多运动、少看屏幕；孩子却只想要"立刻、马上、有趣、看得见"的回报。市面上的打卡 App 普遍设计得过于"工具化"，孩子点几下就腻了，更别提养成习惯。

**作为一个不懂代码的爸爸**，我决定用最朴素的方式解决这个问题：做一款孩子**自己愿意打开、愿意看、愿意"玩"**的成长积分系统——把"完成目标"包装成"账户余额增长"，把"坚持习惯"包装成"宠物养成"。不是教育孩子，而是**和孩子一起玩一个关于成长的游戏**。

这篇文章最初发表于微信公众号《**我，不懂代码，却给俩娃量身定制了一款"暑假成长积分银行"App**》，记录了从灵感到落地的全过程。

## 📚 学习中心

学习中心现在已经不是“顺手塞几个打卡入口”的附属页，而是当前项目里的一个**可扩展资料包平台**。

- `学习` 大 tab 负责先让孩子和家长快速找到“今天学什么、从哪里进”
- `积分 -> 学习单` 子页负责当天勾选、耗时记录、卡点记录和睡前一句话复盘
- 真正发分仍然以 lesson 完成、中文组合奖励、英语连续学习奖励为主，避免“学习单再发一轮管理积分”带来的负担

当前已经接入 3 类学习资料：

- `幼小衔接暑假中文资料包`：60 天晨读 + 45 天识字 + 60 首古诗 + 每周复盘 + A4 打印页
- `Minecraft我的世界英语故事`：当前项目负责计划、进度、计分，外部点读站负责章节阅读
- `幼小衔接学习网站入口包`：把外部学习网站整理成站内入口包，学完再回到当前项目里记录与计分

这套结构保持了一个原则：**学习页负责找入口，学习单负责记录，积分规则尽量轻，后面继续接更多资料包和学习网站时不用重做整站。**

## 🧩 六大成长维度

pet-bank 把孩子的暑假成长拆解为 **6 个相互独立又彼此呼应**的"账户"，每个维度都有专属的图标、配色和任务体系：

### 1. 🧠 学习力 (Learning)
> 阅读、写字、计算、英语……一切"输入型"任务的家。

- 阅读 30 分钟 +10 分
- 写日记一篇 +8 分
- 完成口算练习 +5 分
- ……（任务可由家长自定义）

### 2. 💪 运动力 (Sports)
> 强健体魄是所有成长的地基。

- 跳绳 200 个 +6 分
- 户外跑步 20 分钟 +8 分
- 拍球/投篮练习 +5 分
- 平板支撑 1 分钟 +4 分

### 3. 🎯 自控力 (Self-control)
> 这一项最考验孩子，也最考验家长——克制屏幕、抵制诱惑。

- 主动放下手机 30 分钟 +10 分
- 控制零食摄入 +5 分
- 按计划完成作业不拖延 +8 分
- 按时睡觉 +6 分

### 4. 🔍 探索力 (Exploration)
> 好奇心是孩子最珍贵的火种。

- 观察一种自然现象并记录 +10 分
- 学做一道新菜 +12 分
- 主动问一个"为什么"并寻找答案 +6 分
- 参观博物馆/科技馆 +15 分

### 5. 🛠️ 实践力 (Practice)
> 把"知道"变成"做到"，从手到脑的闭环。

- 自己整理书包 +4 分
- 洗碗/扫地等家务 +6 分
- 修理一件小物品 +10 分
- 完成一个手工作品 +12 分

### 6. 🐾 守护力 (Pet Care) ⭐ 新增
> **这是 pet-bank 最具特色的模块**——孩子不只是积分的"赚取者"，更是一只虚拟宠物的"守护者"。

## 🐾 守护力 · 养宠物制度

> **🆕 v0.3.0 新增「宠物小屋」**：宠物有了自己的家——5 维状态可视化（HP / 饱食 / 快乐 / 亲密 / 经验）、4 互动按钮（喂食 / 玩耍 / 洗澡 / 治疗）、家具摆放、**饥饿 → 倒下 → 救援**情感闭环。

> "5 分钟热度"和"长期承诺"的差距，就是**责任心**的距离。

### 🎯 设计初衷
市面上的任务型 App 普遍存在一个共同问题：**孩子完成任务的"热度"只能维持 3-5 天**。任务打完了、积分刷完了，App 也就被遗忘在角落了。

**守护力模块就是为了解决这个问题而生的**——它不奖励"打几次卡"，而是奖励**"长期持续地照顾一个生命"**。

### 🐶 宠物类型
孩子在初始化时可以选择自己的宠物伙伴：
- 🐕 柴犬（忠诚、活泼）
- 🐈 橘猫（慵懒、亲昵）
- 🐰 兔子（机警、可爱）
- 🐢 乌龟（长寿、沉稳）

### 📋 每日任务
每个孩子每天都要完成以下"守护任务"：

| 任务 | 说明 | 奖励 |
|------|------|------|
| 🍖 喂食 | 给宠物准备食物 | 健康值 +5 |
| 🛁 清洁 | 给宠物洗澡/清理窝 | 健康值 +5 |
| 🤗 陪伴 | 和宠物玩耍 15 分钟 | 快乐值 +8 |
| 💉 健康检查 | 观察宠物状态、记录健康 | 健康值 +6 |
| 🐕 遛宠物 | 带宠物外出散步 20 分钟 | 亲密度 +10 |

### 📊 三维属性
宠物有 **3 个核心属性**，决定它的状态：

- ❤️ **健康值**（0-100）：通过喂食、清洁、健康检查维护
- 😊 **快乐值**（0-100）：通过陪伴、玩耍提升
- 💕 **亲密度**（0-100）：通过长期坚持遛宠物、互动累积

> **如果连续 3 天没有完成任何守护任务**，宠物会"生病"，健康值会持续下降。  
> **如果坚持 30 天**，宠物会"升级"，解锁新的互动动作和外观。

### 💡 设计哲学
1. **责任心**：宠物不是"拥有"，而是"责任"。它不会因为你今天没空就消失。
2. **爱心**：照顾一个比自己更弱小的生命，是孩子学会"给予"的最佳练习。
3. **长期主义**：积分可以一天刷满，但亲密度只能一天天累积——这是对"5 分钟热度"最温柔的反击。
4. **共情能力**：当孩子看到宠物"生病"时，会第一次理解"被照顾"的珍贵。

> "爸爸，宠物今天不开心，是不是因为我昨天没陪它？" —— 当孩子说出这句话时，**教育就已经发生了**。

## 🛠️ 技术栈

pet-bank 采用**最朴素的现代前端方案**——静态 HTML + 模块化 JS，没有框架、没有构建工具，直接打开即可运行。

| 类别 | 技术 | 说明 |
|------|------|------|
| **核心** | Vanilla JavaScript | 零依赖，原生 JS 写就 |
| **样式** | Tailwind CSS (CDN) | 实用优先的 CSS 框架 |
| **图标** | Lucide Icons | 优雅统一的 SVG 图标库 |
| **结构** | 静态 HTML + 模块化 JS | 一个 `index.html` 入口 + 多个 JS 模块协作 |
| **部署** | GitHub Pages / Cloudflare Tunnel / Supabase | 静态前端独立发布，云端数据库与函数单独部署 |
| **数据** | 浏览器本地存储 + Supabase（可选） | 默认可离线本地运行，接入云端后支持账号、家庭、多孩子与互动数据 |

> 当前主站仍保持静态前端形态；配置 Supabase 后，可以渐进开启家长账号、家庭社交、后台管理壳和真实环境部署。

### 为什么保持静态站点？

- ✅ **零构建成本**：直接打开静态站点就能用
- ✅ **跨平台**：Windows / macOS / Linux / iPad / 手机都能开
- ✅ **易维护**：多个 JS 模块分工明确，扩展时不用引入打包链路
- ✅ **可定制**：家长可以随意修改任务、积分、宠物属性
- ✅ **渐进升级**：本地模式与云端模式可以共存，页面发布和数据库发布分开，便于安全升级

## 🚀 本地运行

pet-bank 的运行方式**简单到不能再简单**：

### 方式一：直接打开（最简单）
```bash
# macOS
open index.html

# Linux
xdg-open index.html

# Windows
start index.html
```

### 方式二：本地 HTTP 服务（推荐）
某些浏览器对 `file://` 协议下的本地存储有限制，建议用 HTTP 方式启动：

```bash
# 使用 Python 3（最常见）
python3 -m http.server 8000

# 使用 Python 2
python -m SimpleHTTPServer 8000

# 使用 Node.js（如果已安装）
npx serve .

# 使用 PHP（如果已安装）
php -S localhost:8000
```

然后在浏览器访问：<http://localhost:8000>

> 💡 **Windows 一键启动**：双击项目根目录的 `启动服务.bat`，自动起服务并打开浏览器（访问 <http://127.0.0.1:8765>）。

### 方式三：部署到 GitHub Pages
仓库已配置 `.github/workflows/deploy.yml`，推送到 `main` 后会自动发布到 GitHub Pages。

### 方式四：Cloudflare Tunnel（可选）
如果你还想保留内网穿透访问，也可以继续用 Cloudflare Tunnel 作为备用入口。
