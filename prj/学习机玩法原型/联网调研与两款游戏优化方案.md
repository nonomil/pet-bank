# 拼音赛车与飞机大战：联网调研与优化方案

> 更新时间：2026-07-18
> 适用对象：幼小衔接阶段，正在熟悉中文、拼音和蓝牙键盘的孩子
> 当前状态：方案已落地学习指标；玩法继续以低压力、短反馈、可重复为主。

## 1. 调研结论

本轮读取了以下公开页面：

- [TypingMaster Games](https://www.typingmaster.com/games/)
- [keybr](https://www.keybr.com/)
- [Epistory](https://www.epistorygame.com/)
- [Typing.com Kindergarten Curriculum](https://www.typing.com/curriculum/kindergarten)

得到四条可直接用于本项目的结论：

1. **一个游戏只承担一个主要训练目标。** TypingMaster 将反应、键位定位、整词准确率、持续速度拆成 Strike、KeyMagnet、Bubbles、Wordtris、Cloud Race 等不同玩法。拼音赛车不应再叠加复杂拼写和声调考试；飞机大战不应同时要求躲避、移动、释义问答和默写。
2. **难度应该逐步开放。** keybr 的核心是从少量字母开始，逐步扩展键位。基础档应先让孩子熟悉字母和方向，再增加词长、敌人数或速度，而且一次只提高一个变量。
3. **每次输入都应该立刻改变画面。** Epistory 将键盘输入直接连接到战斗、施法和解谜。当前飞机大战“正确字母 -> 一发子弹 -> 命中反馈”是正确方向；拼音赛车也应让一次选择马上改变路线、速度或奖励。
4. **幼儿课程重视节奏可调与轻成就。** Typing.com 的 Kindergarten 页面强调基础键盘、可调整课程、游戏和成就。对本项目来说，家长可以选基础/进阶/完整，但孩子不需要因为错误失去奖励。

## 2. 低龄设计边界

### 学习目标

- 拼音赛车：看汉字，在少量拼音选项中建立“汉字 -> 拼音”的联系，并练习方向键移动。
- 飞机大战：看英文和中文释义，按顺序输入字母，建立“键盘位置 -> 英文字符串”的手感。

### 不加入的负担

- 不在游戏中强制跟读、释义问答或默写。
- 不把错题直接转换成扣血、扣金币或失去星星。
- 不同时提高敌人数、词长、速度和额外障碍。
- 不用结算页宣称“已经掌握单词”；只展示本局玩过的内容和轻量过程指标。

## 3. 拼音赛车优化规则

### 当前玩法

- 画面给出一个汉字和三个拼音卡片。
- 赛车可以上下左右自由移动，也可以点击赛道定位。
- 接到正确卡片推进赛段；接错后保留同一个汉字并给出温和提示。
- 地图在一整轮内保持稳定，赛段设施负责变化，不每题切换背景。

### 分层

| 档位 | 目标卡 | 速度 | 错误反馈 | 新增内容 |
|---|---:|---:|---|---|
| 基础 | 3 张 | 慢 | 回到同一题，给提示，不结束 | 首声母差异明显 |
| 进阶 | 3 张 | 中 | 保留连击，允许重试 | 位置变化、路线选择 |
| 完整 | 3 张 | 较快 | 只在连续失误时减速 | 声调路牌、奖励门 |

### 学习指标

运行快照中的 `learning` 字段记录：

- `tasksPresented`：本局出现的题目数。
- `tasksResolved`：已经完成的题目数。
- `correctChoice`：正确接卡次数。
- `firstTryCorrect` / `firstTryAccuracy`：第一次就接对的题目数和比例。
- `retryCount`：接错后重新尝试的次数。
- `wrongChoices` / `choiceAccuracy`：选择层面的准确率。
- `voiceAnnounced`：自动播报汉字和拼音的次数。

这些指标用于家长调试和后续观察，不在孩子操作过程中打断游戏。

## 4. 飞机大战优化规则

### 当前玩法

- 一个单词挂在一架敌机上，首字母可以锁定目标，后续字母保持锁定。
- 每个正确字母触发一次发射、弹道和命中反馈。
- 完成整词才击破敌机；错误字母只打断连击，不直接扣护盾。
- 支持方向键或 WASD 移动，但字母输入优先，避免 `W/A/S/D` 同时是拼写字母时抢走输入。

### 难度拆分

| 阶段 | 敌机数量 | 主要训练 | 游戏变化 |
|---|---:|---|---|
| 预热 | 1–2 架 | 找字母、按顺序输入 | 速度慢，无额外障碍 |
| 编队 | 2–3 架 | 目标切换和连续输入 | 目标位置变化、连击奖励 |
| 小高潮 | 3–4 架 | 持续输入和方向躲避 | 只增加一种轻障碍 |
| 完整 | 由完整档开启 | 长词和综合节奏 | 最后再开启 Boss，不与基础档混用 |

实施时遵守“每次只提高一个变量”：先增加词长，再增加敌人，最后才提高速度；Boss、陨石和雷区只属于完整档。

### 学习指标

运行快照中的 `learning` 字段记录：

- `tasksPresented`：生成并展示的单词目标数。
- `wordsCompleted`：完整输入并击破的单词数。
- `firstTryWords` / `firstTryAccuracy`：没有错误字母就完成的单词数和比例。
- `retryWords`：出现错误后仍完成的单词数。
- `correctLetters`、`wrongLetters`、`letterAccuracy`：字母级输入准确率。
- `targetSwitches`：首字母切换到另一架可输入敌机的次数。
- `voiceAnnounced`：自动播报英文和中文的次数。

结算页只显示“首答正确”和“字母准确率”两项，避免让低龄孩子面对过多数字。

## 5. 语音策略

- 英文：优先播放词库中的本地 MP3，缺失时使用浏览器英文语音。
- 中文：优先使用项目已有本地汉字语音映射，缺失时使用中文语音回退。
- 播报顺序：英文词/汉字 -> 短暂停顿 -> 中文或拼音；每个新目标自动播报一次。
- 不要求孩子重复朗读；语音是陪伴反馈，不是通关条件。
- `voiceAnnounced` 只代表已发起播报，不能当作孩子实际听完或掌握的证明。

## 6. 验收方式

```powershell
node prj/学习机玩法原型/scripts/test-learning-metrics.mjs
node prj/学习机玩法原型/scripts/test-pinyin-racer-visual.mjs
node prj/学习机玩法原型/scripts/test-word-shooter-mechanics.mjs
node prj/学习机玩法原型/scripts/test-gameplay-controls.mjs
node prj/学习机玩法原型/verify.mjs
```

本轮新增的 `test-learning-metrics.mjs` 会真实验证：

- 拼音赛车接错后能保留原题并记录重试。
- 拼音赛车重试答对后不会虚增首答正确率。
- 飞机大战错误字母能被记录，但不阻止继续完成单词。
- 飞机大战完成整词后能统计字母准确率和首答情况。
- 两款游戏都暴露自动语音播报次数。

## 7. 下一阶段

1. 先让孩子实际玩 3–5 局，观察是否主动再来一局、是否频繁求助、是否只看动画不看目标。
2. 根据真实体验微调基础档速度和目标数量，不先扩充敌人种类。
3. 如果孩子能稳定完成基础档，再增加“听音找拼音”或“首字母专项”独立模式；不要把它们硬塞进现有赛车和飞机大战。
4. 只有在试玩证明愿意重复后，才考虑把快照指标接入主项目的学习复盘，不直接把它们当作正式掌握度。
