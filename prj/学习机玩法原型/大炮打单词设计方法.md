# 拼音炮台设计方法

## 游戏描述

`word-cannon` 现在是一个固定炮台式拼音打字游戏。玩家看到底部中央的炮台和上方缓慢下落的汉字目标块。按对目标汉字拼音的一个字母，炮台发射一道细长光线；连续打完整个拼音后，目标块碎裂、出现圆形冲击波并计分。这个模式和飞机大战的区别是：飞机大战练英文单词输入，拼音炮台练“看汉字、打拼音”，但仍然保留固定炮台、明确目标、连续弹道和击碎反馈。

## GPT 生图提示词

先用 GPT 生图探索整体方向，不直接把整张图当最终游戏画面。

```text
Create a polished 16:9 web game mockup for a children pinyin typing game called "Pinyin Cannon".

Scene: a fixed cannon turret at the bottom center, a vertical battle lane in the middle, chunky sci-fi hanzi target blocks above, thin white projectile trails shooting upward, circular radar aiming rings around the cannon, and red-orange impact fragments when a target is destroyed.

Style: cool sci-fi arcade, readable, not cute, not crowded. Dark warm arena floor, gold cannon glow, red target blocks, clean target label areas for dynamic words.

Important: do not render long real text in the image. Leave blank label areas or simple short hanzi placeholders. No watermark, no phone frame, no tiny unreadable UI text.
```

圆形探索版：

```text
Create a circular/radial visual concept for "Word Cannon".

A round cannon turret sits at the bottom center with a glowing circular aiming ring. Target blocks appear inside a vertical lane but are visually connected by faint radar arcs and radial guide lines. Thin white beams fire from the cannon toward targets. Keep the word labels readable and leave clear blank areas for dynamic text.
```

元素爆炸图提示词：

```text
Create a separated asset plate for "Word Cannon": cannon base, rotating barrel, circular aiming ring, radar arcs, red word target block, cracked target block, thin beam, muzzle flash, impact ring, explosion shards, score popup, dark vertical arena lane. Keep every element separated with generous spacing for cropping. No watermark. Minimal text.
```

## SVG / DOM / PNG 拆分

| 元素 | 推荐实现 | 原因 |
|---|---|---|
| 汉字、拼音字母高亮 | DOM 文本 | 必须绑定汉字题库，保证可读和可控 |
| 目标块主体 | HTML/CSS 或 SVG | 需要根据拼音长度自适应，不适合烘焙进图片 |
| 炮台圆形瞄准环、雷达线 | SVG / CSS | 几何形状稳定，缩放不糊，方便响应式 |
| 细长弹道 | DOM/CSS transform | 每次按键都要从炮台连到目标，代码生成最稳 |
| 冲击波、碎片、得分弹窗 | DOM/CSS 动画 | 需要跟随目标位置实时生成 |
| 背景质感、炮台皮肤 | GPT PNG | 可以用生图获得质感，但不要包含动态文字 |
| 元素爆炸图 | 网页生成 + 截图 | 用实际 DOM/SVG 生成一张可复核的素材说明图，比纯手工拼图更一致 |

## 当前落地策略

当前原型先采用“网页 DOM/SVG 完成可玩版本”：文字、目标块、炮台、圆形瞄准环、弹道、冲击波和碎片都由网页生成。后续如果 GPT 生图结果更好，只替换背景质感、炮台皮肤和目标块贴图，不替换动态文字和交互层。

## 当前地图方案

当前地图改成四段式战区推进，每 6 个拼音目标左右自动切换一张 Agnes 背景图：

1. 金属走廊
2. 太空机库
3. 未来训练场
4. 熔炉工业区

这样地图切换真正出现在同一局内，而不是刚切图就立刻结算。动态文字、弹道、爆炸和圆形雷达仍然由网页 DOM / SVG 生成；背景质感交给 GPT / Agnes 生图，保持“质感由图片、交互由网页”的分层。
