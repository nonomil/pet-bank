# 学习机玩法原型美术与动效说明

## 视觉定位

这个原型先不做“小霸王学习机机身”，重点验证玩法是否能让孩子愿意敲键盘。视觉方向参考 `prj/GPT生图--UI设计` 里的原则：先有清晰的视觉目标，再让代码实现，不让 Codex 临场猜美术。

当前页面采用“儿童学习机总入口 + 科幻单词射击 + 轻量中文练习关卡”的混合方向：

- 一个大舞台承载当前任务，避免多屏信息争抢注意力
- 图片优先使用现有 Minecraft 英语词卡和汉字游乐场图片
- 背景复用 `assets/ui/playground-bg.webp`、`assets/ui/hanzi-new-bg.webp`、`assets/ui/pg-card-hanzi.webp`
- 控件保持 8px 圆角、强对比文字、短按钮文案，适合幼小衔接孩子快速理解
- 奖励只用星星、进度条、得分、奖励盒和短反馈，不使用长说明
- 其中 `word-shooter` 已根据用户要求切到“更酷炫、更科幻、降低可爱比例”的 Typing Attack 风格；`word-cannon` 使用圆形科幻炮台方向；`pinyin-snake` 使用清爽网格贪吃蛇方向。

## 已生成 Agnes 资产

当前已通过 Agnes 生成项目内图片，统一落在 `prj/学习机玩法原型/assets/generated/`：

- `learning-arcade-stage.png`：主舞台背景，接入页面 body 背景层
- `home-card-word-shooter.png`：首页“飞机大战”选项图，科幻飞船从左向右射击敌机，目标面板留白
- `home-card-word-cannon.png`：首页“拼音炮台”选项图，底部小炮台、圆形雷达和上方空白目标块
- `home-card-pinyin-snake.png`：首页“贪吃蛇”选项图，浅色网格、圆润蛇身和彩色食物点
- `manifest.json`：记录模型、提示词、文件名和生成时间

第一轮主舞台和拼音卡出现了模型乱写字母/拼音的问题，第二轮提示词已改为“所有卡片、按钮、面板必须完全空白；禁止字母、数字、中文、拼音、伪文字和符号”。后续继续生图时要保留这个约束，避免图片里的错误文字干扰孩子。

单词射击已另外产出一套科幻素材包 `assets/generated/word-shooter-scifi-assets/`，包含飞船、目标、武器道具、HUD 面板与爆炸特效。旧 `word-shooter-assets/` 保留为第一轮儿童版方向存档，不再作为当前页面主视觉。

## Lottie 动效引用

动效先引用 `prj/GPT生图--Lottie动画` 已有资产，不在原型里复制一份：

- `animations/task-check.json`：用于过关完成方向，当前页面在 `visual-card` 上保留 `data-lottie-ref`
- `animations/score-stars.json`：用于奖励星星方向，当前页面在 `reward-box` 上保留 `data-lottie-ref`

当前版本没有引入完整 Lottie Web Player，而是先加载 JSON 做资源可达验证，再用 CSS 粒子完成最小可玩的反馈。后续正式合入时，可以把这两个 JSON 接入 Lottie Web 或 Skottie，在过关和奖励盒打开时播放一次。

## 游戏音效

游戏音效使用浏览器本地 `AudioContext` 合成，不依赖额外 mp3：

- 敲对字母：短促上扬音
- 敲错字母：低频短提示，不做惩罚感
- 过关：三音阶完成音
- 奖励盒打开：更明亮的四音阶奖励音
- 切换关卡：轻量切换音
- 播放单词音频：音符提示

右上角提供音效开关，避免孩子或家长觉得吵。英文单词本身的朗读继续复用 Minecraft 词卡里的本地 mp3；如果后续需要补中文引导或拼音朗读，可以接 `prj/tts` 批量生成。

## Agnes 生图方向

Agnes 生图适合在第二轮做“关键视觉帧”和“可裁切素材”，不要先批量生成一堆无法落地的图片。建议先生成 3 类资产：

### 1. 主舞台背景

```text
为一个幼小衔接儿童键盘学习小游戏生成横版网页主舞台背景。
画面是现代儿童学习游戏空间，明亮、柔和、有少量积木、字母、拼音卡片元素，但不要出现真实文字。
风格为高质量 3D clay / soft toy UI，留出中间大面积空白给网页放词卡和按钮。
不要有水印、logo、复杂角色、密集装饰、深色背景。
```

### 2. Minecraft 英语奖励卡

```text
生成一组适合儿童英语拼词游戏的奖励卡视觉参考。
主题是方块世界、草地、石头、木头、门、光等 Minecraft 风格物件，但不要直接使用官方 logo 或受版权保护的角色。
卡片要适合网页 UI：主体居中、背景干净、边缘清楚、无文字、可裁切成 WebP。
```

### 3. 汉字拼音关卡卡

```text
生成儿童拼音练习小游戏的汉字卡片视觉参考。
画面包含单个大汉字卡位、拼音练习氛围、软 3D 教具、浅色课堂游乐场背景。
不要在图片里写具体汉字或拼音，文字留给 HTML 渲染。
```

## 后续替换顺序

1. 继续优先提取参考网页和现有项目里的图片。
2. 缺少关键氛围图时，用 Agnes 生成主舞台背景和奖励素材。
3. 只有简单占位或极小图标才使用 Minecraft 风格 SVG。
4. 动效优先复用 Lottie JSON；无法表达的微反馈再用 CSS。
5. 音效优先用 WebAudio 合成；角色配音或长句朗读再走 `prj/tts`。
