# 多目标弓箭打字 / 数学模式设计

## 推荐方向

把当前单目标玩法升级为“多目标弓箭训练场”。同屏出现 2-3 只怪物，每只怪物头上都有任务牌。孩子开始输入后，游戏根据输入前缀判断正在攻击哪只怪物，弓箭发射器旋转到目标方向，箭沿斜线飞过去。这样“打哪个单词的字母，就向哪个苦力怕射箭”会变成核心反馈，而不是只在底部输入框里看字。

## 英语目标

每只怪物分配一个短词。任务牌显示完整单词，底部键盘高亮当前要敲的字母。孩子输入 `c` 时，所有 `c...` 目标轻微高亮；如果只剩一个候选目标，弓箭立即转向它并发一支小箭。完整输入单词后，该怪物爆炸并得分。若多个目标共享前缀，例如 `cat` 和 `cap`，前两步只高亮候选，第三个字母才锁定目标。

## 数学目标

新增“100以内加法”模式。每只怪物头上显示算式，例如 `34+18`，孩子输入结果 `52` 后攻击该怪物。题目生成约束：两个加数均为 0-100，结果不超过 100；低龄默认不进位或小进位混合，后续可按难度分层。输入规则和英语一致：数字前缀匹配候选答案，完整答案命中。

## 怪物落地

所有怪物改成 lane 模型，而不是单独前后两个 DOM。每个怪物有 `laneX`、`horizonY`、`groundY`、`progress`、`scale`。渲染时以脚底为锚点：`transform-origin: 50% 100%`，图片底部永远贴到当前 lane 的地面点。远处怪物更小、更靠上；靠近时沿 lane 往下走并变大。这样第二只怪物不会悬空。

## 数据结构

```js
enemy = {
  id,
  task: { type: "word" | "math", prompt, answer, hint },
  laneIndex,
  progress,
  state: "walking" | "targeted" | "hit" | "exploding",
  hp: 1
}
```

全局输入 `typed` 不再只绑定一个 `currentTask`，而是通过 `findMatchingEnemies(typed)` 找候选。锁定目标后，`fireArrow(enemy)` 根据弓箭和该敌人的屏幕坐标计算角度、距离和飞行动画。

## 实施顺序

1. 先把单个 `monsterWrap` 改为 `enemyLayer` 渲染列表，保留当前单词模式测试。
2. 加 `findMatchingEnemies`、目标牌和候选高亮，让输入能锁定不同怪物。
3. 改 `fireArrow(enemy)`，弓箭旋转并飞向对应敌人。
4. 加数学模式：100 以内加法题、数字输入、结果匹配。
5. 接入 GPT 生成地图背景，修正所有敌人的脚底锚定。
6. 扩展自动化测试：多敌人不悬空、前缀锁定、数学命中、箭角度变化。
