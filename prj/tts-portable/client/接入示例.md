# 接入示例 · 新项目如何用 voice.js

> `client/voice.js` 是**通用可配置**的语音播报库，不绑定任何业务。接入 = 引入 + init 配置 + （可选）主动 speak。
>
> 默认不监听任何容器（避免乱播报），你告诉它「盯哪些元素」它就盯哪些。

---

## 一、引入 voice.js

把 `client/voice.js` 复制到新项目静态目录，HTML 引入：

```html
<script src="js/voice.js"></script>
```

---

## 二、两种启动方式

### 方式 A：声明式（推荐，最简）

在引入 `voice.js` **之前**定义 `window.VoiceSystemConfig`，DOMContentLoaded 时自动 init：

```html
<script>
window.VoiceSystemConfig = {
  storageKey: 'myapp_voice_settings',        // localStorage 键名（不同项目用不同键，避免串）
  panelSelector: '#voice-mount',             // 设置面板挂载点（CSS 选择器）
  defaultVoice: 'teacher',                   // 默认音色：mom/grandpa/teacher/child
  defaultSettings: {
    serverUrl: 'http://127.0.0.1:9885'       // 后端服务地址（留空只用浏览器 Web Speech）
  },
  selectors: [
    // ↓↓↓ 按你的项目实际 DOM 配置 ↓↓↓
    { sel: '.toast', extract: function (n) { return n.textContent; } },
    { sel: '#dialog-text', extract: function (n) { return n.textContent; } }
  ]
};
</script>
<script src="js/voice.js"></script>
```

### 方式 B：命令式（需控制时机时）

不定义 `VoiceSystemConfig`，`voice.js` 加载后只暴露 API。你在合适时机手动 init：

```html
<script src="js/voice.js"></script>
<script>
// 等业务就绪后再初始化
document.getElementById('startBtn').addEventListener('click', function () {
  VoiceSystem.init({
    storageKey: 'myapp_voice_settings',
    panelSelector: '#voice-mount',
    defaultSettings: { serverUrl: 'http://127.0.0.1:9885' },
    selectors: [
      { sel: '.toast', extract: function (n) { return n.textContent; } }
    ]
  });
});
</script>
```

> `init()` 幂等，重复调用安全（observers 只建一次，面板按 `panelSelector` 重新扫描注入）。

## 三、配置项详解

| 字段 | 类型 | 默认 | 说明 |
|------|------|------|------|
| `storageKey` | string | `'voice_settings'` | localStorage 键名。**不同项目务必改**，否则多项目同域会串设置 |
| `panelSelector` | string | `'.voice-settings-panel'` | 设置面板挂载点。找不到则不注入面板（仍可用 API） |
| `selectors` | array | `[]` | 监听规则，每项 `{sel, extract, throttle?}` |
| `defaultSettings` | object | 见下 | 默认设置（首次无存档时用） |
| `defaultVoice` | string | — | `defaultSettings.voice` 的便捷写法 |

**`selectors` 每项**：
- `sel`：CSS 选择器，匹配要监听的容器。
- `extract(node)`：从匹配节点提取要播报的文本（返回 string，空串则跳过）。
- `throttle`（可选）：节流毫秒数，该选择器在 N ms 内只触发一次（适合高频更新的日志）。

**`defaultSettings`**：
```js
{
  enabled: false,        // 是否启用
  voice: 'mom',          // mom/grandpa/teacher/child（对应后端 config.py 的 VOICE_PRESETS）
  rate: 1.0,             // 语速 0.5~1.5
  pitch: 1.0,            // 音调 0.7~1.3
  serverUrl: ''          // 后端地址，留空走 Web Speech
}
```

## 四、selectors 配置示例（监听自己的容器）

监听规则覆盖「文字变化时自动播报」的场景。常见模式：

```js
selectors: [
  // 1. toast / 提示条：整段文本
  { sel: '.toast', extract: function (n) { return n.textContent; } },

  // 2. 对话框台词容器
  { sel: '#dialog-line', extract: function (n) { return n.textContent; } },

  // 3. 日志只播最后一条（高频更新，加节流）
  {
    sel: '.game-log',
    throttle: 600,
    extract: function (n) {
      var items = n.querySelectorAll('p');
      return items.length ? items[items.length - 1].textContent : '';
    }
  },

  // 4. 卡片取子元素
  {
    sel: '.event-card:last-child',
    extract: function (n) {
      var p = n.querySelector('p');
      return p ? p.textContent : '';
    }
  },

  // 5. 通用标记类：任何带 .tts-speak 的元素变化都播
  { sel: '.tts-speak', extract: function (n) { return n.textContent; } }
]
```

> **匹配规则**：节点本身 `matches(sel)` 或 `closest(sel)` 命中即触发，无需精确选中。`extract` 返回空串自动跳过。

## 五、直接调用 speak（主动播报）

不必依赖 selectors，业务代码随时主动播报：

```js
// 任意时机
VoiceSystem.speak('恭喜你获得了一枚金币！');
VoiceSystem.speak('答对了，真棒！');
```

`speak(text)` 会：① 受 `enabled` 控制（关了不播）；② `cleanText` 清洗（去 emoji、转数学符号、去纯数字）；③ 2s 内同文本去重；④ 进串行队列（不会重叠）。

强制忽略 enabled 临时播报（如系统级提示）：
```js
VoiceSystem.setSettings({ enabled: true });
VoiceSystem.speak('重要通知');
```

## 六、完整最小 HTML 示例

复制即可跑（前提是 `js/voice.js` 已就位，后端可选）：

```html
<!DOCTYPE html>
<html lang="zh-CN">
<head>
  <meta charset="UTF-8">
  <title>Voice 接入示例</title>
  <style>
    body { font-family: sans-serif; max-width: 600px; margin: 40px auto; padding: 0 16px; }
    .toast { padding: 10px; background: #e8f5e9; border-radius: 6px; margin: 8px 0; }
    button { padding: 8px 16px; margin: 4px; cursor: pointer; }
  </style>
</head>
<body>
  <h1>Voice 接入示例</h1>

  <!-- ① 主动播报按钮 -->
  <button onclick="VoiceSystem.setSettings({enabled:true}); VoiceSystem.speak('你好呀，欢迎来到我的应用！');">
    播报欢迎语
  </button>

  <!-- ② 触发被监听的 toast（.toast 在 selectors 里，会自动播报） -->
  <button onclick="showToast('你获得了一颗星星！')">弹 toast</button>

  <div id="toast-area"></div>

  <!-- ③ 设置面板挂载点（panelSelector 指向这里） -->
  <h3>设置</h3>
  <div id="voice-mount"></div>

  <!-- 配置 + 引入 -->
  <script>
    window.VoiceSystemConfig = {
      storageKey: 'demo_voice_settings',
      panelSelector: '#voice-mount',
      defaultSettings: { serverUrl: 'http://127.0.0.1:9885' },  // 没后端就留空，走 Web Speech
      selectors: [
        { sel: '.toast', extract: function (n) { return n.textContent; } }
      ]
    };

    function showToast(msg) {
      var el = document.createElement('div');
      el.className = 'toast';
      el.textContent = msg;
      document.getElementById('toast-area').appendChild(el);
      // 新增的 .toast 被 MutationObserver 捕获 → 自动播报（需先在面板启用语音）
    }
  </script>
  <script src="js/voice.js"></script>
</body>
</html>
```

**验证步骤**：
1. 打开页面 → 「设置」区的面板出现。
2. 勾「启用语音」→ 点「🔊 试听」→ 听到「你好呀，我是你的小伙伴」（这一步激活 Web Speech，见 [踩坑 #10](../docs/踩坑与解决方案.md#10-web-speech-需用户手势)）。
3. 点「弹 toast」→ toast 出现并自动播报「你获得了一颗星星！」。
4. （有后端时）填服务地址 → 点「测试连接」显示「✓ 可达」→ 播报切换为更高音质。

## 七、常见问题

**Q：点试听没声音？**
A：Chrome 首次需用户手势。试听按钮本身就是手势入口，点一次即激活。仍无声检查系统音量/中文语音包。

**Q：toast 出现了但不播？**
A：① 没勾「启用语音」；② selector 的 `sel` 没匹配到（检查 DOM 实际 class/id）；③ `extract` 返回了空串（被 cleanText 过滤，如纯数字）。

**Q：想换音色/语速？**
A：面板里直接调，自动持久化。或代码 `VoiceSystem.setSettings({ voice: 'teacher', rate: 0.9 })`。

**Q：多个项目在同一域名，设置串了？**
A：每个项目用不同的 `storageKey`。

## 八、API 速查

```js
VoiceSystem.init(options)          // 初始化，幂等
VoiceSystem.speak(text, opts?)     // 播报（受 enabled 控制，自动清洗去重）
VoiceSystem.getSettings()          // 读取设置副本
VoiceSystem.setSettings(partial)   // 更新设置（改 serverUrl 会立即重检）
VoiceSystem.checkServer()          // 手动检测服务可达性 → Promise<boolean>
```
