# 移植指南 · 复制到新项目

> 把 tts-portable 搬到新项目的完整流程。目标是 **5 分钟跑起来，不重复踩坑**。

---

## 总览

tts-portable 是**自包含**的：后端（FastAPI）和前端（voice.js）都不依赖宿主项目的业务代码。移植 = 复制目录 + 装依赖 + 配 selectors。

```
宿主项目/
├── (你的业务代码)
└── 你放 tts-portable 的位置/
    ├── tts_server.py
    ├── config.py
    ├── requirements.txt
    ├── client/
    │   └── voice.js          ← 这个会被你复制到宿主的 js/ 下
    └── docs/
```

---

## 步骤 1：复制目录

把整个 `tts-portable/` 复制到新项目的合适位置（建议放 `prj/tts/` 或 `services/tts/`）：

```bash
# 假设新项目根目录是 /new-project/
cp -r tts-portable/ /new-project/prj/tts/
```

> 不需要复制 `cache/`（运行时自动生成，已在 `.gitignore`）。

---

## 步骤 2：后端启动

```bash
cd /new-project/prj/tts/
pip install -r requirements.txt        # fastapi + uvicorn + edge-tts + soundfile
python tts_server.py                    # 或 start.bat / start.sh
```

看到 `[tts] 启动 http://127.0.0.1:9885` 即成功。浏览器开 `http://127.0.0.1:9885/health` 确认 `ok: true`。

要 VoxCPM2 最高音质？先配好 GPU 环境 + 本地模型，见 [快速启动.md Level 2](./快速启动.md#level-2--最高音质voxcpm2-本地-gpu)。

---

## 步骤 3：前端引入

复制 `client/voice.js` 到新项目的静态资源目录：

```bash
cp tts-portable/client/voice.js /new-project/js/voice.js
```

在 HTML 引入：
```html
<script src="js/voice.js"></script>
```

---

## 步骤 4：配置（端口/模型/HF 镜像）

按需改环境变量（不改代码）。推荐复制一份 `.env.example` 为 `.env`（若项目未内置 `python-dotenv`，则用系统环境变量或启动脚本 `set/export`）：

```ini
# .env（示例，需自行接 python-dotenv 才会被读取；不接就用系统环境变量）
TTS_HOST=127.0.0.1
TTS_PORT=9885
VOXCPM2_PYTHON=D:/PythonEnvs/tts-voxcpm/Scripts/python.exe
VOXCPM2_MODEL_PATH=D:/HuggingFaceCache/VoxCPM2
VOXCPM2_TIMEOUT=300
HF_ENDPOINT=https://hf-mirror.com
HF_TOKEN=hf_xxxxxxxxxxxxxxxx
```

> tts-portable **未内置** `python-dotenv`，`.env` 不会自动加载。两种做法：① 用系统环境变量；② 在 `tts_server.py` 顶部加两行 `from dotenv import load_dotenv; load_dotenv()`（需 `pip install python-dotenv`）。默认推荐系统环境变量，最简单。

所有环境变量含义见 [快速启动.md 的「环境变量速查」](./快速启动.md#环境变量速查)。

---

## 步骤 5：接入 selectors

**这是移植唯一需要写代码的地方**。`voice.js` 默认不监听任何容器（避免乱播报），你要告诉它「哪些 DOM 元素的文字变化需要播报」。

在 HTML 里配置（DOMContentLoaded 前定义 `window.VoiceSystemConfig`，voice.js 会自动 init）：

```html
<script>
window.VoiceSystemConfig = {
  storageKey: 'myapp_voice_settings',          // localStorage 键名，按项目改
  panelSelector: '#voice-settings-mount',      // 设置面板挂载点（CSS 选择器）
  defaultSettings: {
    serverUrl: 'http://127.0.0.1:9885'
  },
  selectors: [
    // 你的 toast / 对话框 / 日志容器
    { sel: '.toast', extract: n => n.textContent },
    { sel: '#dialog-text', extract: n => n.textContent },
    { sel: '.game-log p:last-child', extract: n => n.textContent }
  ]
};
</script>
<script src="js/voice.js"></script>
```

详细示例见 [client/接入示例.md](../client/接入示例.md)。

## 移植检查清单

移植完逐条核对：

- [ ] **1. 目录已复制**：`tts-portable/` 在新项目内，`tts_server.py` / `config.py` / `requirements.txt` 齐全。
- [ ] **2. 后端能起**：`pip install -r requirements.txt` 成功；`python tts_server.py` 打印启动日志；`/health` 返回 `ok:true`。
- [ ] **3. 端口不冲突**：9885 没被占；若占用了已设 `TTS_PORT` 并让前端地址一致（见 [踩坑 #1](./踩坑与解决方案.md#1-端口-8765-冲突)）。
- [ ] **4. 前端已引入**：HTML 有 `<script src=".../voice.js">`；浏览器控制台看到 `[VoiceSystem] booted`。
- [ ] **5. selectors 已配**：监听了项目里真实的文字容器；控制台无 `[VoiceSystem]` 报错。
- [ ] **6. 设置面板已挂载**：`panelSelector` 指向的元素存在；面板可见，能勾选「启用」+「试听」。
- [ ] **7. 试听能响**：点「🔊 试听」有声音（验证 Web Speech 激活 + 服务可达，见 [踩坑 #10](./踩坑与解决方案.md#10-web-speech-需用户手势)）。
- [ ] **8. （可选）VoxCPM2 就绪**：`/health` 的 `voxcpm2:true`；首次请求虽慢但能出声（见 [踩坑 #2/#3/#6](./踩坑与解决方案.md)）。

全部打勾，移植完成。

## 常见移植问题

**Q：前端一直走 Web Speech，不走后端服务？**
A：① 设置面板没填 `serverUrl` 或填错；② 没点「测试连接」确认可达；③ 后端没起或端口不对；④ CORS 没配（默认已配，除非你删了）。

**Q：声音出来了但是英文/杂音？**
A：Web Speech 用了系统非中文音色。`voice.js` 的 `_pickBrowserVoice` 已优先选中文音色，若系统没装中文语音包（如裸 Linux），需装 `espeak-ng` + 中文语音数据，或直接上 Level 1 用 edge-tts。

**Q：VoxCPM2 首次请求等了 5 分钟？**
A：冷启动正常（见 [踩坑 #3](./踩坑与解决方案.md#3-subprocess-timeout120-太短)）。第二次起就快了（模型已在显存）。若一直慢，检查显存是否被其他进程占满。

**Q：想改音色怎么办？**
A：改 `config.py` 的 `VOICE_PRESETS`。edge 引擎改 `voice/rate/pitch`，VoxCPM2 改括号里的声线描述（见 [踩坑 #4](./踩坑与解决方案.md#4-voice_description-参数不存在)）。
