# 快速启动 · 5 分钟上手

> 按你的条件选一个梯度，跑不起来换下一个。**任何梯度都能用**，差别只是音质。

---

## Level 0 · 0 配置（浏览器兜底）

**适合**：纯静态站点、Demo、没有 Python 环境、只想先让声音响起来。

**步骤**：

1. 把 `client/voice.js` 复制到你的项目 `js/` 目录。
2. HTML 引入：
   ```html
   <script src="js/voice.js"></script>
   ```
3. 启用并触发（需用户手势，见 [踩坑 #10](./踩坑与解决方案.md#10-web-speech-需用户手势)）：
   ```html
   <button onclick="VoiceSystem.init({}).then(()=>VoiceSystem.speak('你好呀'))">
     开启语音
   </button>
   ```

**效果**：用浏览器自带 Web Speech API 播报。音质取决于系统/浏览器（Windows 中文音色尚可，Linux 多半没装中文语音包）。

**限制**：
- 离线可用，但音色受系统限制。
- Chrome 首次调用需用户交互（点击/按键）激活，否则被静默拦截。

> 需要更好音质？上 Level 1。

## Level 1 · 推荐（edge-tts 云端，高音质，无 GPU）

**适合**：有 Python 环境、能联网、没有 NVIDIA GPU。**这是性价比最高的方案**。

### 步骤

1. **安装依赖**（在 `tts-portable/` 目录下）：
   ```bash
   cd prj/tts-portable
   pip install -r requirements.txt
   ```
   `requirements.txt` 内容：`fastapi` + `uvicorn[standard]` + `edge-tts` + `soundfile`，纯 CPU 包，无 GPU 依赖。

2. **启动服务**：
   ```bash
   python tts_server.py
   # 或用启动脚本（若有）：
   #   Windows:  start.bat
   #   Linux/Mac: start.sh
   ```
   看到这行说明起来了：
   ```
   [tts] 启动 http://127.0.0.1:9885
   ```

3. **健康检查**：浏览器打开 `http://127.0.0.1:9885/health`，应返回：
   ```json
   {"ok": true, "voxcpm2": false, "edge": true, "gpu": false}
   ```
   `edge: true` 即可正常用 Level 1。

4. **前端接入**：HTML 引入 `client/voice.js`，初始化时填服务地址：
   ```html
   <script src="js/voice.js"></script>
   <script>
     window.VoiceSystemConfig = {
       defaultSettings: { serverUrl: 'http://127.0.0.1:9885' },
       selectors: [
         { sel: '.my-toast', extract: n => n.textContent }
       ]
     };
   </script>
   ```
   用户在设置面板勾选「启用语音」+ 点「测试连接」确认可达，即用云端高音质（zh-CN-XiaoyiNeural 等微软神经音色）。

**限制**：
- 需联网（请求微软 Azure TTS 端点，免费）。
- 端口 8765 易被 LLM 代理占用，**默认已改为 9885**，仍冲突时设 `TTS_PORT`（见 [踩坑 #1](./踩坑与解决方案.md#1-端口-8765-冲突)）。

> 有 NVIDIA GPU（≥8GB 显存）？上 Level 2 拿最高音质。

## Level 2 · 最高音质（VoxCPM2 本地 GPU）

**适合**：有 NVIDIA GPU（≥8GB VRAM）、追求最高音质、需完全离线、商用。

### 前置条件

1. **独立 Python 虚拟环境**装 VoxCPM2（PyTorch + CUDA，体积大，**不要装到本项目环境**）：
   ```bash
   # 建一个独立环境，例如 D:/PythonEnvs/tts-voxcpm
   conda create -n tts-voxcpm python=3.10 -y
   conda activate tts-voxcpm
   pip install torch --index-url https://download.pytorch.org/whl/cu121
   pip install voxcpm soundfile
   ```
   记下这个环境的 python 路径，例如 `D:/PythonEnvs/tts-voxcpm/Scripts/python.exe`。

2. **VoxCPM2 模型权重**（4.7GB）。**强烈建议复用已下好的本地模型**，别在线下载（见 [踩坑 #2](./踩坑与解决方案.md#2-voxcpm2-模型下载卡死)）：
   - 已有本地模型目录（扁平结构，含 `model.safetensors` + `audiovae.pth` + `config.json` + `tokenizer.json` 等），例如 `D:/HuggingFaceCache/VoxCPM2`。
   - 没有就得联网下，**务必先设镜像 + token**（否则国内匿名下载会在 53KB 后卡死 15 分钟）：
     ```bash
     set HF_ENDPOINT=https://hf-mirror.com   # Windows CMD
     set HF_TOKEN=hf_xxxxx                   # 解除匿名限速
     # Git Bash 用 export
     ```

### 启动

设三个环境变量后，按 Level 1 的方式启动：

```bash
# Windows CMD
set VOXCPM2_PYTHON=D:/PythonEnvs/tts-voxcpm/Scripts/python.exe
set VOXCPM2_MODEL_PATH=D:/HuggingFaceCache/VoxCPM2
set VOXCPM2_TIMEOUT=300
python tts_server.py
```

```bash
# Git Bash / Linux / Mac
export VOXCPM2_PYTHON=/path/to/tts-voxcpm/bin/python
export VOXCPM2_MODEL_PATH=/path/to/VoxCPM2
export VOXCPM2_TIMEOUT=300
python tts_server.py
```

### 验证

打开 `http://127.0.0.1:9885/health`，应返回：
```json
{"ok": true, "voxcpm2": true, "edge": true, "gpu": true}
```
`voxcpm2: true` 表示本地模型就绪（检查 `model.safetensors` >100MB，不是只看 python.exe 在不在，见 [踩坑 #6](./踩坑与解决方案.md#6-health-假阳性)）。

**首次请求会慢**（冷启动：import torch + 加载 4.7GB 模型 + 扩散生成，常 >120s）。`VOXCPM2_TIMEOUT` 默认 300s 已覆盖，**不要改回 120**，见 [踩坑 #3](./踩坑与解决方案.md#3-subprocess-timeout120-太短)。稳态后 RTF≈0.13（4090 上）。

### 环境变量速查

| 变量 | 默认值 | 作用 |
|------|--------|------|
| `TTS_HOST` | `127.0.0.1` | 服务监听地址 |
| `TTS_PORT` | `9885` | 服务端口（避开 8765） |
| `VOXCPM2_PYTHON` | `D:/PythonEnvs/tts-voxcpm/Scripts/python.exe` | VoxCPM2 独立环境 python |
| `VOXCPM2_MODEL_PATH` | `D:/HuggingFaceCache/VoxCPM2` | 本地模型目录（留空回退在线下载） |
| `VOXCPM2_TIMEOUT` | `300` | 子进程超时秒数 |
| `HF_ENDPOINT` | （未设） | HuggingFace 镜像，国内必设 `https://hf-mirror.com` |
| `HF_TOKEN` | （未设） | 解除匿名限速 |

> 所有变量都可在 `config.py` 顶部找到对应代码，也可写进 `.env`（需自行加 `python-dotenv`，本项目未内置）。

## 前端接入（任意 Level 通用）

无论哪个 Level，前端接入方式一致：

1. **引入** `client/voice.js`。
2. **初始化** `VoiceSystem.init(options)`，配置 `selectors`（监听哪些文字容器）、`defaultSettings.serverUrl`、`panelSelector`（设置面板挂哪）。
3. **或直接调用** `VoiceSystem.speak("文本")` 主动播报。

完整示例见 [client/接入示例.md](../client/接入示例.md)。
