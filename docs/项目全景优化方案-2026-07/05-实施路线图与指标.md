# 实施路线图与指标

## 4-8 周路线图

### 第 1 周：规则止血

- 今日任务按 learner + localDate 保存和重置。
- 定义最小 learner key helper，再显式迁移英语进度与奖励。
- 将 UI 中的 `mastered` 改为“初步学会”。
- 建立 secret scan、静态路由和关键 smoke 的发布门禁。
- 暂停新增大型玩法入口。

验收：跨日、跨档案、重复领奖和发布失败场景都有自动化测试。

### 第 2 周：状态与奖励契约

- 建立 storage registry，首阶段只做审计报告，不改变运行时读写。
- 定义 `PointEvent`、`GameResult`、`LearningAttempt`。
- iframe 游戏停止直接修改成长分。
- 建立每日积分结构、软上限和重玩衰减。

验收：任务、学习、数学 PK、打字和单词地图都通过同一奖励入口。

### 第 3-4 周：真实学习闭环

- 汉字与英语接入统一到期复习队列。
- 分离 completed、provisional、retained。
- 增加会话末复习和跨日出口题。
- 外部阅读增加轻量完成证据。
- 数学记录 skillId、errorTag 和提示级别。

验收：可以计算 1 日和 7 日保持率，并能解释一个孩子为什么收到某道题。

### 第 5-6 周：云同步与事务

- Registry 覆盖率和 Profile round-trip 达标后，快照启用孩子领域白名单。
- 增加 schemaVersion、revision、outbox 和幂等键。
- 邀请、PK、同步改为事务 RPC。
- PK 验证答案事件而非信任客户端总分。

验收：离线刷新后可继续同步；并发请求不会重复计分或部分成功。

### 第 7-8 周：体验与工程收口

- 首页变为单一下一步行动卡。
- 增加全局休息、家长时长和奖励预算。
- 统一 Modal、声音、减少动画和移动验收。
- 抽出 point-ledger、daily-task、game-bridge、review-scheduler。
- 建立 route/asset manifest 和体积预算。

验收：核心流程通过手机、平板、键盘和 reduced-motion 测试。

## 6-12 月蓝图

### 0-3 月

- 三科统一技能分类、尝试事件和复习底座。
- 家长周报以保持率、薄弱点和下一步建议为核心。
- 内容 schema、审核、版本和回滚工具落地。
- 宠物成长从刷 EXP 改为阶段任务和记忆事件。

### 3-6 月

- 识别、回忆、拼写/书写、语境迁移题型贯通。
- 玩法壳与内容包解耦。
- 完成真实的多设备冲突处理。
- 家庭合作任务替代公开竞争导向。

### 6-12 月

- 建立分学龄难度轨道和入门诊断。
- 数据质量充分后再评估 FSRS 或个性化模型。
- 赛季只更新主题、剧情和收藏，不重置学习成长。
- 建立年度儿童隐私、算法公平、防沉迷和资产许可复核。

## 北极星指标

产品北极星：`每周完成 3 天“学习目标 -> 短玩法复习 -> 宠物反馈”闭环的活跃孩子比例`。

学习北极星：`7 日延迟保持率`。

## 指标面板

| 维度 | 指标 |
|---|---|
| 学习 | 1/7/30 日保持率、首答正确率、提示依赖、lapse rate、迁移正确率 |
| 循环 | 今日计划完成率、学习到推荐玩法转化、玩法后回小屋比例、失败后重试成功率 |
| 经济 | 积分来源占比、任务学习与游戏积分比、每日 P50/P90、30 日通胀、上限后刷分次数 |
| 健康 | 单局时长、连续使用超时比例、休息提示接受率、夜间使用、家长限时启用率 |
| 工程 | 未登记存储键、重复积分事件、同步冲突、发布门禁失败、产物体积和未引用资产 |

## 决策 Gate

每个阶段只有满足以下条件才进入下一阶段：

1. 有可重复验证命令，而不是只靠手测截图。
2. 关键数据有版本与迁移方案。
3. 指标能证明学习或可靠性改善。
4. 不扩大儿童数据收集范围。
5. 新玩法复用统一内容、奖励和会话合同。

## 暂不建议

- 不建议立即迁移 React/Vue 或整体 TypeScript 重写。
- 不建议在事件质量不足时引入 AI 推荐或复杂 FSRS 参数优化。
- 不建议继续增加独立货币、排行榜或无限奖励玩法。
- 不建议把 DAU、在线时长、总积分作为学习效果。
- 不建议在云同步边界和儿童隐私未完成前扩大公开用户规模。
