# 学习系统设计

## 从完成量升级到保持率

当前系统已经有内容包、每日学习单、错题加权、数学反馈和阶段奖励，但学习状态仍以“完成”和短时连对为核心。建议把状态分成：

```text
new -> learning -> review -> retained -> transferred
                 \-> lapsed -> review
```

- `completed`：内容流程走完。
- `learning`：同日练习中表现良好。
- `review`：等待跨日复习。
- `retained`：至少两次跨日、无提示正确。
- `transferred`：换题型或语境仍正确。

旧 `mastered` 迁移为 `provisional_mastery`，不能直接等同 `retained`。

## 统一学习数据

核心实体：

- `LearnerProfile`：年龄段、目标、能力起点、每日上限。
- `LearningItem`：学科、技能、难度、先修、内容版本、来源与审核状态。
- `Attempt`：首答、提示、耗时、错误类型、题型与内容版本。
- `MemoryState`：稳定度、难度、重复次数、遗忘次数、`dueAt`。
- `LearningSession`：主动学习时间、新旧题比例、退出原因、休息提示。
- `ContentQuality`：字段完整、答案验证、年龄适配、版权与审核人。

所有记录必须绑定 `learnerId`，优先修复英语固定键造成的跨档案问题。

## 15-20 分钟学习会话

1. 1 分钟：只显示今天一个核心目标。
2. 3 分钟：完成 3-5 个到期旧项目。
3. 6-8 分钟：每批学习 3-6 个新项目。
4. 3-5 分钟：新旧混合主动回忆。
5. 1-2 分钟：无提示出口题决定是否进入跨日复习。
6. 收口：孩子选择“最有把握/还想练”，家长只看一个行动建议。
7. 连续 20 分钟强提醒休息；达到家长上限后停止新任务和连击刺激。

## 首阶段复习算法

先采用可解释的简化 Leitner/SM-2，不急于使用复杂 AI：

- 首次无提示正确：次日复习。
- 首次错误或强提示：会话末再练。
- 第一次跨日正确：3 天后。
- 第二次跨日正确：7 天后。
- 后续间隔：14、30、60 天。
- 任意错误：降低箱位，记录 `lapse` 和 `errorTag`。
- 每日队列：约 60% 到期复习、25% 新内容、15% 薄弱技能。

至少两次跨日正确且一次为异题型，才标为 `retained`。

## 内容治理

- 中文检查空拼音、重复间隔、例句年龄适配和独立汉字覆盖。
- 英语检查音频存在性、翻译口径、词义、难度标定和干扰项质量。
- 数学为题目增加 `skillId`、运算范围、错误类型和生成器版本。
- 外部阅读不能只靠“回来打勾”；增加一道出口题、跟读自评或家长确认。
- 所有内容包进入 schema 验证、抽检、版本和回滚流程。

## 家长反馈

周报只回答四件事：

1. 本周真正保持了什么。
2. 哪一个技能最薄弱。
3. 提示依赖和挫败点在哪里。
4. 下周只做哪一个动作。

避免用总积分、在线时长和排行榜替代学习效果。

## 学习指标

北极星：`7 日延迟保持率`。

辅助指标：

- 每 10 分钟新增 retained items。
- 1/7/30 日保持率。
- 首答正确率与提示后正确率。
- lapse rate 与再次学会所需尝试。
- 识别到回忆、回忆到迁移的落差。
- 会话退出点、连续错误数和休息提醒接受率。
