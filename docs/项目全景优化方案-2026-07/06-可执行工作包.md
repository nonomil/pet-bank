# 可执行工作包

## 执行原则

- 每个工作包可独立验收和回滚。
- 先建立兼容入口和测试，再替换旧行为。
- Storage Registry 先报告、后门禁，避免遗漏键造成数据丢失。
- `GameResult v1` 先统一局结果，掌握证据后续作为可选字段加入。
- 首页推荐必须晚于复习调度器，否则只能继续硬编码。

## 第一批最小变更

### 1. 每日任务日期化

价值最高、范围最小。定义 `learnerId + localDate` 状态，首次读取旧 `petbank_completed` 时迁移到当天。宝箱必须读取同一服务。

最低验收：

- 同一天刷新不丢失。
- 次日任务和宝箱计数归零。
- A/B Profile 不互相影响。
- 旧用户当天状态不突然丢失。

### 2. 英语显式 learner key

使用 active profile ID 构造进度和奖励键，固定旧键只向当前档案迁移一次。本工作包不同时重构学习状态机。

最低验收：

- 两档案同一单词 streak 独立。
- 兑换券奖励独立。
- 旧固定键不会重复迁移。

### 3. CI 无服务快速门禁

首批只加入无需启动本地服务的检查：

```powershell
node scripts/test-static-route-entries.mjs
node prj/runtime_loader_route_base_contract.test.mjs
node prj/route_aware_shell_contract.test.mjs
node prj/profile_isolation_journey_simulation.mjs
```

再对关键 JS 执行 `node --check`。不要立即把依赖本地服务的全量浏览器回归塞进 deploy job。

### 4. 三个游戏奖励 receipt

先给数学 PK、打字防线、单词地图增加持久 receipt/eventId、单局限制和每日上限；后续再演进成完整 `PointLedger`。日期定义必须与 WP1 共用。
