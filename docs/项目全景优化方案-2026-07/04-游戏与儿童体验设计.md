# 游戏与儿童体验设计

## 统一主循环

```text
今日目标
-> 3-5 分钟学习
-> 完成可判定目标
-> 获得熟练星
-> 选择一个短玩法复习
-> 首次掌握获得成长分/宠物经验
-> 回小屋看到宠物反馈
-> 可选探索或结束
```

首页不要陈列所有功能，只展示一张“下一步行动卡”。游戏的作用是复习和反馈，不是独立无限产分器。

## 三层奖励体系

1. `成长分`：家庭任务和经过验证的学习成果，可消费。
2. `熟练星`：学科学习证据，不消费，用于解锁难度与关卡。
3. `宠物经验`：完成计划、首次掌握和探索结算获得。

宝箱、卡牌、徽章和道具属于收藏内容，不再衍生更多货币。

建议每日成长分结构：

- 家庭任务：6-10。
- 学习成果：6-10。
- 小游戏：最多 4-6。
- 全日软上限：20-24。

达到上限后仍可练习，但只记录熟练度和星星，并明确说明当天成长分已收满。

## 奖励合同

游戏只提交 `GameResult`：

```text
gameId, sessionId, learnerId, contentVersion,
attempts, correctFirstTry, hintsUsed, durationMs,
masteryEvidence, completionState, resultHash
```

宿主依据 `RewardPolicy` 决定：首次掌握奖励、每日练习次数、重玩衰减和上限。iframe 不再直接调用全局加分。

## 四套玩法壳

| 玩法壳 | 学习用途 | 可复用实现 |
|---|---|---|
| 快速反应 | 字词识别、口算、拼音 | 数学 PK、汉字题、打字防线 |
| 地图清关 | 单词复习、章节回顾 | 单词记忆地图 |
| 防线生存 | 输入、听辨、计算 | 苦力怕打字、学习机射击 |
| 队伍策略 | 综合复习、长期收集 | 卡牌训练、探索战斗 |

所有壳从同一掌握队列取题，新增学科主要新增内容包，不复制完整游戏。

## 信息架构

```text
今天：今日计划 / 继续学习 / 今日奖励 / 休息
学习：语文 / 数学 / 英语 / 到期复习
伙伴：宠物小屋 / 状态 / 装扮 / 图鉴
冒险：推荐挑战 / 探索 / 卡牌训练
收藏：背包 / 徽章 / 作品
家长：任务规则 / 奖励预算 / 时长 / 报告
```

商店并入伙伴小屋和家长奖励，不作为儿童一级入口；游乐场首页只推荐一项最适合当前学习状态的玩法。

## 失败反馈

统一为三句：

1. 我学会了什么。
2. 这次卡在哪里。
3. 下一局只改一件事。

反馈基于真实错误类型，例如“连续三次在进位处出错，下局开启拆分支援”，避免泛化的“调整阵容再试”。

## 儿童健康与无障碍

- 连续 15-20 分钟显示休息页。
- 每两局插入短暂离屏活动。
- 家长设置工作日/周末时长和学习后可玩局数。
- 不使用断签惩罚、负积分或公开羞辱性排名。
- 统一 Modal 焦点、Escape、返回焦点和 `aria-live`。
- 全站支持静音、减少动画和不依赖颜色的正确/错误信号。
- 固定验收矩阵：390x844、768x1024、键盘、静音、减少动画、200% 文本缩放。

## 资源治理

原型与生成资产体积已经显著提高扫描和发布成本。正式玩法需要独立 manifest、引用分析和预算：

- 单文件图片预算。
- 首屏资源预算。
- 未引用资产报告。
- 远程图片依赖清单与离线 fallback。
- 源资产、生成资产和发布资产分层保存。
