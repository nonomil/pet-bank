# 探索剧情改造设计

## 1. 当前问题

当前探索的基本结构已经完整：

- narrate
- discover
- math
- choice
- encounter

但这些文本大多只是“场景里发生了一点事”，还没有明确说明：

- 你为什么会来这里
- 你在调查谁
- 为什么打赢之后会掉一张宠物卡
- 图鉴页里新增的故事和这次探索有什么关系

## 2. 新结构：图鉴调查版探索

每个场景都改写成同一条主线：

1. **委托目标**
   - 图鉴馆告诉你，这一带有哪类宠物档案缺失
2. **开场叙事**
   - 说明这里的环境、馆方任务和调查重点
3. **发现事件**
   - 捡到宠物留下的工具、脚印、便签、路标、值班表等
4. **数学事件**
   - 不只是开路机关，而是“登记测试”
5. **选择事件**
   - 根据线索判断应该先去哪里、先看什么
6. **遭遇事件**
   - 宠物出现，像在确认你是否有资格把它写进图鉴
7. **战斗结束语**
   - 收卡、登记、补完档案

## 3. 数学事件升级：从算术门槛改成场景任务题

数学题不再只是“剧情中插一道算式”，而是图鉴调查的一部分。每道题都要满足三个条件：

1. **来自现场线索**
   - 题目必须引用当前场景里能看见的脚印、路线、值班表、补给、机关、星轨等线索
   - 避免只把“苹果”换成“贝壳 / 火石 / 星星”的换皮算术
2. **对应明确能力点**
   - 低危区练比较、合计、路线长度、简单分组
   - 中危区练工作表、补给分配、轮值、乘法意义
   - 高危区练规律、倍率、任务规划和多步判断
3. **给出学习反馈**
   - 答对时不只给经验，还要解释“为什么这样算”
   - 答错时给观察提示，而不是简单说“继续探索”
   - 允许孩子从题目里学到一点方法，例如“先找每组数量，再看有几组”

这样数学事件的定位会从“开门机关”变成“用数学完成调查记录”。孩子答题时不是被迫暂停冒险，而是在帮图鉴馆把现场证据整理清楚。

## 4. 低危区和高危区怎么处理

不建议这轮把所有场景都改成 100% 宠物战斗。

建议保留现有节奏：

- 低危险场景：以“调查基础伙伴”为主，战斗像轻量考核
- 高危险场景：更容易遇到真正的 `species` 宠物敌人
- 这样既保住现有数值和地图节奏，也能逐步强化“收卡冒险感”

## 5. 探索与图鉴的明确联动

- 场景里遇到的宠物，需要在图鉴里能找到对应卡片故事
- 图鉴里写到的“来自哪里、在做什么”，需要在场景线索里提前出现影子
- 这样孩子会觉得：
  - “我先在路上认识它”
  - “打赢后才知道它完整的来历”
  - “这张卡不是凭空掉出来的”
