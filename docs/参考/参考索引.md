## 2. 参考项目索引（prj/）

### 2.1 prj/class-pet-garden-1 — 首选蓝本（契合度最高）

- **定位**：现代轻量前后端分离模板，本项目后端化时的直接蓝本。
- **技术栈**：
  - 前端：Vue3 + TypeScript + Vite + Tailwind CSS + Pinia
  - 后端：Node.js + Express + better-sqlite3 + uuid
- **宠物体系**：24 种宠物（19 普通 + 7 神兽）；8 级成长，每级独立图 `lv1-8.png`；`LEVEL_CONFIG=[40,60,80,100,120,140,160]` 累计积分阈值。
- **数据模型**：六表结构 `users / classes / students / badges / evaluation_rules / evaluation_records`。
- **可借鉴点**：
  - `server/db.js` — SQLite 建表 SQL，开箱即用
  - `src/data/pets.ts` — 等级配置与图片命名规范
  - 评价规则 83 条默认数据，规则引擎模板
  - 最精简的前后端分离工程模板
- **关键文件路径**：
  - `prj/class-pet-garden-1/server/db.js`
  - `prj/class-pet-garden-1/src/data/pets.ts`
  - `prj/class-pet-garden-1/package.json`
- **契合度**：★★★★★ — 技术栈现代轻量，schema 清晰，等级图片规范开箱即用；约 302 文件结构简洁。

### 2.2 prj/class-pet-garden-v2 — 次要参考（全栈最复杂）

- **定位**：企业级全栈方案，复杂业务引擎与安全认证的参考来源。
- **技术栈**：
  - 前端：Vue3 + Vite + Pinia + Vue Router + Axios
  - 后端：SpringBoot 3.2 + Spring Security + JWT + JPA + SQLite
  - 部署：Docker / docker-compose
- **宠物体系**：6 级成长（Lv1-6 按 food 进化）；15 Controller / 11 Entity。
- **可借鉴点**：
  - `ShopController.java` + `ShopItem.java` — 商店/装备/进化石/更换卡
  - `PokemonPoolController.java` — 进化规则引擎
  - `species.json` — 物种静态数据规范
  - `PetCard.vue` / `ScoreModal.vue` — 关键 UI 组件
  - JWT 安全方案（Spring Security）
- **关键文件路径**：
  - `prj/class-pet-garden-v2/backend/src/main/java/com/classpet/controller/ShopController.java`
  - `prj/class-pet-garden-v2/backend/src/main/java/com/classpet/controller/PokemonPoolController.java`
  - `prj/class-pet-garden-v2/backend/src/main/resources/data/species.json`
- **契合度**：★★★☆☆ — 当本项目需要复杂商店/装备/进化引擎或安全认证时再深入参考；日常开发过重。

### 2.3 prj/classpet-pro1.0-main — UI/特效灵感（纯前端同构）

- **定位**：纯前端 localStorage 架构，与本项目当前路线同构，UI/特效灵感来源。
- **技术栈**：
  - 纯 HTML/CSS/JS，无框架
  - 存储：localStorage，可选飞书 Bitable 云同步
- **宠物体系**：4 阶段进化（蛋 0 → 幼崽 50 → 成长期 150 → 完全体 300）；三维度宠物状态（饱食/开心/精力）。
- **奖励商店**：免作业券 100 / 座位选择 200 / 班长体验 300。
- **可借鉴点**：
  - `js/data-manager.js` — 数据结构与持久化方案
  - `js/effects.js` — 粒子特效
  - `js/audio.js` — 音效系统
  - 响应式 7 断点方案
  - 进化动画与状态变化反馈
- **关键文件路径**：
  - `prj/classpet-pro1.0-main/js/data-manager.js`
  - `prj/classpet-pro1.0-main/js/effects.js`
  - `prj/classpet-pro1.0-main/js/audio.js`
- **契合度**：★★★★☆ — 纯前端 localStorage 同构（与本项目架构一致），UI/特效/进化动画可直接借鉴。

## 4. 蓝本选择结论

### 4.1 推荐路线

- **首选蓝本**：`prj/class-pet-garden-1`
  - 理由：技术栈现代轻量、schema 清晰、等级图片规范开箱即用、前后端分离工程模板最精简。
- **次选参考**：`prj/class-pet-garden-v2`
  - 理由：仅当需要复杂商店/装备/进化引擎、JWT 安全认证时深入参考；日常开发过重。
- **UI/特效灵感**：`prj/classpet-pro1.0-main`
  - 理由：纯前端架构与本项目同构，粒子特效/音效/进化动画可直接借鉴交互表现。

### 4.2 本项目当前路线

- **当前选定**：纯前端 Vanilla JS（参考 `classpet-pro1.0-main` 同构架构），localStorage 持久化。
- **决策依据**：
  - 与 `classpet-pro1.0-main` 架构同构，可零成本复用其数据层与特效层。
  - 轻量起步、快速迭代，避免早期引入后端复杂度。
- **演进路径**：
  - 当业务需要多端同步、教师管理后台、积分规则引擎化时，切换至 `class-pet-garden-1` 前后端分离模板。
  - 进一步需要企业级安全/部署时，再参考 `class-pet-garden-v2` 的 SpringBoot + Docker 方案。
