# 03-实施路线与验证

## 1. 执行原则

后续如果开始导入，必须按“慢、少、可恢复”的方式做：

- 不并发下载。
- 不循环扫未知路径。
- 优先使用已保存的只读清单。
- 每张图片请求之间加入等待。
- 下载失败立即记录并停止，不用高频重试。
- 不输出、不保存登录 token。
- 只提交本地 `.webp` 和快照，不提交源站 `.png`。
- 新增导入脚本必须支持 `dry-run`，先打印待下载 URL 和输出路径，确认无误再实际下载。
- 新增导入脚本必须支持 `resume`，已存在 `.webp` 时跳过，避免重复请求源站。

## 2. P0 实施路线：甜芽花园册

### 步骤 1：固化植物快照

建议新增：

- `data/source-snapshots/banchong2-plants.json`
- `data/source-snapshots/banchong2-plants-import-manifest.json`

快照来源使用已确认的 `/api/pets` 只读结果，筛选 `category === "PLANT"`。除非清单过期，不重复登录拉取。

### 步骤 2：升级导入脚本

现有脚本：

- `scripts/import_banchong2_animals.py`

建议不要复制粘贴一份植物脚本，而是改造成可配置导入器：

- category：`ANIMAL / PLANT / PERSON`
- source id：`banchong2 / banchong2_plant / banchong2_legend`
- series：`萌爪伙伴族 / 甜芽花园族 / 锦缘奇遇族`
- booklet：`萌爪伙伴册 / 甜芽花园册 / 锦缘奇遇册`
- asset dir：按系列分目录
- delay：默认单线程慢速下载
- resume：已存在 `.webp` 时跳过
- dry-run：只打印将要下载的 URL 和输出路径
- jitter：每次请求之间加入轻微随机抖动
- token hygiene：不打印、不写入、不提交登录 token

如果为了降低风险，也可以先写一个很薄的植物导入脚本，但长期仍应合并成通用导入器。

### 步骤 3：接入图鉴入口

预计需要修改：

- `data/pets.json`
- `data/pokedex-lore-draft.json`
- `js/card-collection.js`
- `js/app.js`
- `js/pet.js`

关键点：

- `card-collection.js`：`阳光花园馆` 增加 `甜芽花园册`。
- `app.js`：认养筛选来源增加 `banchong2_plant` 标签。
- `pet.js`：多阶段图片继承逻辑不能只认 `banchong / banchong2`，应扩展为多阶段来源集合。
- `pokedex-lore-draft.json`：每个植物补 intro、traits、skills、origin、scene 等字段。
- `data/pets.json`：植物 pet id 使用 `bc2p_*` 或 `bc2_plant_*`，不能靠中文名称唯一定位。

### 步骤 4：补测试

建议新增或扩展：

- `prj/pet_banchong2_plant_import_contract.test.py`
- `prj/pet_image_format_contract.test.py`
- `prj/pokedex_lore_data_contract.test.py`
- `prj/pet_asset_integrity.test.mjs`

建议测试点：

- `data/pets.json` 中 `banchong2_plant` 数量等于植物快照数量。
- 所有植物阶段图位于 `assets/banchong2/甜芽花园族/`。
- 所有植物阶段图均为 `.webp`。
- `阳光花园馆` 暴露 `甜芽花园册`。
- `sourceNames` 包含 `banchong2_plant`。
- 新来源可以继承多阶段图。
- 图片格式测试覆盖 `banchong2_plant`，不能只覆盖已导入动物 `banchong2`。

## 3. P1 实施路线：锦缘奇遇册

这一步建议先设计，不马上导入。

### 需要先确认的问题

| 问题 | 推荐答案 |
|---|---|
| 放哪个馆 | `创想课堂馆`，因为含孔子、文昌、文曲星、朝代歌等学习意象 |
| 是否叫宠物 | 不强行叫宠物，称为“奇遇伙伴”更自然 |
| 是否参与战斗 | 先可认养、可图鉴、可展示，战斗能力后置 |
| 文案风格 | Q版国风、轻神话、成长鼓励，不写严肃历史传记 |

### 推荐先产物

- `docs/plans/YYYY-MM-DD-banchong2-legend-booklet-design.md`
- `data/source-snapshots/banchong2-legends.json`
- 命名确认后再开始慢速下载。

## 4. 功能借鉴实施路线

如果你选择做功能，而不是继续导素材，推荐顺序：

1. 阶段图大图预览：小改动，高可见收益。
2. 满级毕业与徽章：让宠物成长有终点和纪念感。
3. 积分规则模板：提高日常使用效率。
4. 兑换使用记录：补齐积分消费闭环。
5. 成长记录导出：作为家长/班级复盘增强项。

## 6. 验收标准

`甜芽花园册`完成时，应满足：

- `阳光花园馆` 出现 `甜芽花园册`。
- 图鉴可筛选并进入植物详情。
- 23 个植物都有 6 阶段 `.webp`。
- 卡片详情和我的宠物页不直连源站图片。
- 图鉴故事不再是模板空话，每个植物有明确性格和陪伴能力。
- 线上 GitHub Pages 不出现 `.png / .jpg / .bmp` 慢加载回退。
- 如果自然成长仍只显示到当前 `PetSystem.STAGES` 的第 5 个阶段，应明确第 6 张阶段图主要用于图鉴详情展示；如果要在宠物页展示第 6 阶，需要另开成长阶段逻辑改造。

`锦缘奇遇册`进入实施前，应满足：

- 名字已确认。
- 所属顶层馆已确认。
- “奇遇伙伴”是否参与战斗已确认。
- 17 个条目的文案口径已确认。
