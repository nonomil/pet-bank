# 02-功能借鉴方案

## 1. 源站可借鉴模块

`班宠乐园2` 的强项不是复杂战斗，而是“积分运营后台 + 成长陪伴”。从前端静态包和只读接口能看到这些模块：

- 家庭/班级成员管理
- 积分奖惩规则
- 宠物领养与成长等级
- 宠物满级毕业与徽章
- 商品兑换与使用记录
- 成长记录与导出
- 成员排行榜与小组排行榜
- 一键随机领养
- 家庭小组

这些功能不需要全部照搬。当前项目已经有更强的探索、PK、图鉴和宠物小屋，所以应该挑能补齐“日常运营闭环”的部分。

## 2. P0 推荐：宠物扩展与图鉴体验

### P0-0：来源路由拆分

目标：先把后续新增来源拆清楚，再导入植物和人物。

建议：

- 已落地动物继续使用 `source: "banchong2"`。
- 植物使用 `source: "banchong2_plant"`。
- 其他/人物使用 `source: "banchong2_legend"`。
- 如果需要溯源，可额外保留 `originSource: "banchong2"`。

价值：

- 避免所有班宠乐园2素材都挤进 `奇趣冒险馆`。
- 避免导入脚本按 source 清理旧数据时误删动物。
- 方便图鉴筛选、性能检查和后续回滚。

### P0-1：甜芽花园册

目标：优先导入植物 23 个，做成 `阳光花园馆 -> 甜芽花园册`。

价值：

- 直接解决“班宠乐园2还有别的宠物没导入”的缺口。
- 植物素材和当前项目花园主题天然契合。
- `.webp` 本地化能继续改善图鉴加载速度。

边界：

- 只做植物，不同时导入人物。
- 不新增大馆。
- 不把植物混进旧 PVZ 系列。

### P0-2：多阶段图片大图预览

目标：宠物详情页、我的宠物节、图鉴卡片详情里，点击任一阶段小图可弹出大图。

价值：

- 新导入的 6 阶段图才真正“看得见”。
- 能改善图鉴详情的收藏感。
- 和你之前提出的“各阶段小图可弹大图”一致。

边界：

- 只做图片灯箱，不做复杂编辑器。
- 先覆盖多阶段宠物来源，再逐步统一旧数据。

## 3. P1 推荐：成长运营闭环

### P1-1：满级毕业与徽章

目标：宠物达到满级后可以“毕业”，生成纪念徽章，并允许重新领养或换宠。

可借鉴点：

- 源站有 `graduate`、`badge-history`、`totalBadges`、`badgeBalance` 等概念。
- 对当前项目来说，它能把“养到满级之后怎么办”补上。

适合改造成：

- 宠物毕业证书
- 图鉴纪念章
- 我的宠物节年度展示

### P1-2：积分规则模板

目标：把常用加分/扣分项做成可编辑模板，而不是每次手动输。

可借鉴点：

- 源站默认规则包括积极发言、作业优秀、乐于助人、进步明显、迟到、未交作业等。

适合改造成：

- 家庭版：阅读完成、数学挑战、整理书包、按时睡觉、主动复盘。
- 班级版：积极发言、小组协作、作业优秀、课堂专注、迟到提醒。

### P1-3：兑换商店使用记录

目标：兑换后不只是扣积分，还能记录“已兑换 / 已使用”。

可借鉴点：

- 源站商品有分类、库存、限购、使用记录。

适合当前项目的轻量版：

- 商品卡：名称、图标、积分、分类。
- 兑换记录：兑换时间、使用状态。
- 使用按钮：确认后变成已使用。

## 4. P2 推荐：管理与报表增强

### P2-1：成长记录导出

目标：把孩子一段时间内的积分、宠物成长、徽章记录导出成 CSV 或本地报告。

适合场景：

- 家长复盘
- 班级期末总结
- 孩子成长档案

### P2-2：家庭/小组排行榜

目标：给多孩子、多家庭、班级场景增加轻量排行榜。

注意：

- 当前项目已经有 PK 和探索，不要把排行榜做成压力源。
- 建议强调“本周进步榜”“坚持榜”“协作榜”，不要只做总分榜。

### P2-3：一键随机领养

目标：给还没有宠物的孩子随机分配一个伙伴。

价值：

- 减少首次使用的选择压力。
- 适合老师或家长快速初始化。

边界：

- 只对未领养成员生效。
- 分配前给预览和撤销。

## 5. 不建议照搬的部分

| 模块 | 不建议原因 | 替代方案 |
|---|---|---|
| 激活码后台 | 当前项目已有自己的部署/账号路线，直接照搬会增加维护成本 | 放到后台专题里单独评估 |
| 完整班级 CRUD | 当前重点是宠物积分和家庭社交，不宜再扩一个班级管理系统 | 只吸收成员、小组、排行榜的轻量形态 |
| 高频实时接口 | 当前项目偏静态前端和轻后端，不应因为参考站点改变架构 | 用本地数据和 Supabase 能力渐进实现 |
| 源站图片直连 | 会慢、会不稳定，也不利于 GitHub Pages | 继续本地 `.webp` 策略 |

## 6. 推荐优先级

| 优先级 | 方案 | 是否需要下载素材 | 适合现在做吗 |
|---|---|---|---|
| P0 | 甜芽花园册 | 需要，慢速下载植物 23 个 | 是 |
| P0 | 阶段图大图预览 | 不需要 | 是 |
| P1 | 满级毕业与徽章 | 不需要 | 是，需先设计数据口径 |
| P1 | 积分规则模板 | 不需要 | 是 |
| P1 | 兑换使用记录 | 不需要 | 是 |
| P2 | 成长记录导出 | 不需要 | 稍后 |
| P2 | 家庭/小组排行榜 | 不需要或少量后端 | 等家庭账号稳定后 |
| P2 | 锦缘奇遇册 | 需要，慢速下载其他/人物 17 个 | 先设计后导入 |
