# 04-只读审计摘要

> 本文记录本轮两个只读子代理的结论摘要。子代理没有编辑文件、没有联网登录、没有下载资源、没有运行会改变仓库状态的命令。

## 1. 现有动物导入管线

当前 `班宠乐园2 -> banchong2` 动物导入已经闭环：

- `data/pets.json` 当前总量为 `261`，其中 `banchong2` 动物 `40` 只，`banchong2_plant` 植物 `23` 个。
- `assets/banchong2/萌爪伙伴族/` 下有 `240` 张 `.webp`，即 `40 x 6` 阶段图。
- `data/pokedex-lore-draft.json` 已有对应 40 条图鉴故事。
- `js/card-collection.js` 已将 `banchong2` 挂到 `奇趣冒险馆 -> 萌爪伙伴册`。

现有脚本 `scripts/import_banchong2_animals.py` 目前强绑定动物导入：

- `CATEGORY` 实际固定为 `ANIMAL`。
- `SOURCE_ID` 固定为 `banchong2`。
- `SERIES_NAME` 固定为 `萌爪伙伴族`。
- `BOOKLET_NAME` 固定为 `萌爪伙伴册`。
- `LEVEL_MAP` 固定为 `[1, 2, 4, 6, 8, 10]`。

因此不能简单改常量直接跑植物，否则有误删、混源和覆盖动物数据的风险。

## 2. 植物导入最小改动

植物导入建议复用动物管线，但先参数化：

| 配置项 | 植物推荐值 |
|---|---|
| category | `PLANT` |
| source id | `banchong2_plant` |
| originSource | `banchong2` |
| 主题册 | `甜芽花园册` |
| series | `甜芽花园族` |
| 顶层馆 | `阳光花园馆` |
| 阶段映射 | `[1, 2, 4, 6, 8, 10]` |

需要新增或修改：

- `data/source-snapshots/banchong2-plants.json`
- `data/source-snapshots/banchong2-plants-import-manifest.json`
- `assets/banchong2/甜芽花园族/`
- `data/pets.json`
- `data/pokedex-lore-draft.json`
- `js/card-collection.js`
- `js/app.js`
- `js/pet.js`
- `prj/pet_banchong2_plant_import_contract.test.py`
- `prj/pet_image_format_contract.test.py`

## 3. 源站功能可借鉴点

班宠乐园2更像“家长/家庭版积分养成系统”，可借鉴点包括：

- 家庭成员与孩子成长数据。
- 积分奖惩规则模板。
- 宠物等级与经验阈值。
- 满级毕业与徽章。
- 商品兑换与使用记录。
- 家庭排行榜和小组排行榜。
- 成长统计与导出。
- 随机成员、随机领养或随机挑战。

建议优先吸收“日常运营闭环”，不要照搬完整后台。

## 4. 关键风险

| 风险 | 应对 |
|---|---|
| 脚本按 source 删除旧数据，复用 `banchong2` 会误删动物 | 植物使用 `banchong2_plant` |
| 当前脚本没有慢速下载控制 | 补 `delay + jitter + resume + dry-run` |
| `向日葵` 等名称可能和现有植物重名 | 使用 `bc2p_*` 唯一 id，不靠 name 定位 |
| 图片格式测试目前只查 `banchong2` | 扩展到 `banchong2_plant` |
| 第 6 张阶段图可能只在图鉴详情出现 | 如果要宠物页展示第 6 阶，另开成长阶段改造 |
| 人物/其他不全是人物 | 用 `锦缘奇遇册` 包住“文化导师 + 奇遇道具” |
