# 班宠乐园2参考借鉴优化方案

> 更新时间：2026-07-06  
> 状态：植物线已落地，人物/其他与功能借鉴仍为候选方案包  
> 来源：已登录只读接口摘要、前端静态包、本项目现有 `banchong2` 动物导入结果

## 1. 当前结论

`班宠乐园2` 不是只有动物素材。当前已确认源站共有 `80` 个宠物条目：

| 分类 | 数量 | 当前项目状态 | 建议处理 |
|---|---:|---|---|
| 动物 `ANIMAL` | 40 | 已导入为 `奇趣冒险馆 -> 萌爪伙伴册`，本地 `240` 张 `.webp` 阶段图 | 保持现状，只做质量维护 |
| 植物 `PLANT` | 23 | 已导入为 `阳光花园馆 -> 甜芽花园册`，本地 `138` 张 `.webp` 阶段图 | 进入质量维护 |
| 其他/人物 `PERSON` | 17 | 未导入 | 先设计成国风/奇遇类子册，确认后导入 |

这次建议不要把三类素材混在同一个来源标签里。动物已经是 `banchong2`，后续植物和人物应使用独立 source id，便于筛选、回滚、性能检查和文案维护。

## 2. 推荐路线

第一阶段建议先做 `植物 23 个`，归入 `阳光花园馆`，新增一个偏可爱的主题册。推荐命名：

| 层级 | 推荐名 | 理由 |
|---|---|---|
| 主题册 | `甜芽花园册` | 可爱、轻松，能同时包住花、果、树、农作物 |
| 系列名 | `甜芽花园族` | 和现有 `萌爪伙伴族` 格式一致 |
| source id | `banchong2_plant` | 独立于动物，便于图鉴筛选 |

工程口径上，后续可以保留 `originSource: banchong2` 表示素材来自班宠乐园2，但运行时筛选和导入必须使用 `banchong2_plant`。这样不会和已落地的动物 `banchong2` 混源，也能避免脚本按 source 删除旧数据时误删动物。

第二阶段建议做 `其他/人物 17 个`，归入 `创想课堂馆` 或 `奇趣冒险馆` 的独立子册。推荐命名：

| 层级 | 推荐名 | 理由 |
|---|---|---|
| 主题册 | `锦缘奇遇册` | 国风但不沉重，可容纳历史人物、神话角色、财神、上上签、朝代歌等 |
| 系列名 | `锦缘奇遇族` | 和“奇遇”定位一致 |
| source id | `banchong2_legend` | 避免直接叫 person，产品感更好 |

## 4. 我的建议

先锁 `甜芽花园册`，做一轮完整小闭环：慢速下载植物素材、转 `.webp`、写入图鉴故事、补入口与测试。它风险最低，也最贴合当前 `阳光花园馆` 的结构。

`锦缘奇遇册`建议先只冻结命名和产品定位，不急着导入。因为它混合了历史人物、神话角色、道具和学习主题内容，最好先决定它是“宠物伙伴”还是“奇遇卡册”，否则后面图鉴文案会散。
