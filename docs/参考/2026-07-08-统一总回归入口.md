# 2026-07-08 统一总回归入口

## 目标

把当前已经分散存在的浏览器模拟、契约测试、云端合同测试，收束成一个可重复执行的统一入口，方便后续整体验收、回归复验和交接。

## 入口脚本

- `scripts/run-full-regression.mjs`

## 配套契约测试

- `prj/regression_runner_contract.test.mjs`

它负责证明：

- 总回归入口文件存在
- 支持 `PETBANK_BASE_URL`
- 已纳入核心玩法、主循环、云端、异常态、排行榜、音效契约等关键测试

## 使用前提

浏览器级模拟依赖本地静态服务先跑起来。

推荐命令：

```powershell
python -m http.server 8765 --bind 127.0.0.1
```

如果服务未启动，`scripts/run-full-regression.mjs` 会先做健康检查并直接报错，不会跑到一半才发现页面打不开。

## 运行命令

```powershell
$env:PETBANK_BASE_URL='http://127.0.0.1:8765'; node scripts/run-full-regression.mjs
```

## 这份入口解决了什么

- 不再需要手工一条一条复制执行分散命令
- 本地浏览器模拟、Node 契约测试、Python 云端合同测试都可以走同一个入口
- 本地服务没启动时会先拦住，避免产生误判
- 后续继续补新的 standalone 模拟时，只要把脚本挂进这个入口，就能自动进入整体验收链路

## 结论

这份统一入口让“整个游戏是否通过当前一轮整体验收”变成了一个可以直接复现的命令，而不是一组分散的人工步骤；截至 2026-07-10，当前工作树已实跑通过 91 项任务。
