## 1. 收口结论

结合当前代码、前序方案和 [项目收口建议 v1](../项目收口建议-v1.md)，本专题确认如下边界：

- 路线地图的职责是“世界导航 + 章节展示 + 进度感承接”，不是独立玩法系统。
- `pet-bank` 的主循环仍然应是“积分 -> 宠物 -> 探索 -> 奖励/小屋”，地图不再新增一套骰子、格子事件、积分奖惩循环。
- 本专题的 **P0** 收口为“视觉与信息架构重构”：
  - 12 场景节点重排
  - 5 章推进更清晰
  - 首页 / 探索双入口视觉统一
  - 移动端更易读
- 下列内容全部移出 P0：
  - DiceWalker 走格状态机
  - 掷骰消耗积分
  - 胜负加减积分
  - 起点圈奖 / 跨圈推进 / Boss 续走
  - 5 类功能地块事件系统

这不是否定地图重构，而是把 P0 从“新玩法系统”收回到“轻量导航重构”。

## 2. 当前代码状态

当前代码和“完整大富翁化”并不一致，实际运行状态更接近轻量版：

- [js/exploration.js](../../js/exploration.js) 当前仍以 `MAP_LAYOUT` 手调坐标渲染 12 个场景节点。
- `BLANK_CELLS` 只是装饰性空白节点，不承载功能事件。
- `renderSceneGridMap(activeSceneId, boardId)` 已经同时服务首页 `#sceneGridMap` 和探索 tab `#sceneGrid`。
- 当前没有骰子 UI、没有 DiceWalker、没有路线格逐格触发、没有地图专属积分结算。
- `startExploration()` 仍是“点场景 -> 扣 HP -> 进故事/战斗/奖励”的轻探索入口。

因此，这组文档的现行方案必须与上述代码现实一致，不能再把“尚未实现的大富翁系统”写成默认主线。

## 4. P0 核心决策

| 维度 | 当前决策 |
|---|---|
| 产品定位 | 地图是导航层，不是第二游戏 |
| 地图结构 | 允许 2-3 个视觉圈层，但本质仍是 12 场景的主线路线图 |
| 节点类型 | 以“场景节点 + 装饰节点 + 章节信息”为主，不新增功能格系统 |
| 交互 | 仍沿用当前点击节点进入场景 / 解锁场景 |
| 首页语义 | 首页是总览与进度展示，不新增复杂操作 |
| 探索页语义 | 探索页是主入口，可高亮当前节点和当前章节 |
| 积分关系 | 不新增地图专属积分产出或扣减规则 |

## 5. 现阶段最值得做的事

- 把 12 个场景节点排成更清晰的外中内层次，而不是继续堆新机制。
- 让“积分、宠物、探索”这几个核心入口在首页和探索页上更直观。
- 提升地图的产品感：章节名、路线感、当前进度、移动端可读性。

## 6. 暂不推进的方向

- 大富翁式逐格走动
- 地图事件格系统
- 掷骰资源消耗
- 地图独立胜负判定
- 基于地图的额外奖励经济
- 为了适配新玩法而重写探索状态机

这些方向如果未来还要评估，应单独开 P2 试验文档，而不是直接并入当前主方案。
