# 路线地图轻量多圈层重构 · 实施计划

> 基于 [01-需求.md](./01-需求.md) 与 [04-方案-大富翁融合.md](./04-方案-大富翁融合.md)。
> 本计划只覆盖“路线地图视觉与信息架构重构”，不包含骰子、逐格、地图积分规则。

| 项 | 内容 |
|---|---|
| 当前目标 | 让路线地图更清晰、更有层次、更像产品入口 |
| 代码现实 | 当前已具备地标散点版、章节主题、双入口复用、boardId 修复 |
| 约束 | 纯前端、零新依赖、优先小步快跑、禁止偷渡第二玩法循环 |

## 4. 执行约束

- 只改路线地图相关层：`MAP_LAYOUT`、`BLANK_CELLS`、`buildRouteSvg`、`buildMapNode`、相关 CSS。
- 不新增 DiceWalker。
- 不新增地图运行时状态机。
- 不新增地图积分消耗/奖励规则。
- 不改 `startExploration()` 的主玩法语义。
- 如果“更像大富翁”与“更易读”冲突，以更易读优先。

---

## 5. 回退原则

如果 S1 截图验证发现“多圈层”反而让移动端更拥挤，则允许回退到：

- 更松的地标式层次布局
- 更弱化的圈层暗示
- 更强调章节分区而非几何圆环

也就是说，**圈层只是手段，不是死目标**。

---

## 6. 暂停项

以下内容全部暂停，不属于当前执行包：

- 掷骰 UI
- 逐格走动
- 功能地块事件系统
- Boss 续走状态机
- 地图专属积分奖惩
- 起点圈奖 / 跨圈奖励

这些方向如要恢复，需另开新文档重新评估。
