# 路线地图轻量多圈层重构（收口方案 v3）

> 文件名沿用“方案-大富翁融合”，但本版已经不是“大富翁玩法方案”。
> 当前方案基于 [01-需求.md](./01-需求.md) 与 [项目收口建议 v1](../项目收口建议-v1.md)，将路线地图重新定义为：
> **更强的导航层，更轻的机制层。**

| 项 | 内容 |
|---|---|
| 目标 | 提升路线地图的章节感、世界感、进度感 |
| 复用 | `MAP_LAYOUT` / `BLANK_CELLS` / `buildRouteSvg` / `buildMapNode` / `renderSceneGridMap` |
| 性质 | 轻量重构，不引入第二玩法循环 |

---

## 1. 产品定位

路线地图在本项目里只做三件事：

- 作为“探索世界入口”
- 作为“章节与进度展示层”
- 作为“首页和探索页的视觉锚点”

路线地图不做的事：

- 不做独立棋盘玩法
- 不做地图专属积分经济
- 不做独立状态机驱动的长链路流程

这一定义能与当前项目主循环保持一致：

`积分 -> 宠物 -> 探索 -> 奖励/小屋`

---

## 2. 方案核心

### 2.1 多圈层，但不是棋盘格系统

建议把当前地图从“地标散点”进一步整理为**多圈层主线路线图**：

- 外层：第 1-2 章，承担“起步与熟悉世界”
- 中层：第 3-4 章，承担“中段推进与挑战升级”
- 内层：第 5 章与终点，承担“收束与奖励感”

这里的“圈层”是**视觉组织方式**，不是需要逐格走动的玩法结构。

### 2.2 场景节点仍然是主角

当前 12 个场景卡片继续保留为主节点，地图不引入新的“功能格”体系。

地图上只保留三类视觉元素：

- 场景节点
- 路线线条
- 装饰节点 / 章节标签 / 分区背景

这样可以避免为了地图引入一整套事件分发、暂停续走、奖励结算逻辑。

### 2.3 首页和探索页共用同一套地图语言

- 首页：总览、展示、唤起兴趣
- 探索页：主操作入口、强调当前节点与当前章节

底层仍通过 `renderSceneGridMap(activeSceneId, boardId)` 统一渲染，只在展示层上区分信息密度和高亮状态。

---

## 3. 视觉与信息架构设计

### 3.1 路线表现

- 保留 SVG 路线线条
- 让路径更明确地围绕外层 -> 中层 -> 内层推进
- 装饰节点继续只做气氛承接，不承载点击后复杂逻辑

### 3.2 章节表现

- 继续保留 `CHAPTER_THEME`
- 允许通过章节标题、章节色带、分区浅底强化 5 章结构
- 终点层需要明显更聚焦，形成“走到中心/终点”的心理预期

### 3.3 节点状态表现

以下四态必须优先保证：

- 当前节点
- 已完成
- 已解锁
- 未解锁

如果多圈层布局与可读性冲突，以可读性优先，不强行追求几何上的“像棋盘”。

---

## 4. 交互策略

### 4.1 保持现有主链路

节点交互继续沿用当前代码：

- 已解锁 -> `goExplore(sceneId)`
- 未解锁 -> `tryUnlock(sceneId, boardId)`

点击节点后仍进入现有探索详情、战斗、奖励回写链路。

### 4.2 不新增地图机制

本轮明确不做：

- 掷骰
- 逐格移动
- 地图即时事件
- 地图专属积分奖惩
- Boss 续走

这些机制会显著提高实现复杂度，并把地图从“导航层”推向“第二玩法层”。

---

## 5. 与当前代码的对应关系

| 模块 | 处理方式 |
|---|---|
| `MAP_LAYOUT` | 作为本轮主要改动点，重排坐标与层级 |
| `BLANK_CELLS` | 继续做装饰点，可少量增减位置 |
| `buildRouteSvg` | 调整路线表现，强化主线路径感 |
| `buildMapNode` | 补强章节/状态视觉，但不改核心进入逻辑 |
| `renderSceneGridMap` | 继续作为首页/探索页共用入口 |
| `startExploration` | 不改玩法语义，不挂地图新状态机 |

如需新增字段，优先只加轻量数据，例如：

- `ring`
- `routeGroup`
- `nodeRole`

不新增地图运行态对象和长期状态持久化。

---

## 6. 分阶段交付

### P0 · 本轮

- 多圈层静态布局
- 章节信息增强
- 首页 / 探索双入口统一
- 桌面 / 移动端可读性修正

### P1 · 后续可选

- 地图背景进一步精修
- 轻量动效
- 章节分区视觉增强

### P2 · 冻结候选

- 掷骰
- 逐格走动
- 地图功能地块
- 地图资源循环

若未来真要恢复 P2，必须先重新评估是否仍符合产品边界，而不是直接沿用历史方案。

---

## 7. 风险与取舍

| 风险 | 取舍 |
|---|---|
| 一味追求“像大富翁”导致可读性下降 | 以导航清晰度优先，宁可不像棋盘也不牺牲可读性 |
| 引入新机制后与现有探索链路耦合过深 | 本轮完全不引入机制扩展 |
| 多圈层布局在移动端拥挤 | 用截图门禁控制，必要时回退到更松的层次布局 |
| 文档继续写得像新游戏方案 | 统一以轻量导航重构表述，不再用“可玩闭环”描述地图 |

---

## 8. 结论

路线地图值得继续做，但更专业的做法不是把它做成一个新系统，而是把它做成：

**更像产品首页入口的世界地图**

而不是：

**一个还要自己维护规则、资源和状态机的小游戏。**
