# 项目优化加速方案

> 基线日期：2026-07-19
>
> 目标：在不破坏本地优先、Profile 隔离、离线玩法和现有奖励契约的前提下，降低首屏加载、玩法首次进入和发布制品维护成本。

## 结论

本项目仍适合以网页为主，但不能继续把所有页面、学习内容、游戏资源和长尾音频当成一个默认发布包。推荐采用：

```text
共享核心契约
  ├─ 主站：任务、积分、宠物、家长
  ├─ 学习中心：学习包、词库、学习音频
  └─ 游戏中心：像素探险、打字防线、数学 PK、卡牌和小游戏

静态文件：同源或版本化 CDN
动态接口：Nginx -> Node.js + SQLite
```

暂不推进微信小程序或原生 Android 重写。先完成资源清单、按玩法分发、CDN、PWA 和真实网络指标，之后再根据使用数据决定平台扩张。

## 重要边界

- `docs/` 是方案和证据资料；当前代码契约以 `docs_project/`、源码和测试为准。
- 不删除当前工作树中用户已有的素材或 Minecraft 学习资源改动。
- 发布制品只能由 `scripts/assemble-pages-artifact.mjs` 生成，不能直接发布仓库根目录。
- CDN 只缓存公开静态资源；账号、Profile、token、快照和 API 一律不进入公共缓存。
- 所有体积结论必须区分：首屏下载量、单玩法首次进入量、完整发布制品大小。
