# 探索地图音频按需加载

## 当前事实

探索地图源目录包含 `960` 个 WAV 和 `960` 个 OGG，WAV 约 `999.71 MiB`，OGG 约 `80.35 MiB`。进入地图不会下载全部音频；旧实现会请求约 `510 KB` 的全量音频 manifest，再在当前句播放时创建 `Audio`。

## 当前改动

1. 新增 `audio-index.json` 根索引，只描述章节索引路径，不携带全部句子音频表；原始 `audio-manifest.json` 只用于生成/校验，发布时不进入 Pages。
2. 新增 `audio-indexes/<chapterId>.json`，进入具体章节时才请求该章的句级音频索引。
3. 当前句仍由 `VoiceSystem.playStoryAudio()` 触发浏览器音频请求，不预加载整章或整季音频。
4. 支持 `window.PetBankConfig.pixelStoryAudioBaseUrl`，配置后把当前句 `.wav` 源 key 映射到 CDN 上的 `.ogg` 文件。
5. CDN 失败时保留现有 TTS 回退，不影响故事文本和进度。

## CDN 目录约定

独立音频包的根地址下保留章节相对目录，例如：

```text
pixel-story-audio/v20260719/
  sf-01/lines/arrival-00.ogg
  sf-01/lines/arrival-01.ogg
```

配置示例：

```html
<script>
  window.PetBankConfig = {
    pixelStoryAudioBaseUrl: 'https://static.example.com/petbank/pixel-story-audio/v20260719/'
  };
</script>
```

索引和音频均为公开静态资源；音频使用 `audio/ogg`、支持 `Range`，版本目录使用长期缓存。账号、Profile、token 和 API 不进入该 CDN。

## 验收

```powershell
node scripts/build_pixel_story_audio_indexes.mjs
node scripts/test-pixel-story-audio-indexes.mjs
node scripts/assemble-pixel-story-audio-artifact.mjs tmp/pixel-story-audio-artifact v20260719
node scripts/test-pixel-story-audio-artifact.mjs tmp/pixel-story-audio-artifact
node scripts/test-pixel-story-publish-contract.mjs
```

通过标准：探索地图入口只请求根索引；进入章节才请求该章索引；播放当前句才请求一条 OGG；发布制品不包含约 510 KB 的全量 `audio-manifest.json`；不会因为进入探索地图下载 80.35 MiB 音频池。
