# 首屏与运行时加载

## 当前问题

- `index.html` 包含全部页面容器，隐藏页面的图片仍可能被浏览器发现。
- 公共 `css/style.css` 和 `js/app.js` 仍承担大量页面逻辑。
- `runtime-loader.js` 已有页面 bundle，但部分 bundle 内部仍存在大型数据副本。
- `js/voice.js` 只属于 `explore` 页面 bundle，不进入首页首屏；其旧预生成语音索引也不应在进入探索页时提前请求。
- `inventory.js`、`treasure.js` 等模块需要检查公共 JSON 是否重复读取。
- `fetchPriority=high` 不能替代真正的懒加载，错误使用会让重资源抢占首屏。
- 本地深层路由必须先跳到共享 `index.html?route=...`，再恢复浏览器地址；否则浏览器会在运行时修正 `<base>` 前预加载 `/app/css`、`/app/js`，随后重复请求根路径资源。
- Minecraft 词卡源池约 `7.2 MiB`，不能随 `/app/learn/minecraft-vocab` 首次进入一起请求。
- `assets/voice/map.json` 约 `44 KB`、旧预生成语音约 `25.1 MiB`；进入探索页不再主动请求索引，第一次自动播报或点击朗读时才请求一次索引，具体 MP3 仍按句播放时请求。
- 学习机首页只需要内容配置和门户缩略图；英语统一词库约 `4.04 MiB`、汉字运行包约 `0.90 MiB` 不应在首页初始化时并行请求。

## P0：首屏减负

1. 根入口只保留首屏需要的图片；隐藏页面图片进入页面后再赋值 `src`。
2. 对非首屏图片设置明确宽高，避免布局抖动。
3. 当前页面必要图片使用普通加载或合理的 `fetchpriority=high`；其余使用 lazy/low。
4. 不在首屏预取学习、游戏、故事和完整音频包。
5. 让错误图片有 emoji/颜色/占位回退，不因懒加载破坏页面结构。

## P1：公共代码拆分

优先从低耦合区域开始：

1. 家长账号和自托管 API 不在孩子端首屏加载。
2. 复盘、作品、工具等低频页面移出主编排器。
3. 将 `style.css` 按核心壳、孩子主线、家长区、地图/探索拆分。
4. 将 `app.js` 的页面渲染拆成模块，但保持 `switchPage`、`PetBankPoints` 等兼容 API。

## P1：数据请求去重

所有公共数据加载应符合：

```js
const promise = fetch(url).then(assertResponse).then(parseJson);
cache.set(url, promise);
return promise;
```

失败的 Promise 必须从缓存清除，避免一次网络失败永久污染运行时；状态变更和奖励结算不能因为缓存重构改变顺序。

当前已落地：

- `js/asset-loader.js` 注册 `PetBankAssetLoader.fetchJson()`，按解析后的资源 URL 缓存公共 JSON Promise。
- `inventory.js` 和 `treasure.js` 对 `data/items.json` 共用同一请求；`learn-center.js` 的目录、学习包和模块 JSON 也进入同一缓存边界。
- 像素故事地图和故事引擎对共享 `manifest.json` 也统一走该 loader；探索入口并发初始化时不再重复请求同一故事清单，独立原型没有 loader 时仍保留原有 fetch fallback。
- HTTP 非 2xx、JSON 解析失败或网络错误都会删除对应缓存项；调用方仍保留各自的空数据/提示 fallback，后续进入页面可以重试。
- 该缓存只处理公开静态 JSON，不处理账号、Profile、token、快照或 API 响应。
- `js/minecraft-vocab-loader.js` 将 Minecraft 词卡拆成 starter 与六个 band；默认幼儿园/桥接/低年级只请求约 `0.44 MiB` starter，切换 Minecraft 初级才请求选中的 band，`all` 仅由用户主动选择时加载六个 band。
- `js/learn-center.js` 的资料包预览改读 starter 分片；Pages 组装器保留源码全量 JSON 在仓库，发布制品中删除该文件，防止学习中心入口无意下载全量词库。
- `data/pets-runtime-index.json` 是由 `data/pets.json` 生成的探索运行索引，只保留 261 个宠物的战斗/基础展示元数据，约 60 KB；`explore`/`forest-map` 按需请求它，首页 `map` 不请求任何宠物目录。
- `data/pets.json` 完整图鉴仍保留，`pet`、`home`、`walk`、`card` 和卡牌对战进入时才调用 `ensurePetCatalog()`；已有存档的 `species_data.imageStages` 会与轻量索引合并，不因延迟目录丢失阶段图。
- 首页专用 `assets/home-bg.webp` 不再由初始 `<body>` 类全站触发；深层 `explore`/`forest-map` 入口不请求首页背景，只有进入 `map` 首页时由路由恢复 `map-page-bg`。隐藏首页同行卡和积分卡的默认 `dog_idle.webp` 回退图也延迟到对应页面激活后再设置。
- 首页轮播 `css/showcase.css`、`js/showcase.js` 不再进入公共 HTML，只在 `map` 首页的 runtime bundle 中加载。`js/minecraft-vocab-session.js` 保留公共轻量 API 预加载，因为今日摘要需要在共享首轮渲染阶段读取它；完整远征模块仍按需加载。
- 孩子端工作台 `css/child-workbench-shell.css`、`js/child-workbench-shell.js` 也移出公共 HTML，只在孩子端经典页面加载；家长/设置/作品/工具深链不下载。动态加载完成后由 `preparePage()` 再同步当前页，保留导航高亮、进度栏和移动端布局。
- 探索战斗特效 `js/battle-fx.js` 不再进入公共 HTML，只在 `explore`/`forest-map` runtime bundle 中加载；探索 bundle 完成后才允许进入战斗，因此不会牺牲战斗首个交互。
- 首页五张常用入口图不再直接引用玩法原图；`scripts/generate-home-image-thumbnails.py` 根据配置生成 640x360 WebP 缩略图，原图继续由实际玩法使用。首页卡片仍由 `data-home-src` 在 `map` 页面激活，并保留 `loading="lazy"`。
- 首页缩略图输出固定在 `assets/ui/home-thumbs/`，生成脚本的 `--check` 会校验 WebP、尺寸和输出小于源图；不得用手工覆盖原图来满足首屏预算。
- 汉字题图 35 张源 PNG 只作为仓库源素材保留；主站和学习机运行时改用同名 WebP，约从 `39.8 MiB` 降到 `2.5 MiB`，发布器不复制 PNG。
- 绘本正文已经独立到 `picturebook-library`；主站只按需加载 25 张书架封面 WebP，正文页和旧全量绘本目录不进入 Pages。
- 学习机按游戏加载数据：打开飞机大战才请求英语词库，打开拼音赛车/贪吃蛇/汉字跳跃才请求汉字数据；`file://` 回退脚本也由同一加载器在点击后注入，语音索引延后到首次汉字朗读。

验收入口：`node --test scripts/test-asset-json-loader.mjs`、`node scripts/test-minecraft-vocab-runtime-shards.mjs`、`node scripts/test-minecraft-vocab-publish-contract.mjs _site`、`node scripts/test-voice-audio-lazy-browser.mjs`、`node scripts/test-pet-runtime-index-contract.mjs`、`node scripts/test-pet-catalog-lazy-browser.mjs`、`python scripts/generate-home-image-thumbnails.py --check`、`node scripts/test-home-image-thumbnails.mjs`、`node scripts/test-learning-arcade-lazy-data-contract.mjs`。

入口 waterfall 使用 `scripts/test-mode-server.mjs`（默认 `7001`）测量，避免未登录访问保护页时跳到家长区；报告同时记录请求 URL 和最终 `finalUrl`，生产环境仍需另测 Nginx 鉴权和 CDN。

## 页面生命周期

页面切换必须继续使用 activation token 或模块 owner 的 stop/abort 机制。懒加载、预取和图片回调不能在 profile reload 或页面切换后写入新页面。

## 目标

- 根首页不请求隐藏玩法大图。
- 深层玩法入口不请求首页背景和隐藏卡片默认占位图。
- 普通页面首次进入不加载其他产品线脚本。
- 重玩法首次进入资源超过预算时必须显示可定位的资源报告。
- 页面切换后旧 timer、音频、iframe 和异步请求不污染新页面。
