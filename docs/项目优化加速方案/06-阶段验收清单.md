# 分阶段验收清单

## 阶段 0：基线

- [ ] `node scripts/local-server.mjs` 可启动。
- [ ] 根首页和代表性页面有资源请求报告。
- [ ] 请求报告包含入口、请求数、失败数、传输字节、资源类型汇总和 Top 大请求。
- [ ] 制品审计输出总量、扩展名、Top 30 和重复组。
- [ ] 制品审计能校验每个 runtime manifest 的预算和至少一个已发布文件覆盖。
- [ ] 记录 commit、工作树状态、浏览器和网络条件。

命令：

```powershell
node scripts/audit-pages-artifact-size.mjs _site_verify --top=30
node scripts/test-pages-artifact-size-audit.mjs
node scripts/test-measure-page-load.mjs
node scripts/test-static-route-entries.mjs

# 本地 HTTP 服务启动后执行；报告写入 tmp，不进入发布目录
node scripts/measure-page-load.mjs --routes=/,/app/today,/app/learn,/app/explore,/app/playground --output=tmp/page-load-baseline.json
```

## 阶段 1：首屏

- [ ] 隐藏页面图片不在根首页默认请求。
- [ ] 首屏图片总量符合预算。
- [ ] 根首页 `app-ready` 正常。
- [ ] 首页、今日、宠物、学习、游乐场无新增图片 404。

命令：

```powershell
node scripts/test-static-route-entries.mjs
node scripts/test-home-browser.mjs
node scripts/test-child-journey-home.mjs
```

## 阶段 2：玩法资源

- [ ] 每个玩法有 manifest 和预算。
- [ ] HTTP/HTTPS 不加载不必要的回退脚本。
- [ ] 音频按章节/词包请求。
- [ ] 探索地图只加载根/当前章节音频索引，当前句才请求 OGG。
- [ ] core/extension 资源不交叉误加载。
- [ ] Pages 组装后的总制品不超过当前阶段 `400 MiB`，各 manifest 预算与覆盖门禁通过。

命令：

```powershell
node scripts/test-asset-fallback-loading.mjs
node scripts/test-runtime-audio-variants.mjs
node scripts/test-runtime-image-variants.mjs
node scripts/assemble-pages-artifact.mjs tmp/_site_verify
node scripts/audit-pages-artifact-size.mjs tmp/_site_verify --top=30 --max-mib=400 --require-manifest-budgets --require-manifest-coverage
```

## 阶段 3：CDN 灰度

- [ ] Minecraft 完整音频由独立 artifact 生成，不进入默认 Pages。
- [ ] 默认无 CDN 配置时不请求完整音频 manifest。
- [ ] CDN 配置后 manifest 映射正确，音频失败可回退浏览器语音。
- [ ] 新版本使用独立版本目录。
- [ ] 静态资源 MIME、Range、CORS 正确。
- [ ] 第二次访问产生缓存命中。
- [ ] API 和 Profile 数据没有公共缓存。
- [ ] 可切回上一版本。

## 阶段 4：应用拆分/PWA

- [ ] `/app/`、`/learn/`、`/arcade/` 可独立构建和回滚。
- [ ] Profile、积分、奖励 receipt 和离线模式跨入口兼容。
- [ ] PWA 只缓存核心壳和用户选择的内容包。
- [ ] Service Worker 升级和失效策略通过浏览器测试。

## 阻断条件

出现以下任一情况，阶段不能标记完成：

- 资源 404、JSON/JS 数据数量不一致或图片自然尺寸为 0。
- Profile 切换、奖励防重、页面生命周期测试失败。
- CDN 返回 HTML 代替 JS/JSON，或 API 被公共缓存。
- 只能通过删除未确认引用的源素材来满足预算。
