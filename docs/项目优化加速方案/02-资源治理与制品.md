# 资源治理与发布制品

## 目标

把“源素材、研究资料、运行时资源、首屏资源、按需资源”分开，发布脚本只复制浏览器确实会请求的文件。

## 分类

| 分类 | 例子 | 发布策略 |
|---|---|---|
| 源素材 | 原始 PNG、生成过程、提示词、参考图 | 保留仓库，不发布 |
| 核心首屏 | 主站壳、当前宠物图、今日页资源 | 主站制品，严格预算 |
| 页面资源 | 小屋、探索、商城、家长页资源 | 页面 bundle 按需发布 |
| 玩法资源 | 像素地图、打字防线、卡牌、数学 PK | 玩法 manifest |
| 长尾内容 | 章节音频、词卡图片、扩展词库 | 章节/主题/词包 manifest |
| 回退资源 | `file://` 专用 JS 数据副本 | 默认不进 HTTP 发布包 |

## Manifest 结构

建议每个玩法具备以下信息：

```json
{
  "id": "pixel-worlds-core",
  "version": "2026-07-19",
  "entry": ["js/pixel-story-map.js", "js/pixel-story-engine.js"],
  "data": ["data/story-packs/05-pixel-worlds-story/manifest.json"],
  "assets": ["assets/story/pixel-worlds-v1/maps/forest.webp"],
  "audio": ["assets/story/pixel-worlds-v1/audio/forest/**"],
  "budgetMiB": 8,
  "fallback": "none"
}
```

实际字段以现有 manifest 和构建脚本为准；此结构用于表达边界，不要求一次性替换所有现有清单。

## Minecraft 词卡清单

当前清单为 `scripts/runtime-asset-manifests/minecraft-vocab.json`。它把 Minecraft 词卡的运行资源分为：

- `normalized` 卡图、卡背、专用单词音频、远征图、学习 UI 和视觉包；
- 24 个基础单词音频；
- 完整词库的 2,168 词旁白源池仍保留在仓库，发布时可转为 OGG/Opus；主站当前只发布 starter/远征首轮的 700 条 OGG，其他 band 使用浏览器语音回退；
- 原始卡图、`minecraft-reference` 参考图和旁白 MP3 源文件只留在仓库，不进入 Pages 制品。

词库 JSON 已完成运行时分片：`minecraft-vocab-runtime-starter.json` 约 `0.44 MiB`，首次进入只加载它；Minecraft 初级再按 `core/basic/building/mobs/world/advanced` 加载单个 band，完整词库只在用户主动选择 `all` 时并行加载六个 band。源文件 `minecraft-vocab.json` 仍保留在仓库，但由组装器从 Pages 制品剔除。旁白也按 starter 收口，当前完整制品约 `417.6 MiB`，这和首屏下载量必须分开统计。

分片覆盖合同为 `scripts/test-minecraft-vocab-runtime-shards.mjs`，发布边界合同为 `scripts/test-minecraft-vocab-publish-contract.mjs`。完整旁白源池仍是后续 CDN/独立音频包的输入，主站不再为用户没有选择的 band 发布长尾音频。

## 像素世界故事清单

`pixel-worlds-story` 清单只允许 4 张地图、80 张场景、32 张角色帧、80 张道具/路线图标和 960 条已转换 OGG/Opus 句级音频进入 Pages。运行时先请求轻量根索引，再按章节请求索引，当前句才请求 OGG；配置 `pixelStoryAudioBaseUrl` 后可把音频移到独立 CDN。`audio-manifest.json` 仅作为生成源保留，不进入 Pages。`assets/story/pixel-worlds-v1/audio` 中约 1.1 GiB 的 WAV 是源素材，发布合同会拒绝 WAV 和地图 PNG；四个 `props-*-4x4` 中间图也不属于运行时。

Minecraft 词库的 `data/vocab/english-minecraft/narration-manifest.json`（约 `3.45 MiB`）和旧 `audio-manifest.json`（约 `70 KB`）同样只服务于生成/审计，不属于浏览器运行时；Pages 只保留 `manifest.json`、必要视图、runtime shards 和 starter/独立 CDN 音频映射。

## 打字防线清单

`typing-defense` 清单把原型的 `web/`、运行时音频、Minecraft 打字词库、打字防线素材和词卡目录登记为唯一发布边界；截图、参考图、提示词、临时目录和开发原型本身不进入 `app/playground/typing-defense-runtime/`。清单同时记录源包到 Pages 发布目录的映射，审计报告按发布目录统计。

## 独立绘本项目边界

主站的 `picturebooks` 页面是独立 `picturebook-library` 的入口和奖励桥接层，不负责承载绘本正文。Pages 只保留 `data/picturebooks/portal.json`、`portal-catalog.json` 和 25 张书架封面 WebP；完整正文页、旧版 `data/picturebooks/catalog.json` 和源 JPG/PNG 由独立绘本项目维护并发布。封面变体由 `scripts/runtime-image-variants.json` 的 `picturebook-portal-covers` 清单生成，不能把正文页重新复制回主站制品。

## 学习机与单词记忆资源边界

学习机和单词记忆仍通过主站 iframe 入口兼容运行，但其运行包必须只携带浏览器实际会使用的文件。源目录中的参考图、`file://` 回退 JS、旧资源包和源 PNG 继续保留，不直接作为 Pages 发布内容。

- 学习机首页 3 张游戏卡和 1 张舞台背景改为同目录 WebP 变体，源 PNG 不覆盖、不删除；清单登记在 `scripts/runtime-asset-manifests/learning-arcade.json`，变体由 `scripts/runtime-image-variants.json` 管理。
- 单词记忆的 Minecraft 宠物姿态图发布时只保留 WebP；40 张 PNG 源图约 `17.68 MiB`，对应 WebP 约 `5.91 MiB`。组装器会阻止姿态 PNG 进入 Pages，并检查全量词卡 JSON 不再引用姿态 PNG。
- 单词记忆默认 `core` 词包仍按原入口加载，扩展和 `all` 词包保持显式选择，不把“全量词卡可选”误写成首屏下载。
- 源素材与离线调试入口仍保留在 `prj/学习机玩法原型/` 和 `prj/单词记忆射击场原型/`；未来独立 CDN 制品应从同一清单组装，不能直接上传整个原型目录。

## 重点治理顺序

1. 音频：按故事章节和 Minecraft 词包拆分，保留 Range 播放能力。
2. 学习图片：按 pack/module/主题发布，不随主站复制完整词卡图。
3. 像素探险：core、extension、地图主题分开，禁止同时携带 JS 回退和 JSON 正式数据。
4. 学习机：只发布 HTTP/HTTPS 真正使用的 JSON 和运行代码。
5. 重复素材：先报告 hash 重复，再确认引用后处理；不批量删除源文件。
6. 生成参考图：禁止进入 Pages 制品，除非 manifest 明确标为 runtime。
7. 已拆分的独立项目正文：只保留主站入口所需的目录和封面，正文由独立 artifact/CDN 发布。

## 构建门禁

每次发布必须输出：

- 总体积和文件数量。
- `assets`、`data`、`app`、`prj` 分布。
- `.ogg`、`.png`、`.webp`、`.json`、`.js` 分布。
- Top 30 大文件。
- 重复内容组。
- 每个 manifest 的文件数和体积。
- 超出预算的具体路径。

构建入口：`scripts/assemble-pages-artifact.mjs`；审计入口：`scripts/audit-pages-artifact-size.mjs`。
