# 后续落地细节

已实现：

- `PetStoryCases` 用 `petbank_pet_story_cases_v1` 保存案件收据。
- 记录以 `storyId + caseId + profileId + petIdentity` 隔离，重复结案返回 duplicate。
- `pet.js` 的宠物状态增加 `pet_id`；旧存档首次加载时自动补写，重新选择宠物会创建新的实例 ID。
- 线索源和照料 API 使用白名单；故事模块不直接改写宠物数值。
- 探索页加载故事 JSON，渲染来信、线索、选择、照料/备用结案和回信。

后续扩展前仍需保持：新案件只能使用 `PetStoryCases.CLUE_SOURCES` 和 `CARE_ACTIONS` 中登记的能力；新增状态字段、奖励或页面入口时同步更新数据契约并补对应测试。第一季不发成长积分，避免为了领奖重复触发照料。
