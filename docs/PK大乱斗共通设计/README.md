# PK 大乱斗共通体验设计

> 日期：2026-07-06
> 参考：`docs/数学PK游戏/宝可梦数学大乱斗.md`
> 范围：数学 PK、卡牌对战、探索故事遭遇战

## 1. 参考稿提炼

参考案例的关键不在具体美术，而在一个孩子很容易理解的战斗闭环：

1. 选关：先知道自己要挑战哪一关、难不难、赢了有什么。
2. 选角色：用自己喜欢或已养成的角色出战。
3. 输入/选择行动：数学题输入答案，或选择技能/道具。
4. 出招反馈：答对或选中技能后，角色立刻发动专属招式。
5. 血条变化：上方或角色旁有清楚 HP，孩子能看到“我打中了”。
6. 胜利结算：有胜利画面、奖励、下一关或再来一局入口。

这套结构可以迁移到当前项目，但美术不需要照抄。当前项目的主角应该继续是“自己的宠物”和已收集的卡牌，敌人则按玩法分别使用机器人、训练对手或探索怪物。

## 2. 三套玩法的定位

| 玩法 | 当前定位 | 应借鉴的部分 | 不应照搬的部分 |
|---|---|---|---|
| 数学 PK | 答题竞速 / 乘法理解练习 | 选难度、答题触发招式、HP/胜场反馈、胜利结算 | 不做复杂卡牌构筑，不让第一次学乘法变成纯战斗 |
| 卡牌对战 | 已经是 2v2 宠物卡牌训练营 | 选关、选队、血条、技能出招、战后奖励 | 不把题目系统硬塞进卡牌战斗 |
| 探索故事 PK | 剧情遭遇战和技能战斗 | 剧情引出敌人、角色登场、HP、技能反馈、结算回到故事 | 不抢主线，不做冗长竞技 UI |

## 3. 共通体验骨架

三套 PK 统一采用同一个体验骨架，但每套玩法只实现自己需要的层级。

```mermaid
flowchart LR
    A["入口 / 关卡"] --> B["角色确认"]
    B --> C["对战舞台"]
    C --> D["行动选择"]
    D --> E["角色出招"]
    E --> F["HP / 进度变化"]
    F --> G{"是否结束"}
    G -- 否 --> D
    G -- 是 --> H["胜利 / 失败结算"]
```

### 3.1 入口 / 关卡

- 数学 PK：难度就是关卡。`加减起步`、`加减进阶`、`乘法启程` 等可以显示为不同训练对手。
- 卡牌对战：沿用现有关卡列表，继续展示区域、敌方、首通奖励和免费次数。
- 探索故事：场景节点就是关卡，遭遇战只在剧情推进到 encounter 时出现。

### 3.2 角色确认

- 数学 PK：左侧固定使用当前宠物，右侧使用分难度机器人。
- 卡牌对战：沿用选队，强化“当前上场”和“替补”的可见性。
- 探索故事：默认当前宠物出战，不再额外让孩子选队，避免打断故事节奏。

### 3.3 对战舞台

- 数学 PK：保留竞技台，题目区在中间，角色左右站位。
- 卡牌对战：保留训练场全屏竞技台，角色卡片信息更战术化。
- 探索故事：保留探索场景背景，战斗只是故事节点的一次遭遇。

### 3.4 行动选择

- 数学 PK：行动就是答题。答对触发宠物攻击，答错显示解释或失去机会。
- 卡牌对战：行动是普攻、重击、防御、必杀、道具、换人。
- 探索故事：行动是技能、道具、逃跑。数学谜题仍属于剧情事件，不直接混成卡牌按钮。

### 3.5 出招反馈

- 数学 PK：答对后宠物侧发光，中间显示“答对，宠物发动技能”；机器人抢先则机器人侧发光。
- 卡牌对战：每个技能有不同动效强度，必杀需要更明显的蓄力和命中反馈。
- 探索故事：沿用 `BattleFx`，但在 UI 上增加“当前回合 / 技能名 / 命中结果”的短提示。

### 3.6 HP / 进度变化

- 数学 PK：P1 可以先保留胜场比分；P2 再把 5 轮胜场视觉化成双方 HP。
- 卡牌对战：已有 HP 条，应继续强化血量数字、克制/冷却提示。
- 探索故事：已有 HP 条，但应补角色名和敌人名的清楚对应。

### 3.7 胜利结算

- 数学 PK：展示得分、连击、成长积分和“继续练习 / 再战 / 返回”。
- 卡牌对战：展示首通奖励、新卡、下一关、再战。
- 探索故事：展示掉落和剧情推进按钮，回到故事事件链。

## 4. P1 优化建议

P1 目标是“统一孩子能看懂的战斗感”，不重写底层规则。

### 数学 PK P1

- 保留现在的乘法练习场。
- 在正式 PK 中增加“答对 = 宠物出招”的文案和短动效。
- 把比分区从纯文字变成更像 HP/能量的 5 格进度。
- 机器人按难度继续使用不同角色。

### 卡牌对战 P1

- 关卡列表顶部增加“选关 -> 选队 -> 对战 -> 奖励”的轻量步骤条。
- 战斗顶部保留回合信息，但加强“当前操作方”。
- 战斗结算页突出新卡、下一关和再战，减少日志占比。

### 探索故事 P1

- 遭遇战进入时增加“敌人出现”登场卡，1 秒内自动进入战斗。
- HP 条旁补上宠物名和怪物名，避免孩子不知道哪条是谁。
- 行动按钮区增加一句短提示：`选择技能，帮宠物突破这一关`。
- 战斗结束后按钮文案回到故事语境：`继续探索`，而不是只写关闭。

## 5. P2 / P3 方向

P2：

- 数学 PK 增加 5 格 HP 风格比分。
- 卡牌对战增加属性克制提示和 Finish 动画。
- 探索故事增加按章节变化的敌人登场框。

P3：

- 三套 PK 都接入统一的“角色登场 / 胜利 / 失败”视觉资产规范。
- 根据宠物种类或卡牌稀有度生成轻量专属技能名。
- 家庭异步 PK 使用同一套结算语言，方便家长理解成绩。

## 6. 美术规则

- 不使用宝可梦原图、名字、UI 字体或明显可识别构图。
- 主角优先使用当前宠物和已有卡牌资产。
- 敌方角色按玩法独立：数学用机器人，卡牌用训练对手，探索用场景怪物。
- 角色图必须透明背景，不能有假棋盘格、白底、灰底。
- 背景服务玩法：数学是竞技台，卡牌是训练场，探索是故事场景。

## 7. 验收标准

- 孩子能在 3 秒内知道：我是谁、对手是谁、现在该做什么。
- 每次答对或出招都有角色反馈，不只是数字变化。
- HP、比分或进度变化可见。
- 结算页给出明确下一步。
- 三套玩法风格相通，但不会互相混淆。
