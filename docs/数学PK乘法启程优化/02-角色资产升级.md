# 数学 PK 角色资产升级

> 日期：2026-07-06
> 决策：左侧使用当前宠物；右侧使用分难度卡通机器人。
> 资产目标：替换 `human-kid.png` 和 `robot-rival.png` 的假透明棋盘格问题。

## 1. 当前问题

`js/math-pk.js` 目前硬编码：

- `assets/arena/human-kid.png`
- `assets/arena/robot-rival.png`

这两张图视觉上像透明背景，但实际是画出来的棋盘格，边缘 alpha 不透明。放在深色竞技台里会露出方格，不适合作为最终 UI 资产。

## 2. 左侧主角

左侧角色改为当前宠物：

1. 优先使用 `PetSystem.getCurrentStageImage()`。
2. 如果拿不到图片，使用 `assets/pets/poses/dog_idle.webp`。
3. 名字优先显示当前宠物物种名；拿不到则显示“我的宠物”。
4. 宠物图不重新生成，复用现有透明姿态图。

当前已验证示例：

- `assets/pets/poses/dog_idle.webp`：边缘 alpha 干净。
- `assets/pets/poses/cat_idle.webp`：边缘 alpha 干净。

## 5. 后处理流程

生成图不直接放进 `assets/arena/math-rivals/`。先经过：

1. 原图保存到临时目录，例如 `.tmp/math-rivals/raw/`。
2. 去除假透明棋盘格、白底或灰底。
3. 按 alpha bbox trim，保留 32-48px padding。
4. 导出 PNG，再转换 WebP。
5. 用脚本检查边缘 alpha。
6. 通过后发布到 `assets/arena/math-rivals/`。

## 6. 透明验收脚本口径

每张最终图需要满足：

```text
edge_opaque_px <= 8
alpha_bbox is not None
alpha_bbox is smaller than full canvas
replacement artifacts / checkerboard pattern not visible
```
