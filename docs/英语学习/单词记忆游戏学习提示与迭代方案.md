# 单词记忆游戏学习提示与迭代方案

> 适用范围：`prj/单词记忆射击场原型` 以及后续并入英语学习资料包后的单词游戏模块。核心目标不是把游戏做成纯射击，而是在移动、拾取、攻击和奖励之间，持续让孩子完成“看见英文、听见发音、理解中文、主动回忆”的闭环。

## 1. 当前问题

现在的俯视单词记忆游戏已经有完整的基础玩法：

- 地图移动
- 中文炸弹
- 英文目标
- 点击索敌
- 敌人追击
- 关卡解锁
- 词库分类和地图主题

但“学习单词”的提示还不够显性。孩子可能知道要点炸弹、点敌人，却不一定知道自己正在练哪个英文词，也可能只把中文炸弹当作游戏道具，没有完成英文、中文、声音和图片之间的连接。

所以后续要补的不是大段说明，而是轻量、即时、可重复触发的学习提示。

## 2. 推荐学习提示设计

### 2.1 单词提示按钮

建议在右上“当前中文球”卡片旁增加一个小按钮，例如 `?` 或 `听`：

- 未拿炸弹时：提示当前场上的一个重点词
- 拿到中文炸弹后：提示该中文对应的英文目标
- 点击后出现小浮层，不暂停或只轻暂停 `1-2` 秒

浮层内容：

```text
英文：hamster
中文：仓鼠
发音：点击播放
图片：当前敌人图 / 词卡图
小句：A hamster is small.
```

对 6 岁孩子，提示不宜一次给太多。默认显示英文、中文、发音和图；例句可以折叠或在后续关卡出现。

### 2.2 拿到炸弹后的即时提示

孩子点击中文炸弹后，主角头顶或右侧任务卡短暂显示：

```text
仓鼠 -> hamster
```

同时播放中文或英文发音。建议顺序是：

1. 第 1-2 关：中文发音优先，帮助理解当前拿到的炸弹。
2. 第 3 关后：英文发音优先，让孩子逐步把注意力转到英文。
3. 设置里允许家长选择 `中文优先 / 英文优先 / 中英都读`。

### 2.3 攻击前提示

拿到炸弹后，若孩子犹豫超过 `3-5` 秒，可以给一个轻提示：

```text
找 hamster
```

地图上的正确敌人可以轻微发光一次，但不要一直高亮。一直高亮会让游戏变成找亮点，降低记忆价值。

### 2.4 答对后的强化

命中正确目标后，不只显示“答对了”，而是做一次完整绑定：

```text
hamster
仓鼠
```

同时播放英文发音，并给星星或连击反馈。这个环节是最重要的记忆强化点，因为孩子刚完成主动选择，反馈会更容易进入记忆。

### 2.5 答错后的纠错

答错时不要重罚，建议：

```text
袋鼠不是 hamster
hamster 是仓鼠
```

配一个轻微停顿和正确目标闪一下。避免直接扣太多血，否则孩子会开始怕错，不敢尝试。

## 3. 学习流程闭环

每个单词在一局里尽量经历 4 个触点：

1. **预告**：进入地图块时，任务卡显示本图 3 个词。
2. **识别**：孩子看到英文目标和图片。
3. **匹配**：孩子拿中文炸弹，选择英文目标。
4. **强化**：答对后显示英文、中文、发音、星星。

后续更高级的关卡再增加第 5 个触点：

5. **回忆**：补全英文、输入首字母、选句型空格。

## 4. 后台可选单词库

### 4.1 目标

后续需要支持家长或老师选择词库，而不是所有孩子都从 3322 张大词库里随机抽。

### 4.2 推荐结构

先不要做复杂后台，建议分三步：

1. **本地词库配置**：继续使用 `data/vocab/word-memory-combined/views/all.json` 作为总仓。
2. **资料包视图**：增加 `starter / animals / minecraft / school / food / review` 等稳定视图。
3. **家长设置页**：允许选择本周启用哪些视图、每局词数、是否混入复习词。

### 4.3 词卡字段建议

```json
{
  "id": "word-hamster",
  "word": "hamster",
  "translation": "仓鼠",
  "category": "animals",
  "level": "starter",
  "image": "https://...",
  "localImage": "./assets/...",
  "audio": "./assets/voice/...",
  "example": "A hamster is small.",
  "tags": ["animal", "kid", "starter"]
}
```

### 4.4 对齐策略

- 根词库负责长期维护。
- 游戏项目只读发布后的视图文件。
- 原型可保留一份兼容导出，但不手工改词。
- 每次合词库后跑词库质量检查：重复词、缺中文、缺分类、图片不可用、音频缺失比例。

## 5. 艾宾浩斯随机复习

### 5.1 目标

不要只让孩子刷新词。真正记住单词需要在快忘的时候再遇到它。

### 5.2 单词状态

每个孩子、每个单词记录：

```text
new -> learning -> review -> mastered
```

建议字段：

```json
{
  "wordId": "word-hamster",
  "seenCount": 4,
  "correctCount": 3,
  "wrongCount": 1,
  "lastSeenAt": "2026-07-09T09:00:00+08:00",
  "nextReviewAt": "2026-07-10T09:00:00+08:00",
  "memoryLevel": 2
}
```

### 5.3 复习间隔

先用简单版本：

| 记忆等级 | 下次复习 |
| --- | --- |
| 0 | 10 分钟后 |
| 1 | 1 天后 |
| 2 | 3 天后 |
| 3 | 7 天后 |
| 4 | 15 天后 |
| 5 | 30 天后 |

答对则升级，答错则降级并更快复习。

### 5.4 游戏内混词比例

每局建议：

- `60%` 当前关卡新词
- `30%` 到期复习词
- `10%` 已掌握词，用来建立成就感

如果到期复习词不足，再用当前分类词补齐。

## 6. 难度提升路线

难度提升不要一开始上打字，否则会让“理解单词”和“键盘操作”同时变难。建议分层：

### 6.1 P0：看中文打英文目标

当前玩法：

```text
中文炸弹：仓鼠
英文目标：hamster
```

适合入门和低压力复习。

### 6.2 P1：首字母提示

命中前或提示按钮里只显示：

```text
h _ _ _ _ _ _
```

孩子可以按 `H` 选择目标，蓝牙键盘开始介入。

### 6.3 P2：单词补全

出现补全任务：

```text
ham_er
```

孩子输入缺失字母 `st`，再发射。

### 6.4 P3：听音选词

不显示中文，只播放英文发音，让孩子找对应图片或英文。

### 6.5 P4：句型填空

面向更后期：

```text
A ____ is small.
```

选项：`hamster / sword / lava`

这一步才真正开始从单词走向句子。

## 8. 统一语音策略

这条线现在不建议再拆成“英文一套、中文一套”。对原型页来说，最稳的是统一成一条播放解析链：

1. **本地静态 mp3**
2. **实时 TTS 服务**
3. **浏览器语音兜底**

### 8.1 为什么这样更合适

- 基础高频词提前生成本地 mp3，页面响应最快
- 中文和英文都走同一个页面端入口，后面切换 Edge-TTS、VoxCPM2、Cosy 类服务时，前端不用重写
- 离线或服务挂掉时，还能退回浏览器语音，不阻断孩子继续玩

### 8.2 页面端责任

游戏页只做三件事：

- 先查 `assets/voice/map.json`
- 没命中时请求统一 TTS endpoint
- endpoint 不可用时退回 `speechSynthesis`

不要在页面里硬编码“中文一定 edge，英文一定浏览器”这种分叉逻辑。

### 8.3 服务端责任

统一由 `prj/tts` 承接实时生成，服务内部再自己做引擎降级：

```text
VoxCPM2 -> edge-tts -> fallback JSON
```

页面不需要知道当前究竟用了哪个模型，只需要拿到可播音频，或拿到 `fallback:true` 后继续走浏览器语音。

### 8.5 体验护栏

- 本地开发环境可默认直连 `http://127.0.0.1:9885/tts`
- 远端 TTS 不应该无限等待，前端应设置短超时，再退回浏览器语音
- 对儿童游戏来说，“马上有声音”比“死等更自然的声音”更重要

### 8.4 后续落地方向

- 把 `单词记忆射击场原型`、`消灭苦力怕打字游戏`、后续学习机玩法都收口到同一个 TTS endpoint 配置方式
- 高频词继续沉淀成本地 mp3，降低联网依赖
- 中文提示语优先补 Edge-TTS / VoxCPM2 生成，减少浏览器中文发音偏硬的问题

### 7.2 奖励策略

不要一开始把所有皮肤给孩子。建议通过学习解锁：

- 通关第 3 关：解锁一个主角皮肤
- 连续 3 天复习：解锁一个炸弹特效
- 掌握 20 个动物词：解锁动物主题地图装饰
- 掌握 50 个 Minecraft 词：解锁 Minecraft 主题徽章

### 7.3 美术流水线

继续沿用当前项目的 GPT 生图 / Agnes / 透明 PNG 资产流程：

1. 先生成整体参考图。
2. 再生成无文字透明背景的多元素爆炸图。
3. 裁切、去空白、去棋盘格。
4. 保存到项目资产文件夹。
5. 网页按 manifest 引用。

## 8. UI 和流畅度优化

### 8.1 UI 原则

- 游戏台保持全屏。
- 说明文字尽量收进按钮、浮层和任务卡。
- 主信息只保留：当前关、星星、进度、护盾、当前中文炸弹。
- 学习提示用小浮层，不要做长说明侧栏。

### 8.2 操作流畅度

重点优化：

- 主角移动手感
- 敌人追击不要突然贴脸
- 炸弹飞行轨迹更明显
- 命中特效更爽
- 拿到炸弹后球跟随主角更稳定
- 提示按钮点击后不卡顿
- 语音播放不打断连续操作

### 8.3 儿童可理解性

每个界面只回答一个问题：

- 我现在拿着什么？`仓鼠`
- 我要打谁？`hamster`
- 我还差几个？`0 / 3`
- 我有没有答对？星星和发音反馈

不要同时展示大量分类、词库、后台概念给孩子。后台能力应该藏在家长设置里。

## 9. 分阶段路线图

### V1：学习提示闭环

目标：让当前游戏真的像单词学习。

- 增加提示按钮
- 拿炸弹后显示 `中文 -> 英文`
- 答对后显示英文、中文、发音
- 答错后给正确提示
- 增加本局重点词预告

实施状态：

- 已在 `prj/单词记忆射击场原型` 增加 `?` 提示按钮。
- 已支持提示卡显示英文、中文、图片和发音按钮。
- 已支持答对后的短暂强化卡。
- 已支持答错后的正确词提示。
- 已支持本局重点词预告：当前地图块显示 3 个英文 / 中文小词条，答对后打勾。
- 尚未做家长可配置的提示策略。

### V2：词库选择

目标：让家长能控制练什么。

- 增加稳定词库视图
- 增加分类/等级/主题筛选
- 支持每局词数和复习比例
- 文档化词卡格式和同步脚本

### V3：记忆曲线复习

目标：让孩子不断遇到快忘的词。

- 为每个词记录掌握状态
- 计算 `nextReviewAt`
- 每局混入到期复习词
- 增加复习完成奖励

### V4：难度进阶

目标：从认词过渡到拼写和句型。

- 首字母提示
- 单词补全
- 听音选词
- 句型填空
- 蓝牙键盘输入模式

### V5：套装和奖励生态

目标：长期吸引孩子回来。

- 背景套装
- 主角皮肤
- 炸弹特效
- 通关章
- 热门游戏灵感奖励皮肤
- 更顺滑的动画和音效

## 10. 推荐下一步

优先做 V1，不要先做后台。

原因：

- 目前最大缺口是“孩子是否知道自己在学哪个单词”。
- 提示按钮和答对强化成本低，收益高。
- 做完 V1 后再接词库和记忆曲线，数据记录才有意义。

建议下一次开发任务：

```text
给单词记忆射击场原型增加学习提示按钮：
点击后展示当前炸弹/目标的英文、中文、发音、图片；
答对后自动弹出 1 秒学习强化卡；
答错后提示正确词；
补自动化测试和 README。
```
