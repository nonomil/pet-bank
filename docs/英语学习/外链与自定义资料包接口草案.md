# 外链与自定义资料包接口草案

本草案用于给 `LearnCenter` 预留更通用的资料包来源能力，让当前项目后续既能接：

- 外部英语点读网站
- 当前仓库内置 lesson
- 家长自定义 JSON 资料包

而不需要每新增一种资料就重写一套学习中心。

## 1. 设计原则

### 1.1 积分、进度、资料来源分离

无论 lesson 内容来自哪里：

- 成长分只走当前项目的 `window.addGrowthPoints(delta)`
- 进度只走当前项目的 `petbank_learning_*`
- 资料来源只决定“打开哪里、渲染什么辅助信息”

也就是说：

- `来源` 不控制积分系统
- `来源` 不控制多孩子隔离
- `来源` 只负责 lesson 的内容入口和少量展示差异

### 1.2 pack 级别抽象，不把规则写死在某个网站

第一期虽然主要接 `mayihaoke`，但数据结构不应出现：

- “只支持 mayihaoke”
- “只支持 mcbook56”
- “只支持英语外链”

应该统一抽象为：

- pack 类型
- lesson 来源类型
- adapter 标识

## 2. 建议增加的 pack 元信息

### 2.1 catalog.json 层

每个 pack 除了当前已有字段，建议额外支持：

```json
{
  "id": "english-mc-hybrid-2026",
  "title": "Minecraft我的世界英语故事",
  "packType": "hybrid",
  "audience": "Minecraft英语故事",
  "coverEmoji": "🧩",
  "summary": "当前项目负责计划和积分，外部章节负责点读和阅读。",
  "sourceAdapter": "mayihaoke-reader",
  "capabilities": [
    "external-open",
    "manual-complete",
    "weekly-review",
    "custom-pack-ready"
  ]
}
```

### 2.2 manifest.json 层

建议扩展字段：

```json
{
  "id": "english-mc-hybrid-2026",
  "packType": "hybrid",
  "contentMode": "external-plus-local",
  "sourceAdapter": "mayihaoke-reader",
  "customPackEnabled": true,
  "rewardRules": {
    "story-chapter": 2,
    "starters-chapter": 2,
    "triple-streak": 1,
    "weekly-review": 5,
    "default": 1
  }
}
```

字段含义：

- `packType`
  - `internal`
  - `hybrid`
  - `external-gateway`
  - `custom`
- `contentMode`
  - `internal-only`
  - `external-only`
  - `external-plus-local`
- `sourceAdapter`
  - 指定 runtime 应该走哪套来源解析逻辑
- `customPackEnabled`
  - 明确当前学习中心是否允许这个 pack 被复制为自定义模板

## 3. 建议增加的 module 元信息

模块层建议支持：

```json
{
  "id": "mcbook56-story",
  "title": "Minecraft我的世界英语故事",
  "type": "external-reader",
  "adapterId": "mayihaoke-reader",
  "source": {
    "provider": "mayihaoke",
    "series": "mcbook56",
    "baseUrl": "https://mayihaoke.com/mcbook56/read",
    "routePattern": "https://mayihaoke.com/mcbook56/read/{chapterSlug}"
  }
}
```

模块 `type` 建议支持：

- `internal-reading`
- `internal-vocab`
- `review`
- `external-reader`
- `external-vocab`
- `custom-mixed`

## 4. Lesson 数据结构建议

### 4.1 通用 lesson 结构

```json
{
  "id": "chapter-01",
  "title": "第一章：新世界的开始",
  "summary": "点开外部故事页，完成 1 次听读或亲子共读。",
  "estimatedMinutes": 10,
  "rewardKey": "story-chapter",
  "tags": ["Minecraft英语", "故事点读"],
  "objectives": [
    "完整听读 1 遍",
    "尝试跟读 3 个词",
    "回到当前项目打勾"
  ],
  "source": {
    "kind": "external-chapter",
    "adapterId": "mayihaoke-reader",
    "provider": "mayihaoke",
    "chapterSlug": "chapter-1",
    "url": "https://mayihaoke.com/mcbook56/read/chapter-1"
  },
  "completion": {
    "mode": "manual-check",
    "points": 2
  },
  "support": {
    "keywords": ["hello", "world", "block"],
    "parentTip": "先听一遍，再跟读关键词，不要求逐句背。"
  },
  "customPayload": {}
}
```

### 4.2 自定义资料包 lesson 结构

自定义资料包应允许 `source.kind` 切换：

```json
{
  "source": {
    "kind": "custom-link",
    "label": "打开老师发来的共读页",
    "url": "https://example.com/custom-lesson"
  }
}
```

或者：

```json
{
  "source": {
    "kind": "internal-content"
  }
}
```

建议支持的 `source.kind`：

- `external-chapter`
- `external-link`
- `internal-content`
- `custom-link`
- `custom-mixed`

## 5. Runtime 预留接口建议

当前 `LearnCenter` 已有 pack 和 lesson 渲染能力，后续建议预留这些函数：

```javascript
resolvePackCapabilities(manifest)
resolveLessonSource(pack, module, lesson)
resolveLessonLaunchUrl(pack, module, lesson)
resolveLessonReward(pack, module, lesson)
renderLessonSupportCard(pack, module, lesson)
renderCustomPackExtras(pack, module, lesson)
```

### 5.1 这些接口分别做什么

- `resolvePackCapabilities(manifest)`
  - 判断这个 pack 是否支持外链、打印、家长提示、自定义扩展
- `resolveLessonSource(...)`
  - 统一解析 lesson 来源
- `resolveLessonLaunchUrl(...)`
  - 统一返回外部打开地址
- `resolveLessonReward(...)`
  - 允许 pack 级默认奖励和 lesson 级覆盖
- `renderLessonSupportCard(...)`
  - 在 lesson 页渲染“目标词 / 家长提示 / 外部入口”
- `renderCustomPackExtras(...)`
  - 给自定义 pack 预留扩展区

## 6. 当前项目需要保持不变的底层能力

以下底层能力不应因英语扩展而改变：

- `window.addGrowthPoints(delta)`
- `petbank_learning_progress`
- `petbank_learning_rewards`
- `petbank_learning_catalog_state`
- `ProfileManager` 的多孩子隔离逻辑

也就是说，新增的是：

- 来源层抽象
- lesson 类型抽象
- 展示层和动作层接口

而不是：

- 新积分系统
- 新进度系统
- 新账号系统

## 7. 自定义资料包的最低要求

未来如果要允许“家长自定义 pack”，建议最低要求是：

1. 必须有唯一 `pack.id`
2. 必须有 `manifest.json`
3. 至少有一个模块
4. 每个模块至少一节 lesson
5. 每节 lesson 必须可解析出：
   - 标题
   - 完成模式
   - 积分规则
   - 来源入口

## 8. 示例：外链英语包与自定义包如何共存

### 8.1 外链英语包

```json
{
  "id": "english-mc-hybrid-2026",
  "packType": "hybrid",
  "sourceAdapter": "mayihaoke-reader"
}
```

### 8.2 家长自定义包

```json
{
  "id": "custom-english-home-2026",
  "packType": "custom",
  "sourceAdapter": "custom-json-pack"
}
```

只要 runtime 先看 `packType` 和 `sourceAdapter`，同一个学习中心就能同时承载两种资料包。

## 9. 第一阶段实现建议

第一阶段不要一次把所有接口都做成复杂插件系统，推荐做到“可扩展但不过度设计”：

1. 先让 `manifest.packType`、`manifest.sourceAdapter` 生效。
2. 先支持 `source.kind = external-chapter`。
3. 先预留 `custom-link` 和 `internal-content` 的结构，但不要求全部做完 UI。
4. 先把 runtime 函数做成普通函数映射，而不是完整插件框架。

## 10. 这份接口草案的价值

它的价值不是“字段变多了”，而是避免以后出现这类返工：

- 英语扩展只能接一个网站
- 自定义资料包无法复用学习中心
- 每新增一个 pack，就得单独写一套页面和积分逻辑

有了这层接口，学习中心未来才能逐步变成真正可扩展的资料包平台。
