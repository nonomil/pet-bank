# 英语学习资料包文档包

本目录沉淀当前项目里已经落地的英语学习扩展方案。

截至 `2026-07-05`，英语部分已经不是“待规划”，而是已经作为学习中心里的一个可扩展资料包模块接入主站：

- 学习页上层入口卡已接入 `Minecraft我的世界英语故事`
- 资料包 id 为 `english-mc-hybrid-2026`
- 当前项目负责入口、计划、进度、积分与复盘
- 外部学习站负责章节点读 / 阅读主体
- 同时预留 `custom-entry-slot`，方便后续继续接更多英语网站或家长自定义资料

## 当前已落地范围

### 1. 学习页入口

- 英语资料包与中文资料包、学习网站入口包并列展示
- 学习入口卡片前置到学习页上方，孩子能更快找到英语入口
- 当前英语入口更偏“轻量起步”，先让幼小衔接暑假阶段敢听、敢跟读

### 2. 当前资料包结构

`english-mc-hybrid-2026` 当前包含 4 个模块：

1. `mcbook56-story`
2. `mcbookstarters-reader`
3. `english-weekly-review`
4. `custom-entry-slot`

其中前两项是英语章节主入口，第三项负责每周复盘，第四项专门留给后续扩展。

### 3. 当前计分规则

- `mcbook56-story`：每节 `+2`
- `mcbookstarters-reader`：每节 `+2`
- `english-weekly-review`：每次 `+5`
- 连续完成 3 个英语章节：额外 `+1`

英语资料包仍然沿用学习中心现有的防重复奖励机制，不额外开第二套积分系统。

### 4. 当前产品边界

这期没有做这些重方案：

- 没有整站搬运第三方英语内容进仓库
- 没有依赖跨域页面自动判断“是否读完”
- 没有把英语拆成一个独立新项目或独立新站点

当前选择的是更稳的“学习壳 + 外部阅读源”模式，既能接住英语学习，又能继续复用当前项目的积分与进度底座。

## 这套文档解决什么问题

1. 明确为什么英语学习继续挂在当前学习中心里，而不是拆成单独项目。
2. 明确为什么第一期采用“混合式资料包”，而不是纯外链或全内置。
3. 明确当前英语模块已经落地到什么程度，避免把已完成内容误判成“待规划”。
4. 明确后续如何继续扩到更多英语网站、老师链接和自定义资料包。

## 后续扩展建议

如果后面继续扩英语学习，建议沿当前数据结构往前走：

1. 先补一个真实可用的 `custom-entry-slot` 样例。
2. 再增加更多英语网站入口或老师分发链接。
3. 最后再考虑更重的 pack 类型，例如单词卡包、点读绘本包或校内同步复习包。
