# 混合式英语学习资料包方案

## 1. 目标

在当前 `pet-bank` 学习中心内，新增一套适合幼小衔接暑假的英语学习资料包，采用“当前项目做学习壳，外部网站做章节内容”的混合模式：

- 当前项目负责：
  - 英语资料包入口
  - 今日推荐
  - 章节计划
  - 打勾计分
  - 进度统计
  - 每周复盘
- 外部网站负责：
  - 点读页面
  - 章节阅读主体
  - 可能存在的配图、音频、互动内容

## 2. 为什么选混合式，而不是另外两种

### 2.1 不选纯外链

纯外链虽然快，但会带来几个明显问题：

- 学习过程几乎都发生在当前项目之外
- 当前项目只能做一个“导航页”
- 用户会觉得英语学习不是真正属于这个系统的一部分
- 后续“我的进度”里英语部分会显得很弱

### 2.2 不选全内置

全内置的体验最好，但第一阶段不适合直接做：

- 第三方内容复用和维护成本更高
- 自建整套英语内容制作量很大
- 当前项目已经有可复用的学习中心壳，没有必要第一步就重做内容平台

### 2.3 选混合式的核心原因

混合式刚好平衡三件事：

1. 孩子仍然在当前项目里选课、看计划、拿积分。
2. 真正的章节阅读可以复用已经存在的优质外部内容。
3. 后面可以很自然升级成“部分内置 + 部分外链 + 自定义资料包”。

## 3. 当前观察到的外部站点形态

截至 `2026-07-05`，对 `mayihaoke.com` 的静态检查显示：

- 站点是 SPA 前端壳
- 根页面和 `/mcbook56/read` 返回的是统一 HTML 壳
- 前端 bundle 内可见以下路由线索：
  - `/mcbook56/read/:chapter?`
  - `/mcbookstarters/read/:chapter?`
  - `/mcbook78/read/:chapter?`
  - `/cambridge-vocab`

这说明它更适合被当作：

- 章节阅读源
- 分级英语阅读入口
- 英语故事点读外链资源

而不适合在第一阶段被“整站内容采集后重建”。

## 4. 第一阶段产品形态

建议新资料包命名为：

- `幼小衔接英语启蒙资料包`

建议数据 id：

- `english-mc-hybrid-2026`

### 4.1 包含的模块

第一阶段推荐 4 个模块：

1. `我的世界英语故事`
   - 来源优先：`mcbook56`
   - 目标：每天读 1 个故事章节
2. `Starters 英语启蒙`
   - 来源优先：`mcbookstarters`
   - 目标：每天读 1 个适合启蒙的章节或词汇页
3. `每周英语复盘`
   - 当前项目内置轻量复盘页
   - 家长可记录：本周最熟的词、最爱的一章、下周继续读什么
4. `自定义资料入口`
   - 不是立即开放给孩子
   - 先预留为后续 pack 类型扩展位

### 4.2 单节学习路径

单节课的推荐流程：

1. 在当前项目打开英语资料包
2. 进入某一节英语 lesson
3. 在 lesson 页看到：
   - 本节标题
   - 难度/年龄段
   - 预计时长
   - 建议目标词
   - 打开外部点读页按钮
   - 读完打勾按钮
4. 点击“打开点读页”跳到外部章节
5. 阅读或点读完成后，回到当前项目
6. 点击“读完打勾”
7. 当前项目发成长分、记录进度、更新“我的进度”

### 4.3 为什么不做自动判定“读完”

第一阶段不建议实现自动读完判定，原因很明确：

- 外部网站跨域，当前项目无法稳定读取真实阅读状态
- iframe 是否稳定可用仍需浏览器实测
- 即便能嵌入，也不等于能可靠判断孩子真的读完

因此第一期采用：

- `手动打勾`
- `当前项目负责防重复发分`
- `后续可增加家长确认、最短停留时长、章节问答等轻量校验`

## 5. 积分建议

建议和现有中文学习体系保持同一量级，避免英语模块一上来破坏成长分平衡。

### 5.1 建议规则

- 每完成 1 个英语章节：`+2 分`
- 每连续完成 3 个英语章节：额外 `+1 分`
- 每周英语复盘完成：`+5 分`

### 5.2 这样设计的原因

- `+2` 与当前中文晨读一致，容易理解
- 连续 3 节再奖励 `+1`，能鼓励连贯学习，但不至于刷分过快
- 周复盘 `+5` 保持和“家长参与”价值相匹配

## 6. 页面设计建议

### 6.1 学习中心首页

学习中心首页新增一个英语区卡片：

- 英语今日推荐
- 今日建议打开的章节
- 最近英语累计积分
- 继续学习按钮

### 6.2 资料包页

资料包页应包含：

- 包简介
- 适合年龄
- 模块卡片
- 当前完成率
- 家长提示
- 进入计划 / 打印 / 继续学习

### 6.3 Lesson 页

lesson 页保留当前中文学习中心已经跑通的交互逻辑，只调整为英语语境：

- 目标词卡片
- 跟读提醒
- 外链章节按钮
- 读完打勾按钮
- 家长提示

### 6.4 进度页

“我的进度”中新增英语进度统计：

- 总完成章节数
- 英语累计成长分
- 本周英语学习次数
- 最近完成记录

## 7. 自定义资料包能力为什么现在就要预留

如果当前实现只围绕 `mayihaoke` 写死，后面会立刻遇到这些问题：

- 不能接入别的英语网站
- 不能接入家长自己整理的 PDF / 网页 / 图片资料
- 不能做校内英语同步复习包
- 不能做 KET / Starters / Movers 的不同层级包

所以第一阶段就要把“资料包来源”抽象成可切换模式：

- 外链章节型
- 内置内容型
- 自定义混合型

## 8. 阶段路线

### Phase 1：文档与接口

- 明确混合式英语资料包设计
- 明确外链 lesson 和自定义 pack 的接口合同
- 形成 implementation plan

### Phase 2：英语资料包 MVP

- 新增英语资料包卡片
- 新增英语计划、lesson、进度
- 接入 `mcbook56` / `mcbookstarters` 外链
- 完成打勾计分

### Phase 3：自定义资料包样例

- 增加一个 `custom-demo-pack`
- 验证同一 LearnCenter 能同时承载：
  - 外链章节包
  - 自定义 JSON 包

### Phase 4：增强能力

- 家长备注
- 自定义封面图
- 打印词汇卡
- 简单词汇问答
- 音频和点读辅助

## 9. 第一阶段明确不做

- 不采集外站全文进仓库
- 不做外站内容自动解析和重建
- 不做云端同步
- 不做自动判断是否真的读完
- 不做开放式用户上传后台

## 10. 成功标准

英语学习第一阶段成功，不看“内容做了多少”，而看这几件事是否成立：

1. 英语资料包能像中文资料包一样进入学习中心。
2. 孩子可以在当前项目里进入某一节英语学习。
3. 可以从当前项目打开外部章节。
4. 回来后可以打勾，并稳定获得积分。
5. 进度与积分能够进入现有“我的进度”和主成长系统。
6. 代码层没有把学习中心绑死在某一个外部站点上。
