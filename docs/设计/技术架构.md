# 技术架构

> 本文为「成长伙伴 · 萌宠冒险岛」`pet-bank` 的当前技术架构总览，描述技术栈、运行时分层、目录结构、数据组织、部署方式，以及 2026-07-05 新增的“学习中心”产品域如何接入现有项目。
>
> **源码依据**：`index.html`、`js/*.js`、`css/*.css`、`data/**/*.json`、`scripts/learning-center-smoke.mjs`

---

## 1. 技术栈

项目仍然坚持**纯前端、零构建、源码即产物**路线，目标是“浏览器打开即用，家长可本地部署，单仓库持续扩展”。

| 层次 | 当前选型 | 说明 |
|---|---|---|
| 入口 | `index.html` 单入口 | SPA 页面骨架、导航、容器、脚本顺序都在这里定义 |
| 样式 | Tailwind CSS CDN + 本地 CSS | 当前 `css/` 下 11 份样式表，覆盖主站、游乐场、汉字、排行榜、学习中心等专题 |
| 脚本 | Vanilla JS + IIFE 全局模块 | `index.html` 当前直接加载 26 个本地脚本，无打包器、无 ES Module |
| 图标 / 字体 | Lucide CDN、Google Fonts | 继续保持轻量外链依赖 |
| 数据 | `data/**/*.json` | 静态内容、题库、宠物库、学习资料包都走 JSON 驱动 |
| 持久化 | `localStorage` | 所有运行态统一写入 `petbank_*` 前缀键 |
| 部署 | GitHub Pages / 任意静态站 | 本地也可用 `python -m http.server 8000` 预览 |

**关键约束：**

- 无强制后端依赖，学习中心本身可完全离线在本地数据上运行
- 无 npm 构建步骤，新增能力需兼容脚本串行加载模型
- 所有新能力优先通过 `data/*.json + localStorage + window.*` 契约接入

## 2. 单页应用外壳

### 2.1 页面与导航

当前项目是一个**单页应用（SPA）**，`index.html` 中共有 **22 个 `.page` 页面容器**，顶部有 **6 个一级导航 tab**：

| 一级导航 | 主要承载页面 |
|---|---|
| `首页` | `map` |
| `积分` | `today` / `review` / `reward` / `shop` / `inventory` |
| `学习` | `learn` / `learn-pack` / `learn-plan` / `learn-lesson` / `learn-print` |
| `宠物` | `pet` / `home` / `card` |
| `探索` | `explore` |
| `游乐场` | `playground` / `mathpk` / `hanzi` / `leaderboard` |

此外还保留 `works`、`tools`、`settings` 等次级页面。

### 2.2 路由机制

页面切换仍由 `js/app.js` 中的 `switchPage(page)` 统一负责：

1. 关闭当前 `.page.active`
2. 激活目标 `#page-${page}`
3. 依据 `PAGE_TO_TAB` 同步一级导航高亮
4. 触发对应模块的渲染钩子

学习中心的 5 个页面已接入该机制：

- `learn` -> `LearnCenter.renderHub()`
- `learn-pack` -> `LearnCenter.renderPack()`
- `learn-plan` -> `LearnCenter.renderPlan()`
- `learn-lesson` -> `LearnCenter.renderLesson()`
- `learn-print` -> `LearnCenter.renderPrint()`

## 4. 学习中心如何接入现有系统

“学习中心”不是独立仓库，也不是第二个 SPA，而是当前项目内新增的一个**产品域**：

```text
任务 / 学习
   ↓
成长积分（window.addGrowthPoints）
   ↓
宠物 / 探索 / 商店 / 图鉴
   ↓
多孩子档案（ProfileManager 快照隔离）
```

### 4.1 数据入口

学习中心使用独立的 `data/learn/` 目录：

```text
data/
└── learn/
    ├── catalog.json
    └── packs/
        └── summer-chinese-bridge-2026/
            ├── manifest.json
            ├── plan.json
            ├── char-library.json
            └── modules/
                ├── morning-reading.json
                ├── literacy-45days.json
                ├── weekly-review.json
                ├── poems.json
                └── classics.json
```

### 4.2 进度与发分

学习中心新增 4 个 `localStorage` 键：

- `petbank_learning_catalog_state`
- `petbank_learning_progress`
- `petbank_learning_rewards`
- `petbank_learning_print_prefs`

因为它们仍然使用 `petbank_*` 前缀，所以会自动被 `ProfileManager` 纳入多孩子切换快照，实现“每个孩子各学各的、互不串档”。

### 4.3 与主积分的关系

学习中心不自建第二套积分账本，统一走：

```javascript
window.addGrowthPoints(delta)
```

当前资料包奖励规则：

- `60 天晨读`：每节 `+2`
- `45 天识字`：每节 `+3`
- `每周复盘`：每节 `+5`
- 同一天完成晨读 + 识字：额外 `+1`

## 5. 数据架构

### 5.1 三层载体

```text
静态 JSON（data/**/*.json）
        ↓ fetch
运行时内存（各 IIFE 模块 state）
        ↓ localStorage
浏览器本地持久化（petbank_*）
```

### 5.2 当前主要数据目录

| 目录 / 文件 | 作用 |
|---|---|
| `data/pets.json` | 宠物库 |
| `data/scenes.json` | 场景与怪物配置 |
| `data/items.json` | 物品定义 |
| `data/skills.json` | 技能配置 |
| `data/point-items.json` | 积分加减项目 |
| `data/hanzi-questions.json` / `data/hanzi-hsk.json` | 汉字玩法题库 |
| `data/learn/**` | 学习资料包平台与内容 |

### 5.3 localStorage 命名策略

运行态键统一用 `petbank_*` 前缀，当前主要分为 4 类：

1. 成长主循环：积分、任务、宠物、背包、场景、商店、宝箱
2. 多孩子档案：`petbank_profiles_meta`、`petbank_active_profile`、`petbank_profile_data_{id}`
3. 学习中心：`petbank_learning_*`
4. 游乐场与学习玩法：`petbank_hanzi_*`、`petbank_lb_*` 等

## 7. 部署与验证

### 7.1 部署

- 线上默认部署：GitHub Pages
- 本地预览：`python -m http.server 8000`

因为大量数据通过 `fetch()` 加载，**不能直接双击 `index.html` 用 `file://` 打开**。

### 7.2 验证

学习中心当前有专用冒烟脚本：

```bash
python scripts/generators/gen_summer_chinese_pack.py
node scripts/learning-center-smoke.mjs
```

该脚本覆盖：

- 学习导航与页面壳
- 资料包加载
- 60 天晨读 / 45 天识字数量
- lesson 完成与一次性发分
- 同日组合奖励
- 打印页渲染
- 多孩子隔离
