## B. 关键接口契约

### B1. 页面路由契约

`js/app.js` 通过 `PAGE_TO_TAB` + `switchPage(page)` 管理页面切换。学习中心当前接入的页面 key 为：

- `learn`
- `learn-pack`
- `learn-plan`
- `learn-lesson`
- `learn-print`

对应渲染钩子：

```javascript
LearnCenter.renderHub('learn-container');
LearnCenter.renderPack('learn-pack-container');
LearnCenter.renderPlan('learn-plan-container');
LearnCenter.renderLesson('learn-lesson-container');
LearnCenter.renderPrint('learn-print-container');
```

### B2. 主积分契约

所有跨域玩法统一通过 `app.js` 导出的接口回写成长积分：

```javascript
window.addGrowthPoints(delta)
window.spendPoints(n)
window.deductGrowthPoints(n)
```

学习中心只使用 `window.addGrowthPoints(delta)` 发放奖励，不自建第二套积分系统。

### B3. 多孩子隔离契约

`ProfileManager` 使用“快照切换”模型：

- 当前孩子活跃时，业务模块继续直接读写 `petbank_*`
- 切换孩子时，`ProfileManager` 将整套 `petbank_*` 业务键存入 `petbank_profile_data_{id}`
- 再恢复目标孩子的快照

因此学习中心只要继续使用 `petbank_learning_*` 前缀，就会自动获得多孩子隔离。

### B4. LearnCenter 对外接口

`js/learn-center.js` 当前导出：

```javascript
window.LearnCenter = {
  init,
  getCatalog,
  getPack,
  getModule,
  loadAllModules,
  renderHub,
  renderPack,
  renderPlan,
  renderLesson,
  renderPrint,
  openPack,
  openLesson,
  openPlan,
  openPrint,
  completeLesson
};
```

职责边界：

- 负责学习目录、资料包、lesson 与打印页渲染
- 负责学习进度持久化与奖励防重复
- 不负责主积分存储本身
- 不负责多孩子切换逻辑本身

### B5. 学习中心发分规则

当前 `summer-chinese-bridge-2026` 资料包配置为：

| 模块 | 单次奖励 |
|---|---|
| `morning-reading` | `+2` |
| `literacy-45days` | `+3` |
| `weekly-review` | `+5` |
| 同日晨读 + 识字组合 | 额外 `+1` |

奖励唯一键格式：

```text
{packId}:{moduleId}:{lessonId}
{packId}:daily-bundle:{lessonId}
```

## C. 数据合同

### C1. 核心静态数据

| 文件 | 用途 |
|---|---|
| `data/pets.json` | 宠物库、稀有度、形态、来源 |
| `data/scenes.json` | 探索场景、怪物、掉落、解锁门槛 |
| `data/items.json` | 物品定义 |
| `data/skills.json` | 技能定义 |
| `data/point-items.json` | 积分加减项目 |
| `data/hanzi-questions.json` / `data/hanzi-hsk.json` | 汉字玩法题库 |

### C2. 学习中心数据目录

```text
data/learn/
├── catalog.json
└── packs/
    └── summer-chinese-bridge-2026/
        ├── manifest.json
        ├── plan.json
        ├── char-library.json
        └── modules/
            ├── morning-reading.json
            ├── literacy-45days.json
            ├── weekly-review.json
            ├── poems.json
            └── classics.json
```

### C3. `catalog.json`

总目录，声明学习中心当前有哪些资料包。

最小结构：

```json
{
  "packs": [
    {
      "id": "summer-chinese-bridge-2026",
      "title": "幼小衔接暑假中文资料包"
    }
  ]
}
```

### C4. `manifest.json`

单资料包清单与奖励规则。

关键字段：

- `id`
- `title`
- `audience`
- `description`
- `rewardRules`
- `modules[]`

### C5. `plan.json`

周计划与每日节奏。

关键字段：

- `summary`
- `dailyRoutine[]`
- `weeks[]`

### C6. `modules/*.json`

每个学习模块一个 JSON，内部以 `lessons[]` 存放条目。

当前模块类型：

- `morning-reading`
- `literacy-45days`
- `weekly-review`
- `poems`
- `classics`

## E. 当前验证基线

学习中心新增后，当前最直接的回归脚本是：

```bash
python scripts/generators/gen_summer_chinese_pack.py
node scripts/learning-center-smoke.mjs
```

冒烟脚本当前覆盖：

1. 学习导航与页面壳存在
2. 学习中心首页能看到资料包
3. `60` 天晨读与 `45` 天识字数量正确
4. lesson 完成会写入 `petbank_learning_progress`
5. 奖励只发一次
6. 同日晨读 + 识字组合奖励生效
7. 打印页存在 A4 友好容器
8. 切换孩子后学习进度隔离

---

*后续若继续新增资料包，优先保持“新 pack 只新增 `data/learn/packs/*` 数据和少量展示文案，不改动主积分与档案底座”的扩展方式。*
