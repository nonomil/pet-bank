> 这份文档用于快速回答 5 个问题：项目是什么、现在做到哪了、还有什么没做、接下来准备做什么、哪些事情暂时不做。

---

## 1. 一句话定位

`pet-bank`（**成长伙伴 · 萌宠冒险岛**）是一个面向家庭/课堂陪伴场景的 **“成长积分 + 宠物养成 + 探索冒险” 纯前端静态网页**。

它的核心不是“记账”，而是把孩子每天的任务打卡，转成一个能持续反馈的成长循环：

**任务打卡 / 数学 PK → 获得积分 → 宠物成长 → 探索与战斗 → 背包 / 商店消费 → 回到宠物小屋继续养成**

---

## 2. 当前快照

| 维度 | 当前状态 |
|---|---|
| 产品形态 | 纯前端 SPA，GitHub Pages 静态部署 |
| 技术路线 | `HTML + Tailwind CDN + Vanilla JS + localStorage + data/*.json` |
| 一级导航 | **5 个主 tab**（积分 / 宠物 / 探索 / 兑换 / 更多），13 个页面收纳于各 hub 内（不删页） |
| 功能模块 | **12 个功能模块**（任务、宠物、宠物小屋、探索、战斗、背包、商店、宝箱、遛弯、数学 PK、图鉴、工具箱） |
| 当前主链路 | 可演示：任务 → 积分 → 宠物 → 探索 → 战斗 → 奖励 → 背包 / 商店 |
| 当前阶段 | **M0 基本收口完成**，已从“铺模块”进入“做产品收口与深化”阶段 |

## 4. 还没做完的

这些是当前最明确、最值得继续推进的未完成项。

### 4.1 活跃缺口

- ~~商店家具联动未打通~~ **已打通（M0.5）**：商店「家园装饰」分类购买家具 → 写入 `petbank_home_furniture` → 小屋固定槽位摆放 → 替换 / 收回 → 刷新持久化，纯装饰闭环已落地。
- ~~战斗深度不足~~ **已完成基础深化（v0.3.9）**：技能面板、道具快捷栏、CD、防御减伤已落地；当前不再是优先缺口。
- ~~探索故事治理不足~~ **已收口（R1 单一源）**：`data/stories/*.json` 为唯一正式故事源，`exploration-detail.js` 移除 140 行硬编码兜底与 `SCENE_ENDING`，`_loadStories` 读 `events`+`ending_text`，加载失败明确提示。
- ~~数学场景化不足~~ **已完成（R3 数学场景化）**：12 场景 `math` 事件改为固定场景题（森林蘑菇加减 / 瀑布露珠剩余 / 雪山冰柱乘 / 洞穴水晶除 / 太空等差规律 / 星空花园斐波那契等），`genMathQuestion` 优先 `event.question`，统一 galgame-word 样式。
- **盲盒奖池仍未完全数据化**：`shop.js` 里的盲盒奖池仍是内置常量，没完全统一到 `items.json`。
- **探索端到端验证不足**：虽然功能能跑，但缺少稳定的端到端冒烟与回归清单。
- **角色口径仍有历史漂移**：统一口径 `147`（历史 `139` 已修正），文档表达持续统一中。

### 4.2 非阻断尾项

- ~~宠物小屋和商店的 AC-19 / AC-20 仍未兑现~~ **已完成基础闭环**：商店购买家具 → 写入 ownership → 小屋摆放 / 替换 / 持久化 已打通；后续不再是“是否联动”，而是是否继续扩展更深层家园玩法。
- 家具系统当前已完成“纯装饰闭环”；后续如继续推进，也只建议做轻量体验打磨，不建议直接上 buff、拖拽摆放或多房间。
- 生病机制目前只有“倒下 / 救援”闭环，还没有独立的长期病症系统。

## 5. 准备做什么

按“产品级、不过度设计、刚好满足需求”的标准，推荐优先级如下：

### 5.1 推荐顺序

> ✅ **更新（2026-06-30）**：① 商店家具联动、② 首页 Tab 收口、③ 战斗深化、④ 探索叙事收口（R3 数学场景化 + R1 单一源 + R4 去模板 + voice 调整）均已落地，当前推荐下一步切换为“数据一致性与系统性回归验证”。

1. **商店家具联动**（✅ 已落地）
   - 这是最自然的产品收口项。
   - 也和 [banchong 分析](./参考/banchong宠物互动分析与合入方案.md) 的结论一致：先把装饰经济做成闭环，不要先扩战斗深度。
   - 能把积分消费、小屋、家具摆放真正串成闭环。
   - 改动范围相对独立，风险比战斗深化更低。
   - 本轮已补齐正式文档： [plans/README](./plans/README.md)、[设计稿](./plans/2026-06-29-shop-furniture-linkage-design.md)、[实施计划](./plans/2026-06-29-shop-furniture-linkage-implementation.md)。

2. **首页 Tab 收口**（✅ 已落地）
   - 先把一级导航压到 `积分 / 宠物 / 探索 / 兑换 / 更多`。
   - 不删功能，只做入口重组，让主线更清楚。
   - 本轮已补齐正式文档： [设计稿](./plans/2026-06-29-home-tab-navigation-design.md)、[实施计划](./plans/2026-06-29-home-tab-navigation-implementation.md)。

3. **探索叙事收口与内容治理**（✅ 已完成）
   - ✅ 12 场景数学场景化 + 5 章教育目标（`data/stories/README.md` 目标表：ch1 number_sense / ch2 add_sub_apply / ch3 life_apply / ch4 mul_div_mech / ch5 pattern_logic）。
   - ✅ 故事单一源收口（`data/stories/*.json`）、结束文案并入场景数据（`ending_text` 顶层）、文档同步。
   - ✅ 遭遇/对手去模板化（12 场景章内专属对手 + 36 怪物立绘）、choice 即时环境反馈（6 场景）、死代码清理（删 `disableEventActions`）。
   - ✅ 语音调整：`VoiceSystem.AUTOPLAY_RULES` 仅保留 `#galgameText`（探索 galgame 对话），其他模块（battleLog/walk/math/toast）不播报。

4. **数据一致性与回归验证**
   - 统一盲盒奖池与 `items.json`
   - 梳理探索奖励与图鉴触发
   - 补冒烟 / 回归检查清单

## 6. 暂停 / 延后

这些不是“不要做”，而是**当前阶段不建议投入**。

### 6.1 延后项

- 多孩子账户 / 多档案切换
- 云同步 / 后端化 / 登录体系
- PWA 离线安装
- 数据导出 / 导入
- 成长报告 PDF
- AI 任务推荐 / 家长社区
- 更完整的宠物疾病、繁殖、社交体系

### 6.2 已停止项

- **暂无明确废弃项**

当前更准确的说法不是“停了很多”，而是：
**项目已经从大铺功能，切换到了“优先收口核心体验，延后大平台能力”的阶段。**
