# 项目现状总览

> 这份文档用于快速回答 5 个问题：项目是什么、现在做到哪了、还有什么没做、接下来准备做什么、哪些事情暂时不做。
>
> **更新时间**：2026-07-05
> **适用场景**：重新接手项目、判断下一步优先级、和需求/代码/进度文档做快速对照

---

## 1. 一句话定位

`pet-bank`（**成长伙伴 · 萌宠冒险岛**）是一个面向家庭/课堂陪伴场景的 **“成长积分 + 学习资料包 + 宠物养成 + 探索冒险” 纯前端静态网页**。

它的核心不是“记账”，而是把孩子每天的任务打卡，转成一个能持续反馈的成长循环：

**任务打卡 / 学习资料包 / 数学 PK → 获得积分 → 宠物成长 → 探索与战斗 → 背包 / 商店消费 → 回到宠物小屋继续养成**

---

## 2. 当前快照

| 维度 | 当前状态 |
|---|---|
| 产品形态 | 纯前端 SPA，GitHub Pages 静态部署 |
| 技术路线 | `HTML + Tailwind CDN + Vanilla JS + localStorage + data/*.json` |
| 一级导航 | **6 个主 tab**（首页 / 积分 / 学习 / 宠物 / 探索 / 游乐场），共 22 个页面容器 |
| 功能模块 | **主循环 + 学习 + 游乐场 + 档案壳** 已形成稳定运行时，包含学习中心、汉字、排行榜、多孩子本地切换等能力 |
| 当前主链路 | 可演示：任务 / 学习 → 积分 → 宠物 → 探索 → 奖励 → 背包 / 商店 |
| 当前阶段 | **M0 基本收口完成**，已从“铺模块”进入“做产品收口与深化”阶段 |

## 4. 还没做完的

这些是当前最明确、最值得继续推进的未完成项。

### 4.1 活跃缺口

- ~~商店家具联动未打通~~ **已打通（M0.5）**：商店「家园装饰」分类购买家具 → 写入 `petbank_home_furniture` → 小屋固定槽位摆放 → 替换 / 收回 → 刷新持久化，纯装饰闭环已落地。
- ~~战斗深度不足~~ **已完成基础深化（v0.3.9）**：技能面板、道具快捷栏、CD、防御减伤已落地；当前不再是优先缺口。
- ~~探索故事治理不足~~ **已收口（R1 单一源）**：`data/stories/*.json` 为唯一正式故事源，`exploration-detail.js` 移除 140 行硬编码兜底与 `SCENE_ENDING`，`_loadStories` 读 `events`+`ending_text`，加载失败明确提示。
- ~~数学场景化不足~~ **已完成（R3 数学场景化）**：12 场景 `math` 事件改为固定场景题（森林蘑菇加减 / 瀑布露珠剩余 / 雪山冰柱乘 / 洞穴水晶除 / 太空等差规律 / 星空花园斐波那契等），`genMathQuestion` 优先 `event.question`，统一 galgame-word 样式。
- **盲盒奖池仍未完全数据化**：`shop.js` 里的盲盒奖池仍是内置常量，没完全统一到 `items.json`。
- **探索端到端验证不足**：虽然功能能跑，但缺少稳定的端到端冒烟与回归清单。
- **路线地图仍有体验层优化空间**：当前探索地图实际仍是手调散点版；路线地图专题已完成文档收口，确认下一步只做“轻量多圈层重构”，不做骰子/逐格/第二积分循环。
- **角色口径仍有历史漂移**：统一口径 `147`（历史 `139` 已修正），文档表达持续统一中。
- **学习中心第一套资料包虽已可用，但内容仍有继续打磨空间**：当前数据包已完整可学，后续可以继续细化晨读文案、识字节奏、每周复盘提示与更多资料包模板。

### 4.2 非阻断尾项

- ~~宠物小屋和商店的 AC-19 / AC-20 仍未兑现~~ **已完成基础闭环**：商店购买家具 → 写入 ownership → 小屋摆放 / 替换 / 持久化 已打通；后续不再是“是否联动”，而是是否继续扩展更深层家园玩法。
- 家具系统当前已完成“纯装饰闭环”；后续如继续推进，也只建议做轻量体验打磨，不建议直接上 buff、拖拽摆放或多房间。
- 生病机制目前只有“倒下 / 救援”闭环，还没有独立的长期病症系统。

## 5. 准备做什么

按“产品级、不过度设计、刚好满足需求”的标准，推荐优先级如下：

### 5.1 推荐顺序

> ✅ **更新（2026-07-05）**：学习中心与“幼小衔接暑假中文资料包”已经完成落地实现，当前建议从“做学习中心”转为“打磨资料包质量 + 继续扩展第二套资料包 / 学习入口”。

1. **学习中心内容打磨与资料包扩展**
   - 当前框架已经成立，后面不需要再重搭底座。
   - 下一步更值得做的是：
     - 优化首套资料包文案与节奏
     - 新增第二套资料包（如拼音、英语晨读、数学口算）
     - 补充“学习网站入口型资料包”
   - 现有底座：
     - `学习` 一级入口
     - `60 天晨读`
     - `45 天识字`
     - `每周复盘`
     - `A4 打印友好页`
     - `学习进度 + 一次性积分发放 + 多孩子隔离`

2. **盲盒奖池与数据一致性**
   - 统一盲盒奖池与 `items.json`
   - 梳理探索奖励与图鉴触发
   - 避免学习中心接入后，整体数据源继续分裂

3. **学习中心回归验证常态化**
   - `scripts/learning-center-smoke.mjs` 已补齐并通过。
   - 后续新增资料包时，继续沿用这条回归链路，避免“内容加了，发分/隔离/打印页坏了”。

4. **遛弯页细节体验继续打磨**
   - 当前大结构已经收口完成，后续重点应放在体验层细节：
     - 场景互动反馈和气泡节奏
     - 好友邀约文案与状态提示
     - 路线记录的可读性和复盘感
   - 不建议再回到“大说明区 + 多面板平铺”的重布局。

5. **路线地图轻量多圈层重构**
   - 这是探索入口的体验层优化，不是新玩法系统。
   - 目标是把首页 + 探索页共用地图做得更清晰、更有章节推进感。
   - 当前已完成需求 / 方案 / 实施计划 / CC handoff 文档收口。
   - 如果下一轮希望优先做“用户第一眼感知到的体验提升”，可以直接执行这包：
     [路线地图专题索引](./路线地图大富翁融合/README.md)。

## 6. 暂停 / 延后

这些不是“不要做”，而是**当前阶段不建议投入**。

### 6.1 延后项

- 云同步 / 后端化 / 登录体系全面闭环
- 外部学习网站深度嵌入或 iframe 化
- 专用 PDF 生成管线
- 成长报告 PDF / 家长周报
- AI 任务推荐 / 家长社区
- 更完整的宠物疾病、繁殖、社交体系

### 6.2 已停止项

- **暂无明确废弃项**

当前更准确的说法不是“停了很多”，而是：
**项目已经从大铺功能，切换到了“优先收口核心体验，延后大平台能力”的阶段。**
