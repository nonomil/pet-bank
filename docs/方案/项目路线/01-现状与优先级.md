# 现状与优先级

> 当前基线：2026-07-12。历史扫描中的 Supabase/旧账号描述已失效；具体证据和已落地项见 [07-当前扫描与优化方案](./07-当前扫描与优化方案-2026-07-12.md)。

## 产品定位

这是一个本地优先的家庭成长 H5：任务与积分是入口，宠物/小屋提供长期反馈，学习中心承载语文、英语、数学内容，探索、卡牌、PK 和多个小游戏承担练习与娱乐。静态发布低成本和离线可用性是当前架构的有效优势。

当前不是“完整家庭云平台已上线”：浏览器不加载 Supabase/旧账号社交运行时；自托管 API 已提供认证、家庭、邀请、孩子和 revision 快照，家长设置页已接入账号/家庭/孩子管理，但孩子状态自动快照同步尚未接入。

## 已确认的优先级

### 已落地：不重复建设

- 每日任务和日宝箱已经由 `PetBankDailyState` 统一读写，并有损坏存储、跨日和 Profile 测试。
- 英语词汇进度/奖励已经有显式 profile 后缀及旧键一次性迁移。
- 游戏 receipt、核心奖励策略、奖励反馈和任务事件日志已经存在。
- 本地-only 运行门禁、回归 runner 完整性、静态路由和 Pages fast gate 已接入。
- 家长端已接入自托管注册/登录、家庭邀请码和孩子映射；快照 API 已有端到端合同，但尚未接入 Profile 自动 push/pull。

### P0：现在先做

1. **Storage Registry 审计**：把 `petbank_*` key 的 owner、scope、schema、迁移和未来云快照资格机器化；第一阶段只报告，不接管运行时。
2. **奖励入口审计/收口**：盘点所有直接改 `totalPoints`/`petbank_points` 的路径，统一 eventId、profileId、source、receipt 和失败语义。
3. **时间入口收口**：新逻辑统一本地 `YYYY-MM-DD`；兼容读取宝箱/遛弯/竞技场旧日期格式，补边界测试。

### P1：P0 稳定后做

1. 把 `app.js` 的每日状态、奖励桥和存储访问渐进抽为纯逻辑 service，保留旧 window facade。
2. 把学习尝试从短时 `mastered` 扩展为可解释的 provisional/retained 语义和复习调度。
3. 评估 `learn-center.js`、CSS 和页面壳的拆分，按边界拆，不做全量技术栈重写。
4. 如果产品确实需要跨设备，复用现有 SQLite 认证/家庭/孩子/快照 API，只补 Profile 生命周期、outbox、冲突处理、恢复验证和 canary；不要重复建设账号接口。

### P2：工程收口

- 根目录补最小工具清单、批量 `node --check`、JSON/schema 检查和 secret scan。
- Pages 制品输出 route/asset manifest、体积和未引用资源报告。
- 做 keyboard/mobile/reduced-motion、iframe 会话和语音取消的统一验收。

## 决策原则

1. 先修规则可信度，再扩展内容和玩法数量。
2. 保留 Vanilla JS、静态发布和本地优先优势，不以换框架为目标。
3. 孩子状态、设备设置、家长设置、认证和未来云快照必须分 scope。
4. 学习效果用跨日保持和可解释尝试衡量，不用积分、时长或总答题数替代。
5. 新玩法必须复用统一内容、时间、奖励和 session 合同。
6. 清理/删除必须有引用证据、可恢复来源和用户授权；目录看起来杂乱不是删除理由。
