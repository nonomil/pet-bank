# 实施路线图与指标

> 2026-07-12 重排。已经落地的每日状态、英语 scope、部分奖励 receipt 和 Pages fast gate 不再列为待实施工作；具体证据见 [07-当前扫描与优化方案](./07-当前扫描与优化方案-2026-07-12.md)。

## 第一阶段：规则审计与止血（1-2 周）

- 生成 Storage Registry，报告未登记 key、owner 冲突、profile scope 和快照资格。
- 盘点所有积分/经验/物品发放路径，补 eventId、source、profileId、receipt 和重复事件测试。
- 统一新逻辑使用本地 `YYYY-MM-DD`，兼容读取宝箱/遛弯/竞技场旧日期。
- 为高风险旧入口加 contract，而不是继续新增玩法。

验收：registry 报告可重复；重复按钮/刷新/iframe 消息不重复发奖；同日/跨日/profile 场景可自动验证；已有用户数据不被重写。

## 第二阶段：兼容服务层（2-4 周）

- 从 `app.js` 抽出 daily state service、reward bridge、storage gateway。
- 保留 `window.addGrowthPoints`、`window.PetBankDailyState` 等兼容 API，迁移一个调用域后再删除直写。
- 补写入失败、坏 JSON、跨 tab、Profile reload 和旧 key migration 测试。

验收：领域规则可脱离 DOM 单测；页面仍能通过核心流程；Profile round-trip 不丢数据；新 key 和旧 key 的行为有明确版本。

## 第三阶段：学习可信度（4-6 周）

- 引入 `LearningAttempt`、contentVersion、skillId、hintLevel、errorTag。
- 将短时 `mastered` 解释为 provisional；先汉字或英语试点跨日 retained。
- 复习调度器稳定后再改首页推荐和家长复盘指标。

验收：能计算 1 日/7 日保持率，能解释推荐原因，提示依赖和错误类型可追踪；不收集超出必要范围的儿童数据。

## 第四阶段：产品需要时补齐自托管快照生命周期（单独项目）

- 复用现有认证、家庭、孩子和 revision 快照 API，不重复实现账号后端。
- 将 `SelfHostedApi.latestSnapshot/pushSnapshot` 接入 Profile 启动恢复、切换前 push 和切换后 pull。
- 增加 outbox、幂等、冲突提示、恢复验证、安全/迁移/回滚/端到端测试。
- 完成 canary 和失败降级后，才把孩子主流程标记为“已同步”。

验收：离线刷新可重试、重复提交不重复计分、冲突不静默覆盖、恢复可验证；否则保持孩子端 local-only，家长端只显示账号/家庭/孩子 API 状态。

## 第五阶段：工程与体验收口

- 拆 `learn-center.js`、`app.js` 和 CSS 的高变区域。
- 生成 route/asset manifest、体积预算、最大文件和未引用资源报告。
- 统一 iframe session/source/seq、timer 清理、语音取消、keyboard/mobile/reduced-motion 验收。

## 指标面板

| 维度 | 指标 | 用途 |
| --- | --- | --- |
| 数据 | 未登记 key 数、迁移失败数、Profile round-trip 丢失数 | 判断状态治理是否变好 |
| 奖励 | duplicate/rejected 事件、来源占比、每日 P50/P90、重复发奖数 | 判断经济是否可控 |
| 学习 | 1/7 日保持率、首答正确、提示依赖、errorTag 分布 | 判断学习而非刷分 |
| 循环 | 今日计划完成、玩法转化、玩法后回小屋、失败后重试 | 判断主循环是否成立 |
| 工程 | fast gate 失败、回归失败、制品体积、未引用资源 | 判断发布质量 |
| 健康 | 单局时长、连续使用时长、休息提示、reduced-motion 通过率 | 判断儿童体验风险 |

## Gate

进入下一阶段前必须满足：

1. 有可重复命令，不只靠截图手测。
2. 有 schema/version/migration 和回滚策略。
3. 没有扩大儿童数据收集范围。
4. 新玩法复用现有时间、内容、奖励和 session 合同。
5. 变更通过对应 contract，并能解释失败属于环境还是代码。

## 暂不做

- 不做 React/Vue 或 TypeScript 全量重写。
- 不在学习事件质量不足时引入 AI 推荐或复杂 FSRS 调参。
- 不增加新的货币、排行榜和无限奖励来源。
- 不把 DAU、在线时长、总积分当作学习效果。
- 不恢复 Supabase/旧社交运行时；自托管快照同步必须以现有 API 合同和本地降级为边界。
