# 可执行工作包

> 当前版本只列仍然需要做的工作。已落地能力见 07 文档，不要重复实现。

## 执行原则

- 先报告/测试，再改变运行时行为。
- 每个包有明确文件边界、迁移、回滚和验收命令。
- 兼容旧 `window.*` API 和旧 key，分阶段切换。
- 不把本地存储治理、奖励经济、学习状态和 SQLite 同步绑成一个不可回滚大改动。

## 工作包

| ID | 工作包 | 文件范围 | 前置 | 最小验收 | 回滚 |
| --- | --- | --- | --- | --- | --- |
| P0-A | Storage Registry 审计 | 新 registry、扫描脚本、`localstorage-keys.md` | 现有 key 家族清单 | 未登记/多 owner/scope 缺失报告可重复 | 只新增报告，不接管运行时 |
| P0-B | 奖励入口收口 | `app.js`、奖励服务、游戏桥、math/treasure/shop | receipt 事件合同 | 重放、刷新、重复消息至多一次 | 保留旧 facade 与来源开关 |
| P0-C | 时间入口收口 | `app.js`、treasure、walk、arena、learn | 共享 date helper | 同日/跨日/旧格式/profile/timer 测试 | 旧值只读兼容 |
| P1-A | DailyState service | `app.js`、新增纯逻辑模块 | P0-C 合同 | 无 DOM 单测 + 旧 API 回归 | 旧 `PetBankDailyState` facade |
| P1-B | Reward bridge/storage gateway | 奖励模块、各发放调用点 | P0-A/P0-B | 所有新入口走统一合同，写失败可见 | 按来源逐步迁移 |
| P1-C | LearningAttempt 试点 | 汉字或英语、学习中心、复盘 | profile scope + 时间合同 | 1/7 日保持、提示/错误、推荐原因 | 保留旧 progress 字段 |
| P1-D | 大模块渐进拆分 | `app.js`、`learn-center.js` | P1-A/B/C 稳定 | 纯函数测试、页面 smoke、无行为回归 | 旧 facade |
| P2-A | 发布/工具治理 | `scripts/`、workflow、registry | P0-A | node check、JSON/schema、secret、manifest | 门禁分层，先报告 |
| P2-B | 自托管快照生命周期（条件项目） | `js/self-hosted-api.js`、`js/profiles.js`、家长 UI、`prj/petbank-server/`、部署文档 | 产品确认跨设备需求；现有 API 合同 | Profile push/pull、revision/outbox/冲突/恢复 E2E、canary | 家长管理保留；孩子端继续 local-only |
| P2-C | UI/儿童健康 | app shell、CSS、声音、iframe | session/lifecycle 稳定 | mobile、keyboard、reduced-motion、timer cleanup | 页面/样式独立回退 |

## 第一迭代建议

只承诺 `P0-A + P0-B + P0-C` 的 registry、合同、报告和测试夹具。第一迭代不改存量孩子数据、不接入快照自动同步、不做大模块重构；复用现有 SQLite API，不重复建设认证/家庭接口。
