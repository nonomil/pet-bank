## 2. 详细设计

### 2.1 螺旋布局（FR1/FR2 · 决策 a,d）

新增 `generateSpiralLayout(count=12)`：阿基米德 `r = r0 + b·θ`，但 **r0 加大**（避开 r<15% 中心死区），θ 步长使 12 点均匀展开约 2.5 圈；中心最后 3-4 点因半径小、190px 卡片必重叠 → **手动微调**这几点坐标（偏离纯螺旋公式，保证不覆盖）。返回 `[{x,y},...]` 顺时针向内，填入 `MAP_LAYOUT` 的 x/y，保留 `size/chapter/id`。`buildRouteSvg` 的 `<polyline>` 按数组顺序连点 → 自动成螺旋臂（已确认成立）。

**视觉门禁**：S1 先只做布局 + 桌面/移动端截图审查，确认螺旋形态、卡片不重叠、中心手调点合理，再继续。

### 2.2 骰子推进机制（FR3 · 决策 b,f）

- `renderSceneGridMap(activeSceneId, boardId)`：`boardId==='sceneGrid'`(探索 tab) 才渲染「🎲 掷骰子(10积分)」按钮 + 角色位置标记；首页 `sceneGridMap` 仅渲染螺旋 + 进度。
- 流程：`spendPoints(10)` 预检 → 扣 10 → 生成 1–6 → CSS 翻滚动画 → 角色标记从 `currentIndex` 沿数组前进 N 步到 `currentIndex+N`（越界停末位）→ 落入目标场景。
- 当前位置：读写 `localStorage['exploration_current_pos']`，渲染时贴角色头像到对应节点。
- **位置同步规则（v3 补丁·F4）**：① 骰子推进落格 → 写位置为目标 sceneId；② 按序点选 `goExplore` 确认进入场景 → 同步写位置为该 sceneId；③ 切 tab 重渲（首页/探索 tab）→ 读位置贴标记。即所有「到达场景」动作（骰子/按序）统一更新位置，保证双入口、两模式位置一致。**注**：仅「实际进入场景」才更新位置；仅打开场景详情页（未确认进入）不改位置，消除 `goExplore` 语义歧义。

### 2.3 积分奖惩（FR4 · 决策 c,e,h,i）

骰子落入 → 带 `entryFlag='dice'` 调 `startExploration`。**积分结算挂在 app.js 战斗结果消费层**（覆盖双入口），仅当 `entryFlag==='dice'` 时触发：

| 结果 | 现有 | 骰子模式新增 | API |
|---|---|---|---|
| 战斗胜利 | +EXP/掉落 | +15 积分 | `addGrowthPoints(15)` |
| 战斗失败 | 宠物倒下 | −10 积分 | `deductGrowthPoints(10)`（不截断） |
| 无怪安全通过 | +EXP | +15 积分 | `addGrowthPoints(15)` |

按序入口（`entryFlag` 非 dice）不触发积分奖惩，零误伤。`entryFlag` 经 `showBattleModal` → 结果消费层透传（galgame 与非 galgame 两路均补传）。

**无怪分支单独钩子（v3 补丁·F1）**：无怪场景不进战斗、不走 `closeBattleModal`，故 +15 必须额外钉在 no-battle 分支——`startExploration` 无怪返回处（[exploration.js:298-304](../../js/exploration.js#L298-L304)）的消费端（[app.js:1048-1067](../../js/app.js#L1048-L1067) / [exploration-detail.js:261-282](../../js/exploration-detail.js#L261-L282)），同样靠 `entryFlag==='dice'` 触发。即 **战斗胜负** 与 **无怪通过** 两条路径各自挂一次积分钩子，且都受 entryFlag 门控，覆盖无遗漏。

**落地方式（S4 执行现场确认）**：在 `startExploration` 返回值 `result` 的消费处定位——若该处已有 `if(result.battle)` 战斗分支，则在其 `else`（无怪）分支挂 `addGrowthPoints(15)`；若无显式分支判断，则在消费函数内新增 `if(!result.battle && entryFlag==='dice') addGrowthPoints(15)`。具体消费函数名在 S4 现场定位（候选 [app.js:1048-1067](../../js/app.js#L1048-L1067) `goExplore` 消费处 / [exploration-detail.js:261-282](../../js/exploration-detail.js#L261-L282)）。方案层只定规则与门控，不写死实现行号。

### 2.4 未解锁格（FR5 · 决策 g）

骰子落入未解锁 `targetScene`：达 `min_level` 且 `unlock_cost>0` → 弹确认「花 {unlock_cost} 解锁？」同意→扣解锁→进入；拒绝→停原位（骰子 10 不退）。未达 `min_level` → 提示 + 停原位。复用 `tryUnlock`/`isSceneUnlocked`。

### 2.5 与既有机制兼容（FR5）

全部保留：HP 扣减、战斗、掉落、章末精英怪、解锁。骰子只是「换种到达方式」，到达后链路与按序一致。

### 2.6 边界规则（草案，执行前可微调 · 修订 F6）

| 边界 | 处理 |
|---|---|
| 积分不足 10 掷骰 | `spendPoints(10)` 预检失败 → toast「积分不足」，**不掷不扣** |
| 宠物倒下 / HP 不足掷骰 | 前置 `pet.hp<=0` 或 `< 最小 hp_cost` → toast「宠物需要休息」，**不掷不扣** |
| 落同格（落点=已通关场景） | 正常走 `startExploration`（遇怪/安全），积分奖惩照常 —— 靠消耗 10 / 成功 +15 / 失败 −20(净) 的期望防刷，不额外限制 |
| 骰子越界（超出第 12 章） | 停在最后一个场景 |
