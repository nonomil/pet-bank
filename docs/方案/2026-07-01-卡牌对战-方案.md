## 1. 背景与目标

### 1.1 现状（调研）
- 147 只宠物卡（四维 hp/atk/def/spd + 3 技能 power_strike×1.8/defend×0.5/ultimate×3 + 稀有度/系列）。
- 探索战斗 `exploration.js battleTurn`：回合制，atk 接入，**def/spd 未接入**，单怪物硬编码，职责耦合（伤害+CD+掉落+showEnding+UI日志依赖）。
- 战斗 UI `app.js showBattleModal`：DOM 中立外观但**硬编码 1v1**（PetSystem.getState/monster/两 HP 条/左右 fighter）。
- 掉卡 25%（`app.js battleAction` won 分支）。

### 1.2 目标
1. 新增卡牌对战：3v3 队伍，回合制，PvE 闯关。
2. 接入 def/spd（救活两维）——**卡牌对战必接，探索独立开关**。
3. battle-engine 通用化（纯结算）+ mode adapter（探索/对战各管业务）。
4. PvE 闯关 + PvP 热座彩蛋。

## 3. 对战机制设计

### 3.1 回合制 4 动作
| 动作 | 效果 | CD |
|------|------|----|
| 普攻 | `max(1, atk-def/2)` 伤害 | 0 |
| 技能 power_strike | `×1.8` 倍率 | 2 |
| 防御 defend | 下次受击 `×0.5` | 3 |
| 必杀 ultimate | `×3` 倍率 | 5 |

### 3.2 数值/技能接入【C3 修正：def/spd 独立开关】
> ⚠️ 审查修正（Codex）：def 接入会静默降探索难度（敌人反击变轻）+ 改时序（玩家先动敌人反击模型）。**def/spd 不强加给探索**。

- **伤害公式**（卡牌对战 + 可选探索）：`dmg = max(1, round((atk - def/2) × skillMult + rand(0..2)))`
  - def=0 退化纯 atk（缺省安全）
- **减伤**：defend 激活 → 受击 ×0.5（一次性，乘法叠加）
- **先手**：spd 高者先手（spd=0 → 平手，沿用现有"玩家先动敌人反击"）
- **【开关】探索战斗 def/spd**：默认 **关**（保持探索现状：玩家先动+敌人反击+不掉 def 减伤），通过配置 `combat.useDefSpd` 控制；卡牌对战 **必开**。
  - 探索若开启 def/spd = 难度调整，单独评估（声明：敌人反击变轻，探索变简单）。

### 3.3 3v3 轮换【C8 修正：换人写死】
- **选队**：玩家从收集卡选 3 只（SSR≤1）。
- **入场**：按 spd 排序，spd 最高首发。
- **换人**（写死规则）：
  - **自动换**：当前宠物 HP 归零 → 自动换下一只（不耗回合）。
  - **手动换**：玩家主动换人 → **耗本轮行动**（换人=本回合动作，不出招）。
- **胜负**：3 只全灭 = 负；敌方 3 只全灭 = 胜。

### 3.4 PvE 闯关
- 关卡 = AI 队伍（3 只预设宠物/怪物）。
- `data/arena-stages.json`（关卡×难度，复用 5 章主题）。
- AI：贪心规则（血低防御/CD好放技能/血量阈值换人），分难度档防 ultimate 滥用。
- 通关解锁 + 奖励（积分/新卡/成就）。
- PvP：本地热座彩蛋（同设备轮流），异步/联机不做。

## 4. 架构分层【C1 修正：纯结算 + mode adapter】

> ⚠️ 审查修正（Codex）：battleTurn 职责耦合，通用引擎不能只留 execAction 否则丢行为。

```
┌─ battle-engine.js（新，纯战斗结算，mode 无关）
│   - combatant 抽象 + execAction(actor,target,action)
│   - decideOrder（spd 先手）+ checkFaint + checkWin
│   - def 减伤公式 + 事件 emit（actionResolved/faint/battleEnd/fleeSuccess）
│   - 【不负责】掉落/EXP/showEnding/CardCollection/UI
│
├─ mode adapter: exploration.js（探索，1v1）
│   - 监听 battle-engine 事件 → 触发掉卡25%/EXP/showEnding/逃跑奖励
│   - def/spd 开关（默认关）
│
├─ mode adapter: card-arena.js（卡牌对战，3v3）
│   - 选队/轮换/换人/关卡进度/AI
│   - 监听 battle-engine 事件 → 更新 3v3 UI/奖励
│   - def/spd 必开
│
└─ app.js showBattleModal（UI，听事件）
    - 探索：1v1（现有 DOM）
    - 对战：3v3（复用 modal 外壳 + 新增替补/上场/换人区）
```

## 5. 技术设计

### 5.1 事件契约【C2 修正：UI 不解析日志】
battle-engine emit 事件（非中文日志），adapter/UI 监听：
- `actionResolved {actor, target, action, dmg, isCrit}` — 一次行动结算完成
- `faint {combatant, side}` — 一只倒下
- `battleEnd {result: 'win'|'lose'|'flee'}` — 战斗结束
- `fleeSuccess` / `fleeFail` — 逃跑

UI 听事件 → 更新 HP/浮动伤害/换人，不解析 `appendBattleLog` 中文文本判伤害。

### 5.2 combatant 抽象
```js
combatant = { id, name, img, side:'player'|'enemy', level,
  hp, maxHp, atk, def, spd,
  skills:[{id,mult,cd,currentCd}], defending:false }
```
- 怪物 skills 空数组时分叉（只普攻），combatant 补 level 字段。

### 5.3 3v3 UI【C4 修正：不硬塞现有 DOM】
- **复用**：modal 外壳 + 浮动伤害 + UI 锁。
- **新增 DOM**（card-arena 专属，不硬塞 .battle-fighters）：
  - 上场区（双方当前各 1 只立绘+HP）
  - 替补队列（玩家剩余 2 只小头像+HP，可点换人）
  - 换人操作区（手动换=耗回合提示）

### 5.4 入口【C7 修正】
- 卡牌对战挂 **"宠物" tab** 子页（与 pet/home/card/walk 同 tab，新增 arena 子页），不占新一级 tab。

### 5.5 模块划分
| 模块 | 职责 |
|------|------|
| `js/battle-engine.js`（新） | 纯结算 + 事件 emit |
| `js/exploration.js`（重构） | adapter：迁移 battleTurn 调 battle-engine，掉落/showEnding 保留 |
| `js/card-arena.js`（新） | adapter：选队/3v3/轮换/AI/关卡/奖励 |
| `js/app.js`（改） | showBattleModal 听事件；card-arena 用新 3v3 DOM |
| `data/arena-stages.json`（新） | 关卡数据 |
| `js/pet.js`（已就绪） | getTotalDef/Spd 已暴露 |

## 8. 风险与边界

- **探索不回归**（最高风险）：P0b 兼容迁移，行为契约保不变；def/spd 独立开关默认关。
- **职责拆分**：battle-engine 纯结算，掉落/showEnding/CardCollection 留 adapter，避免通用引擎吞业务。
- **事件契约**：UI 听事件不解析日志，降低耦合脆性。
- **3v3 UI**：复用 modal 外壳 + 新 DOM，不硬塞 1v1。
- **def/spd 缺省安全**：=0 退化纯 atk 不报错。
- **平衡**：SSR≤1；AI 分难度档；关卡递增。
- **纯前端**：PvE 离线；PvP 仅热座。
