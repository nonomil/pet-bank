# 方案与变更索引

> 本目录存放两类文档：
>
> 1. **方案文档**：描述某个功能准备怎么做
> 2. **变更文档（Change Request）**：记录方案或范围为什么改了

---

## 1. 编号规则

### 1.1 方案文档

- 建议编号前缀：`PLAN-*`
- 用于冻结功能边界、数据结构、验收标准

### 1.2 变更文档

- 固定前缀：`CR-###`
- 当前仓库沿用文件名 `change-request-###-主题.md`

### 1.3 检查清单

- 建议前缀：`CHECKLIST-*`
- 用于记录方案验收项、执行前检查点、回归要点

## 3. 使用建议

- 想看“准备怎么做”：
  优先读 `PLAN-*`
- 想看“为什么当前阶段要先做叙事收口而不是继续扩功能”：
  先读 [叙事收口与文案优化-方案](./叙事收口与文案优化-方案.md)
- 想看“为什么中途改方案”：
  读 `CR-*`
- 想看“下一阶段正式设计稿 / 实施计划”：
  去看 [../plans/README.md](../plans/README.md)
- 想看“卡牌系统现在到底怎么收口、哪些文档最该读”：
  去看 [../卡牌系统/README.md](../卡牌系统/README.md)
- 想看“路线地图为什么不继续做成大富翁玩法、现阶段到底该怎么改”：
  去看 [../路线地图大富翁融合/README.md](../路线地图大富翁融合/README.md)
- 想看“版本发布 / CR / 进度之间怎么对应”：
  去看 [../changes/README.md](../changes/README.md)
- 想看“本轮是否真做完了”：
  配合 [../进度/README.md](../进度/README.md) 一起看
