# 2026-07-06 C 轮加载治理与发布瘦身方案

## 目标

在上一轮“首页按需加载功能模块”的基础上，继续减少首屏外部依赖、首屏大图体积和 Pages 发布包体积，为后续继续新增页面保留更稳定的加载边界。

本轮不做大爆炸重构，不拆业务能力，不改积分/宠物/云同步数据语义。重点是把仍然影响首屏和发布链路的资源治理问题先收口。

## 当前观察

1. `index.html` 和 `admin.html` 仍在首屏使用 `https://cdn.tailwindcss.com` 与 Google Fonts。
2. `css/style.css` 的首页地图样式仍引用两个大 PNG：`data/GPT生图/背景图片-1.png` 和 `assets/home-bg/map-board.png`。
3. `css/style.css` 和 `js/app.js` 仍偏大，但上一轮已经通过 `runtime-loader.js` 把非首页业务模块延后，本轮先不继续大拆。
4. GitHub Pages 工作流仍整体复制 `assets` 和 `data`，虽然已经排除部分源图目录，但仍容易把 `.bak`、源快照、语音生成资源等非运行时文件带到发布包。
5. 现有测试主要守住“非首页脚本/CSS 不回到首屏”，还缺少外链、本地兼容 CSS、首屏大 PNG、发布包白名单的合同测试。

## 推荐范围

### P0：首屏外链本地化

把 Tailwind CDN 和 Google Fonts 从 `index.html`、`admin.html` 移除，改成本地兼容样式：

- 新增 `css/vendor/tailwind-lite.css`，只覆盖项目实际用到的轻量 utility class。
- 新增或扩展本地字体声明，不再请求 Google Fonts。
- 保留现有视觉主样式 `css/style.css`，避免引入完整构建系统。

取舍：不引入 Tailwind CLI，也不生成完整 Tailwind 产物，避免为了一个静态站优化引入额外 Node 构建链。

### P1：首屏地图图片转码

把当前首屏地图相关 PNG 转成 WebP，并改 CSS 引用：

- `data/GPT生图/背景图片-1.png` -> `assets/home-bg/map-hero-texture.webp`
- `assets/home-bg/map-board.png` -> `assets/home-bg/map-board.webp`

取舍：保留原 PNG 源文件，不删除用户资产；发布包可以通过白名单排除不再引用的源图。

### P1：Pages 发布包白名单化

新增一个可复用的 Pages artifact 组装脚本，替代 workflow 里的散落 `cp` / `rm -rf`：

- 只复制运行时入口：`index.html`、`admin.html`、`cloud-config.local.js`、`css`、`js`、`assets`、`data`。
- 再统一排除源文件、备份文件和非运行时资源。
- 明确排除 `assets/voice`，因为当前前端运行时代码没有引用它，语音生成资产应单独发布或按功能懒加载。

取舍：本轮先做“排除清单 + 测试”，不做全量精确 manifest。全量 manifest 更细，但风险和工作量更高。

### P2：防回归测试

补充静态合同测试：

- 首页和管理页不再请求 Tailwind CDN / Google Fonts。
- 首页样式不再引用首屏大 PNG。
- 新 WebP 资源存在。
- Pages 组装脚本不会把源图目录、`.bak`、`data/source-snapshots`、`assets/voice` 带入发布包。

## 非目标

1. 不拆 `data/pets.json`。
2. 不继续拆 `js/app.js` 的业务渲染函数。
3. 不接入 Service Worker。
4. 不处理当前未提交的语音 mp3 内容本身。
5. 不改变线上 Supabase / 家庭账号能力。

## 验收标准

1. `index.html` 与 `admin.html` 不包含 `cdn.tailwindcss.com`。
2. `index.html` 与 `admin.html` 不包含 `fonts.googleapis.com`。
3. `css/style.css` 不再引用 `data/GPT生图/背景图片-1.png` 和 `assets/home-bg/map-board.png`。
4. `assets/home-bg/map-hero-texture.webp` 与 `assets/home-bg/map-board.webp` 存在。
5. Pages artifact 组装结果不包含 `assets/voice`、`assets/pets/originals`、`assets/pets/poses-originals`、`data/source-snapshots`、`*.bak`。
6. 现有静态性能合同、cloud config 回归、smoke、UI 对齐回归继续通过。

### 已知风险

1. `tailwind-lite.css` 是项目定制兼容层，不是完整 Tailwind。后续如果新增大量 Tailwind utility，需要同步补充或引入真正的构建产物。
2. Pages artifact 仍约 `181MB`，主要体量来自运行时宠物、卡牌、场景等 WebP 资源。下一轮若继续瘦身，建议做资源 manifest 或按功能拆分资源包。
3. 当前工作区存在语音资源与其他文件的未提交变更，本轮没有修改这些内容；发布脚本会排除 `assets/voice`，但语音功能若未来要上线，应单独设计懒加载/外部存储策略。

### 后续建议

1. 为新增页面建立约定：不得直接在 `index.html` 增加外链脚本、外链字体或功能页 CSS。
2. 下一轮优先考虑 `data/pets.json` 摘要/详情拆分，降低进入宠物相关页面的首次数据负担。
3. 如果继续追求发布包体积，可以把 `assets/cards`、`assets/banchong*`、`assets/pets/poses` 做按功能 manifest 发布。
