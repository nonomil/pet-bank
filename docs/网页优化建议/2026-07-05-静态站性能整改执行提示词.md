# 2026-07-05 静态站性能整改执行提示词

下面这段可以直接丢给 AI 助手、外包开发，或者未来的自己继续执行：

```text
请你作为资深前端性能工程师，对当前项目做“静态站首屏性能优化 + 静态托管适配”整改。

先不要大改功能，目标是：
1. 提升首页打开速度
2. 降低首屏请求数
3. 让项目更适合部署到 EdgeOne / 帽子云 / GitHub Pages 这类静态托管平台

项目已知问题：
- 首页 head 里直接使用 cdn.tailwindcss.com、unpkg、Google Fonts
- 首页一次性加载大量 CSS 与 JS
- 首页启动时直接加载完整 data/pets.json
- 线上 cloud-config.local.js 在不同站点分别出现：
  - 返回 HTML
  - 返回 404
- 当前公开站点不是最优静态托管链路

请按下面的顺序工作：

第一阶段：配置与发布修复
- 检查 cloud-config.local.js 的线上请求来源
- 确保 cloud-config.local.js 只在 localhost / file 模式下尝试读取
- 生产环境改用 js/cloud-config.public.js 或运行时注入配置
- 检查部署平台的 SPA fallback 规则，确保 .js/.css/.json/.webp 不会被错误回退到 index.html

第二阶段：首屏减负
- 去掉生产环境运行时 tailwind CDN，改为本地构建 CSS
- 将首页底部脚本按功能拆分，首页只保留必需脚本
- 探索、卡牌、汉字、社交、语音、云同步模块改为首次进入页面时懒加载
- 减少首页首轮 CSS 文件数量

第三阶段：数据减负
- 将 data/pets.json 拆成首页摘要数据和详情分片
- 首页只加载 summary
- 图鉴、宠物详情、战斗等页面再按需加载 detail 数据
- 排查 stories、voice、社交类数据是否在首页被提前加载

第四阶段：缓存与部署适配
- 给 HTML 设置短缓存
- 给版本化 JS/CSS/图片设置长缓存
- 给 JSON 设置合理缓存时间
- 确保静态资源适配 EdgeOne / 帽子云 / GitHub Pages

输出格式：
1. 问题清单
2. 根因分析
3. P0 / P1 / P2 改造步骤
4. 涉及文件清单
5. 验收指标

实施要求：
- 小步提交
- 不做无关重构
- 每一步都说明为什么改
- 保证首页功能不回归
```
