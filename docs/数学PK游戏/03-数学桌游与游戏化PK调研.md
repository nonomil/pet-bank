# 数学 PK 桌游/卡牌/游戏化调研与优化建议

> 日期：2026-07-07
> 目标：参考市面上的儿童数学桌游、卡牌数学游戏、数学冒险产品，整理一套适合当前 `数学 PK` 的优化方向。
> 当前用户画像：低龄儿童；已掌握 100 以内加减法；乘法处于“等组 + 重复加法”起步阶段。

## 1. 先说结论

当前 `数学 PK` 不应该直接朝“更像格斗游戏”一路加复杂战斗系统，而应该走：

1. `学习环节` 更慢、更具体、更可视化。
2. `PK 环节` 更短、更有反馈、更少纯随机。
3. `奖励系统` 更强调“掌握和坚持”，不是只奖励“答得快”。
4. `技能卡` 可以做，但推荐先做成“学习支援卡”而不是复杂伤害卡。

最适合当前产品的方向，不是照搬某一款桌游，而是组合三类成熟思路：

- `数学认知进阶`：从等组、阵列、重复加法进入乘法。
- `卡牌/桌游选择感`：给孩子少量、有意义的选择，而不是纯运气。
- `游戏化奖励闭环`：即时反馈、小目标、阶段奖励、可见成长。

## 2. 外部调研里最值得借的东西

### 2.1 数学学习规律

从课程标准和教学资源看，乘法入门的核心不是“背表”，而是先理解：

- `5 × 7` 是 `5 组 7 个`。
- 等组可以用图形、阵列、重复加法和乘法式互相表示。
- 乘法流利度最终要到 `100 以内`，但起步阶段靠策略和表示，不该一上来只做限时口算。

这和当前我们已经做的 `乘法启程` 方向是一致的，但还可以更进一步：把 PK 里的战斗节奏也跟认知阶段绑定，而不是所有孩子都被同一套竞技节奏推进。

### 2.2 数学游戏的有效原则

公开教育资源反复提到几件事：

- 好的数学游戏应更多依赖 `策略和选择`，而不是只靠运气。
- 游戏要 `容易学会`，规则不能太重。
- 即时反馈、进度可见、适应性练习，会明显提升坚持度和信心。
- 卡牌/桌游类数学游戏的强项，不只是练熟练度，还能促成“说出思路”和“看见数量关系”。

这意味着我们的 `数学 PK` 最应该补的，不是更多华丽按钮，而是：

- 每局开始前给一个简单选择
- 每局结束后给清楚成长反馈
- 让题目策略和战斗结果之间有更直观的映射

### 2.3 市面产品常见的吸引点

几类产品的共同点很明显：

- `Prodigy / Boddle`：用宠物、战斗、任务、奖励轨道维持长期兴趣。
- `Imagine Math Facts`：用自适应、即时反馈、可视化总结推动熟练度。
- `Eureka Math / Wild Side` 这类卡牌资源：强调一副牌多玩法、视觉支持、可谈论数学。

可以借，但不能照搬的地方：

- 可借：宠物成长、任务、奖励轨道、轻量卡牌、阶段目标。
- 不建议直接照搬：超长任务链、过多战斗属性、太重的养成数值、过深的抽卡/随机掉落。

## 3. 对当前数学 PK 的判断

当前数学 PK 已经有几个正确方向：

- 有 `5 档难度`
- `乘法启程` 已经从纯竞速改成了练习 + PK
- 角色改成了 `宠物 vs 机器人`
- 战斗反馈开始变强

但它离“低龄数学战斗体验真正成立”还差四个关键层：

### 3.1 缺少“战前选择”

现在进入对局后，孩子几乎没有策略选择，只有答题。
这会让 PK 变成“做题外壳上的即时演出”，而不是“我在带着宠物想办法赢”。

### 3.2 缺少“掌握型奖励”

现在更偏向单局结果和即时表现。
但对低龄数学来说，更关键的是：

- 我今天看懂了几组几个
- 我已经稳定掌握了 2 和 5 的乘法
- 我连续三天完成了乘法启程

这类成长反馈现在还不够强。

### 3.3 难度分级还可以更细

当前 5 档已经比以前好很多，但从认知角度看，`乘法启程` 内部还应再拆：

- 看图理解
- 重复加法求和
- 2/5/10 的简单乘法
- 5 以内乘法
- 6-9 的策略乘法

### 3.4 PK 回合结构还偏单一

目前基本是“答对 -> 出招 -> 结算”。
对孩子来说，最容易产生兴趣的其实是：

- 我先选一个支援
- 我打中了会触发哪种效果
- 连续答对能不能攒出一次大招

## 4. 推荐的优化方向

这里我给三条路线，推荐先做 `路线 A`。

### 4.1 路线 A：学习支援卡 PK

这是我最推荐的版本，因为它最贴合当前产品状态，也最符合低龄数学认知。

每局开始前，孩子从 `3 张支援卡` 里选 `1 张`，只带一张进本局。

卡牌不主打伤害，而主打“帮助理解”和“调节节奏”：

| 卡牌类型 | 示例 | 对学习的意义 |
|---|---|---|
| 看图卡 | `看阵列`：本题自动显示等组图 | 帮孩子回到表示 |
| 拆分卡 | `拆一拆`：显示 `8×7 = 8×5 + 8×2` | 帮孩子学策略 |
| 缓冲卡 | `慢一点`：机器人本轮多思考 2 秒 | 给低龄孩子时间 |
| 容错卡 | `再试一次`：本轮首次答错不丢攻击机会 | 降低挫败感 |
| 连击卡 | `小爆发`：连续答对 2 题，本轮攻击更强 | 奖励稳定表现 |

这类卡适合当前阶段，因为它不会把孩子的注意力从数学上拉走，反而会把数学策略做成“技能”。

### 4.2 路线 B：回合制奖励轨道

把每场 PK 从“单点胜负”改成“能量积累”。

孩子在对局中不只是打掉对方 HP，而是在积累三种资源：

- `理解星`：看图题、解释题、应用题答对
- `速度星`：在合理时间内完成
- `连击星`：连续答对

每局结束后不只给胜负，还给一条短奖励轨道：

- 3 星：解锁一个宠物表情或上场动作
- 6 星：解锁一个支援卡
- 9 星：解锁一个小宝箱
- 12 星：解锁该难度的徽章

这样就算输了一局，孩子也会觉得“我还是在前进”。

### 4.3 路线 C：关卡型 Boss PK

把普通机器人 PK 和 Boss 关分开。

- `普通关`：3 轮短对局，节奏快
- `Boss 关`：每通过 3 次普通关出现一次
- Boss 机制不是更快答，而是带一个学习要求：
  - 只出等组题
  - 只出 2/5/10 乘法
  - 只出需要拆分的乘法

Boss 奖励可以更明显，比如：

- 新机器人图鉴
- 宠物招式动画升级
- 本难度通关勋章

这条路线更有游戏感，但比 `支援卡` 更晚做更稳。

## 5. 我对“技能卡”这件事的具体建议

`可以做，但要克制。`

如果技能卡一上来就做成：

- 造成双倍伤害
- 偷对方血量
- 强制跳过对方回合
- 高概率暴击

它会很快把注意力从数学迁到“战斗数值”，孩子会开始追求卡强不强，而不是题会不会。

更适合当前产品的是两层卡牌：

### 第一层：学习支援卡

用于低龄和新知识起步阶段。

推荐优先做：

- `看图卡`
- `慢一点`
- `再试一次`
- `拆分卡`
- `提示卡`

### 第二层：战斗效果卡

等孩子已经进入 `综合闯关 / 乘除挑战` 再加。

推荐后续再做：

- `护盾`
- `连击加成`
- `小暴击`
- `冻结机器人 1 回合`

也就是说：

- `乘法启程` 以学习卡为主
- `综合闯关` 开始混少量战斗卡
- `乘除挑战` 才允许更强的竞技卡

## 6. 奖励系统怎么设计更合理

### 6.1 即时奖励

每题结束就要有反馈：

- 答对：出招、星点、连击条、轻音效
- 答错：温和提醒、解释、再试机会

### 6.2 单局奖励

每局结束不只显示输赢，建议显示：

- `本局答对率`
- `本局用了哪些策略`
- `本局获得 1-3 星`
- `距离下个奖励还差几星`

### 6.3 阶段奖励

建议用 `小目标 + 明确解锁`，而不是纯随机掉落：

- 连续 3 天完成练习：解锁新进场动作
- 某个乘法族掌握 90%：解锁该族徽章
- 胜利 5 次：解锁一张新支援卡
- 完成某个阶段：解锁新的机器人对手

### 6.4 不建议过重的随机宝箱

宝箱可以有，但不应成为主驱动。
低龄数学里最稳定的动力，仍然是：

- 我做对了
- 我看懂了
- 我升级了
- 我的宠物变厉害了

## 7. 按数学认知规律重新分级

推荐把 `数学 PK` 的难度理解成 `大阶段`，而不是只有题库档位。

| 大阶段 | 当前建议内容 | PK 节奏建议 |
|---|---|---|
| A 数感起步 | 10/20 以内合成拆分、凑十 | 可以不做强 PK，偏练习 |
| B 加减熟练 | 20/100 以内加减 | 短回合 PK |
| C 乘法启程 | 等组、阵列、重复加法、2/5/10、5 以内乘法 | 练习优先，PK 放慢 |
| D 乘法策略 | 6-9 乘法、拆分、交换律、分配策略 | 支援卡 + 正式 PK |
| E 综合闯关 | 加减乘混合、简单应用题 | 混合型 PK |
| F 乘除挑战 | 乘除熟练度、未知因数、短时判断 | 竞技型 PK |

其中 `C` 阶段一定不要直接做成“乘法背表竞速”。

## 8. 我最推荐的一版具体方案

如果只做一轮最值的优化，我推荐这版：

### 第一优先级

- `乘法启程` 增加 `支援卡三选一`
- 卡牌只做学习支援卡
- 机器人节奏继续按题目难度变慢
- 每局结算增加 `本局获得星星`
- 星星累计解锁新卡和新机器人

### 第二优先级

- 把 `普通关 / Boss 关` 分出来
- 每通过 3 局普通关进入 1 局 Boss
- Boss 不是更快，而是更强调某一数学策略

### 第三优先级

- 宠物做“招式成长”
- 同一只宠物随着胜场和掌握度，攻击动画更丰富
- 让孩子感觉不是自己在刷题，而是在“训练自己的宠物”

## 9. 建议避免的设计

- 不要让奖励只和 `速度` 绑定。
- 不要让卡牌效果压过数学本身。
- 不要让随机性太大，避免“运气比策略重要”。
- 不要把 `乘法启程` 做成大量限时竞速。
- 不要把新手和熟练玩家丢进同一套 PK 节奏。

## 10. 下一轮实施建议

推荐按这个顺序做：

1. 先做 `学习支援卡` 文档和 UI 草案。
2. 再做 `星星奖励轨道` 和 `本局结算增强`。
3. 之后再做 `Boss 关`。
4. 最后再做更复杂的战斗技能或养成联动。

也就是说，下一轮最值得落地的不是“更多伤害数值”，而是：

- `会帮助孩子学会题目的卡`
- `输了也能前进的奖励轨道`
- `按认知阶段区分的 PK 节奏`
