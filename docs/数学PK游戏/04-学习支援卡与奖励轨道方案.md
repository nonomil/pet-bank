# 数学 PK 学习支援卡与奖励轨道方案

> 日期：2026-07-07
> 基于：`03-数学桌游与游戏化PK调研.md`
> 目标：给 `数学 PK` 增加低龄友好的战前选择、局内支援、局后奖励轨道，让孩子既有“在战斗”的感觉，也有“在学会”的路径感。

## 1. 方案目标

这轮不做复杂数值战斗系统，不做抽卡，不做大量随机掉落。

本方案只解决三件事：

1. 让孩子在开局前有一个 `简单但有意义的选择`。
2. 让每局结束后有 `输赢之外的成长反馈`。
3. 让数学 PK 更符合 `数学认知进阶`，而不是只偏向竞速。

## 2. 设计原则

### 2.1 先学会，再变强

支援卡的第一职责是帮助孩子理解题目、坚持作答、减少挫败，不是单纯提高伤害。

### 2.2 一局只带一张卡

低龄孩子不适合同时处理多张技能卡和复杂资源系统。
每局 `3 选 1` 已经足够形成“我的选择会影响结果”的体验。

### 2.3 奖励强调可见进步

奖励重点不是“爆稀有”，而是：

- 我今天完成了
- 我已经会这类题了
- 我解锁了一个新帮助
- 我的宠物更会战斗了

### 2.4 新手和高手分层

`乘法启程` 只开放学习支援卡；
`综合闯关` 再加入少量战斗向卡；
`乘除挑战` 才允许更竞技的效果。

## 3. 整体流程

推荐的数学 PK 新流程：

```text
选难度
  -> 选本局支援卡（3 选 1）
  -> 开始对局 / 练习
  -> 答题与出招
  -> 结算星星
  -> 推进奖励轨道
  -> 解锁卡片 / 对手 / 宠物动作
```

其中：

- `练习场` 和 `正式 PK` 都可以使用支援卡
- `练习场` 更偏学习卡
- `正式 PK` 更偏节奏卡和连击卡

## 4. 支援卡系统

## 4.1 系统结构

每次进入一局前，系统随机给出 `3 张当前阶段可用卡牌`，孩子选择 `1 张`。

卡牌字段建议：

```js
{
  id: 'slow_robot',
  name: '慢一点',
  stage: ['medium_mul', 'medium_mix'],
  rarity: 'common',
  type: 'learning_support',
  timing: 'auto_round_start',
  description: '本局机器人每题多思考 2 秒',
  effect: { robotThinkMsBonus: 2000 }
}
```

最关键的不是稀有度，而是：

- `在哪个阶段可用`
- `什么时候生效`
- `它帮助孩子解决什么学习问题`

## 4.2 第一批推荐卡牌

### A. 乘法启程专用学习卡

这是优先级最高的一组。

| 卡牌 | 作用 | 适合对象 |
|---|---|---|
| `看阵列` | 本局乘法题自动显示行列图 | 第一次接触乘法的孩子 |
| `慢一点` | 机器人每题多思考 2 秒 | 还在用加法算乘法的孩子 |
| `再试一次` | 本局首次答错不立即失去出手机会 | 容易紧张的孩子 |
| `拆一拆` | 乘法题答错后自动显示拆分提示 | 刚进入乘法策略阶段 |
| `提示火花` | 连续 2 题答错后高亮关键数 | 需要轻提醒的孩子 |

### B. 综合闯关过渡卡

这组从 `medium_mix` 开始开放。

| 卡牌 | 作用 | 方向 |
|---|---|---|
| `小连击` | 连续答对 2 题后本局多 1 星 | 奖励稳定表现 |
| `护盾叶片` | 本局首次被机器人抢先时少掉 1 格 | 降低失败感 |
| `快热身` | 第一题机器人额外变慢 | 帮孩子进入状态 |
| `应用题灯泡` | 应用题显示关键词高亮 | 帮孩子读题 |

### C. 乘除挑战后期卡

这组留到后面。

| 卡牌 | 作用 | 风险 |
|---|---|---|
| `冻结 1 回合` | 机器人一题不攻击 | 竞技感强，慎早放 |
| `暴击星尘` | 连击时额外攻击表现 | 易偏战斗，后放 |
| `双倍能量` | 本局第一颗星翻倍 | 更适合熟练玩家 |

## 4.3 当前阶段最适合先做的 5 张

如果只做第一版，建议只做这 5 张：

1. `看阵列`
2. `慢一点`
3. `再试一次`
4. `拆一拆`
5. `小连击`

原因：

- 已经覆盖了 `理解支持 / 节奏支持 / 容错支持 / 成就支持`
- 不需要引入很重的局内状态管理
- 非常适合当前的 `乘法启程 -> 综合闯关` 阶段

## 5. 支援卡如何影响对局

## 5.1 局内表现规则

卡牌效果尽量遵循三个模式：

- `自动生效`
- `条件触发`
- `一次性护航`

避免做成“对局中反复点很多按钮”。

推荐：

- `自动生效`：进入本局就持续起作用
- `条件触发`：答错一次、连对两题、遇到应用题时触发
- `一次性护航`：只救一次，简单易懂

不推荐：

- 手动点技能释放
- 多资源消耗
- 多张卡连锁
- 大量状态叠加

## 5.2 对当前代码的映射建议

现有 `MathPKGame` 已经有：

- 难度
- 题目对象
- 机器人思考时间
- 连击与分数
- 练习场 / PK 分支

所以第一版卡牌落地时，最自然的接入点是：

- `开局时写入 activeSupportCard`
- 出题时修饰 question payload
- 算机器人时间时叠加 bonus
- 答错和答对时检查一次性卡牌是否触发
- 结算时根据表现追加星星

也就是说第一版不需要重构整个对战系统。

## 6. 奖励轨道系统

## 6.1 为什么要做奖励轨道

现在数学 PK 更像“单局得分”。
但低龄孩子更需要的是：

- 就算输了，也知道自己拿到了什么
- 就算今天还不会，也知道离解锁还差多少

奖励轨道正好解决这个问题。

## 6.2 推荐的轨道结构

建议采用 `星星轨道`，不是金币轨道。

每局最多获得 `3 颗星`：

- `胜利星`：赢得本局或完成练习目标
- `理解星`：关键学习目标达成
- `稳定星`：答题正确率或连击达标

### 乘法启程的 3 星规则

- `1 星`：完成本局练习或对战
- `1 星`：至少 3 题真正答对
- `1 星`：至少 1 道乘法题独立答对

### 综合闯关的 3 星规则

- `1 星`：赢得对局
- `1 星`：正确率达到 70%
- `1 星`：出现连击或完成应用题

### 乘除挑战的 3 星规则

- `1 星`：赢得对局
- `1 星`：正确率达到 80%
- `1 星`：在时间要求内稳定完成

## 6.3 轨道奖励节点

先做一条很短的轨道：

| 累计星数 | 解锁 |
|---|---|
| 3 星 | 一张新支援卡 |
| 6 星 | 一个宠物战斗表情 / 入场动作 |
| 9 星 | 一个机器人图鉴条目或徽章 |
| 12 星 | 当前阶段完成印章 |

这样孩子 3-4 局内就能感知到成长。

## 6.4 奖励不要太随机

建议默认用 `明确解锁`：

- 攒到 3 星，必得卡牌
- 攒到 6 星，必得动作

宝箱可以作为表现层出现，但内容最好是可预期解锁，而不是纯抽奖。

## 7. 难度与卡牌开放节奏

## 7.1 开放表

| 难度 | 卡牌数量 | 卡牌类型 |
|---|---|---|
| 加减起步 | 0-2 | 只给最简单提示卡 |
| 加减进阶 | 2-3 | 缓冲、再试一次、快热身 |
| 乘法启程 | 5 | 全部学习支援卡 |
| 综合闯关 | 6-8 | 学习卡 + 少量连击卡 |
| 乘除挑战 | 8+ | 再加入少量竞技卡 |

## 7.2 乘法启程内部建议

| 子阶段 | 开卡重点 |
|---|---|
| 等组理解 | `看阵列`、`慢一点` |
| 重复加法 | `再试一次`、`提示火花` |
| 5 以内乘法 | `小连击`、`拆一拆` |
| 乘法策略起步 | 再逐步开放更强支援 |

## 8. UI 建议

## 8.1 开局选卡

不要做成复杂卡组页。

推荐：

- 进入本局前弹出 `3 张大卡`
- 每张只显示：
  - 卡名
  - 一句作用说明
  - 小图标
  - “适合慢慢想 / 适合连击 / 适合看图”这类轻标签

孩子点一张就进入对局。

## 8.2 局内展示

当前生效卡只占一个很小的位置：

- 顶部 round pill 下方
- 或左下角小徽章

不要做成占中间的大面板。

推荐文案示例：

- `本局支援：慢一点`
- `再试一次已使用`
- `小连击待触发`

## 8.3 结算展示

结算页比现在多一块：

```text
本局获得 ★★☆
累计星轨：6 / 12
解锁：新支援卡「拆一拆」
```

如果没解锁，也要给明确反馈：

```text
再拿 1 颗星，就能解锁新卡
```

## 9. 数据结构建议

第一版建议完全本地化，先不接云端。

推荐在本地保存：

```js
{
  unlockedSupportCards: ['show_array', 'slow_robot'],
  selectedSupportCard: null,
  mathPkStarProgress: {
    medium_mul: 4,
    medium_mix: 2,
    hard: 0
  },
  mathPkRewardsClaimed: {
    medium_mul: [3],
    medium_mix: []
  }
}
```

这版足够支撑：

- 解锁哪些卡
- 各阶段攒了多少星
- 哪些节点已经领过

## 10. 第一版实施范围

为了稳，建议第一版只做：

### 做

- `3 选 1` 开局支援卡
- 5 张卡牌
- `乘法启程` 和 `综合闯关` 两档先接
- 每局 `3 星制`
- 简单星轨
- 解锁新卡提示

### 暂不做

- 卡牌升级
- 多张卡组
- 稀有度抽卡
- Boss 关
- 云端同步
- 竞技型强控制卡

## 11. 实施顺序建议

### P1

- 卡牌数据结构
- 开局选卡弹层
- `慢一点 / 再试一次 / 看阵列` 三张卡
- 局后 3 星结算

### P2

- 加入 `拆一拆 / 小连击`
- 增加星轨 UI
- 解锁新卡与动作

### P3

- 加入 Boss 关
- 卡牌与机器人类型更深联动
- 宠物技能成长和卡牌解锁联动

## 12. 推荐下一步

如果下一步要继续实现，我最建议马上落的是：

1. `支援卡数据结构`
2. `开局三选一 UI`
3. `慢一点 / 再试一次 / 看阵列` 这三张卡
4. `本局 3 星结算`

这是最小但最有体感的一版。
