## 1. 背景与问题

### 1.1 现状
- 图鉴两级（`js/card-collection.js`）：`_renderCategory`（4大类）→ `_renderGrid`（瀑布流）→ `showDetail`（详情模态）。
- 战斗数值：`data/pets.json` 有 `base_hp/base_atk/rarity/series/source`（13字段），**缺 def/spd**。
- 战斗公式（`js/exploration.js battleTurn`）：普攻 `max(1,atk+rand)` / 技能 `atk×mult` / 反击 `monster.atk+rand`；**玩家 def 不参与**；装备 weapon(加atk)/armor(加max_hp)。

### 1.2 问题
1. **卡牌"半透明看不见"**：`css/card-collection.css:111-114` `.card-item.uncollected{opacity:.4;filter:grayscale(1)}`。去白底后透明 PNG 叠在 `rgba(255,255,255,.05)` 极透明背景上，融入深色页面。探索场景已改纯 grayscale（commit 1c100b4），**图鉴策略不一致**。
2. **卡牌无信息密度**：仅 emoji+名字，无战力数值。
3. **数值单薄**：仅 hp/atk，无 def/spd，战斗策略单一。

### 1.3 目标
1. 卡牌重设计：稀有度边框 + 宠物立绘 + HTML 数值文字四层叠加（不生成含文字图）。
2. 保留战斗数值系统：hp/atk/**def/spd** + rarity/series，为 3v3 轻训练营提供基础能力。
3. 双生成机制：A) Agnes 边框图（装饰增强）；B) 规格文档供 GPT-IMAGE-2。
4. 未收集策略与探索统一（grayscale+暗化，非 opacity）。
5. 将展示口径控制在“儿童可理解的能力展示”，而不是重战力、重稀有追逐。

## 2. 轻训练营数值系统设计（数据层）

### 2.1 数值维度（新增 def/spd，不加 crit）

| 维度 | 字段 | 现状 | 说明 |
|------|------|------|------|
| 生命 | `base_hp` | ✅ | 战斗已有 |
| 攻击 | `base_atk` | ✅ | 战斗已有 |
| **防御** | `base_def` | 🆕 | 减伤（卡牌展示 + P4可选接入） |
| **速度** | `base_spd` | 🆕 | 先手/闪避（卡牌展示 + P4可选接入） |
| 稀有度 | `rarity` | ✅ | common/rare/epic/legendary |
| 系列 | `series` | ✅ | 19 系列 |

> 不加 crit/skill：避免数值膨胀 + 战斗过度改造。数值应服务短局训练营，而不是复杂深养成。

### 2.2 数值生成（rarity 基线 + 倾向修正 + 种子 + 分布校验）【C5 修正】

> ⚠️ 审查修正：现有 base_hp/base_atk **不完全按 rarity 单调**（存在 common 攻击 > rare 样本），直接按 rarity 随机补 def/spd 会固化失衡。

**生成规则**：
1. **rarity 基线区间**：

| rarity | def 基线 | spd 基线 |
|--------|---------|---------|
| common | 2~5 | 3~6 |
| rare | 5~8 | 6~9 |
| epic | 8~11 | 9~12 |
| legendary | 11~15 | 12~16 |

2. **宠物倾向修正**：读现有 base_atk 相对同 rarity 均值的偏差，高攻宠物 def 偏低（玻璃大炮）、低攻宠物 def 偏高（坦克），spd 类似反相关 atk。避免"全维碾压"。
3. **固定种子**：`seed = hash(pet.id)`，保证可复现（多次跑结果一致）。
4. **分布校验**：生成后统计每 rarity 档 def/spd 均值，确认单调（common < rare < epic < legendary 均值），失衡则调整。

**脚本**：`.tmp/gen-pet-stats.py`，读 pets.json → 算偏差 → 填 def/spd → 校验 → 写回（备份原文件）。

### 2.3 PetSystem state 扩展【C3/C6 修正】

> ⚠️ 审查修正：`js/pet.js` state（`:56-78`）仅 hp/atk，`chooseSpecies`（`:107-109`）映射不读 def/spd，公开 API 仅 getTotalAtk/getTotalMaxHp。

**改动**：
1. `js/pet.js` state 加 `def/spd`，`chooseSpecies` 映射 `base_def/base_spd` → state。
2. 新增 `getTotalDef()` / `getTotalSpd()`（基础值 + 装备加成预留，armor 可加 def）。
3. **旧档兼容**：`load()` 时若 state 无 def/spd，从 species.base_def/base_spd 回填（避免旧存档 undefined）。
4. `getState()` 返回含 def/spd。

## 3. 卡牌规格设计（视觉层）

### 3.1 卡牌比例与尺寸

- **比例**：3:4 竖卡。
- **网格卡**：180×240px | **详情卡**：360×480px | **大类卡**：保持 160×宽。

### 3.2 四层叠加架构（CSS 为主，PNG 仅装饰增强）【C4 修正】

> ⚠️ 审查修正（Codex）：Agnes/AI 生图难稳定产出"中央/四角精确透明"PNG，**不作定位基准**。Layer1 以 CSS 边框+底纹为主，PNG 边框只作装饰皮肤。

```
卡牌容器(相对定位, 3:4, 统一百分比坐标)
├─ Layer1: 稀有度边框 [CSS优先: border+box-shadow+渐变底纹; PNG边框仅增强皮肤]
│   ※ 坐标基准=容器百分比, 不依赖PNG透明洞
├─ Layer2: 宠物立绘 (透明PNG, 居中 object-fit:contain, 占60-70%)
├─ Layer3: 暗角渐变 (::after, 增强可读性)
└─ Layer4: HTML 文字数值 (绝对定位 z-index最高, 相对容器)
    - 顶部:系列标签 | 底部:名称栏(深色条带)
    - 四角数值角标: HP左上 / SPD右上 / DEF左下 / ATK右下 (相对容器)
```

**关键**：立绘 object-fit:contain，四角数值/名称栏**相对卡牌容器**定位（非 PNG 透明洞），保证 Agnes 图未生成时 CSS 边框仍对齐。HTML 文字清晰、可本地化、数值即时更新。

### 3.3 稀有度配色（统一单一源）【C1 修正】

> ⚠️ 审查修正：pets.json `rarity_config` / CSS `card-rarity-*` / 方案 三处配色不一致，P0 统一。

**统一配色源**（WoW 体系，写入 pets.json rarity_config 为权威，CSS/方案引用）：

| 档位 | 主色 Hex | 光效 | 现有 CSS 类 |
|------|---------|------|------------|
| common | `#9D9D9D` 灰 | 无 | `.card-rarity-common` |
| rare | `#0070DD` 蓝 | 内发光 | `.card-rarity-rare` |
| epic | `#A335EE` 紫 | 内发光+边缘光 | `.card-rarity-epic` |
| legendary | `#FF8000` 橙金 | 外发光+金粒子 | `.card-rarity-legendary` |

**P0 同步**：pets.json rarity_config legendary `#f39c12`→`#FF8000`；CSS `#f1c40f`→`#FF8000`（或保留 CSS 现值，仅以 pets.json 为准——gate 确认）。

### 3.4 数值角标布局（四角）

```
┌──[HP红]──────[SPD绿]──┐
│                        │
│      🐾 宠物立绘        │
│      (透明居中)         │
│                        │
└──[DEF蓝]──────[ATK橙]──┘
       │  名称栏  │
```
- 角标圆形（网格 28px / 详情 44px）。
- 配色：HP `#C0392B`红 / ATK `#D35400`橙 / DEF `#2C7BE5`蓝 / SPD `#27AE60`绿。
- 白色粗体 + 黑色描边。
- 未收集：角标 `?`，立绘 grayscale+暗化，名称 `???`。

### 3.4.1 儿童展示口径

数据层保留 `HP / ATK / DEF / SPD`，但展示和文案建议转译为：

- HP = 体力
- ATK = 勇气
- DEF = 防守
- SPD = 灵巧

原因：

- 孩子更容易理解
- 减少“纯战力压制”的竞争心智
- 与“宠物训练营”定位更一致

### 3.5 未收集策略（与探索统一）

去掉 `opacity:.4`，改 `grayscale(1) brightness(.5)`（与 commit 1c100b4 一致）。立绘剪影化（grayscale+暗化保留轮廓悬念），角标 `?`，名称 `???`，底部"探索解锁"。

### 3.6 无图宠物 emoji 兜底【C2 修正】

> ⚠️ 审查修正：classpet 80条（`pets.json:3476+`）无 stages/imageStages，方案须显式兜底。

- Layer2 立绘：有图 → `<img>` 透明 PNG；**无图 → 大 emoji**（species.emoji，font-size 适配卡牌区，同 pet 页方案）。
- 角标数值仍显示（def/spd 等不受图影响）。
- 兜底判断：`species.imageStages?.['2'] || species.imageUrl || species.emoji`。

## 4. 双生成机制

### 4.1 机制 A：Agnes 边框图（装饰增强，非主路径）【C4 修正】

> ⚠️ 审查修正：Agnes 难精确透明，**降级为装饰增强皮肤**，不承载定位基准。

- **产物**：4 档稀有度边框皮肤图（`assets/cards/frame-{rarity}.png`，480×640，装饰外框+底纹，**不要求精确透明区**，作为 Layer1 增强 background-image 叠在 CSS 边框上）。
- **要求**：稀有度配色外框 + 底纹质感 + **无文字**（中央/四角是否透明不强制，因为坐标用容器百分比）。
- **提示词**：`ornate card frame, {rarity配色} border, {底纹} texture, no text, no character, decorative only, 3:4 portrait`。
- **接入**：生图后作 Layer1 增强（CSS 边框 + PNG 皮肤叠加），未生图则纯 CSS 边框。

### 4.2 机制 B：规格文档供 GPT-IMAGE-2

- 维护 `docs/卡牌系统/资源/卡牌边框生图提示词.md`：边框规范、配色 hex、尺寸、提示词模板、主题变体（节日/系列限定）。合成脚本 `docs/卡牌系统/资源/compose-card.py`、数值生成 `docs/GPT生图/gen-pet-stats.py`。
- GPT-IMAGE-2 按规格批量生成主题边框（春节/星空等）。
- 两机制互补：Agnes 出基础 4 档；GPT-IMAGE-2 出主题变体。

### 4.3 降级策略（图鉴立即修复不阻塞）

- **P0 纯 CSS 边框**（border + box-shadow + 渐变底纹），立即可用，不依赖生图。
- P2 边框 PNG 作增强皮肤叠加。
- 保证图鉴 P0 即修复"看不见"。

### 4.4 稀有度边界

保留稀有度边框，是为了：

- 让卡牌更好看
- 让图鉴更有层次
- 强化收藏成就感

不建议把稀有度继续扩成：

- 抽卡保底
- 强随机追逐
- 长线数值压制

本项目中，稀有度的产品角色应是“外观和收藏层次”，不是“沉迷驱动器”。

## 6. 验收标准

1. **图鉴可见**：未收集 grayscale+暗化（非 opacity 0.4），轮廓可见。
2. **卡牌式**：稀有度边框（CSS/PNG）+ 立绘（或 emoji 兜底）+ 四角数值（HP/ATK/DEF/SPD）+ 名称栏。
3. **数值系统**：147 只宠物都有 base_def/base_spd（rarity 单调+倾向修正），PetSystem getTotalDef/Spd 可取，旧档兼容。
4. **无图兜底**：classpet 等无图宠物显示 emoji，角标数值正常。
5. **不回归**：图鉴两级 + 详情模态正常，认养/小屋/战斗不受影响。
6. **双机制**：Agnes 4 档边框皮肤 + 规格文档产出。
7. **产品口径**：图鉴展示强调收藏和训练，不强化抽卡/战力攀比。

## 7. 风险与边界

- **classpet 40~80 只缺图**：emoji 兜底（§3.6），角标数值不受影响。
- **数值平衡**：def/spd 生成做分布校验（§2.2），避免 common 压过 rare。
- **不破坏战斗**：P1 仅加数据+展示，不改 battleTurn（P4 可选才改，且先定义 def 进 takeDamage / spd 首回合先手边界）。
- **产品边界**：数值展示服务轻训练营，不继续外扩成复杂卡牌养成系统。
- **边框图不阻塞**：P0 CSS 边框，P2 生图增强。
- **配色统一**：P0 同步 pets.json/CSS/方案三处（§3.3）。
