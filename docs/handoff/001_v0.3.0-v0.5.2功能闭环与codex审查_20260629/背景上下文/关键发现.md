## 3. 关键设计决策（值得 codex 审查）

1. **多孩子 swap vs profileId 前缀**：选 swap（业务零改动）而非前缀（改所有 localStorage 调用点）。权衡：swap 简单但切换时全量快照/恢复（性能 O(n) 键数）；前缀侵入大但无切换开销。
2. **战斗 CD 时序**：cdStartedThisTurn 跳过当回合 tick。需审查：多技能连用/道具后 tick 的边界。
3. **defending 一次性**：敌人反击减伤后清除。需审查：连续 defend/被未反击清除的边界。
4. **盲盒奖池硬编码 vs items.json rarity**：shop.js 用硬编码 4 层奖池（COMMON/RARE/EPIC/LEGENDARY），未动态读 items.json rarity。权衡：硬编码简单可控，但与 items.json rarity 可能漂移。
5. **feed homeContext 分流**：旧入口（feedPet）不传 homeContext 走 heal；home.js 传 homeContext 走扣分+饱食。需审查：两套语义是否清晰。
