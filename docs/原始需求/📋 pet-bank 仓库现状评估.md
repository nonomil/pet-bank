### 📋 pet-bank 仓库现状评估

> 评估基准：`2026-0628-宠物积分银行.md` + `pet-bank-全系统方案v2.md` + 当前代码 + 本地 `index.html` 渲染
>
> 结论：仓库已经不是“早期 demo”，而是一个功能面很宽的多模块系统；但与文档相比，短板主要仍在养成深度、家园承载和体验收口，不在“有没有功能”。

> 本轮补丁已把地图首页、探索入口、战斗弹窗和宠物终极阶段可达性补齐。

#### 1. 结构化需求拆分

|---|---|---|---|---|

| 宠物小屋 / 家园感 | 独立空间、摆设、家园承载 | 当前只有宠物页 | 未落地 | 这是原始需求里最明显的空白之一 |

#### 2. 已经落地的内容

- `index.html` 已经不是单页小 demo，而是 11 个 Tab 的多模块入口，主页、任务、宠物、探索、数学 PK、背包、图鉴、商店、工具箱都在。
- `data/items.json` 已整理到 26 个物品，覆盖消耗品、装备、材料三类。
- `data/scenes.json` 已有 12 个场景，数量上已经超过 v2 方案的 5 场景目标。
- `data/pets.json` 已有 147 种宠物，来源分布为 banchong 91、classpet 40、minecraft 8、original 8。
- `assets/pets` 已同步 Minecraft 宠物资源，当前可见 `poses/mc_*.png` 40 张和 `strips/mc_*.png` 8 张，总文件数 112。
- 首页首屏已在本地确认能渲染，截图不是空白页。

#### 3. 关键缺口

- `宠物小屋 / 家园感` 仍未落地，这会是和原始需求最明显的一块差距。
- `守护力` 现在还能再往深做一层：`hunger`、`intimacy`、`生病/恢复` 的自然衰减与关怀反馈还不够完整。
- `战斗系统` 目前能跑通探索 -> 遭遇 -> 结算，但技能面板、道具快捷栏和更细的战斗反馈还可以再补。
- `首页地图` 已经比早期 demo 更接近参考图，但还可以继续往“更像插画地图 / 节点更清晰”的方向收口。

#### 4. 和参考方案的差距

- 从“功能广度”看，当前仓库已经不比 v2 少，甚至更宽：宠物数据、场景数量、物品数量、Tab 页面都已经超过了 v2 的最小目标。
- 从“闭环稳定性”看，差距还明显：探索、图鉴、工具箱、宝箱、数学 PK 这些模块都已经存在，但彼此之间仍有不少接线问题，真正稳定串起来的只有任务积分、宠物基础养成和背包主链路。
- 从“参考对齐”看，v2 重点是 16 角色 × 5 阶段、5 场景、26 道具、一个顺滑的养成-探索-消费循环；当前仓库更像是把“模块库”先铺开了，再等接线收口。
- 这意味着差距不是“有没有做”，而是“能不能稳定演示一整套流程”。本地网页已经确认首页可打开，但切入核心页时会遇到运行时错误，这就是当前和参考方案之间最实际的距离。
- README 已同步改成“静态 HTML + 模块化 JS”，并补上 GitHub Pages 发布方式。

#### 5. 优先级建议

- 先修 P0：补齐家园/小屋入口、战斗技能与道具快捷体验、守护力自然衰减反馈。
- 再修 P1：补 `CardCollection.init()`、`ToolboxSystem.init()`、`petbank_tasks_completed_today` 的写入点、缺失物品 id 对齐。
- 然后修 P2：继续把地图首页做得更像“插画地图”，并把探索/遛弯/宝箱串成更顺滑的成长闭环。
