# 一期最小可用版

> 冻结日期：2026-07-05
> 当前口径：`minimal-v1`
> 目标：先把“多家长账号 + 多孩子 + 宠物小屋互访互动”做成一个用户能看懂、能跑通、能坚持使用的版本。

---

## 1. 为什么先收缩

当前家庭账号社交体系已经接上了很多能力，包括：

- 家长账号登录与注册邀请码
- 家庭创建、家庭邀请码、多个孩子同步
- 好友码、串门记录、轻互动
- 数学 / 汉字异步 PK
- 联调诊断、部署脚本、排障工具

但对一期试运行来说，信息量已经偏大。

如果首页、设置页和文档都同时暴露这些能力，家长很容易不知道：

- 现在到底先用什么
- 哪些是必须
- 哪些只是后面联调或二期能力

所以这一版先明确：

`先把“账号 + 家庭 + 多孩子 + 小屋互动”跑顺，再决定要不要继续放开 PK 和复杂联调工具。`

---

## 2. 一期只做什么

一期默认只聚焦这 5 件事：

1. 家长可以用邀请码注册 / 登录账号
2. 一个家庭下可以有 1 个或多个孩子
3. 多个孩子可以从本地档案同步到同一个家庭
4. 孩子之间可以看小屋、串门、打招呼、送花、一起遛弯
5. 宠物小屋背景保持可用，至少包含 `清晨阳光房` 和 `花园阳台`

一句话总结：

`先验证“多个孩子玩在一起，会不会更愿意坚持”这个核心假设。`

---

## 3. 一期暂不主推什么

以下能力先保留底层，不作为一期默认主流程：

- 数学异步 PK
- 汉字 / 识字异步 PK
- 复杂云端诊断面板
- 双设备联调取证工具
- 大量部署 / 运维脚本

这不代表这些代码没用，而是：

- 一期先不让普通家长先看到它们
- 先避免产品心智过重
- 等真实试运行需要时再打开

---

## 4. 一期主路径

推荐按下面这条路径理解和验收：

1. 家长 A 用注册邀请码创建账号
2. 家长 A 创建家庭
3. 家长 A 同步 1 个或多个孩子到家庭
4. 家长 B 注册并加入同一家庭，或者另一个家庭用好友码建立关系
5. 孩子进入宠物小屋，看到好友 / 家庭同伴，执行串门与轻互动
6. 家长观察孩子是否更愿意连续回来玩

---

## 5. 一期界面收口原则

为了让用户更容易看懂，默认界面按这个口径展示：

- 设置页优先展示：账号、家庭、孩子同步、好友码、串门互动
- 设置页默认不展开：复杂联调诊断、PK 相关难度设置
- 社交面板默认强调：好友码、串门、送花、一起遛弯
- 社交面板默认弱化：PK 权限和 PK 文案

---

## 6. 一期验收标准

满足下面这些，就算一期成立：

- 家长能成功注册 / 登录
- 家长能创建家庭并看到家庭信息
- 至少 2 个孩子可以同步到同一个家庭
- 至少 1 次串门互动能成功写入并回显
- 小屋背景 `清晨阳光房` / `花园阳台` 正常展示
- 用户在不理解 PK 和联调工具的前提下，也能独立跑通主流程
