# 一期最小可用版执行计划与任务清单

> 适用范围：`minimal-v1`
> 当前目标：先把账号、家庭、多孩子和宠物小屋互动做成一个简洁稳定的一期版本。

---

## 1. 当前已收口

- 已新增 `family-social-scope` 收口开关，默认进入 `minimal-v1`
- 已把设置页里的复杂联调诊断从一期默认界面隐藏
- 已把社交面板默认文案收口到“好友码、串门、轻互动”
- 已把 PK 相关前台控制从一期默认社交面板里隐藏

---

## 2. 当前主任务

### P0：先跑通用户主链路

- 验证注册邀请码可用
- 验证家长登录 / 退出
- 验证创建家庭
- 验证家庭下同步多个孩子

### P1：把互动主链路跑通

- 验证家庭内多个孩子可以互相显示为社交同伴
- 验证好友码可以建立跨家庭关系
- 验证 `打招呼 / 串门 / 送花 / 一起遛弯` 都能成功写入并回显
- 验证宠物小屋来访记录展示正常

### P2：把体验问题收平

- 验证 `清晨阳光房` 和 `花园阳台` 背景可见
- 验证设置页不会再把一期无关能力堆给普通用户
- 验证本地旧孩子档案导入路径可理解

---

## 3. 当前不作为一期阻塞项

以下内容先不阻塞一期：

- 数学异步 PK
- 汉字 / 识字异步 PK
- PK 公平性联调
- 诊断 JSON 导出 / 对比
- Supabase 运维脚本全量演练

这些内容仍然保留代码和文档，但先视为二期或联调期能力。

---

## 4. 还剩的真实硬门槛

即使产品范围收缩，一期仍然还有 3 个必须拿到的真实证据：

1. 一套真实 Supabase 项目可用
2. 至少一轮双家长 / 双账号实际操作记录
3. 至少一轮真实串门互动成功记录

---

## 5. 二期再决定的内容

等一期主链路稳定后，再决定是否继续打开：

- 数学 PK
- 识字 PK
- 更复杂的好友权限
- 云端诊断与排障工具
- 更完整的灰度运营与后台巡检
