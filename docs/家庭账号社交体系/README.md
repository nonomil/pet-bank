# 家庭账号社交体系

> 主题目标：把当前“单设备、多孩子本地切换”的养成产品，升级为“多家长账号、多孩子、多家庭互动”的家庭社交养成系统。
>
> 创建日期：2026-07-05
> 适用场景：评估账号体系、邀请码注册、家庭成员关系、跨家庭串门、数学/汉字异步 PK 的产品与技术落地路径。

---

## 1. 当前结论

- 当前仓库已经支持 **本地多孩子切换**：`js/profiles.js` 通过 `ProfileManager` 在 `localStorage` 中做 profile 快照切换。
- 这套能力能解决“同一台设备里多个孩子轮流玩”，但**不能**解决“多个家长账号、多家庭、跨账号互动”的目标。
- 一旦进入“孩子和别人家的宠物互动、串门、PK、留记录”的层级，系统必须有**云端真源**，不能再只靠浏览器本地数据。
- 推荐产品模型是：`家长账号 -> 家庭 -> 孩子 -> 宠物 -> 跨家庭好友/互动/PK`。
- 推荐技术路线是：**保留当前静态前端**，新增 **Supabase Auth + Postgres + RLS + Edge Functions** 作为轻后端。

## 4. 与当前仓库的关系

### 4.1 可以直接沿用

- `js/profiles.js`：可作为“本地孩子档案”与“首次云端导入”的桥接层
- `js/math-pk.js`：可改造成“共享题组 + 异步提交成绩”
- `js/hanzi-game.js` / `js/hanzi-progress.js`：可改造成“共享题组 + 异步识字 PK”
- `js/home.js`：可扩展“串门、留言、礼物、来访记录”

### 4.2 必须升级

- `localStorage` 不能再是真实数据源，只能做缓存或离线兜底
- 当前 `ProfileManager` 只认识“本地 profile”，未来要增加“云端 childId / householdId / memberId”语义
- 现有排行榜是“孩子自己的本地成绩”，未来要支持“好友对战记录”和“共享题组结算”

### 4.3 非阻塞但顺手可做的修复

- `清晨阳光房` 与 `花园阳台` 背景图已经接回 `js/home.js`，当前更值得优先补的是“真实 Supabase 联调、双设备恢复和访问边界验收”。
