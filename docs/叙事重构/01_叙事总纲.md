# 叙事总纲

> **定位**：这不是一个完整 galgame，也不是纯功能菜单。它是一个“成长积分 + 宠物养成 + 对话式探索”的家庭陪伴产品。
>
> **核心判断**：项目的叙事价值来自“孩子做对事情后，宠物和地图会发生变化”，而不是来自复杂剧情分支。

---

## 1. 这项目到底在干嘛

一句话：

**用积分驱动宠物成长，用探索承接故事，用兑换和小屋承接消费与归属感。**

这意味着它的重点不是“讲一个长篇故事”，而是建立一个稳定的产品感受：

- 完成任务，会得到积分
- 积分可以喂养、兑换、解锁
- 宠物会变强、会生病、会进化
- 探索会像一段小冒险，而不是纯弹窗
- 每次回来，世界都应该有一点变化

---

## 2. 叙事核心

### 2.1 主情绪

这个项目要传达的不是战斗感，而是：

- 被看见
- 被照顾
- 有一点冒险
- 有一点收集
- 有一点成长期待

### 2.2 主体验

用户打开产品时，应该感觉自己在进入一个“会慢慢长大”的世界，而不是一个工具堆。

### 2.3 叙事承载方式

- 首页负责“我现在有什么”
- 宠物负责“我在照顾谁”
- 探索负责“今天去哪里看看”
- 兑换负责“我能换到什么”
- 更多负责“补充功能和低频入口”

---

## 3. 产品边界

### 3.1 要保留的

- 成长积分
- 宠物养成
- 探索冒险
- 数学 PK
- 家园 / 小屋
- 商店消费

### 3.2 不要扩成的

- 不做完整恋爱式 galgame
- 不做多结局大主线
- 不做复杂对话树引擎
- 不做后端剧情状态机

### 3.3 适合的形态

最合适的是：

**绘本式世界观 + 对话式探索 + 少量选择 + 轻量数学谜题**

---

## 4. 这套叙事的成功标准

1. 孩子能看懂自己在做什么。
2. 每次探索都有“进入感”和“结束感”。
3. 角色和场景有记忆点。
4. 数学题不是割裂的，而是故事的一部分。
5. 视觉上比普通网页更像“一个小世界”。
