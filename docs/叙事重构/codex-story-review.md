# 成长伙伴·萌宠冒险岛 探索故事审查报告

审查范围：`data/stories/*.json` 12 个场景故事 + `docs/叙事重构/01_叙事总纲.md` ~ `05_事件规则.md`。

## ✅ 优点

- 整体风格统一，符合“轻童话 + 轻冒险 + 低压力”的总纲要求。像 `forest`、`beach`、`stargarden` 这类开场句都短，儿童可读。
- 多数场景都有明确视觉锚点，记忆素材不错。比如 `forest` 的发光蘑菇、`candy` 的棒棒糖树、`castle` 的彩虹桥、`stargarden` 的月亮树。
- 事件链基本完整，几乎每个场景都能走出 `narrate -> discover -> math -> choice -> encounter`，符合“探索有进入感和结束感”的方向。
- 数学节点数量稳定，且大多数都放在场景语义里，不是孤立题面。比如海边、海底、绿洲、青蛙、舱门密码，方向是对的。

## ⚠️ 问题

### P0

- `forest.math2`、`mountain.math2`、`space.math2` 缺少 `mathType` 字段。按 `05_事件规则.md`，数学节点至少要明确是 `arithmetic / word / logic`，否则后续渲染、题型分流和题库扩展都会断档。
- `12` 个场景的章节密度明显失衡：第 1 章只有 `forest`，第 4 章却塞了 `castle / cave / desert / mountain / volcano` 五个场景。这样会让前期过轻、后期过挤，违背 `03_章节结构.md` 里“每章有成长点和结算点”的节奏控制。

### P1

- 数学事件现在更像“门槛提示”，还不够像“题目本身”。例如 `beach.math2` 的“举着一块牌子，上面写着一道应用题考验你……”，`candy.math2` 的“挂着一道应用题牌子……”都能过场，但题面语义太泛，教育性弱，和 `05_事件规则.md` 里“题目与场景语义一致”还有距离。
- 结尾遭遇句型过于重复，记忆点被模板化削弱。比如 `beach`、`underwater`、`volcano`、`castle` 都是“出现一个身影/黑影/脚步声”，情绪一致但缺少各章独有的收束画面。
- 第 2、3 章场景氛围偏“漂亮”，但推进差异不够大。`beach -> candy -> underwater -> waterfall` 都在做“好看、发现、过路”的同一节奏，章节边界感不够强。

### P2

- 部分旁白句式偏模板化，建议减少“你发现了/前方有/突然有”的连续使用。当前文本短是优点，但重复开头会让孩子听久了觉得像同一套句子换皮。
- `choice` 事件的分支结果目前只在形式上分岔，叙事反馈还不够明显。比如 `forest.choice3`、`castle.choice3`、`desert.choice3` 选项很清楚，但缺少“选了以后世界变了什么”的即时回响。
- `math` 的难度梯度有，但没有明显章节教育目标。现在只是 easy/medium/hard 递增，还没形成“森林练加减、海边练应用题、高地练乘除、终点练逻辑”的稳定认知路径。

## 💡 优化建议

- 补齐 3 个缺失的 `mathType`，并按章节固定题型。示例：`forest.math2` 明确为 `arithmetic`；`mountain.math2` 保留 `arithmetic` 但把题面写成冰壁加减/乘法；`space.math2` 明确为 `logic` 或 `arithmetic` 中的一种，避免后续系统分流歧义。
- 把第 4 章拆成“高地洞窟主线 + 两个支线场景”或减少一个场景。示例：`castle -> cave -> mountain -> volcano` 四连已经够重，`desert` 可以挪到第 3 章做过渡，或合并为高地前哨。
- 给每章补一个独特的“收束意象”。示例：`forest` 结尾用“树洞里亮起一盏小灯”；`beach` 结尾用“贝壳拼成一条路”；`underwater` 结尾用“水母群给你让开”；`stargarden` 结尾用“星路完整点亮”。
- 把数学题写得更像场景事件，不是通用牌子。示例：`waterfall.math2` 可改成“青蛙守着 3 片荷叶，第一片有 2 颗露珠，第二片有 3 颗，第三片有几颗”；`candy.math2` 可改成“糖果树今天结了 2 串草莓糖，每串 3 颗，一共有几颗”。
- 减少重复的“黑影/身影/脚步声”，改成章内专属对手。示例：`castle` 用“披着旧披风的守门影子”，`volcano` 用“滚着火星的石怪”，`underwater` 用“泡泡里游来的灯鱼”，更容易记住。

## 优先级

- P0：补齐 `mathType`；调整章节密度，至少把第 4 章从 5 场压回更均衡的结构。
- P1：把数学题从“提示句”升级成“场景题面”；强化各章结尾意象和敌人辨识度。
- P2：继续压缩模板句式，补强 `choice` 的局部反馈，让每次探索的结束感更明确。
