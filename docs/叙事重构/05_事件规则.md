# 事件规则

> 这是探索层最关键的一份文档。它定义每个事件类型该怎么写、怎么显示、怎么推进。

---

## 1. 事件类型

### 1.1 `narrate`

用途：
- 纯旁白
- 介绍场景
- 过渡节奏

规则：
- 默认可点击推进
- 文字要短
- 适合开场和收尾

### 1.2 `discover`

用途：
- 发现道具
- 发现线索
- 发现气氛点

规则：
- 必须有一个“看见了什么”
- 可以给物品奖励
- 文字要带一点惊喜感

### 1.3 `choice`

用途：
- 提供二选一或三选一
- 让用户参与分岔

规则：
- 选项数建议 2 个，最多 3 个
- 每个选项都要有明确结果
- 不做大分支，只做局部反馈

### 1.4 `math`

用途：
- 把数学 PK 融入探索
- 作为解谜门槛或机关门

规则：
- 题目要像故事的一部分
- 对答错结果要有明确反馈
- 不要把数学题做成纯考试界面

### 1.5 `encounter`

用途：
- 引出战斗
- 让探索有一点风险感

规则：
- 先预警，再切战斗
- 文字要简短有力

---

## 2. 事件节奏

推荐顺序：

```text
narrate -> discover -> math -> choice -> encounter -> reward
```

不是每个场景都必须全套出现，但这是一条最完整的标准链。

---

## 3. 文案规范

- 每个事件最多 2-3 句
- 句子尽量短
- 儿童可读
- 不写长篇解释
- 情绪要明确

---

## 4. 数学事件规则

- 必须服务故事
- 可以奖励经验
- 可以作为开门、解锁、拿线索的条件
- 题目要与场景语义一致

例如：

- 森林：树上的算式
- 海边：贝壳上的题
- 山地：冰壁上的题
- 太空：舱门密码题
