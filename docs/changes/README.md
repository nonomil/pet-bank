# 变更索引

> 本目录不重复存放所有变更文件，而是负责回答一个实际问题：
> **“这次到底改了什么、为什么改、在哪个层级看最合适？”**

---

## 1. 三种“变更”文档的分工

| 类型 | 位置 | 用途 | 适合什么时候看 |
|---|---|---|---|
| 版本变更 | [../../CHANGELOG.md](../../CHANGELOG.md) | 版本级发布记录，回答“某个版本交付了什么” | 查版本发布、标签、资源修复 |
| 范围变更（CR） | [../方案/README.md](../方案/README.md) | 方案执行中为什么改范围、改优先级 | 查“为什么中途改方案” |
| 阶段进度 | [../进度/README.md](../进度/README.md) | 某一轮会话或阶段到底做了什么 | 查“这次具体做了哪些事” |

## 3. 推荐读法

- 想知道“当前项目已经发布过什么”：
  先看 [CHANGELOG.md](../../CHANGELOG.md)
- 想知道“为什么会出现某个范围调整”：
  再看 [../方案/README.md](../方案/README.md)
- 想知道“某一轮会话具体改了哪些文件和文档”：
  再看 [../进度/README.md](../进度/README.md)
- 想知道“下一步准备怎么做”：
  去看 [../plans/README.md](../plans/README.md)
