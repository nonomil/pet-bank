# 01-参考项目拆解

## 1. 班宠乐园 1 的安排

参考地址：[班宠乐园 1](https://banchong.cn/auth/login)

实测后，它的顶部主导航包含：

- `宠物大厅`
- `宠物小屋`
- `排行榜`
- `成长记录`
- `徽章`
- `小卖部`
- `老师工具箱`
- `班级管理`

它的特点是：孩子和老师高频看到的是主流程页面，真正的管理操作继续向内收。

### 小屋管理

`宠物小屋` 页面不是直接跳到后台，而是在当前页面里打开 `管理小屋 / 管理玩具` 弹层。

这个设计可以借鉴：  
小屋装饰、家具购买、玩具管理都属于“当前功能里的管理弹层”，不用单独占主导航。

### 小卖部管理

`小卖部` 里有 `兑换记录 / 班币总览 / 抽奖 / 拍卖结算 / 班币管理 / 管理商品`。

这个设计说明：  
商品和兑换相关管理，应该留在兑换/商店内部，不要放到游乐场或学习页。

### 老师工具箱

`老师工具箱` 里放：

- 宠物图鉴
- 随机点名
- 静音挑战
- 朗读挑战
- 作业消消乐
- 课堂计时器
- 监考时钟
- 远程喊话
- 抽奖游戏
- 荣誉证书
- 期末评语助手
- 宠物养育统计

它不是系统设置，而是“课堂辅助工具集合”。

### 班级管理

`班级管理` 再拆：

- 花名册
- 积分规则
- 等级规则
- 小组管理
- 标签管理
- 家长查看
- 徽章管理
- 小屋管理
- 科任老师
- 班级设置

这说明规则、学生、标签、徽章、小屋配置都应该属于更深一层的管理区。

## 2. 班宠乐园 2 的安排

参考地址：[班宠乐园 2](https://daydayupjfgl.com/parent)

实测后，它的顶部只有：

- `主页`
- `小卖部`
- `光荣榜`
- `工具`
- `设置`

主页里有一些批量动作：

- `一键分配领养`
- `一键锁屏`
- `批量管理`
- `编辑座位`
- `撤回模式`

设置页集中放：

- 账号名、密码、退出登录
- 系统名称、主题、音效
- 成长等级配置
- 加分/扣分规则管理
- 解锁密码
- 危险操作面板

工具页只放轻工具，例如随机点名、家庭留言墙。

## 3. 借鉴结论

两个参考项目虽然风格不同，但分层逻辑一致：

- 主导航给高频使用场景。
- 工具页放轻量辅助工具。
- 设置页放规则、账号、密码、成长等级。
- 危险操作继续折叠，不直接展示。
- 数据导出、批量管理、覆盖类操作不能和普通工具并列。

当前项目应该学习这个分层，而不是把所有入口都塞进 `游乐场` 或一个大杂烩 `工具箱`。
