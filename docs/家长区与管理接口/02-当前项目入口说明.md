# 02-当前项目入口说明

## 1. 顶部 `家长区`

`家长区` 是右上角的小入口，靠近孩子切换和成长分。它不是孩子日常玩的主导航，而是家长偶尔进入的管理区域。

当前它承接三个页面：

- `设置`
- `工具箱`
- `成长作品`

## 2. 设置

设置页当前承担最多管理职责。页面容器在 `index.html` 的 `page-settings`。

当前页面已经按家长心智拆成五个分区：

- `账号与孩子`：孩子档案、账号列表、增删改名。
- `家庭云端`：家长登录、家庭同步、好友与串门边界。
- `学习与题目`：学习打勾模式、数学 PK 难度等低频配置。
- `规则模板`：积分规则的规划占位，暂不开放批量编辑。
- `高级与危险操作`：诊断、备份、覆盖、导入导出等高风险能力的收纳区。

### 账号与孩子档案

由 `SettingsPage` 和 `ProfileManager` 管理，渲染在 `settings-account-list`。

主要能力：

- 新建孩子
- 孩子改名
- 删除孩子
- 当前孩子标记
- 和右上角孩子切换器联动

这部分相当于参考项目里的 `花名册 / 学生管理` 的家庭轻量版。

### 家长账号与云端配置

由 `AuthSystem` 管理，渲染在 `auth-root`。

主要能力：

- 登录家长账号
- 注册家长账号
- 保存 Supabase 云端配置
- 清空本地保存的云端配置
- 签发注册邀请码
- 退出云端账号

这部分是家庭账号体系的入口，不应该放在孩子主导航里。

### 家庭与孩子同步

由 `HouseholdSystem` 管理，渲染在 `household-root`。

主要能力：

- 创建家庭
- 同步当前孩子到云端
- 同步全部孩子
- 生成 / 撤销家庭邀请码
- 从云端导入孩子档案
- 用云端覆盖本地数据

其中 `用云端覆盖本地数据` 风险较高，后续建议折叠到高级区并加强确认。

### 社交与好友设置

由 `SocialSystem` 管理。

主要能力：

- 好友码
- 串门权限
- 小屋可见性
- 异步 PK 权限
- 好友互动记录

这部分是家长控制孩子社交边界的地方，不应该散落在游乐场和宠物页。

### 学习与数学配置

设置页还包含：

- `学习打勾模式`
- `数学 PK 难度`

这类配置影响学习和游戏体验，但修改频率低，适合放在家长区。

### 规则模板

规则模板当前是说明占位区。它用于承接后续的加分/扣分规则管理，但在没有角色权限、备份确认和规则编辑设计之前，不直接开放批量编辑。

### 高级与危险操作

高级区用于收纳云端诊断、备份恢复、覆盖类操作和数据管理说明。普通模式不默认展示 `数据管理`；需要临时启用时，使用 `?parentAdmin=1` 或本地高级开关，并先导出备份。

## 3. 工具箱

工具箱默认只显示轻量工具：

- `随机点名`
- `番茄计时`

它对应参考项目里的 `工具` 或 `老师工具箱`，但当前家庭版不应默认展示所有维护接口。

### 默认隐藏的数据管理

`数据管理` 包含：

- 导出全部 `petbank_*` 本地数据为 JSON。
- 从 JSON 导入并覆盖当前本地数据。

因为导入会覆盖数据，它已经被隐藏为高级接口。详见 [03-隐藏接口与权限边界](./03-隐藏接口与权限边界.md)。

## 4. 成长作品

成长作品页用于收集孩子作品与成长档案。

它不是危险接口，也不是系统设置，但属于低频家长查看内容，所以放在 `家长区`，不放在 `游乐场`。

后续建议把它做成：

- 本周作品
- 学习单精选
- 探索截图或宠物纪念
- 可导出的成长档案

## 5. 游乐场现在只管游戏

`游乐场` 下拉只保留：

- 游乐场首页
- 数学 PK
- 汉字游戏
- 卡牌对战
- 排行榜

这条规则很重要：  
以后新增孩子玩的小游戏，可以进游乐场；新增设置、规则、导出、账号，不要进游乐场。
