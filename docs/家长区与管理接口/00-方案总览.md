# 00-方案总览

## 1. 这次到底解决什么

之前 `游乐场` 里混进了 `成长作品 / 工具箱 / 设置`。孩子点进游乐场，本来期待看到数学 PK、汉字游戏、卡牌对战、排行榜，却会看到偏后台、偏家长管理的入口，心智很乱。

这次的目标不是删除这些功能，而是重新分层：

- 孩子高频玩的内容留在主导航和游乐场。
- 家长低频管理的内容收进右上角 `家长区`。
- 会覆盖数据或暴露诊断信息的接口默认隐藏。

## 2. 推荐信息架构

| 层级 | 入口 | 放什么 | 给谁看 |
|---|---|---|---|
| 主导航 | 首页 / 积分 / 学习 / 宠物 / 探索 / 游乐场 | 日常成长、学习、宠物、游戏 | 孩子 + 家长 |
| 家长区 | 设置 / 工具箱 / 成长作品 | 账号、孩子档案、家庭云端、轻工具、作品档案 | 家长为主 |
| 高级接口 | 数据管理 / 诊断 / 覆盖恢复 / 清空配置 | 备份、恢复、联调、危险操作 | 维护者 / 明确知道风险的家长 |

## 3. 为什么叫“家长区”

不推荐继续叫 `管理`。  
`管理` 听起来像后台系统，也容易让孩子觉得这里有很多不能碰的复杂东西。

`家长区` 更准确：

- 语气更柔和，符合儿童产品。
- 能包含设置、家庭账号、孩子档案、作品、工具。
- 不会把它误解成完整后台。

## 4. 隐藏不等于安全权限

当前项目是静态前端应用，很多数据还在浏览器本地和 Supabase 客户端侧。  
所以这里说的“隐藏接口”主要是产品信息架构隐藏，不是安全鉴权。

换句话说：

- 隐藏能防止普通用户误点。
- 隐藏能减少孩子看到后台入口。
- 隐藏不能替代真正的服务端权限判断。

如果以后做真实后台角色，需要再升级为 `parent / admin / child` 权限模型。

## 5. 当前 P0 已完成

- `游乐场` 下拉只保留游戏项。
- `works / tools / settings` 统一归属右上角 `家长区`。
- `工具箱` 默认隐藏 `数据管理`。
- `数据管理` 可通过 `?parentAdmin=1` 或本地开关打开。
- 新增回归测试，防止以后又把管理入口放回游乐场。

## 6. 当前 P1 已推进

- `设置` 页已经从单一面板拆成五个家长可理解的区域：`账号与孩子 / 家庭云端 / 学习与题目 / 规则模板 / 高级与危险操作`。
- 现有挂载点 `settings-account-list / auth-root / household-root / social-root / settings-learning-mode / settings-math-diff / diagnostics-root` 保持不变。
- `规则模板` 先作为说明区保留，后续再接入可编辑积分模板。
- `高级与危险操作` 明确提示 `?parentAdmin=1`，继续把数据管理和诊断类能力作为高级接口处理。
