# 调研：当前战斗系统全貌（卡牌对战前置调研）

> 调研日期：2026-06-29
> 目的：摸清现有 PVE 战斗系统的回合流程、数值参与、UI 结构、卡牌收集关系，
> 判断卡牌对战（PVP/PVE 复用）能否复用 `battleTurn` / 战斗 UI。
> 范围：只读调研，不改代码。

---

## 0. 结论速览（可复用 vs 需新建）

| 维度 | 现状 | 卡牌对战可复用性 |
|------|------|------------------|
| 回合逻辑 `battleTurn` | 单体 PVE，硬编码"1 玩家 + 1 怪物" | **部分复用**：伤害公式/技能/CD/defend 框架可抽离；但 `currentBattle.monster` 硬编码、`PetSystem.takeDamage` 单体调用，需参数化为"对战双方" |
| 战斗 UI（modal） | 左右对峙 + VS + HP 条 + 浮动伤害 + 战斗日志 | **高度可复用**：DOM 结构中立，已支持双方 HP 条；卡牌对战只需换双方立绘/数据源 |
| 数值系统 | atk 接入；**def/spd 未接入伤害**（注释明写 P4） | 需补 def 减伤 / spd 抢先后手，卡牌对战更敏感 |
| 技能系统 | 3 通用技能（攻/防/大），CD 机制完善 | **可复用**：技能定义表 + CD tick 框架中立，卡牌对战加卡牌技能即可 |
| 卡牌收集关系 | 战斗胜利 25% 概率掉卡（按稀有度加权） | 卡牌对战可对称掉卡（胜方掉败方卡碎片） |

## 1. 战斗核心：`js/exploration.js`

### 1.1 模块结构
- IIFE 模块 `ExplorationSystem`，对外暴露 `startBattle` / `battleTurn` / `endBattle` / `getCurrentBattle` 等（`exploration.js:460-479`）。
- 闭包内 `let currentBattle = null`（`exploration.js:30`）—— 全局唯一战斗实例。

### 1.2 触发链路
```
renderExplorePage / goExplore
  → ExplorationSystem.startExploration(sceneId)     (exploration.js:274)
       ├─ PetSystem.takeDamage(scene.hp_cost)        探索扣 HP
       ├─ scene.monsters 随机选一只                  (exploration.js:287)
       └─ return { battle: { scene, monster } }
  → app.js:1041  ExplorationSystem.startBattle(scene, monster)
  → app.js:1042  showBattleModal(battle)
```
另有 galgame 入口：`exploration-detail.js:277-278`（样板场景从 galgame 末事件 triggerBattle）。

### 1.3 `startBattle(scene, monster)` —— `exploration.js:300-318`
```js
currentBattle = {
  scene,                                    // 场景对象（含 danger_level/rare_drops）
  monster: { ...monster, current_hp: monster.hp },   // 深拷贝 + 当前 HP
  pet_max_hp: pet.hp,                       // 战斗开始时宠物 HP 快照
  log: [],                                  // 战斗日志 [{type,text}]
  turn: 1,
  status: 'ongoing',                        // ongoing|won|lost|fled
  chapter: arenaChapter                     // 章节号（决定 arena 背景）
};
```
开始时 `PetSystem.resetBattleState()` 重置 CD/defending（`exploration.js:303-305`）。

### 1.4 `battleTurn(action)` —— `exploration.js:323-454`（核心回合流程）

`action` 归一化为对象：`'attack' | 'flee' | {type:'skill',skillId} | {type:'item',...}`（`exploration.js:333`）。

```
battleTurn(action):
  if 战斗非 ongoing → return null
  battle.turn += 1
  cdStartedThisTurn = []   // 本回合刚启动 CD 的技能，回合末 tick 跳过

  ┌─ action.type === 'flee' ──────────────────────────────────────┐
  │  fleeChance = 0.3 (硬编码, 对齐 combat.json)                   │
  │  成功: status='fled', exp = monster.exp × 0.3                  │
  │  失败: 敌人白嫖一击 dmg = max(1, monster.atk - level/2)         │
  │        takeDamage(dmg, applyDefend:true)                       │
  │        可能 → status='lost'                                     │
  │  tickCooldowns → return                                        │
  └────────────────────────────────────────────────────────────────┘

  玩家行动 playerDmg:
  ├─ 'skill': getSkill → canUseSkill(CD校验)
  │     ├─ type='defend': setDefending(true), 不造伤, startCooldown(cd=3)
  │     └─ type='attack': playerDmg = max(1, floor(atk×mult) + rand(0..2))
  │                       monster.current_hp -= playerDmg
  ├─ 'item': 仅记日志（实际使用已在 app.js useItemInBattle 完成）
  └─ 'attack'(普攻): playerDmg = max(1, atk + rand(0..2) - 1)
                     monster.current_hp -= playerDmg

  ┌─ 胜利判定 monster.current_hp <= 0 ────────────────────────────┐
  │  status='won', addWin()                                        │
  │  exp = monster.exp + scene.danger_level×3 → addExp            │
  │  monster.drops[]: 每条 rate 概率 addItem                       │
  │  scene.rare_drops[]: 每条 rate 概率 addItem                    │
  │  tickCooldowns → return                                        │
  └────────────────────────────────────────────────────────────────┘

  敌人反击（普攻/技能攻击型/道具/defend 后均触发）:
     enemyAtk = max(1, monster.atk + rand(0..1) - 1)
     wasDefending = isDefending()
     ko = takeDamage(enemyAtk, applyDefend:true)   // defending 态 ×0.5 并清除
     if ko → status='lost'

  tickCooldowns(cdStartedThisTurn)   // 本回合刚启动的 CD 不递减
  return battle
```

### 1.5 三个伤害公式

| 公式 | 位置 | 表达式 |
|------|------|--------|
| 普攻（玩家→怪物） | `exploration.js:398` | `max(1, getTotalAtk() + rand(0..2) - 1)` |
| 技能（玩家→怪物） | `exploration.js:384` | `max(1, floor(getTotalAtk() × multiplier) + rand(0..2))` |
| 敌人反击（怪物→玩家） | `exploration.js:437` | `max(1, monster.atk + rand(0..1) - 1)` |
| 逃跑失败惩罚（怪物→玩家） | `exploration.js:348` | `max(1, monster.atk - floor(level/2))` |
| defending 减伤 | `pet.js:202-204` | `dmg = max(1, floor(dmg × 0.5))`，一次性清除 |

**关键：四条公式都不读 def / spd。** 仅 atk + 随机扰动。

### 1.6 胜负结算
- 胜利：`status='won'` + `addWin()` + exp + drops + rare_drops（`exploration.js:407-434`）。
- 失败：`status='lost'`，仅日志"💀 宠物倒下了"，**不扣分不掉物**（combat.json 的 `lose_consequences` 配置存在但未在代码实现）。
- 逃跑成功：`status='fled'` + 30% exp。
- `endBattle()` 把 `currentBattle=null`（`exploration.js:456-458`），由 `closeBattleModal`（`app.js:1317`）调用。

### 2.2 动作面板 `renderBattleActions`（`app.js:1094-1141`）
- 技能行：`grid-cols-2`，普攻 + `pet.skills` 各一按钮，CD 中禁用并显示 `(CD:n)`。
- 道具快捷栏：固定 4 槽 `BATTLE_ITEM_SLOTS = ['potion_small','potion_large','pet_food_basic','revive_potion']`（`app.js:1091`）。
- 逃跑行：单独一行。
- 全局锁 `battleUILocked`（`app.js:1092`）：玩家行动后→敌人反击前禁用所有按钮，防连点。

### 2.3 行动派发
- `battleAction(action)`（`app.js:1156-1217`）：归一化 → 锁 UI → `battleTurn` → `appendBattleLog` → 取最后一条伤害做 `showBattleDamage` → 胜负态显示结算按钮 / 否则解锁。
- `useItemInBattle(itemId)`（`app.js:1269-1312`）：`InventorySystem.useItem` → `battleTurn({type:'item'})`（道具消耗 1 回合，触发反击）。
- `showBattleDamage(dmg, target)`（`app.js:1237-1245`）：1.2s 上浮淡出，`target='pet'/'monster'` 决定左右。
- `handleBattleAnimate(e)`（`app.js:1247-1266`）：监听 `battle-animate` 事件，玩家攻击→怪物抖 `.battle-hit`，敌人攻击→宠物抖 + modal 红闪 `.battle-flash-red`。
- `updateBattleUI`（`app.js:1219-1234`）：仅更新两条 `.hp-fill` 宽度 + 注册动画监听。
- `closeBattleModal`（`app.js:1314-1332`）：解锁 → `endBattle` → 关 modal；galgame 入口走 `ExplorationDetail.showEnding/exit`，普通入口 `renderAll`。

**注意：`battle-animate` 事件在 exploration.js 中未见 dispatch（搜索无果），handleBattleAnimate 实际靠 battleTurn 内部？—— 复核：exploration.js 全文无 `dispatchEvent`，该动画监听当前是死代码 / 预留。**

### 2.4 HP 比例计算
- 宠物：`pet.hp / pet.total_max_hp`（`app.js:1076,1224`）—— 用 total_max_hp（含护甲）。
- 怪物：`monster.current_hp / monster.hp`（`app.js:1077,1227`）。

## 3. 宠物数值：`js/pet.js`

### 3.1 核心数值字段（`pet.js:56-77`）
```
state = {
  atk: 5,          // 基础攻击（选宠时覆盖为 species.base_atk, pet.js:119）
  def: 0,          // 注释明写"卡牌展示，P4 可选接入战斗"
  spd: 0,          // 注释明写"卡牌展示，P4 可选接入战斗"
  max_hp, hp,
  weapon: null,    // 装备槽（effect.atk）
  armor: null,     // 装备槽（effect.max_hp / effect.def 预留）
  skills: ['power_strike','defend','ultimate'],   // 已开放技能 id
  defending: false,
  ...
}
```

### 3.2 数值聚合函数
| 函数 | 位置 | 公式 | 是否接入战斗 |
|------|------|------|--------------|
| `getTotalAtk()` | `pet.js:325-331` | `state.atk + weapon.effect.atk` | **是**（battleTurn 行328） |
| `getTotalDef()` | `pet.js:334-340` | `state.def + armor.effect.def` | **否**（仅导出，伤害公式不读） |
| `getTotalSpd()` | `pet.js:343-345` | `state.spd` | **否**（仅导出，无抢先后手逻辑） |
| `getTotalMaxHp()` | `pet.js:418-424` | `state.max_hp + armor.effect.max_hp` | **是**（HP 条 / equip） |
| `takeDamage(amount,{applyDefend})` | `pet.js:200-209` | `hp -= dmg`；defending 态 ×0.5 并清除 | **是** |
| `heal(amount)` | `pet.js:212-215` | `hp = min(max_hp, hp+amount)` | 是（道具） |

### 3.3 def/spd 来源
- `pets.json` 物种字段：`base_def` / `base_spd`（`pet.js:502-504` 读取）。
- **但宠物初始 8 物种（pet.js:41-48）只有 `base_hp/base_atk`，无 base_def/base_spd** → 默认 0。
- 即便 pets.json 147 种里有 base_def/base_spd，也只落到 `state.def/spd` 做展示，**不参与任何伤害计算**。

### 3.4 装备（`equip`，`pet.js:427-440`）
- `item.slot === 'weapon'` → `state.weapon`（effect.atk 接入 atk）。
- `item.slot === 'armor'` → `state.armor`（effect.max_hp 接入 max_hp；**effect.def 预留未接入**）。
- 装备时若 max_hp 提升，差额补到当前 hp。

### 3.5 技能 / CD / defend
| 函数 | 位置 | 作用 |
|------|------|------|
| `getSkillsData()` | `pet.js:362` | 返回 data/skills.json |
| `getSkill(id)` | `pet.js:367` | 单技能定义 |
| `getSkills()` | `pet.js:372` | 玩家已开放技能 id 列表 |
| `getCooldown(id)` | `pet.js:377` | 当前 CD 剩余（0=可用） |
| `startCooldown(id,cd)` | `pet.js:382` | 启动 CD |
| `tickCooldowns(exclude)` | `pet.js:388-394` | 回合末递减所有 CD（跳过本回合刚启动的） |
| `canUseSkill(id)` | `pet.js:397-400` | 已开放 + CD≤0 |
| `resetBattleState()` | `pet.js:403-407` | 每场战斗重置 `cooldowns={}` + `defending=false` |
| `setDefending/isDefending` | `pet.js:410-415` | defend 技能标志 |

CD 存闭包变量 `cooldowns`（非 state），**不持久化**，每场战斗重置（注释 `pet.js:79`）。

## 6. 卡牌对战复用判断

### 6.1 可直接复用
- **战斗 UI 模板**（`showBattleModal` 的 DOM 结构 + CSS）：左右对峙/VS/HP 条/浮动伤害/日志，全部中立。卡牌对战只需换双方数据源（宠物→卡组）和立绘。
- **技能系统框架**（`data/skills.json` + `pet.js` CD tick + defend 标志）：定义表中立，加 `card_skills` 即可。
- **伤害公式骨架**：`max(1, atk×mult + rand)` 形式可直接用于卡牌对战，但需补 def/spd。
- **UI 锁 / 防连点**（`battleUILocked`）：回合制通用，直接复用。
- **浮动伤害 + 抖动动画**（`showBattleDamage` / `handleBattleAnimate`）：通用。

### 6.2 需重构 / 新建
- **`battleTurn` 去硬编码**：当前 `currentBattle.monster` 是单怪物，`PetSystem.takeDamage` 单宠物。卡牌对战需抽象为"双方 combatant"（各持 hp/atk/def/spd/skills），`battleTurn(action, side)`。
- **接入 def/spd**：卡牌对战对 def 减伤、spd 抢先后手更敏感。建议：
  - 改 `takeDamage(dmg)` 内部 `dmg = max(1, dmg - getTotalDef())` 或比例减伤；
  - `battleTurn` 开头按双方 spd 决定行动顺序（当前固定玩家先手）。
- **胜负结算分叉**：PVE 掉物/掉卡逻辑（`exploration.js:417-430` + `app.js:1193`）硬编码在 PVE 流程里，卡牌对战需独立的结算钩子。
- **`battle-animate` 事件当前是死代码**（exploration.js 无 dispatch），若卡牌对战要用动画需补发事件。
- **combat.json 配置漂移**：exp 公式、lose_consequences 未对齐，卡牌对战应新建配置而非复用。

### 6.3 推荐路径
**抽离 `battleTurn` 为通用回合引擎**（双方 combatant + 行动队列 + def/spd），PVE 和卡牌对战各传自己的 combatant 工厂；UI 复用 `showBattleModal` 模板，数据源参数化。技能表扩展为 `skills.json`（通用）+ `card_skills.json`（卡牌专属）。
