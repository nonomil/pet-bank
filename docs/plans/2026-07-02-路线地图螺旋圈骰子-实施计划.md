## 4. 执行约束

- 纯前端静态，零新 npm/外部依赖；CSS 动画用 transform 不触发重排。
- 数值参数化（骰子消耗/奖惩/骰子范围用 const，便于调参）。
- 失败扣分**必须** `deductGrowthPoints`（不可 `addGrowthPoints(-10)`，会截断）。
- 每 Step diff ≤200 行；超 200 行拆分。
- 执行结束回写本文件 steps 状态 + `handoff`/`review` 收口。
