# 2026-07-06 首页轻入口改版

## 背景

本轮首页调整目标不是继续堆模块，而是把首页收敛成一个更容易“想点开”的入口页。用户最新反馈集中在三点：

1. 旧版首页左侧绿色区域没有实际吸引力。
2. 中间大块任务/宠物仪表盘太重，文字太多，像管理后台。
3. 右上轮播消失后，首页失去了最直观的视觉吸引点。

## 本次改版方向

首页改为三段式：

1. 左侧：一句话引导 + 一个明确的“今天先做”入口 + 少量个人状态。
2. 右侧：恢复可点击轮播，用真实栏目图做主视觉，直接跳到对应页面。
3. 下方：保留 4 个代表性图片入口卡，不再铺长文介绍。

核心原则：

- 首页不是功能总表，而是启动页。
- 先展示想点的内容，再展示可管理的信息。
- 文字收短，图片优先，交互直接。

## 落地内容

### 结构调整

- 去掉首页主视觉里的重型 sidebar 感知，首页统一走全宽内容。
- 删除首页旧的“大标题 + 大任务卡 + 大宠物卡 + 宝箱仓库”组合。
- 改成左文案右轮播的双栏结构，下方仅保留 4 张图片卡。

### 轮播恢复

恢复首页轮播容器与脚本接入，并改成当前栏目的直达入口：

- 学习中心 -> `learn`
- 宠物伙伴 -> `pet`
- 探索冒险 -> `explore`
- 游乐场 -> `playground`
- 今日打卡 -> `today`

轮播支持：

- 自动轮播
- 左右切换
- 点击整张图跳转

### 文案收敛

删除不适合首页的长句：

- 不再使用“首页不用先铺一堆模块。先完成今天最重要的一件任务，再带上宠物去下一站冒险。”

改为更直接的首页语言：

- “先选一件开始，剩下的路就会亮起来。”

## 后续建议

如果继续打磨首页，优先级建议如下：

1. 换掉底部入口卡的临时图片，统一成同一套生成风格。
2. 把轮播图替换成更儿童向、更统一的栏目封面，而不是现有页面截图混搭。
3. 继续收首页左侧标题字号，做一版更偏一年级低龄感的轻量版本。

## 2026-07-06 回到原首页基础微调

用户继续反馈新版首页“框太大、信息量少、排版太简单”，整体不如原首页。基于这次反馈，本轮不再沿用“大左侧介绍 + 大右侧轮播 + 4 张图片入口卡”的 landing page 方向，而是回到原本更均衡的 dashboard 骨架，只做轻量微调。

调整原则：

- 保留原首页的项目感：顶部轮播、成长概览、同行宠物、快捷入口、宝箱仓库。
- 缩小视觉块体积，避免大色块和空面板占据首屏。
- 快捷入口回到 6 个明确模块，不再使用一堆“不知道干嘛”的图片卡。
- 首页恢复左侧任务/积分侧栏，保留“积分系统 + 宠物项目”的上下文。
- 轮播继续使用站内统一风格图片，不再使用外部蓝色网站截图。

本轮落地：

- `index.html`：首页恢复为 `showcase` + `map-hero` + `dash-launcher` + `map-warehouse-panel`。
- `css/showcase.css`：轮播恢复横幅比例，并调整为米色、森林绿、柔和光影体系。
- `css/style.css`：压缩首页 hero、宠物卡、入口卡和宝箱区尺寸，新增学习/游乐场入口卡色调。
- `index.html`：在原 hero 左侧补一个紧凑“今日建议”入口，增加真实任务信息密度。
- `js/app.js`：恢复首页侧栏，仅其他页面保留全宽；补齐原首页 `mapPoints`、`mapPetLevel`、`mapWins` 数据刷新。

新的判断：这个项目首页不适合做极简启动页。更合适的是“有一点游戏大厅密度的儿童成长 dashboard”，让孩子一眼看到宠物、积分、任务和去处，但不要把功能说明写得像后台。
