# 学习中心与幼小衔接暑假中文资料包设计稿

> **ID**：`DESIGN-2026-07-05-01`
>
> **状态**：已落地（v0.7.3）
>
> **目标**：在当前 `pet-bank` 静态前端项目内，新增一个可扩展的“学习中心”一级能力入口，先完整接入一套适合**幼小衔接暑假**的“暑假中文学习资料与计划”资料包，并与现有积分、档案和进度体系联动。

---

## 1. 背景

当前项目已经具备这些稳定能力：

- `任务 -> 积分 -> 宠物 -> 探索 -> 奖励` 的主循环
- 多孩子档案切换（`ProfileManager`）
- 主积分统一入口（`addGrowthPoints(delta)`）
- 汉字答题与学习进度的局部经验
- 纯前端静态站 + `data/*.json` + `localStorage` 的轻量架构

但它目前仍更像“成长打卡 + 游戏化冒险”，还没有一条完整的“学习主线”：

1. 没有一个可以长期接入不同学习资料包的统一入口。
2. 现有学习内容偏玩法化，不等于“成套学习资料与计划”。
3. 用户提出的“暑假中文学习资料”需求，本质上不是加一个小游戏，而是接入一个新的产品域：
   - 资料包
   - 学习计划
   - 日进度
   - 完成打卡
   - 进度汇总
   - 打印友好页面

所以这轮不适合把内容继续塞进现有 `hanzi` 页面，而应该新增一个**学习中心**，让项目从“会打卡”升级到“既能打卡，也能真正学”。

## 2. 核心产品判断

### 2.1 不是新仓库，而是“当前仓库内的独立学习子站”

本轮推荐方案不是新开独立项目，而是在当前 SPA 中新增“学习中心”产品域。

对用户来说：

- 它像一个新网站/新专区
- 可以从当前项目直接进入
- 学完后成长分和全局进度能汇总回主项目

对代码来说：

- 仍然复用现有 `index.html` + `switchPage()` + `localStorage`
- 不需要先拆仓库、拆部署、拆账号体系
- 后续若学习域单独成熟，再考虑物理拆分

### 2.2 第一版优先做“内置资料包”，外部网站作为扩展位

第一版的目标不是做一个学习网站导航页，而是做一个**真正能用的内置学习资料包系统**。

优先级：

1. 内置资料包：能学、能打卡、能计分、能汇总
2. A4 打印页：适合浏览器直接打印
3. 外部学习网站扩展位：后续接入
4. PDF 导出：第二阶段补

### 2.3 资料包平台先行，不把“暑假中文”做成一次性死页面

本轮虽然先落地“暑假中文学习资料与计划”，但架构层必须先支持多个资料包。

否则后面新增：

- 拼音启蒙包
- 英语晨读包
- 古诗积累包
- 数学口算包
- 外部学习网站导航包

都要重搭一遍。

## 3. 产品边界

### 3.1 第一阶段要做

1. 新增一级入口 `学习`
2. 新增学习中心首页
3. 新增通用资料包加载框架

## 4. 一级信息架构

### 4.1 顶部导航

当前顶部导航为：

- `首页`
- `积分`
- `宠物`
- `探索`
- `游乐场`

本轮建议升级为：

1. `首页`
2. `积分`
3. `学习`
4. `宠物`
5. `探索`
6. `游乐场`

理由：

- “学习”已经成为和“积分 / 宠物 / 探索”并列的主能力
- 不适合埋在“游乐场”里
- 也不该落到“更多”这种低频收口位

### 4.2 页面层级

学习中心建议新增 5 个页面：

| 页面 key | 用途 | 说明 |
|---|---|---|
| `learn` | 学习中心首页 | 总览、今日任务、资料包目录 |
| `learn-pack` | 单资料包主页 | 资料包介绍、模块入口、完成率 |
| `learn-plan` | 学习计划页 | 周视图 / 日视图 / 节奏总览 |
| `learn-lesson` | 学习内容页 | 晨读、识字、复习等单日内容 |
| `learn-print` | 打印友好页 | A4 风格排版，浏览器直接打印 |

叶子页统一归属 `learn` 一级 tab。

## 5. 资料包平台设计

### 5.1 总体思路

学习中心不是写死一个页面，而是通过：

- 一个总目录
- 多个资料包
- 每个资料包下多个模块
- 每个模块下多个 lesson 条目

去驱动页面渲染。

### 5.2 目录结构

推荐目录如下：

```text
data/
└── learn/
    ├── catalog.json
    └── packs/
        └── summer-chinese-bridge-2026/
            ├── manifest.json
            ├── plan.json
            ├── modules/
            │   ├── morning-reading.json
            │   ├── literacy-45days.json
            │   ├── weekly-review.json
            │   ├── poems.json
            │   └── classics.json
            └── assets/
```

说明：

- `catalog.json`：学习中心总目录
- `manifest.json`：单个资料包总说明与模块清单
- `plan.json`：整套学习计划的周节奏与日节奏
- `modules/*.json`：每个模块的 lesson 数据
- `assets/`：这个资料包自己的封面和插图资源

### 5.3 为什么不拆成 100+ 个 day 文件

虽然“每天一个 JSON 文件”很直观，但第一版会制造大量零碎文件，不利于快速落地和维护。

第一阶段推荐：

- 每个模块一个 JSON
- 模块内用 `lessons[]` 存 45 天 / 60 天条目

这样既保留了扩展性，也控制了文件复杂度。

## 6. 第一套资料包：幼小衔接暑假中文

### 6.1 资料包定位

资料包名称建议：

- UI 名称：`幼小衔接暑假中文资料包`
- 数据 id：`summer-chinese-bridge-2026`

适用对象：

- 幼小衔接
- 暑假场景
- 单次学习时间短
- 家长可陪同，也可孩子自己点读浏览

### 6.2 第一版必须包含的模块

1. `60 天晨读`
2. `45 天识字`
3. `每周复盘`
4. `古诗积累`
5. `经典短句 / 弟子规选读`

### 6.3 60 天晨读

每条 lesson 建议包含：

- `day`
- `title`
- `subtitle`
- `content`
- `pinyinContent`
- `focus`
- `estimatedMinutes`
- `tags`

目标：

- 每天 1 份短内容
- 内容轻、可朗读、可直接完成
- 支持“显示拼音 / 隐藏拼音”切换

### 6.4 45 天识字

每条 lesson 建议包含：

- `day`
- `chars[]`
- `pinyin[]`
- `words[]`
- `sentences[]`
- `reviewPrompt`
- `estimatedMinutes`

第一版重点不是做复杂互动题，而是把它先做成稳定的资料卡学习页。

### 6.5 每周复盘

面向家长和孩子共读：

- 这一周学了什么
- 哪些字掌握了
- 哪些内容要回看
- 下周重点是什么

### 6.6 古诗与经典短句

它们既可以作为晨读来源，也可以单独作为可打印资源集合。

## 7. 页面与交互

### 7.1 学习中心首页 `page-learn`

首屏建议展示：

- 今日学习任务
- 当前在学资料包
- 最近完成记录
- 学习累计贡献成长分
- 资料包目录卡片

### 7.2 资料包主页 `page-learn-pack`

展示：

- 封面
- 适用对象
- 学习目标
- 总天数
- 模块卡片
- 当前完成率
- 进入今日任务按钮

### 7.3 学习内容页 `page-learn-lesson`

必须支持：

- 上一天 / 下一天
- 标记完成
- 拼音显示切换
- 朗读型大字排版
- 当前已完成状态展示

### 7.4 学习计划页 `page-learn-plan`

建议提供：

- 周卡片视图
- 日序列视图
- 今日进度标记
- 补学提示

### 7.5 打印页 `page-learn-print`

第一版采用浏览器打印友好策略：

- A4 宽度排版
- 大字号
- 可显示拼音
- 每页底部预留打勾区
- 浏览器原生打印即可导出 PDF

这样可以尽快达到“可直接打印”的效果，而不用现在就维护 PDF 生成链路。

## 8. 进度与积分联动

### 8.1 存档键

建议新增：

| 键名 | 用途 |
|---|---|
| `petbank_learning_catalog_state` | 学习中心总状态 |
| `petbank_learning_progress` | 资料包/模块/lesson 完成进度 |
| `petbank_learning_rewards` | 已发放学习积分记录，防重复 |
| `petbank_learning_print_prefs` | 拼音显示等打印偏好 |

这些键仍然沿用 `petbank_*` 前缀，因此会自动被 `ProfileManager` 的快照切换机制纳入多孩子隔离。

### 8.2 进度模型

推荐结构：

```json
{
  "summer-chinese-bridge-2026": {
    "modules": {
      "morning-reading": {
        "completedLessons": ["day-01", "day-02"]
      },
      "literacy-45days": {
        "completedLessons": ["day-01"]
      }
    },
    "lastLessonId": "morning-reading:day-02",
    "updatedAt": 1780000000000
  }
}
```

### 8.3 计分规则

第一版建议简单、稳定、可防重复：

- 完成 1 天晨读：`+2`
- 完成 1 天识字：`+3`
- 完成当日整套计划：额外 `+1`
- 完成每周复盘：`+5`

所有加分只允许走：

```javascript
window.addGrowthPoints(delta)
```

### 8.4 防重复发分

`petbank_learning_rewards` 记录唯一奖励键，例如：

- `summer-chinese-bridge-2026:morning-reading:day-01`
- `summer-chinese-bridge-2026:literacy-45days:day-01`
- `summer-chinese-bridge-2026:daily-bundle:day-01`

一旦发过，就不能再次领取。

## 9. 与现有模块的关系

### 9.1 复用

- 档案隔离：`ProfileManager`
- 主积分：`addGrowthPoints()`
- 路由切换：`switchPage()`
- 页面风格：沿用当前项目的卡片、hub、页面容器

### 9.2 不复用的部分

- 不把学习中心强塞进 `hanzi-game.js`
- 不把资料包进度混进 `HanziProgress`
- 不把打印页塞进积分/作品页

### 9.3 后续和汉字系统的关系

本轮“45 天识字”先做成资料卡学习页。

后续第二阶段再考虑：

- 从识字卡跳转到 `hanzi` 专项练习
- 把识字 lesson 和 `HanziProgress` 复习状态更深度联动

这样可以避免第一版同时做“资料包平台”和“课程化识字引擎”的双重复杂度。

## 11. 验收标准

1. 顶部导航新增 `学习` 一级入口。
2. 学习中心首页能展示至少 1 个资料包卡片。
3. `幼小衔接暑假中文资料包` 可以进入资料包主页。
4. 资料包至少包含 `60 天晨读` 与 `45 天识字` 两个核心模块。
5. 每个模块都可以逐天打开 lesson。
6. lesson 可以标记完成，并正确写入进度。
7. lesson 完成后能通过 `addGrowthPoints()` 发放成长分，且不能重复刷分。
8. 学习进度在切换孩子档案后隔离生效。
9. 资料包有 A4 友好打印页，可直接浏览器打印。
10. 第一版不要求原生 PDF 生成，但打印页必须达到“家长能直接打印”的可用程度。

---

## 12. 一句话结论

**这轮不是给项目再塞一个“学习小游戏”，而是给 `pet-bank` 新增一个真正可扩展的“学习中心”产品域：先落地幼小衔接暑假中文资料包，再把未来更多学习内容和学习网站入口都纳入同一套资料包平台。**
