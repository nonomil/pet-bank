# Minecraft 单词学习视觉素材设计

## 目标

为主站 Minecraft 单词远征和独立 Anki 词卡工作台补齐必要的视觉素材，使“词库、学习节奏、练习选项、反馈结果”成为一个统一的学习产品，同时保持文字可读、资源体积可控和离线可用。

已有词卡图片优先复用，不为 11,000 余张卡片逐张重复生图。AI 图片只承担场景、氛围和交互状态的视觉表达；单词、中文、音标、短语、句子、按钮标签全部由 HTML/CSS 渲染。

## 视觉方向

选择“明亮方块世界学习营地”：天空蓝、草地绿、木材棕和矿物青作为场景色，使用现代儿童游戏插画与轻像素块面，不做暗黑洞穴主导，也不在图片内嵌文字、数字、Logo、水印或伪 UI。

主站和独立工作台共用素材语言，但信息密度不同：主站允许更强的场景氛围和陪伴感；工作台使用低对比度、低干扰背景，保证目录、搜索和卡片阅读优先。

## 素材包

### 主站 Minecraft 单词远征

1. `study-camp-hero`：首屏学习营地，预留左侧文字区和右下角宠物/卡片区。
2. `warmup-grove`：复习热身，带已点亮的词语方块和短路径。
3. `new-word-mine`：新词输入，明亮矿洞、矿物晶体和空白展示台。
4. `recall-bridge`：主动回忆，跨越方块桥，预留中央答题区。
5. `scene-village`：场景句，村庄、告示板和可放置物品，但告示板保持空白。
6. `reward-word-stars`：完成反馈，词语星、宠物伙伴和奖励宝箱，预留 HTML 奖励文案区。
7. `card-frame-sheet`：无文字卡片边框/角标/状态装饰；若生成结果不稳定，退回 CSS 边框，不强行接入。

### 独立 Anki 词卡工作台

1. `workbench-bg`：低干扰的浅色方块工作台纹理或远景，不能影响目录和正文对比度。
2. `detail-bg`：卡片详情区域的轻量纸张/地图纹理，可被正文面板覆盖。

实际词卡媒体继续来自 `prj/anki-minecraft-vocab/assets/media` 和主站已审查的 `assets/learn/english-vocab`。缺图时先使用现有 fallback，不自动向外站请求。

## 接入边界

- 主站素材放在 `assets/learn/english-vocab/generated/`，使用语义文件名和 `manifest.json`。
- 独立工作台素材放在 `prj/anki-minecraft-vocab/assets/ui/`，不把原始 Anki 媒体重新复制到 Pages 制品。
- `js/minecraft-vocab-page.js` 只新增素材引用和状态类，不改变 11 步队列、Profile 进度或奖励接口。
- `css/minecraft-vocab.css` 负责背景叠加、卡片状态、响应式裁切和无障碍对比度；不得把文字烘焙进图片。
- `prj/anki-minecraft-vocab/styles.css` 只使用工作台低干扰素材，目录/搜索/详情仍保持独立布局。
- 生图 prompt、原始响应、质量检查和发布清单与运行时图片分离；不提交密钥、响应中的授权信息或临时目录。

## 生成和质量门禁

使用 TOKEN24 的 OpenAI 兼容 `gpt-image-2` 接口，统一生成无文字 PNG/WebP。每张图必须检查：文件头可解码、实际尺寸、色彩模式、无明显文字/Logo/水印、可作为 CSS 背景裁切，且在 390px 与桌面宽度下不遮挡正文。

生成失败不能写入运行资源；保留已有素材和 fallback。透明素材必须额外检查 alpha；不可信的“透明背景”描述不能直接接入。

## 验证

1. 静态质量检查：manifest、图片尺寸、PNG/WebP 文件头和发布白名单。
2. 主站浏览器流程：学习中心入口、远征首页、四阶段卡片、选择/输入选项、完成奖励、桌面与移动端无横向滚动。
3. 独立工作台浏览器流程：目录折叠、搜索、卡片详情、图片/音频加载、移动端目录抽屉。
4. 回归检查：Minecraft 内容完整性、学习中心 smoke、Anki Python 测试、Pages 制品和全量回归。

## 非目标

- 不为每一张词卡重新生成专属插图。
- 不把 AI 图片中的文字当作可交互 UI。
- 不改变 Anki 数据结构、词卡字段、学习积分账本或 Profile 快照边界。
- 不把完整 Anki 工作台或未审查外部媒体复制进主站发布制品。
