## 4. 功能性需求（FR）

### FR1 · 螺旋圈布局（从外到内）
- 12 个场景沿「从外圈到中心」的螺旋路径排布，替代当前手写散点折线。
- 视觉上能看出「圈层 + 旋臂」结构，章节顺序（第 1 章 → 第 12 章）沿螺旋走向递进。
- 当前角色位置、已通关、已解锁、未解锁四种状态在螺旋节点上仍可区分。

### FR2 · 双入口同步
- 首页 `#sceneGridMap` 与探索 tab `#sceneGrid` 渲染同一螺旋地图，布局一致。
- 两处的差异（如探索 tab 可能有当前选中态高亮）需保留，不因布局改造丢失。

### FR3 · 骰子推进机制（与按序并存）
- 保留现有「按序点选进入场景」模式。
- 新增「消耗积分扔骰子」模式：扔出 1–6 点 → 角色按点数沿螺旋前进 N 步 → 落入目标场景。
- 两种模式用户可选用（具体触发入口见 §6 待澄清）。

### FR4 · 积分奖惩
- 骰子模式落入场景后，依结果结算：**成功 +积分 / 失败 −积分**。
- 「成功 / 失败」的判定口径、积分增减数值见 §6 待澄清（候选：沿用战斗胜负 / 场景难度检定）。

### FR5 · 与既有机制兼容
- 骰子推进不得破坏现有：HP 扣减、战斗触发、物品掉落、场景解锁（`unlock_cost`/`min_level`）、章末精英怪逻辑。
- 未解锁场景被骰子落入时的处理（弹窗提示 / 弹跳回退 / 自动解锁）见 §6。
