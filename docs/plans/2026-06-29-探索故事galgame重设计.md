## 2. Galgame 风格设计

### 2.1 整体布局
```
┌─────────────────────────────────────┐
│  [背景图 全屏]                       │
│                                       │
│  [左立绘]            [右立绘]         │  ← 事件角色/宠物立绘（左右）
│                                       │
│                                       │
├─────────────────────────────────────┤
│  🍄 蘑菇仙子                    ▶   │  ← 底部对话框：角色名 + 文字 + 推进
│  你小心翼翼地走进森林，脚下的落叶      │
│  发出沙沙的声音……                    │
│  [选项1] [选项2]                      │  ← choice 时弹出选项（可选）
└─────────────────────────────────────┘
```

### 2.2 核心元素
- **底部对话框**（`.galgame-box`）：固定底部，半透明深色（rgba(0,0,0,0.75) + blur），圆角顶部，min-height 30vh
  - 角色名（`.galgame-name`）：金色/品牌色，16-18px 粗体
  - 文字（`.galgame-text`）：白色，**18-20px**，line-height 1.8，可加逐字打字机效果
  - 推进指示（`.galgame-next`）：右下角 ▶ 闪烁，点击对话框任意处或 ▶ 推进
- **左右立绘位**（`.galgame-portrait-left/right`）：事件角色立绘（forest=蘑菇仙子/beach=海鸥船长/...），淡入淡出切换
- **选择卡片**（`.galgame-choices`）：choice 时从对话框上方弹出 2 个选项卡片（galgame 风：半透明白底 + 边框 + hover 高亮）
- **宠物立绘**（可选）：玩家宠物常驻一侧（用 PetSystem.getCurrentStageImage）

### 2.3 事件类型映射
| 事件 | galgame 呈现 |
|------|--------------|
| narrate | 对话框显示旁白（无角色名，或"旁白"）+ ▶ 推进 |
| discover | 左/右立绘显示发现物（🍄）+ 对话框显示文字 + "获得物品"高亮 |
| choice | 对话框显示情境 + 弹出选择卡片 → 选中后对话框显示结果 |
| encounter | 立绘切换为怪物 + 对话框红字警告 + ▶ 触发战斗 |

### 2.4 战斗触发
encounter ▶ → triggerBattle → showBattleModal（战斗 modal 覆盖 galgame 对话框，战斗后回 galgame 或 exit）

## 3. 实施方案（exploration-detail.js + style.css）

### 3.1 exploration-detail.js 改造
- `show()`：渲染 galgame 框架（背景 + 左右立绘位 + 底部对话框），不再 events 堆叠
- `showNextEvent()`：**单条**渲染到对话框（非 innerHTML += 堆叠），事件类型映射立绘/文字
- `choose()`：弹出选择卡片 + 结果回对话框
- `next()`：点击对话框/▶ → 推进（单条替换）
- 加打字机效果（可选，文字逐字显示）

### 3.2 CSS（galgame 对话框 + 立绘）
```css
.galgame-box { position: fixed; bottom: 0; left: 0; right: 0; min-height: 30vh;
  background: rgba(0,0,0,0.78); backdrop-filter: blur(10px);
  border-top: 2px solid rgba(255,210,120,0.4); border-radius: 20px 20px 0 0;
  padding: 20px 28px; z-index: 10; cursor: pointer; }
.galgame-name { color: #ffd87a; font-size: 17px; font-weight: 700; margin-bottom: 8px; }
.galgame-text { color: #fff; font-size: 19px; line-height: 1.8; }
.galgame-next { position: absolute; right: 24px; bottom: 16px; color: #ffd87a;
  font-size: 20px; animation: blink 1.2s infinite; }
.galgame-portrait-left, .galgame-portrait-right { position: fixed; bottom: 30vh;
  max-height: 50vh; transition: opacity 0.4s; }
.galgame-portrait-left { left: 4vw; } .galgame-portrait-right { right: 4vw; }
.galgame-choices { display: flex; flex-direction: column; gap: 10px; margin-top: 12px; }
.galgame-choice { background: rgba(255,255,255,0.12); border: 1px solid rgba(255,210,120,0.5);
  border-radius: 10px; padding: 12px 16px; color: #fff; font-size: 17px; cursor: pointer; }
.galgame-choice:hover { background: rgba(255,210,120,0.25); }
@keyframes blink { 50% { opacity: 0.3; } }
```

### 3.3 字体优化
- 故事文字 14px → **19px**
- 角色名/选项 17px
- line-height 1.8（宽松易读）

## 8. 开源 VN 引擎调研 + 借鉴（web 搜索）

### 8.1 主流开源 VN 引擎
- **[Monogatari](https://monogatari.io/)**（[GitHub](https://github.com/Monogatari/Monogatari)）：纯 web VN 引擎（HTML5+CSS3+JS），Ren'Py 语法，**最契合本项目纯前端定位**，免费开源可商用
- **[Ren'Py](https://www.renpy.org/)**：最主流 VN 引擎（Python），2026-05 起 HTML5/WebAssembly（beta）。功能最全，但偏 Python 生态
- itch.io / GameMaker / Godot 也有 VN 社区资源与教程

### 8.2 VN UI 最佳实践（搜索结论）
| 维度 | 最佳实践 |
|------|---------|
| 对话框 | 高对比可读；多场景多风格（旁白/对话/内心独白）|
| 说话人指示 | nameplate + 色彩编码 + 立绘高亮 + 非说话人 dim |
| 立绘 | scaling/positioning/轻微动画/镜头移动让静态角色生动；全身立绘肢体语言更丰富 |
| 选择 | 清晰 hover 反馈 + 逻辑分组 + 不可选项 disable/隐藏 |
| 参考工具 | [LlamaGen galgame UI 生成器](https://llamagen.ai/zh-tw/text-to-image/galgame)（AI 生成 VN 风格 screen，灵感参考）|

### 8.3 借鉴结论
本项目纯前端（Vanilla JS），**借鉴 Monogatari 的呈现模式**（底部对话框 + 立绘 + 选择 + 推进 + 历史 log），但**不引入 Monogatari 引擎**（避免依赖）。自实现轻量 galgame 呈现层（exploration-detail.js + CSS ~120 行）。

## 9. 数学 PK 融入探索（解谜设计）

### 9.1 概念
把数学 PK 从独立 Tab **融入探索**，作为**解谜节点**（galgame 式题目卡片）。探索故事中遇数学谜题，答对奖励/解锁，答错触发战斗或扣 HP。把"智力关卡"（解谜）与"体力关卡"（战斗）交替。

### 9.2 新事件类型 `math`（sceneEvents）
```js
{ type: 'math', text: '一扇石门挡住去路，门上刻着古老算式……', difficulty: 'easy',
  reward: { exp: 20, item: 'crystal_shard' }, failAction: 'battle' }
```
- 呈现：galgame 对话框显示情境 + 数学题卡片（算式 + 输入/选项）
- 答对：奖励（exp/item）+ 推进故事
- 答错：按 failAction 触发战斗 / 扣 HP / 重试

### 9.3 难度分级（复用 math-pk.js 出题）
- easy：加减（探索前段：forest/beach/waterfall）
- medium：乘除（中段：mountain/candy/castle）
- hard：混合/应用题（后段：space/volcano/stargarden）

### 9.4 融入探索流程
`narrate → discover → math(解谜) → choice → encounter(战斗)`
- 复用 math-pk.js 的出题/判定逻辑（不重写）
- 数学 PK 独立 Tab 保留（连击/练习），探索解谜是"融入式"

### 9.5 解谜 UI（galgame 风题目卡片）
- 对话框上方弹出题目卡片（半透明，大字算式）
- 输入答案 / 多选选项
- 答对：金色光效 + 奖励 toast；答错：红光 + failAction
