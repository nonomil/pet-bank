# 宠物积分核心循环 v2 设计

## 目标

让孩子在完成任务、游戏、宝箱或小屋动作后，立即看懂“得到了什么、宠物变成什么样、下一步做什么”，并能在第二天继续回来。

## v2 范围

### N1 统一奖励结果卡

新增与 DOM 无关的奖励结果模型和一个轻量 UI 渲染器。输入来自 `CoreRewardService.claim()` 的结果，统一展示成长分、宠物经验、亲密度、道具、升级/进化和下一步建议。渲染器只负责反馈，不改变发放逻辑。

覆盖顺序：宝箱、游戏结果、任务完成、小屋动作。无法接入的旧路径保留原 toast/alert 作为降级。

### N2 任务可撤销兼容事件

任务勾选继续允许撤回积分，但每次勾选和撤回都写入独立操作记录：完成是正向事件，撤回是补偿事件。事件 ID 包含任务、日期、操作序号，不能把“撤回后再次完成”误判为重复领取。

### N3 成长日志与回访提示

使用独立的本地日志保存最近 30 条成长事件，不复制完整业务状态。首页和小屋显示最近一次成长、连续照料天数和下一成长目标。日志不可用时不阻断主流程。

### N4 浏览器验收

增加无浏览器 DOM 契约测试，并用现有本地网页入口做一次桌面/移动宽度检查。重点检查卡片位置、溢出、刷新持久化和旧存档兼容。

## 不做的事

- 不修改现有积分奖励数值。
- 不增加每日积分上限。
- 不重做游戏玩法和词库。
- 不把任务撤回语义改成不可逆扣分。

## 验收标准

- 四类来源都能生成统一奖励结果模型，至少有降级反馈。
- 同一正向事件重复提交不会重复发放；任务撤回和再次完成可区分。
- 奖励结果能显示宠物经验和升级/进化结果。
- 刷新后成长日志和照料连续天数仍可恢复。
- 现有 v1 专项测试和 Pages/路由/边界测试继续通过。
