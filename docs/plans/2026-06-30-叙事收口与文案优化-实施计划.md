# 叙事收口与文案优化实施计划

> **给 Claude:** 必需子技能：使用 `superpowers:executing-plans` 来逐任务实施此计划。

**目标：** 把探索故事收口成单一正式来源，并完成 12 场景数学场景化、章节教育目标和文档同步。

**架构：** 以 `data/stories/*.json` 为唯一正式故事文本源，把 `chapter_skill` 与 `ending_text` 并入场景数据，运行时只负责读取和渲染，不再长期维护硬编码故事文本。实现顺序采用“先数据、后运行时、再文档”的收口策略，避免边改边漂移。

**技术栈：** `Vanilla JS`、`JSON`、`Markdown`、GitHub Pages 静态部署、本地 HTTP server 冒烟验证。
