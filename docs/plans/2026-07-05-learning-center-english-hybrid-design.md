# 学习中心混合式英语资料包设计稿

> **ID**：`DESIGN-2026-07-05-EN-01`
>
> **状态**：待实施
>
> **目标**：在当前 `pet-bank` 学习中心内新增一套“混合式英语资料包”能力，让孩子在当前项目里选课、看计划、拿积分，在外部网站里完成点读或章节阅读，并把进度和成长分稳定汇总回当前项目。

---

## 1. 背景

学习中心已经完成两种基本形态：

1. `内置资料包`
   - 例：幼小衔接暑假中文资料包
2. `网站入口型资料包`
   - 例：幼小衔接学习网站入口包

用户当前提出的新需求不是“再加一个入口卡”，而是：

- 在当前项目里形成一条完整的英语学习主线
- 支持章节式学习
- 支持学完计分
- 不排斥外部点读内容
- 未来还能接入更多英语网站和自定义资料包

因此这轮最合适的不是纯内置或纯跳转，而是**混合式英语资料包**。

## 2. 核心产品判断

### 2.1 推荐方案：B. 混合式资料包

推荐方案不是：

- 把英语内容完全复制进当前仓库
- 也不是只在学习中心挂一个普通外链

而是：

- 当前项目承担“学习壳”
- 外部网站承担“章节点读页”
- 资料包完成情况、成长分、复盘仍保留在当前项目

### 2.2 为什么这条路最适合当前阶段

当前项目已经具备：

- 学习中心总入口
- pack / module / lesson 数据驱动结构
- 进度和奖励防重复
- 多孩子隔离
- “读完打勾”与成长分联动

所以英语第一阶段不需要再发明新底座，只需要把 lesson 来源扩展为：

- 内置内容
- 外部章节
- 未来的自定义内容

## 3. 外部来源判断

截至 `2026-07-05` 的静态检查结果：

- `https://mayihaoke.com/` 与 `/mcbook56/read` 返回 SPA 壳
- 页面 HTML 本身不直接包含章节正文
- 站点前端 bundle 中能看到这些路由线索：
  - `/mcbook56/read/:chapter?`
  - `/mcbookstarters/read/:chapter?`
  - `/mcbook78/read/:chapter?`
  - `/cambridge-vocab`

这说明：

1. 外部站点确实具备章节化阅读结构。
2. 适合被当作章节阅读来源。
3. 不适合在第一阶段做“自动采集重建内容”。
4. 当前项目第一阶段应采用“打开外部章节 + 回来打勾”的稳妥方案。

---

## 4. 第一阶段产品边界

### 4.1 要做

1. 新增一套英语资料包
2. 在学习中心首页能看到英语入口
3. 资料包可进入计划页、lesson 页、进度页
4. lesson 页能打开外部章节
5. 学完回来可打勾得分
6. 为自定义资料包保留统一字段和 runtime 入口

### 4.2 明确不做

- 不自动判定外站是否读完
- 不采集外站全文入库
- 不做登录态联动
- 不做开放式后台上传界面
- 不做复杂插件系统

## 6. Lesson 体验设计

### 6.1 单节结构

一节英语 lesson 由两部分组成：

1. 当前项目内的学习壳
   - 标题
   - 目标词
   - 家长提示
   - 读完打勾
   - 积分说明
2. 外部章节入口
   - 打开外部点读页
   - 返回后手动打勾

### 6.2 为什么仍然保留当前项目内的 lesson 页

因为 lesson 页不是“多余的一层”，而是英语学习真正被纳入当前系统的关键：

- 没有 lesson 页，英语只剩下网站跳转
- 有了 lesson 页，英语就能进入：
  - 今日推荐
  - 我的进度
  - 成长分
  - 最近完成记录

## 8. Runtime 扩展判断

第一阶段不需要完整插件系统，但需要把 `LearnCenter` 内部判断从“只看 module.type”升级为“同时看 source 类型和 adapter”。

推荐在 runtime 中逐步引入：

```javascript
resolvePackCapabilities(manifest)
resolveLessonSource(pack, module, lesson)
resolveLessonLaunchUrl(pack, module, lesson)
resolveLessonReward(pack, module, lesson)
renderLessonSupportCard(pack, module, lesson)
```

这样可以在不推翻当前学习中心的前提下，逐步支持更多 pack 来源。

## 9. 奖励规则建议

第一阶段建议：

- 故事章节：`+2`
- Starters 章节：`+2`
- 连续 3 章奖励：`+1`
- 每周英语复盘：`+5`

这些规则有三个目的：

1. 与当前中文学习分值同量级
2. 保证英语资料包有成长感
3. 不至于成为刷分捷径

## 10. 风险与应对

### 10.1 风险：外部站点页面结构变化

应对：

- 当前项目只依赖明确 URL，不依赖全文抓取
- 后续如站点改版，仅调整 `routePattern` 或 lesson URL

### 10.2 风险：iframe 不稳定或跨域受限

应对：

- 第一阶段默认使用新窗口 / 当前窗口打开
- 不把 MVP 建立在 iframe 成功的前提上

### 10.3 风险：写死某个站点，后续难扩展

应对：

- 文档先定义 `packType` 与 `sourceAdapter`
- runtime 先做最小映射层

## 12. 成功标准

英语学习这一轮设计成功，至少要满足：

1. 学习中心新增英语资料包入口。
2. 英语 lesson 可以稳定打开外部章节。
3. 回来打勾能够稳定计分且不重复发分。
4. 英语进度可进入当前“我的进度”。
5. 数据结构可继续承载外链包和自定义 pack。
