# 探险终端首页设计

## 设计决策

采用 A 方案：一个统一的“探险终端”首页，三张地图共享同一套入口壳层。切换世界时只替换地图背景、节点、宠物状态和主题色，不改变按钮位置和主要交互。

## 当前问题

现有首页仍然像普通业务页面：顶部导航、侧边信息、标题、切换条、地图卡片多层叠加，地图没有成为主角。三张地图虽然可以切换，但视觉上像三个嵌在页面里的模块，缺少游戏主菜单的连续感。

## 目标结构

```text
顶部：孩子身份 / 积分 / 家长入口
主区域：当前世界的全屏地图预览
顶部浮层：森林 / 科幻 / 方块三个大型世界入口
左侧浮层：宠物头像和少量状态
右侧浮层：成长进度和当前世界进度
底部浮层：继续冒险主按钮 + 侦探小游戏辅助入口
```

地图画布必须占据主要视觉面积。首页不再出现重复的页面标题、状态卡、说明段落、刷新路线按钮或第二套世界 Tab。

## 视觉方向

- 基础气质：明亮、科技感、儿童游戏主菜单，不使用旧式棕色像素面板。
- 统一外壳：深蓝到青蓝的空间终端底色、浅色 HUD、珊瑚和暖黄色行动强调色。
- 森林：阳光、溪流、荧光路线、森林绿和青绿色科技导航。
- 科幻：明亮星空、青蓝空间站、玻璃穹顶、橙黄色推进器光源，避免阴暗赛博朋克。
- 方块：明亮村庄 + 地下水晶洞窟，保留方块识别度，但使用更清晰的高亮节点和进度线。
- 生图只做构图和氛围参考；真实中文、积分、节点名称、按钮状态全部由 HTML/CSS 渲染。

## 交互约束

- 三个世界入口始终位于地图上方或地图顶部浮层，位置固定。
- 当前世界通过高亮边框、指示箭头和明确的 `aria-selected` 表达，不只依赖颜色。
- “继续冒险”是唯一主行动；侦探小游戏是次级入口，不作为第四张主地图。
- 地图节点直接进入故事章节；地图分页、解锁和完成状态沿用现有数据契约。
- 进入故事后继续使用全屏舞台，不显示首页导航和世界选择器。

## 响应式

- 桌面端：地图占满内容区，左右状态面板悬浮在地图边缘。
- 手机端：世界入口变为横向滚动的视觉卡片；左、右状态面板折叠为顶部或底部紧凑条；地图保持可缩放/可滚动，不产生页面级横向溢出。
- 主按钮和世界入口保持稳定尺寸，不能因标题或完成状态改变布局。

## 验收标准

- 用户进入首页即可看到当前地图，而不是先看到多组说明卡片。
- 三个世界入口一眼可见且切换位置不变。
- 三张地图在同一套壳层中有明显主题差异，但不会让用户重新学习导航。
- 桌面 1440px 和手机 390px 均无横向溢出；地图、按钮、宠物和进度信息不互相遮挡。
- 生图中的伪文字、伪数字和装饰性按钮不直接进入产品 UI。
