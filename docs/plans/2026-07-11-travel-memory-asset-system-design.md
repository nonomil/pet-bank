# 旅行纪念资产系统设计稿

**状态：** 方案阶段，尚未生成或接入新的图片资产
**日期：** 2026-07-11
**范围：** 旅行纪念徽章、旅行纪念卡、冰箱贴、宠物旅行卡牌，以及它们在网页中的收集闭环

## 1. 目标与边界

孩子完成一次探索后，不只看到一段结算文字，而是能看到宠物带回来的具体东西：一枚徽章、一张卡片或一枚冰箱贴。纪念物要推动三个动作：

1. 当场想点开：奖励在完成页立即出现，先看宠物动作和物品，再看一句短说明。
2. 愿意回家摆放：冰箱贴进入宠物小屋库存，由孩子主动选择摆放，不自动覆盖现有家具。
3. 愿意继续收集：卡片图鉴显示已获得、下一站和未解锁的轮廓，形成“再完成一次就多一张”的期待。

本阶段不改变积分、经验、宠物战斗数值、卡牌对战规则和学习题目算法。图片只负责情绪、识别和收藏价值；地点名称、日期、稀有度、学习结果和奖励数量全部由 HTML/数据渲染，避免把文字烧录进图片。

## 2. 儿童体验闭环

### 2.1 30 秒普通旅行

```text
进入地图 → 看见宠物动作 → 选择一个图标行动 → 得到纪念物 → 回家展示 → 看到下一站
```

- **看见（0~3 秒）：** 场景大图、宠物表情和一个明显的纪念物轮廓先出现。
- **选择（10~20 秒）：** 每屏只保留两个图标化选项；数学题或战斗是可插入挑战，不是每次必经门槛。
- **带回家（5~10 秒）：** 纪念物以“飞回背包/贴到相册”的动效出现，文字最多两行。
- **继续（2 秒）：** 只给一个主按钮“去看下一站”，旁边保留“回小屋摆放”。

### 2.2 文字规则

- 主提示默认 8~16 个汉字；超过 24 个汉字拆成两次点击。
- 先说动作，再说原因，例如“蘑菇亮起来了！带回家做纪念。”。
- 详细故事放在“听故事/更多”抽屉中，不阻塞领取奖励。
- 关键奖励用图标、颜色和数量表达，避免连续阅读。
- 图片不出现汉字、数字、英文、拼音、按钮和水印。

## 3. 资产分层与网页位置

| 类型 | 首要用途 | 首批主题 | 规格 | 当前接入位置 |
| --- | --- | --- | --- | --- |
| 旅行纪念徽章 | 地图已完成标记、完成页、成长档案 | 森林、海滩、星光花园 | 透明 PNG，512x512 | `js/exploration.js`、旅行结算区 |
| 旅行纪念卡 | 完成页大卡、成长档案收藏 | 森林、海滩、星光花园 | 卡底 1200x800；贴纸 768x768 | `js/travel-memory.js`、成长/收藏入口 |
| 冰箱贴 | 小屋装饰库存 | 发光蘑菇、贝壳、星花 | 透明 PNG，512x512，白边/软阴影 | `js/home.js`、`data/furniture.json` |
| 宠物旅行卡牌 | 长期收藏和宠物图鉴 | 统一卡框 + 宠物姿态 | 卡框 768x1024；宠物主体独立 | `js/card-collection.js` |

首批只做森林、海滩、星光花园三个样板。通过浏览器验收后，再扩展其余探索场景，避免一次生成 12 套而导致画风、比例和奖励规则同时漂移。

## 4. 统一视觉语言

- 明亮、软 3D 玩具质感，轮廓厚、主体大、背景干净，适合 5~10 岁儿童。
- 森林使用叶绿/蘑菇暖红，海滩使用海蓝/贝壳奶白，星光花园使用蓝紫/星星金黄；颜色承担场景识别，不承担文字信息。
- 物品必须“一眼能说出是什么”，禁止复杂写实、密集纹理、恐怖表情、尖锐攻击性造型。
- 同一批次统一光照方向、阴影软硬度、描边厚度和圆角语言。
- 卡牌背景、徽章、冰箱贴和宠物姿态分层生成；网页负责组合，方便日后换主题、换宠物和换文案。

## 5. 数据契约建议

现有 `data/travel-rewards.json` 只保存标题、图标、物品 ID 和下一站文案。后续增加可选资产字段，缺失时必须保持旧 emoji 回退：

```json
{
  "forest": {
    "memoryTitle": "森林贴纸",
    "memoryIcon": "🍄",
    "itemId": "mushroom",
    "asset": "assets/generated/travel-memory/badges/forest-badge.png",
    "cardAsset": "assets/generated/travel-memory/cards/forest-card-bg.png",
    "fridgeAsset": "assets/generated/travel-memory/fridge-magnets/forest-mushroom-magnet.png",
    "petCardAsset": "assets/generated/travel-memory/pet-cards/pet-travel-card-frame.png",
    "assetVersion": "v1",
    "assetSource": "manual",
    "assetStatus": "placeholder"
  }
}
```

约束：

- `assetSource` 只能是 `existing`、`Bee/Grok`、`ChatGPT-web` 或 `manual`，实际来源必须与生成记录一致。
- `assetStatus` 至少区分 `placeholder`、`candidate`、`verified`、`published`。
- localStorage 只保存 scene ID、资产路径和完成时间，不保存图片二进制。
- 图片加载失败时继续显示 emoji/纯 CSS 卡片，不阻塞领取奖励。
- 宠物旅行卡牌初期只进入收藏展示，不改变 `PetSystem` 或 `CardArena` 数值。

## 6. Grok/Bee 生成策略

`docs/GPT生图/Grok生图.md` 是历史文件名，不代表 Grok 官方接口已可用。按该文件规定，每一批生成前必须：

1. 只在本机进程读取 key，不把 key 写入 prompt、manifest、截图、日志或回复。
2. 重新探测模型列表和图像接口；404、502、无 enabled channel 都算失败。
3. 保存状态码、错误消息和 request id 等脱敏证据；失败目录保留，不能冒充成功素材。
4. Bee 通道不可用时，可改用已登录 ChatGPT 页面，来源必须写成 `ChatGPT-web`，不能写成 Grok/Bee。

生成批次：

### Batch 0：风格参考板

只生成一张横向参考板，展示森林、海滩、星光花园三种纪念物和宠物回家场景。评审色彩、轮廓、孩子能否一眼理解，以及图片压在网页卡片上是否留有 HTML 文案空间。

### Batch 1：徽章

生成三枚独立徽章或 3 格爆炸图：发光蘑菇、贝壳漂流瓶、星花小月亮。每个组件互不重叠、留足透明边缘、无文字，裁切后统一为 512x512。

### Batch 2：冰箱贴

沿用 Batch 1 的主体，但增加统一白边、磁贴厚度和贴合阴影。每张只放一个物件，确保在小屋缩小到 64~96px 时仍可识别。

### Batch 3：旅行卡/宠物卡牌

先生成无文字卡底、边框和宠物主体，再由网页叠加地点、日期、阶段和收藏状态。卡底和主体分离，允许同一张卡框套用不同宠物。

### Batch 4：网页集成

只接入通过 alpha、尺寸、`naturalWidth`、浏览器截图和无 console error/warning 检查的资产。先接三场景，再扩展其余九场景。

## 7. 目录、命名与 manifest

建议目录：

```text
prj/学习机玩法原型/assets/generated/travel-memory/
  reference/
    travel-memory-reference-prompts.md
    batch-00-style-board.*
  badges/
  cards/
  fridge-magnets/
  pet-cards/
  manifest.json
  README.md
```

文件名必须表达语义，例如 `forest-badge.png`、`beach-shell-magnet.png`，禁止 `final2.png`、`part-01.png` 等临时命名。manifest 至少记录：

- `source`、`model`、`promptFile`、`generatedAt`
- 原始文件、选中版本、裁切输出和处理命令
- `width`、`height`、`colorMode`、`bbox`、`opaquePixels`
- alpha 四边检查、绿色残留检查、网页加载检查结果
- 当前 `assetStatus` 和下一轮只改动的一个变量

## 8. 验收门槛

### 生成/像素门槛

- 真正 RGBA 透明，不接受棋盘格伪透明。
- 四边 alpha 最大值为 0，阴影不越过组件边界。
- 运行时图片无可读文字、字母、数字、拼音、汉字和水印。
- 主体在目标尺寸下仍有清晰轮廓，不能依赖大图细节。

### 网页门槛

- 完成页领取幂等：重复打开不会重复发放旅行记忆。
- 图片失败仍能领取奖励和返回地图。
- 宽度 `1280x720` 与 `390x844` 无横向溢出、按钮可点击、文字不遮住物品。
- `naturalWidth > 0`，无相关 console error/warning。
- 旅行卡、冰箱贴和宠物卡牌都能从对应入口再次找到。

### 儿童体验观察

每批只改一个变量：轮廓、颜色、材质、表情或比例。记录三个问题：孩子是否三秒内知道奖励是什么、是否愿意点开看大图、是否知道下一步该去哪里。

## 9. 风险与回退

| 风险 | 处理 |
| --- | --- |
| Bee/Grok 通道失效 | 保留探测失败证据；使用 `manual` 占位或 `ChatGPT-web`，不伪装来源 |
| 图片含文字/伪透明 | 不进入运行时；重新生成或人工后处理并更新 manifest |
| 资产加载失败 | 使用 `memoryIcon` 和 CSS 卡片回退，不阻塞结算 |
| 小屋库存与家具规则冲突 | 纪念物先进入独立收藏库存，主动摆放后才写入家具槽位 |
| 卡牌资产误改战斗平衡 | 旅行卡牌只读展示，数值继续来自现有 `data/pets.json`/卡牌系统 |
| 12 个场景风格漂移 | 三场景样板验收后锁定风格板，再分批扩展 |

## 10. 成功标准

第一阶段完成时，森林/海滩/星光花园各有一套可识别的徽章、卡片和冰箱贴；完成页能即时展示，宠物小屋能领取并主动摆放，卡片图鉴能回看；没有新积分货币、没有强制长文本、没有未验证素材进入运行时。后续每次扩展只需新增数据和 manifest，不需要重写奖励流程。
