# 宠物图鉴单宠物详情样板设计

## 目标

把当前偏“战斗卡牌弹窗”的宠物详情，升级成更像“宝可梦图鉴”的单宠物介绍页样板，整体以图鉴书版为主，局部加入展馆展示感。

## 结论

采用“图鉴书展开页”作为详情主结构：

- 页面整体延续首页统一背景，不新增单独大背景图
- 弹窗内容改为米白图鉴书纸面
- 左侧突出宠物主展示与阶段观察
- 右侧突出故事、特征、技能与基础资料
- 下方保留并强化数值阅读区

首个样板宠物选择 `向日葵`，因为素材、辨识度和亲和力最适合作为第一版视觉参考。

## 设计原则

1. 图鉴首先是“认识宠物”，不是“战斗结算弹窗”。
2. 主视觉、故事、阶段、属性要有清晰主次，避免所有信息等权堆叠。
3. 视觉气质必须回到当前项目的绘本成长系，不继续沿用厚重暗色竞技卡风。
4. 详情页的结构要能复用到其他宠物，只是文案与局部标签不同。
5. 第一版先完成一个高质量样板，再决定是否批量扩展所有宠物的故事数据。

## 页面结构

### 顶层容器

- 继续沿用现有 `cardDetailModal`
- 但内部结构改成“图鉴书页”式双栏详情卡
- 尺寸从当前小弹窗升级为中大尺寸，适合展示阶段区和介绍区

### 左栏：主展示区

- 宠物大立绘
- 稀有度徽章
- 系列标签
- 编号 / 来源角标
- 一句简短副标题，像宝可梦图鉴的分类名

这一栏要带一点展馆感，像“本期观察对象”的展示台。

### 右栏：档案介绍区

- 名称
- 别名 / 图鉴称号
- 宠物故事
- 性格特征标签
- 代表技能
- 收集状态 / 稀有度 / 系列 / 来源

这一栏更像图鉴书正文页，强调可读性与信息节奏。

### 底部：成长与能力

- `成长阶段`
  - 横向显示已有多阶段图
  - 当前阶段高亮
  - 每阶段带简短阶段名
- `能力面板`
  - HP / 攻击 / 防御 / 速度
  - 用更像图鉴读数卡的布局呈现

## 视觉语言

### 保留的项目共性

- 继续使用首页统一背景
- 继续使用暖白、浅金、草木绿这一组柔和颜色
- 保留圆角、玻璃感、浅阴影这些当前项目成熟语言

### 新增的图鉴识别度

- 主卡像一本展开的图鉴书
- 左栏像展馆展示牌
- 右栏像纸面记录页
- 阶段区像观察记录条
- 技能与特征标签像贴纸或印章

### 明确不再使用的旧风格

- 大面积深紫黑战斗底
- 过重的四角数值角标
- 只像对战卡面而不像宠物百科

## 内容策略

### 第一版数据

先不大规模改 `pets.json`，而是在图鉴详情模块中提供一层“详情扩展文案映射”：

- `向日葵` 先写完整样板内容
- 其他宠物先用现有 `desc / rarity / series / source / imageStages` 生成兜底内容

这样可以先快速验证 UI 和信息结构，再决定是否批量扩展 198 只宠物的故事库。

### 向日葵样板内容

展示以下栏目：

- 图鉴称号：阳光培育师
- 故事描述：更像图鉴介绍的小传
- 特征：温暖、耐心、群体辅助、晨光亲和
- 技能：阳光结晶、鼓舞光环、晨曦补给
- 阶段说明：从种子到完全盛放的成长观察

## 验收标准

1. 点击宠物卡片后，不再出现旧式小型战斗弹窗，而是图鉴书式详情页。
2. `向日葵` 能展示完整样板：大图、故事、特征、技能、阶段示意、能力值。
3. 其他宠物即使没有专属故事，也能用统一结构正常打开详情。
4. 详情页视觉与首页、游乐场、汉字页、排行榜保持统一，不再显得像另一套游戏。
5. 布局在桌面端清晰分栏，在窄屏下可自然折叠为上下结构。
