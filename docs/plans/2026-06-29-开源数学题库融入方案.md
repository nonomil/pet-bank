## 1. 背景

v0.5.3 探索已落地「数学解谜」（forest easy 算式 + genMathQuestion）。但：

- 题型只有**基础运算**，缺**应用题 / 数独 / 逻辑 / 幼小衔接**。
- 数字靠随机生成，缺**真实情境化应用题**（买书/种树/郊游）。

→ 调研开源数学题库，融入探索解谜，覆盖幼小衔接 / 低年级 / 思维启蒙。

### 选型结论

- **应用题 → CMATH**：唯一同时满足「中文 + 分年级 + 有答案 + 可商用(CC BY 4.0) + 体量适中」的现成库。**已 clone 到 prj/cmath**。
- **基础运算 / 数独 / 逻辑 / 幼小衔接 → 自生成**：开源现成库要么协议不清，要么题型不符。自生成最可控（数字小、情境可控、无版权风险）。
- SuperCLUE / 散布数据集：协议不明或题型偏评测，**暂不引入**。

> 详见 [prj/cmath 合入说明 §5 契合与局限](../../prj/cmath/合入宠物积分银行说明.md)。

## 5. 接入实现（genMathQuestion 扩展）

```js
// js/exploration-detail.js
// 现状：function genMathQuestion(difficulty)
// 扩展：function genMathQuestion(type, difficulty)

let CMATH_POOL = null;  // 懒加载 data/math-cmath.json
async function loadCmathPool() {
  if (CMATH_POOL) return CMATH_POOL;
  const r = await fetch('data/math-cmath.json');
  CMATH_POOL = (await r.json()).grades;
  return CMATH_POOL;
}

function pickFromCmath(difficulty) {
  const g = difficulty === 'easy' ? '1' : '2';   // hard 复用 2 年级 + 干扰集(后段)
  const pool = CMATH_POOL[g];
  const q = pool[Math.floor(Math.random() * pool.length)];
  return { text: q.q, answer: q.a, options: q.opts };  // 直接用题库自带选项
}

function genMathQuestion(type = 'arithmetic', difficulty = 'easy') {
  switch (type) {
    case 'arithmetic': return genArithmetic(difficulty);  // 现有
    case 'word':       return pickFromCmath(difficulty);  // 新增：CMATH
    case 'sudoku':     return genSudoku4x4();             // 新增
    case 'logic':      return genLogic();                 // 新增
  }
}
```

场景事件配置 `math` 节点时指定 `type`：

```js
{ type: 'math', mathType: 'word', difficulty: 'easy', reward: { exp: 20 } }   // beach 用 CMATH
{ type: 'math', mathType: 'arithmetic', difficulty: 'medium', reward: { exp: 15 } }  // mountain 用算式
```

## 7. 边界

- **不引入协议不明的数据集**（散布 JSON / 网盘 PDF），避免版权风险。
- CMATH 只取 **1-2 年级 + 单步 + 答案≤50**（3 年级+ 含分数/多步，对儿童偏难）。
- 幼小衔接（学前）**全自生成**，CMATH 不覆盖。
- 数独只做 **4×4**（不做 9×9）。
- 探索解谜（融入式单题）与数学 PK Tab（连续练习）**共存不合并**。
