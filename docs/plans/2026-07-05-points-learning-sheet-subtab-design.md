# 积分区学习单独立子页设计稿

## 1. 背景

当前项目已经把学习页收口为“入口大厅 + 资料包 + 学习网站 + 打印讲义 + 我的进度”，方向是对的。

但 `积分 -> 今日打卡` 页面里仍然直接挂着 `今日学习单`，会带来两个问题：

1. `今日打卡` 同时承担“日常积分任务”和“学习积分收口”，页面职责开始混杂。
2. 学习单和日常任务都挤在一个页面里，家长切换心智不清楚，也不利于后续继续扩学习单。

## 2. 目标

把学习单从 `今日打卡` 页里拆出来，变成 `积分` 大 tab 下一个独立的同级子页。

最终形成：

- `今日打卡`：日常积分任务
- `学习单`：学习积分 / 学习收口
- `每周复盘`：周级复盘
- `奖励兑换` / `兑换商店` / `背包`：积分消费与资产

## 3. 方案选择

### 方案 A：继续留在 `今日打卡` 页，做左右两列

优点：

- 桌面端同时可见

问题：

- 移动端会变成长页面
- “日常积分”和“学习积分”仍然混在一个子页里
- 学习单后续继续扩展时，`今日打卡` 页会越来越重

### 方案 B：学习单独立成 `积分` 大 tab 下的同级子页

优点：

- 信息架构最清楚
- 家长心智最稳定
- 学习页继续保持轻入口，不被重新做重
- 后续可以单独扩学习单，不影响日常打卡页

问题：

- 需要新增一个 page 壳和积分区子导航项

### 方案 C：把学习单搬回 `学习` 大 tab，下方左右布局“学习单 + 入口大厅”

优点：

- 学和记放在一个大区里

问题：

- 学习页又会重新变重
- 和当前“学习页负责入口、积分页负责收口”的收口方向冲突

## 4. 最终决定

采用 **方案 B**。

### 页面职责

- `学习` 大 tab：继续负责“找到学什么”
- `积分` 大 tab：继续负责“今天完成了什么”
- `学习单`：作为 `积分` 大 tab 里的学习专属收口子页

### 用户路径

1. 先去 `学习` 页找到内容入口
2. 打开 lesson / 资料包学习
3. 再回到 `积分 -> 学习单` 做当天收口
4. `积分 -> 今日打卡` 继续只做日常积分任务

## 5. 具体交互

### 积分区子导航

新增：

- `学习单`

调整后顺序建议：

1. `今日打卡`
2. `学习单`
3. `每周复盘`
4. `奖励兑换`
5. `兑换商店`
6. `背包`

### 今日打卡页

- 保留宠物加减分卡
- 保留 `今日成长任务`
- 移除当前内嵌的 `今日学习单`

### 学习单页

- 独立渲染当前 `LearnCenter.renderDailyCheckin(...)`
- 标题可明确区分“学习积分收口”
- 页面底部可保留一个轻量按钮，跳到 `学习` 页入口大厅

## 6. 技术方案

### 路由

新增 page：

- `learning-sheet`

并把它归入 `PAGE_TO_TAB.today`

### 页面壳

新增：

- `#page-learning-sheet`
- 独立容器，例如 `#points-learning-sheet-container`

### LearnCenter 接线

`renderDailyCheckin()` 保持不变，继续只负责渲染学习单本体。

变化点是：

- 不再默认渲染到 `today-learning-checkin`
- 改为渲染到 `points-learning-sheet-container`

### 设置页口径

设置页文案从：

- “积分页显示哪种学习单”

微调为：

- “积分区学习单显示哪种模式”

避免用户误以为学习单还在 `今日打卡` 页里。

## 7. 验收标准

1. `积分` 区出现独立的 `学习单` 子导航。
2. `今日打卡` 页不再直接展示学习单。
3. `学习单` 页能完整展示模板 A/B/C。
4. 设置页切换模板后，`学习单` 页立即跟随变化。
5. lesson 打勾后，`学习单` 页仍然正确同步完成状态。
6. 打印页与学习页入口大厅不受影响。

## 8. 不做什么

这轮不做：

- 学习单左右双列常驻布局
- 把学习单搬回学习页
- 新增学习单额外奖励
- 重写 `LearnCenter` 学习单数据模型
