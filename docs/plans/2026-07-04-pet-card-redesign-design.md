# 宠物卡牌版式重设计

日期：2026-07-04

## 目标

把当前图鉴卡从“四角圆形数值角标 + 名称栏”的旧式读法，升级成更像正式集换式卡牌的阅读结构：

- 卡片背景由 Agnes 重新生成
- 数值不再只放在四个角落
- 数值必须带中文属性名称
- 页面优先展示整张合成卡图，而不是 HTML 拼接半成品

## 设计结论

### 1. 保留 4 个核心属性，但改成中文属性区

当前数据仍然保留：

- `生命`
- `攻击`
- `防御`
- `速度`

但不再使用四个角上的孤立圆点。

改成底部信息区统一呈现，读法更接近正式卡牌：

- 左右分区或 2×2 属性块
- 每项属性都有中文名和数值
- 颜色只做辅助，不再承担全部信息表达

### 2. 卡牌结构改成“主图 + 信息牌面”

新版卡牌结构：

1. 顶部：系列 / 来源小标签 + 稀有度标识
2. 中部：宠物主立绘展示区
3. 名称区：宠物名 + 简短副标题或来源说明
4. 底部：中文属性区（生命 / 攻击 / 防御 / 速度）

这样卡牌会更像“可以阅读的信息卡”，而不只是“带数值的宠物贴纸”。

### 3. Agnes 负责卡框气质，脚本负责真实数据

Agnes 只生成卡牌底图模板：

- 有统一留白和装饰分区
- 不写真实宠物名
- 不写真实数值
- 不放真实角色

脚本负责叠加：

- 宠物立绘
- 中文名称
- 系列 / 来源
- 稀有度标签
- 生命 / 攻击 / 防御 / 速度

这样后续批量出 198 张卡时，不需要手工修每一张。

### 4. 第一轮先做“1 张样卡 + 全链路”

先做一张高质量样卡验证方向，推荐优先使用 `向日葵`：

- 辨识度高
- PVZ 风格明确
- 适合验证参考图 2/3/4 的卡牌读法

第一轮重点不是一次性把所有卡都做完，而是先把：

- Agnes 新底图
- 脚本合成
- 页面接入
- 浏览器实看

全部跑通。

## 推荐方案

采用“Agnes 新卡框 + 脚本叠中文属性 + 页面优先使用整卡图”的方案。

不推荐继续沿用当前四角角标方案做微调，因为那样只能“稍微不难看”，很难真正达到你给的参考图气质。
