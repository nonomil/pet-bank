# 家长区与隐藏管理接口设计

> 日期：2026-07-06  
> 状态：P0 已实施  
> 范围：设置、工具箱、低频管理入口、危险操作隐藏规则

## 参考项目结论

### 班宠乐园 1

班宠乐园 1 的主导航包含 `宠物大厅 / 宠物小屋 / 排行榜 / 成长记录 / 徽章 / 小卖部 / 老师工具箱 / 班级管理`。其中孩子和老师都可能高频使用的页面放在主导航；真正偏管理的功能再向内收：

- `宠物小屋` 里用弹层处理 `管理小屋 / 管理玩具 / 自定义价格`，不是单独放到主流程。
- `小卖部` 里把 `班币管理 / 管理商品 / 拍卖结算` 收在小卖部内部。
- `老师工具箱` 放随机点名、静音挑战、朗读挑战、计时器、证书、评语助手、统计导出等教师工具。
- `班级管理` 再细分 `花名册 / 积分规则 / 等级规则 / 小组管理 / 标签管理 / 家长查看 / 徽章管理 / 小屋管理 / 科任老师 / 班级设置`。

### 班宠乐园 2

班宠乐园 2 的家长页顶部只有 `主页 / 小卖部 / 光荣榜 / 工具 / 设置`。主页里有 `一键分配领养 / 一键锁屏 / 批量管理 / 编辑座位 / 撤回模式`。设置页集中放：

- 账号、修改账号名、修改密码、退出登录。
- 个性化、系统名称、主题、音效。
- 搭子成长等级配置。
- 加分/扣分规则管理。
- 解锁密码。
- 危险操作面板。

这说明它把“孩子可见玩法”和“家长管理能力”分层：日常导航保持短，规则、密码、危险操作集中到设置，并且危险操作需要再打开面板。

## 当前项目规划

当前项目已经把 `成长作品 / 工具箱 / 设置` 从 `游乐场` 拆到右上角 `家长区`。下一步不是继续扩一级导航，而是把家长区内部再分层：

- `设置`：账号、孩子档案、云端家庭、学习/数学配置、规则模板、危险操作。
- `工具箱`：默认只显示轻工具，例如随机点名和番茄计时。
- `成长作品`：保留为孩子作品与成长档案展示。
- `高级数据接口`：导入/导出、诊断 JSON、云端覆盖、本地重置等都不进儿童路径，默认隐藏或折叠。

## P0 决策

`工具箱` 默认隐藏 `数据管理` 卡片。这个卡片仍然保留，但只有满足任一条件时出现：

- URL 带 `?parentAdmin=1`。
- 控制台或内部脚本设置 `localStorage.petbank_parent_admin_tools = "1"`。
- 本次会话设置 `sessionStorage.petbank_parent_admin_tools = "1"`。

这样做的原因是数据导入会覆盖本地全部 `petbank_*` 数据，属于高风险接口。它不应该和随机点名、番茄计时并列展示。

## 后续建议

- P1：把设置页整理成卡片分区：`账号与孩子 / 家庭云端 / 学习与题目 / 规则模板 / 高级与危险操作`。
- P1：把云端诊断继续保持隐藏开关，不默认给普通家庭用户展示。
- P1：导入数据前增加“必须先导出备份”的显式二次确认。
- P2：如果后续接入真实登录角色，再把隐藏接口从本地开关升级为 `parent/admin` 权限判断。
