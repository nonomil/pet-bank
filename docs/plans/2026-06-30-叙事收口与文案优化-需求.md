## 3. 需求清单

### R1 [P0] 故事单一数据源收口

- **现状**：`_loadStories`（exploration-detail.js）fetch `data/stories/*.json` 成功覆盖 `sceneEvents`；失败回退**硬编码兜底**（exploration-detail.js `let sceneEvents = {...}`，140 行）。两套并行，fetch 失败显示旧兜底 → 漂移风险。
- **需求**：文件夹为**唯一源**，移除/精简硬编码兜底。
- **方案**（/solution-loop 决策）：A. 删兜底，fetch 失败显示"请用 http server 打开"+重试；B. 保留极简兜底（仅 forest 冒烟）注释"仅 file:// 兜底"。
- **验收**：改 `data/stories/forest.json` → 运行时即时生效，无兜底覆盖。
- **涉及**：`js/exploration-detail.js`

### R2 [P0] 叙事文档同步

- **需求**：修正 3 份过期文档：
  - `08_CC执行-文案整理与样板.md`：现写"3 样板/6 缺 math/forest 第二章" → 实际"12 全有 math+ending/forest 第一章"，更新或归档。
  - `03_章节结构.md`：第二章"森林边界"与实现（forest 第一章）漂移 → 对齐章节映射。
  - `07_CC执行提示词.md`：写"先 3 样板不铺全量" → 实际已全量 36 怪物+12 math，补"执行状态"段。
- **验收**：文档与代码/数据零冲突。
- **涉及**：`docs/叙事重构/08、03、07、README`

### R3 [P1] 数学场景化 + 章节教育目标 ★最优先

> 三份审查核心共识 + 用户明确"只选一个主题就做这个"。

- **现状**：math 事件偏"门槛提示"（"螃蟹举牌子应用题""青蛙念应用题"），离 05「题目与场景语义一致」远；数据只有 mathType/difficulty，无章节教育约束。
- **需求（精确方案，只改数据 + 少量字段，不改引擎）**：

  **(a) 5 章教育目标表**（写入 `data/stories/README.md` + 每场景 json 加 `chapter_skill` 字段）：

  | 章 | 章名 | 教育目标 | 题型 |
  |----|------|---------|------|
  | 1 | 起点花园 | 数感 / 数量关系 | arithmetic（10-20 以内加减）|
  | 2 | 森林边界 | 加减应用 | word（CMATH 1-2 年级加减）|
  | 3 | 海边集市 | 生活情境应用题 | word（CMATH 买卖/分配）|
  | 4 | 高地洞窟 | 乘除与机关题 | arithmetic（乘除）+ 场景机关 |
  | 5 | 星空终点 | 规律 / 逻辑 | logic（找规律/等式）|

  **(b) 每场景重写 1 个 math 事件**（题面=场景内具象计数/分配，非通用牌子）。示例：
  - `waterfall.math`：荷叶上露珠数量变化（第一片 2 颗、第二片 3 颗，选对下一片才能过河）—— 加法应用。
  - `candy.math`：糖果树结 2 串草莓糖每串 3 颗，一共几颗 —— 乘法应用。
  - `forest.math`：3 朵发光蘑菇 + 又长 2 朵 —— 数感加减。
  - `stargarden.math`：星座 2,5,8,11,? —— 规律。

  **(c) 不新增系统**：复用现有 genMathQuestion（arithmetic/word/logic），场景化针对**自生成题的题面包装** + **新增固定场景题模板**；CMATH word 随机题库不动（已是应用题）。
- **验收**：12 场景 math 题面含场景具象物（蘑菇/露珠/糖果/冰晶/星星…）；5 章 chapter_skill 字段 + README 教育目标表；章节推进有稳定教育认知。
- **涉及**：`data/stories/*.json`（math text + chapter_skill）+ `data/stories/README.md` + 可能 `js/exploration-detail.js`（固定场景题模板，若需）
