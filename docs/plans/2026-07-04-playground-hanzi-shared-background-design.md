# 游乐场、汉字与排行榜统一背景设计

## 目标

把游乐场页、汉字游戏页和排行榜页的背景统一回首页路线地图的同一视觉体系，消除页面之间的画风断裂。

## 结论

采用“直接复用首页背景图”的方案，不再为游乐场、汉字页和排行榜页各自维护独立大背景图。

统一使用项目现有首页背景资源：

- `assets/home-bg.webp`
- 视觉参考同 `assets/home-bg.jpg`

## 设计原则

1. 背景风格统一优先于单页差异化。
2. 中心内容区由页面卡片和内容容器承担信息密度，背景不再抢主视觉。
3. 页面局部差异通过卡片样式、按钮、插图和玩法内容表达，不再通过整张背景图换画风表达。
4. 背景实现要尽量复用现有 `map-page-bg` 体系，避免游乐场、汉字页、排行榜页继续分叉出独立背景机制。

## 页面策略

### 游乐场

- 移除当前 `pg-bg.webp` 独立背景依赖。
- 改为复用首页同款 `home-bg.webp`。
- 保留当前入口卡片和吉祥物资源，让“内容层”承担游乐场辨识度。

### 汉字游戏

- 移除当前 `hanzi-new-bg.webp` 独立背景依赖。
- 改为复用首页同款 `home-bg.webp`。
- 保留现有大厅卡片、答题卡、按钮的玩法层视觉，但不再让整页背景切到另一套森林书房画风。

### 排行榜

- 移除当前 `leaderboard-bg.png` 独立背景依赖。
- 改为复用首页同款 `home-bg.webp`。
- 保留排行榜卡片、趋势块、tab 和装饰元素，但不再让排行榜单独切换到另一张整页背景。

## 实现方式

1. 给 `#page-playground`、`#page-hanzi` 与 `#page-leaderboard` 统一使用透明页面容器。
2. 三页都通过固定定位的伪元素引用 `../assets/home-bg.webp`。
3. 三页都使用一致的层级和隔离规则，避免再出现背景被页面底色、负层级或自定义遮罩覆盖的问题。
4. 保留内容容器的可读性样式，但不新增强烈的全屏色罩。

## 验收标准

1. 游乐场、汉字页和排行榜页切换时，背景画风与首页一致。
2. 三页不再引用 `pg-bg.webp`、`hanzi-new-bg.webp` 和 `leaderboard-bg.png` 作为页面主背景。
3. 中心内容阅读性正常，不出现新的蓝罩、灰罩或大面积色偏。
4. 样式回归测试能检测三页是否继续复用首页背景资源。
