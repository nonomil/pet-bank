# 游乐场与家长区导航收口设计

> 日期：2026-07-06  
> 状态：已实施  
> 范围：顶部导航、游乐场下拉菜单、低频管理页归属

## 背景

`游乐场` 原本替换了旧的 `更多` tab，但历史遗留把 `成长作品 / 工具箱 / 设置` 继续挂在游乐场下拉里。结果是孩子进入游戏区时，会看到偏后台、偏数据管理的入口，和 `数学 PK / 汉字游戏 / 卡牌对战 / 排行榜` 的游戏心智混在一起。

参考 `docs/汉字游乐场/2026-07-02-汉字答题与排行榜-设计稿.md` 的决议，游乐场的定位应是“游戏类玩法 Hub”，而不是低频功能收纳箱。早期 `更多` 设计里也明确写过，`成长作品 / 工具箱` 属于低频能力，应降低主导航存在感。

参考项目 `prj/class-pet-garden-v2/frontend/src/views/SettingsView.vue` 也采用类似思路：班级信息、学生管理、评分项、主题与退出登录集中在独立 `系统设置` 页面，不混入学生首页或宠物玩法路径。因此当前项目也应把这些偏家长/配置/资料的入口从 `游乐场` 拆走。

## 收口方案

`游乐场` 只保留游戏入口：

- 游乐场首页
- 数学 PK
- 汉字游戏
- 卡牌对战
- 排行榜

`成长作品 / 工具箱 / 设置` 归入右上角独立 `家长区` 入口。这个入口靠近孩子切换和积分胶囊，语义上更接近“家长/设备/资料管理”，不会打断孩子玩游戏的路径。

`工具箱` 暂不删除。里面包含点名器、番茄钟、数据导入导出等能力，仍可从 `家长区` 入口进入，但不再作为游乐场子项暴露。这里的“隐藏”是信息架构层面的隐藏：儿童游乐路径不展示这些入口，但旧的 `switchPage('works'|'tools'|'settings')` 深链继续保留。后续如果要进一步隐藏高风险接口，可以把工具箱里的“数据管理”再折叠到高级区，或加家长确认。

## 验收

- 顶部游乐场下拉不再出现 `成长作品 / 工具箱 / 设置`。
- `mathpk / hanzi / leaderboard / playground` 仍高亮游乐场。
- `works / tools / settings` 高亮右上角 `家长区`。
- 页面本体保留，不破坏旧的 `switchPage('works'|'tools'|'settings')` 深链。
