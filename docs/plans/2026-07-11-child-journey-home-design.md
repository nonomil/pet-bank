# 儿童主线首页设计

**目标：** 让孩子打开网页后先看到一件明确、可完成的事，再自然进入伙伴反馈或短冒险，而不是在功能大厅里自行筛选。

## 范围

- 保留所有既有路由、任务、学习、游戏、宠物和商店页面。
- 首页新增一个视觉优先级最高的“今天先做这件事”行动卡。
- 现有六个同权快捷入口调整为三条儿童主线：今天、伙伴、冒险；低频兑换和探索保留在“更多冒险”。
- 本轮不改积分规则、词库、游戏结算或 localStorage。

## 孩子路径

```text
首页主线卡（今天先做什么）
  -> 今日打卡 / 学习
  -> 完成后回首页
  -> 伙伴或一局推荐冒险
```

主线卡复用当前首页的今日任务优先级：默认去 `today`，不引入新的状态来源。三入口只负责降低选择压力：

| 主线 | 入口 | 内容 |
| --- | --- | --- |
| 今天 | 今日打卡、学习中心 | 家庭任务与学习计划 |
| 伙伴 | 我的宠物 | 宠物、小屋、遛弯、图鉴 |
| 冒险 | 游乐场 | 数学、汉字、打字、像素探险 |

## 视觉与交互

- 主线卡使用较大的圆角、明确序号、预计时间、奖励提示和单一 CTA。
- 三入口使用一致的卡片尺寸；不隐藏原入口对应的功能。
- “更多冒险”默认收纳探索和兑换商店，避免与今天的核心目标竞争。
- 小屏改为单列主线卡和三列/双列入口，不依赖 hover 才能理解可点击性。
- 所有按钮保留原生 `button` 语义、可见焦点和现有 `switchPage` 导航。

## 验收

1. 首页源码存在主线卡、三个主入口和“更多冒险”区域。
2. 主线卡 CTA 到 `today`；三个入口分别到 `today`、`pet`、`playground`。
3. 原 `explore`、`shop` 仍可从更多入口直达。
4. 40 条静态路由、现有 Pages 资源制品和 JS 语法不回归。
