# 三样板短旅行章节设计

**日期：** 2026-07-12
**范围：** `forest`、`beach`、`stargarden`
**目标：** 让孩子在少量点击内看到宠物反应、做一次选择并带回纪念物；数学题和战斗变成可选挑战，而不是每次旅行的必经阅读链。

## 现状问题

当前页面虽显示“看见 / 选择 / 带回家”三节点，但运行时仍按五个故事事件顺序推进：旁白、发现、数学题、选择、遭遇战，然后统一进入战斗。结果是三节点只是进度标签，孩子仍要经过长链条才能看到旅行完成反馈。

## 设计选择

### 选择 A：只缩短文案

不改事件流程，只把五个事件的文字变短。实现风险最低，但数学题和遭遇战仍然强制出现，无法达到 30–90 秒短旅行目标。

### 选择 B：三节点主路径，可选挑战（推荐）

在三个样板 JSON 顶层增加 `chapter_flow`：

```json
{
  "chapter_flow": {
    "mode": "short",
    "see": [0, 1],
    "choose": 3,
    "challenge": { "math": 2, "battle": 4 }
  }
}
```

运行时默认展示 `see` 事件，然后显示选择事件。选择反馈后出现两个动作：`带回家` 和 `挑战一下`。带回家直接记录旅行纪念物并显示完成卡；挑战一下才展示数学题，答题完成后再进入遭遇战。跳过挑战不扣本轮探索体力，不改变积分、经验或战斗数据；选择挑战时沿用现有 `startExploration()` 和战斗结算。

### 选择 C：重写成全新三事件 JSON

删除旧数学题和遭遇事件，数据最干净，但会破坏已有配音、测试、战斗入口和旧进度兼容，不适合当前阶段。

## 数据流

```text
show(scene)
  ├─ chapter_flow.mode=short → see[0..1]
  ├─ choose → choice feedback
  ├─ 带回家 → recordTravelMemory → showEnding
  └─ 挑战一下 → math → encounter → existing battle → showEnding
```

三样板之外没有 `chapter_flow` 时继续走现有五事件兼容路径。旅行记忆仍幂等，重复完成不重复发放物品；卡牌、积分、经验、竞技场状态不受跳过挑战影响。

## 文字与视觉规则

- 默认主路径每屏最多一句旁白和一个动作按钮。
- `shortText` 仍是默认文案，`detailText` 收进“想知道更多”。
- 选择按钮保持 2–6 字动词短标签；选择后的奖励使用图标、短句和纪念物预览。
- 完成页只突出宠物表情、带回来的物品和下一站预告。

## 错误与兼容

- 缺少 `chapter_flow`、索引越界或字段非法时，自动回退旧五事件流程。
- 刷新时沿用 `ExplorationProgress` 的场景进度；短章节状态额外保存 `flowMode` 和 `challengeStatus`，不迁移旧 key。
- 故事加载失败仍显示明确的本地服务器提示，不生成隐形兜底文本。

## 验收

- 三样板默认 3 次以内点击出现选择，选择后 1 次点击可完成并看到旅行纪念物。
- 可选挑战路径仍能答数学题、进入战斗、胜利后生成同一旅行记忆。
- 跳过挑战不创建战斗、不扣额外 HP、不改变卡牌/积分/经验。
- 刷新短章节可恢复到选择或挑战状态；退出清理临时状态。
- `1280x720` 和 `390x844` 无横向溢出，控制台无非预期错误。
