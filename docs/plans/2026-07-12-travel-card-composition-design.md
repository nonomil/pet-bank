# 旅行宠物卡牌组合展示设计

**状态：** 已落地并通过浏览器验收
**日期：** 2026-07-12
**范围：** 旅行记忆收藏区中的“场景底图 + 宠物旅行卡框 + 宠物快照”组合展示

## 目标

孩子完成探索后，能看到一张真正属于自己宠物的旅行卡，而不是只有一张地点背景图。卡牌是收藏展示和回访线索，不参与宠物战斗、积分、经验、稀有度或系列收集奖励。

## 设计选择

采用三层 DOM 组合，而不是重新生成带文字的整图：

1. 场景卡底图作为背景层，保留森林、海滩、星光花园的颜色识别。
2. Agnes 统一卡框作为装饰层，保证三张卡的视觉语言一致。
3. 完成探索时保存宠物快照；卡面中央显示当前阶段宠物图片，图片缺失时回退到 emoji。

文字（地点、宠物名、阶段、下一站）继续由 HTML 渲染，避免图片文字无法修改、无法读屏和无法本地化。

## 数据契约

`TravelMemory.record()` 接受可选 `pet` 对象，并只保存展示字段：

```json
{
  "speciesId": "pet-id",
  "name": "宠物名称",
  "emoji": "🐾",
  "image": "assets/pets/example.webp",
  "stage": "幼崽"
}
```

旧记录没有 `pet` 时保持兼容，卡牌使用当前宠物或 emoji 回退。字段只用于 DOM 展示，不进入 CardArena。

## 页面行为

- 卡片图鉴顶部仍显示旅行记忆数量和下一站预告。
- 每张旅行记忆卡显示组合卡面、地点名、宠物阶段和回家文案。
- 图片加载失败时，卡框仍保留；宠物图片失败显示 emoji；底图或卡框失败显示 CSS 卡面。
- 移动端保持单列卡片，不产生横向滚动。

## 验收指标

- 三场景各显示一张组合旅行卡，卡框和宠物层均可验证加载。
- 旧版无宠物快照记录仍可渲染，不抛异常。
- 旅行卡不改变 `petbank_cards`、系列套票、竞技场队伍和积分经验。
- 桌面 `1280x720`、移动 `390x844` 无横向溢出；图片 `naturalWidth > 0`。
- 自动测试覆盖快照归一化、旧记录回退、展示源码契约；浏览器测试覆盖真实入口和两种 viewport。
