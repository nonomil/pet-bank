# 宠物进化与探索留存优化方案

> 目标：让孩子愿意继续点下一步、愿意照顾宠物、愿意再次进入探索，而不是要求孩子读完一段长文字。

## 1. 审查结论

### 1.1 当前已有基础

- `PetSystem` 已有 5 个等级阶段和 `imageStages` 图片阶段，等级会自动推进进化。
- 宠物小屋已有喂食、玩耍、洗澡、休息、每日照料、成长日志和统一奖励反馈。
- 探索已有 12 个场景、每场 5 个事件、选项、发现、数学题和战斗结算。
- 探索页面已有背景图、宠物立绘、对话框和场景结束语，具备视觉叙事骨架。

### 1.2 主要问题

1. **进化是结果，不是目标**：孩子看到当前等级和进度条，但不知道“下一阶段会长什么样、还差哪一步、今天能做什么”。
2. **成长反馈偏文字**：喂食、玩耍、探索主要通过 toast 和长消息反馈，宠物本身的动作、表情和场景变化不足。
3. **探索像连续阅读**：每个场景固定 5 个事件，叙事、发现、选择、答题、战斗按长链条推进，低龄孩子容易在第二三步退出。
4. **旅行没有持续记忆**：探索结束后有结束语，但缺少“宠物寄回来的照片/物品/下一封信”，无法形成回访理由。
5. **文字承载过多功能**：故事、规则、奖励、解析、下一步提示常堆在同一个对话框里；孩子需要读，系统也需要解释，互相争夺注意力。

## 2. 设计原则

- **先看后读**：先出现宠物动作、表情、物品或地图变化，再用一句短句解释。
- **一句一动作**：一屏只保留一个主要动作和一个主要选择。
- **短句优先**：故事文本默认 12~24 个汉字；超过 30 字拆成两次点击；规则文字收进“？/详情”。
- **奖励可见**：经验、进化碎片、旅行纪念物使用图标和数字呈现，文字只说明原因。
- **每次离开都留下东西**：探索必须产生照片、贴纸、材料、故事页或宠物新动作中的至少一种。
- **不靠失败惩罚留人**：答错提供可立即尝试的提示；旅行失败也留下线索，不清空进度。

## 3. 推荐的核心循环

```text
照料宠物 → 获得一枚成长印记 → 解锁短旅行
      ↓                         ↓
看到进化预览 ← 收集旅行纪念物 ← 30~90 秒探索小事件
      ↓
宠物回家展示新动作/新装扮 → 产生下一站预告
```

### 3.1 进化改为“预览 + 印记”

保留现有等级和自动进化规则，增加一层孩子可理解的表达：

- 当前阶段显示一张宠物大图、阶段名和一个“还差几枚印记”。
- 下一阶段显示半透明剪影、一个可收集的视觉特征（翅膀、颜色、帽子、能力图标）。
- 喂食、玩耍、洗澡、完成任务、探索分别获得不同颜色的成长印记；印记只是展示层，不改变现有积分算术。
- 达到阈值时先播放 1~2 秒进化预告，再进入完整进化反馈；不要直接跳过。

### 3.2 旅行改为“短章节”

每次旅行由 3 个节点组成，而不是固定读完 5 个事件：

1. **看见**：一张场景图 + 宠物动作 + 8~16 字旁白。
2. **选择**：两个图标化选项，选择后立即显示宠物反应和一个奖励。
3. **带回家**：得到纪念物/故事页/装扮，出现下一站预告。

数学题或战斗作为可插入节点，不再每次都强制出现。普通旅行 30~60 秒，完整挑战 2~3 分钟。

### 3.3 探索故事用“图卡 + 气泡”替代长对话框

- 每个事件数据增加 `visual`, `petMood`, `shortText`, `detailText`。
- 默认只渲染 `shortText`；长解释放入点击后展开的“想知道更多”。
- `petMood` 映射到开心、惊讶、害怕、得意等短动画/滤镜，先让孩子看到反应。
- 选项按钮使用图标 + 2~6 字短标签，避免把完整句子塞进按钮。
- 对话框保留无障碍文本和读屏顺序，但视觉主路径只突出短句。

## 4. 建议的页面结构

### 4.1 首页/宠物小屋

保留现有照料卡，新增一个紧凑的“下一件小事”区域：

- `今天再做 1 件事，宠物就能去看新地方`。
- 展示单个 CTA，如“去看海边”；不要同时列出多个任务。
- 进化卡点击后打开预览，不跳转复杂图鉴。

### 4.2 探索页

- 地图节点显示场景插图、宠物剪影和一枚未领取纪念物。
- 已完成节点显示照片/贴纸，不只显示勾选状态。
- 地图首屏只展示当前节点、下一节点和一个锁定预告，其他节点折叠。

### 4.3 旅行完成页

只展示三件事：宠物表情、带回来的物品、下一站预告。详细奖励日志和故事原文放在“查看记录”。

## 5. 数据与工程方案

### Phase A：低风险表现层

- 新增 `js/pet-evolution-preview.js`：读取现有 `STAGES`/`imageStages`，提供下一阶段预览和进度文案。
- 在 `home.js` 增加单一 CTA 和进化预览弹层；不改经验、积分、存档字段。
- 在 `exploration-detail.js` 增加短句优先渲染和宠物情绪 class；旧 `text` 字段继续兼容。

### Phase B：旅行纪念物

- 新增 `data/travel-rewards.json`，定义照片、贴纸、材料和下一站预告。
- 在现有探索完成结算中增加 `travelMemory` 事件，沿用现有库存/成长日志机制。
- 每个场景先改 3 个样板：森林、海滩、星空花园；验证后再迁移其余 9 个场景。

### Phase C：短章节与分支

- 为故事事件增加 `chapter`, `shortText`, `detailText`, `petMood`, `rewardPreview`。
- 普通路线控制在 3 节点；数学题/战斗按场景标签抽取，不再固定强插。
- 增加断点存档，刷新页面后可从当前节点继续，不重复整段故事。

## 6. 文字预算

| 区域 | 默认可见文字预算 | 处理方式 |
| --- | ---: | --- |
| 首页下一步 | 18 字 | 只说行动和结果 |
| 进化预览 | 24 字 | 名称 + 还差多少 |
| 探索旁白 | 24 字 | 超过则拆卡 |
| 选项按钮 | 6 字 | 图标 + 动词 |
| 奖励反馈 | 16 字 | 图标 + 数字 + 一句原因 |
| 详细解释 | 不限 | 默认折叠，主动展开 |

## 7. 验收指标

- 首次进入探索后，孩子在 3 次点击内看到宠物反应或获得物品。
- 单次普通旅行完成时间 30~90 秒。
- 探索主路径任何一屏不出现超过 2 个句号分隔句。
- 移动端 390px 宽度下，选项按钮不换成三行，不出现横向滚动。
- 进化预览能明确回答：当前阶段、下一阶段、还差什么、下一步做什么。
- 旧故事 JSON 只含 `text` 时仍可正常运行，避免一次迁移造成回归。

## 8. 实施顺序

1. 先做 Phase A：进化预览、单一下一步 CTA、短句渲染。
2. 用森林/海滩/星空花园三个场景做真实浏览器截图验收。
3. 再做 Phase B：旅行纪念物和回家展示。
4. 最后迁移其余场景到短章节数据格式。

不建议第一轮就加入复杂分支、随机结局或新的积分货币；先验证“孩子是否愿意继续点击”和“是否会回来查看宠物变化”。
