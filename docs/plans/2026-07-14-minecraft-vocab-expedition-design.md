# Minecraft 单词学习远征设计

**状态：** 已确认，进入实施

**目标：** 将现有 Minecraft 词汇模块升级为孩子端默认的 11 卡每日学习远征，并把参考站精选词卡、Anki 清洗词库、本地媒体和宠物积分奖励连成一条可验证的学习闭环。

## 方案选择

已比较三种方案：

1. 只增强独立 Anki 工作台。覆盖最完整，但与宠物积分主线分离，孩子端学习节奏不够集中。
2. 只复刻参考站 500 卡。上线成本低，但无法复用已提取的 11,241 张 Anki 卡和本地媒体。
3. 主站远征 + 双数据池。主站 `minecraft-vocab` 承担日常学习；参考站 500 卡作为精选池；完整 Anki 数据作为独立探索入口。采用第三种方案，以保持日常任务简单，同时保留内容深度。

## 用户流程

- 默认进入 `今日学习远征`，显示当天 11 张卡、预计约 10 分钟、宠物陪伴和可获得奖励。
- 顺序固定为：热身听音、新词卡、主动回忆、场景句。用户可以重新查看已学卡，但不能跳过未完成步骤后直接结算。
- 卡片显示本地或参考站图片、英文单词、音标、中文释义、短语、中英文例句；单词和例句提供播放按钮。
- 用户回答错误不扣分，展示正确答案并将卡片标记为稍后复习；完成全部卡片后才允许结算。
- 每个 profile 每个本地日期最多领取一次远征奖励；重复打开完成页不会重复增加积分或宠物经验。

## 页面与视觉

现有 `page-minecraft-vocab` 和 `MinecraftVocabPage` 继续作为唯一主站入口。页面分为：

- 远征首页：今日进度、奖励、宠物状态、开始/继续按钮，以及精选池与完整词库入口。
- 单卡学习：稳定宽度的聚焦卡片、步骤进度、听音按钮、显示答案/自评操作、下一张按钮。
- 完成页：完成数量、正确率/回顾数量、奖励结果、继续探索和返回学习中心。

视觉使用深色矿洞背景、草方块绿和钻石青作为强调色，主体卡片保持高对比的纸张/面板表面。所有标题、单词、中文和按钮文本由 HTML/CSS 渲染，生成图片不得包含文字、数字、标志或水印。桌面使用两列信息布局，移动端改为单列，所有触控目标不小于 44px，支持明显键盘焦点和 `prefers-reduced-motion`。

## 数据边界

- 参考站数据来自 `data/learn/external/mayihaoke/word-cards.json`，固定保留来源字段和抓取说明，不运行时依赖外网。
- Anki 主数据保留在 `prj/anki-minecraft-vocab/data/cards.json`，媒体保留在该工作台的 `assets/media/`；主站只引用已经筛选、轻量化的学习包。
- 学习包继续放在 `data/learn/packs/english-mc-hybrid-2026/modules/`，生成脚本负责从主数据构造 starter/core/typing/memory 视图。
- 新增业务状态必须使用 `petbank_` 前缀，并登记 `docs_project/data-contracts/localstorage-keys.md` 和 registry。远征进度、当天日期、错题和奖励状态按 Profile 隔离；设备、账号和连接凭证不得进入 Profile 快照。

## 奖励与失败处理

- 完成事件使用稳定的 `eventId`、当前 `profileId`、`source=minecraft-vocab` 和明确 reward 类型。
- 结算调用 `GameRewardReceipts.claim()`，由统一积分 API `PetBankPoints.add()` 实际入账；不得直接修改 `petbank_points`。
- 任何持久化或奖励失败都保留可重试状态，并显示可定位的页面反馈；网络或媒体加载失败使用本地内容回退，不把未完成显示成完成。
- 页面离开时调用现有 `MinecraftVocabPage.stop()`，清理音频、计时器和事件引用，避免旧页面回调污染新 profile。

## 验证与交付

- 单元/契约：词卡字段与数量、清洗结果、重复奖励、日期和 Profile 隔离。
- 浏览器：现有 Minecraft session/browser contract、主站学习 smoke、桌面和 320/375/768/1440 宽度，无横向滚动、可操作焦点、页面切换清理。
- 发布：Pages 制品组装、静态路由入口、资源白名单和运行时路径；后端独立测试不因本功能改变。
- 文档：更新版本记录、运行/验证说明，并补充 `docs_project/runbooks/self-hosted/` 下 Hermes 可执行的部署、数据刷新、验收和回滚步骤。
