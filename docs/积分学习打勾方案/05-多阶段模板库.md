# 05-多阶段模板库

---

## 1. 为什么要做多个模板

> 说明：当前项目里，`模板 A / B / C` 的主要作用是控制字段密度和记录深度，**不是**切换不同的额外奖励规则。

同样是“每日学习单”，幼小衔接和小学标准版的负担感完全不同。

如果一开始就上：

- 每行时长
- 每行错题数
- 每日评分
- 家长评语
- 错题整理卡

对幼小衔接会太重，不容易坚持。

所以推荐直接做成 **三个阶段、三个模板**。

---

## 2. 模板 A：幼小衔接超轻量版

### 适用阶段

- 当前最推荐
- 幼小衔接暑假
- 重点是“形成每天都能完成的节奏”

### 页面结构

1. 日期
2. 今日 3 到 4 件事
3. 每项一个打勾框
4. 今天用了多久
5. 睡前一句话
6. 家长一句夸奖

### 推荐任务行

| 顺序 | 任务 | 默认值 |
|---|---|---|
| 1 | 晨读 | 自动生成 |
| 2 | 今日古诗 | 自动生成 |
| 3 | 识字 | 自动生成 |
| 4 | 睡前复盘 | 固定项 |

### 建议字段

| 字段 | 是否保留 |
|---|---|
| 完成勾选 | 保留 |
| 每行计划分钟 | 不保留 |
| 每行实际分钟 | 不保留 |
| 总时长 | 保留 |
| 每行错题数 | 不保留 |
| 今天卡住的 1 个点 | 保留 |
| 家长一句话 | 保留 |

### 当前项目里的积分口径

- lesson 正常完成继续沿用原有规则
- 学习单当前不额外发模板 A 奖励
- 模板 A 的价值在于“孩子更容易坚持”

### 这版最重要的特征

- 轻
- 快
- 每天能坚持

---

## 3. 模板 B：轻量标准版

### 适用阶段

- 幼小衔接后半段
- 家长希望开始记录时间
- 但还不想做完整错题本

### 页面结构

1. 日期与今日目标
2. 今日任务表
3. 总时长或 3 行内轻量时长
4. 今天卡住的点
5. 睡前复盘

### 推荐任务行

| 顺序 | 任务 | 默认值 |
|---|---|---|
| 1 | 晨读 | 自动生成 |
| 2 | 今日古诗 | 自动生成 |
| 3 | 识字 | 自动生成 |
| 4 | 学习网站 / 拓展 | 可选 |
| 5 | 睡前复盘 | 固定项 |

### 建议字段

| 字段 | 是否保留 |
|---|---|
| 完成勾选 | 保留 |
| 每行计划分钟 | 可选 |
| 每行实际分钟 | 只保留 2 到 3 行 |
| 总时长 | 保留 |
| 每行错题数 | 仍不建议全开 |
| 今天卡住的 1 到 2 个点 | 保留 |
| 家长一句话 | 可选 |

### 当前项目里的积分口径

- 模板 B 目前只增加字段，不增加独立学习单分值
- 仍然沿用 lesson 主奖励

### 这版最重要的特征

- 比模板 A 多一点记录能力
- 但仍然不追求“全字段”

---

## 4. 模板 C：错题加强版

### 适用阶段

- 小学一年级更稳定之后
- 家长明确需要错题追踪
- 或者孩子已经能接受更完整的学习单

### 页面结构

1. 日期、目标、状态
2. 任务表
3. 行级时长
4. 错题数
5. 错题整理块
6. 睡前复盘块
7. 家长反馈

### 建议字段

| 字段 | 是否保留 |
|---|---|
| 完成勾选 | 保留 |
| 每行计划分钟 | 保留 |
| 每行实际分钟 | 保留 |
| 每行错题数 | 保留 |
| 错题整理明细 | 保留 |
| 睡前复盘 | 保留 |
| 家长评语 | 保留 |

### 当前项目里的积分口径

- 模板 C 当前也只增加记录密度
- 错题整理和复盘目前不单独奖励
- 后续如果真要加，也应另行验证，不作为默认方案

### 这版最重要的特征

- 适合“真正要管理学习质量”的阶段
- 不适合幼小衔接第一版

---

## 5. 当前项目最推荐模板

如果只选一个模板先落地，推荐：

**模板 A：幼小衔接超轻量版**

原因：

1. 和当前 `读完打勾` 逻辑最好接
2. 家长每天维护成本最低
3. 最适合暑假场景
4. 最容易坚持 30 到 60 天

---

## 6. 三个模板如何升级

推荐升级路径：

1. 先上模板 A
2. 家长觉得稳定后，切到模板 B
3. 真正需要错题管理时，再开模板 C

这样做的好处是：

- 不会一上来把页面做太重
- 也不会把后续扩展堵死

---

## 7. 一句话建议

对幼小衔接来说，最重要的不是“记录得多完整”，而是：

**每天能顺顺当当地学完、打上勾、得到一点成就感。**
