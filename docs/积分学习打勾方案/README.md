# 积分学习打勾方案

> 目的：把参考“每日一页学习本”的优点，收口成当前项目里已经落地的轻量学习单方案。
>
> 最终口径：`学习页负责入口`，`积分页负责学习单打勾`，`设置页负责选择模板模式`。
>
> 更新时间：2026-07-05

---

## 1. 当前最终结论

当前项目没有把学习单塞回学习页，也没有单独再做一个新网站。

已经落地的方式是：

1. `学习` 页重点负责“快速找到学什么”
2. `积分` 页重点负责“今天学完了没有、勾到哪里了、睡前怎么收口”
3. `设置` 页负责切换学习单模板模式

这条路径比“把所有东西都堆进学习页”更轻，也更符合家长真实使用顺序。

---

## 2. 当前已落地范围

### 学习页

- 上方前置学习入口卡
- 中文资料包、英语资料包、学习网站入口包并列展示
- 今日推荐入口更轻量，优先解决“今天先点哪里”

### 积分页

- 展示 `今日学习单`
- 已支持 `模板 A / 模板 B / 模板 C`
- lesson 打勾结果会同步到学习单主任务
- 支持总时长、卡点、睡前复盘、拓展勾选等轻量记录

### 设置页

- 默认突出 `模板 A：幼小衔接超轻量版`
- `模板 B / 模板 C` 收在进阶模式里
- 切换后，积分页与打印页会一起跟随

---

## 3. 当前积分规则

学习单当前**不额外发“管理奖励”**。

主积分继续由这些行为结算：

- 中文 lesson 完成
- 中文晨读 + 识字同日组合奖励
- 英语 lesson 完成
- 英语连续学习奖励

学习单本轮重点负责：

- 汇总当天任务
- 记录总时长
- 标记今天卡住的点
- 做睡前简短复盘
- 给打印页提供同源数据

这样可以避免“同一项学习因为多点几个按钮又多得一次分”。

---

## 4. 为什么这样收口

参考图很适合启发结构，但不适合原样照搬给幼小衔接。

当前最终方案保留了参考图里最有价值的部分：

- 一天有一张总表
- 学完就能打勾
- 晚上能快速收尾
- 家长能一眼看懂今天进度

同时主动砍掉了第一阶段太重的部分：

- 不做满页字段
- 不做每行分钟必填
- 不做完整错题本必填
- 不做学习单额外发分

---

## 5. 当前最推荐的模式

对幼小衔接暑假来说，当前最推荐：

1. 默认使用 `模板 A`
2. 学习页只负责选入口，不在里面塞重表格
3. 积分页完成今天的学习单收口
4. 先坚持 15 到 20 分钟的短节奏，再考虑升级模板

## 7. 继续扩展时的边界

后续如果继续扩学习单，建议仍然守住这 4 条边界：

1. 学习页继续做“入口页”，不要重新变成重管理页。
2. 积分页继续承接“今日打勾 + 晚上收口”。
3. 模板升级优先加字段密度，不优先加新的奖励层。
4. 所有扩展都继续复用当前 `petbank_learning_*` 数据底座和多孩子隔离方式。
