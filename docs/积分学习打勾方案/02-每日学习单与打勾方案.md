# 02-每日学习单与打勾方案

---

## 1. 目标

把当前学习中心升级成：

- 打开学习页就知道今天该做什么
- 每做完一项都能打勾
- 能记录计划时间与实际时间
- 错题能单独整理
- 睡前 3 分钟做一个简单复盘

---

## 1.1 幼小衔接版本的约束

幼小衔接不适合把学习单做成“满页表格 + 满页字段”。

第一版必须控制住：

1. 每天核心任务不超过 4 到 5 行
2. 孩子自己能看懂
3. 家长 1 分钟内能填完
4. 睡前复盘只保留一句话到三句话

所以本文件里的“完整字段”更适合作为上限设计，
真正推荐落地时请优先看 [05-多阶段模板库](./05-多阶段模板库.md)。

---

## 1.2 2026-07-05 当前落地口径

这份文档前半段保留了早期探索过程，但当前项目已经收口为：

1. `学习页` 负责资料入口，不再承接重学习单
2. `积分页` 负责 `今日学习单` 打勾与收尾
3. `设置页` 负责切换 `模板 A / B / C`

因此下面的字段设计可以继续参考，但页面落位以当前实现为准。

---

## 2. 页面结构建议

当前最终采用的是下面这组分工：

### 当前已采用

- `学习页`：负责“今天学什么、从哪里开始”
- `积分页`：负责“今天完成了哪些、有没有收口”
- `设置页`：负责“学习单显示哪种模式”

### 为什么不是继续放学习页

- 学习页已经承担了资料入口和内容跳转
- 再塞学习单会让页面重新变重
- 家长真正需要打勾和收尾时，更自然会回到积分页

所以现在的学习单不是“学习中心里的重表格页”，而是“积分页里的今日收口层”。

---

## 3. 每日学习单字段

每日学习单建议固定为“一天一张卡 / 一天一页表”。

### 页头区

| 字段 | 含义 |
|---|---|
| 日期 | 当前学习日期 |
| 星期 | 自动显示 |
| 今日主题 | 例如“第 1 周第 5 日” |
| 孩子状态 | 可选：精神好 / 一般 / 累了 |
| 今日目标 | 例如“轻量完成晨读 + 识字 + 睡前复盘” |

### 任务表区

每行任务建议包含：

| 字段 | 说明 |
|---|---|
| 类型 | 晨读 / 古诗 / 识字 / 网站 / 错题整理 / 复盘 |
| 任务标题 | lesson 标题或自定义任务 |
| 计划分钟 | 系统建议值，可改 |
| 实际分钟 | 学完后填写 |
| 完成状态 | 勾选 |
| 错题数 | `0/1/2/...` |
| 积分状态 | 已发 / 待发 / 不发 |
| 操作 | 打开内容 / 继续学习 / 查看错题 |

### 收尾区

| 模块 | 内容 |
|---|---|
| 错题合计 | 当天错题总数 |
| 错题整理 | 是否已整理 |
| 睡前复盘 | 今天最好的一项、最难的一项、明天先做什么 |
| 家长一句话 | 例如“今天专注不错，识字有点慢” |

---

## 4. 当天自动生成哪些任务

建议系统每天自动生成 5 到 6 行，不需要家长从零填写。

### 核心任务

1. 晨读 `day-xx`
2. 古诗 `day-xx`
3. 识字 `day-xx`
4. 错题整理
5. 睡前复盘

### 可选任务

1. 学习网站入口
2. 打印讲义完成
3. 家长加的一条自由任务

---

## 5. 推荐的默认行模板

| 顺序 | 类型 | 默认来源 | 默认计划分钟 |
|---|---|---|---|
| 1 | 晨读 | `morning-reading` | 5 |
| 2 | 古诗 | `poems` 当日条目 | 3 |
| 3 | 识字 | `literacy-45days` | 8 |
| 4 | 学习网站 | `guided-sites` 或留空 | 10 |
| 5 | 错题整理 | 当天错题汇总 | 5 |
| 6 | 睡前复盘 | 日终固定项 | 3 |

这样更贴近“幼小衔接暑假”的节奏：短、轻、能完成。

---

## 6. 打勾方案

推荐采用 **三层打勾结构**。

### 第 1 层：lesson 页打勾

这是当前已经实现的能力：

- 一页读完点“读完打勾”
- 发该 lesson 的积分
- 写入学习进度

这层继续保留，不动。

### 第 2 层：每日学习单任务行打勾

这是新增层。

规则建议：

- 行任务如果直接对应某个 lesson，那么它的完成状态应尽量自动同步 lesson 打勾结果
- 如果是“错题整理”或“睡前复盘”，则由该行独立打勾
- 如果是外部学习网站，则学完回来手动打勾

### 第 3 层：今日完成状态

当核心行全部完成后，页面顶部出现：

- `今日学习单已完成`
- `今日成长分已结算`
- `明天继续加油`

这层不对应单节内容，而是对应“一整天是否收口”。

---

## 7. 错题整理方案

错题不建议只用一个数字带过，建议拆成两个层级。

### 轻量版本

每行只记：

- `错题数`
- `是否已整理`

适合先快速落地。

### 完整版本

再补一个“错题抽屉”或“错题小卡”：

- 来源任务
- 错字 / 错词 / 错句
- 是否再次读对
- 是否需要明天再看

推荐第一阶段先上轻量版本，第二阶段再补错题明细卡。

---

## 8. 睡前复盘方案

睡前复盘不建议做成长问卷，只需要 3 个简短问题：

1. 今天最顺的一项是什么？
2. 今天最容易错的是哪一项？
3. 明天准备先做什么？

这样可以控制在 2 到 3 分钟内完成。

---

## 9. 打印版建议

网页版和打印版建议共用同一套字段，只是表现形式不同。

网页版重点：

- 自动带出任务
- 可打勾
- 可计分
- 可联动 lesson

打印版重点：

- 一页一日
- 空格足够写时间和错题数
- 家长可以直接签字或写一句话

---

## 10. 数据模型建议

当前已经落地的核心学习单存储有两组：

| 键名 | 用途 |
|---|---|
| `petbank_learning_daily_sheet` | 某天的任务行快照 |
| `petbank_learning_sheet_mode` | 当前模板模式（A / B / C） |

如果后续继续加深，再考虑拆分：

- `petbank_learning_mistake_log`
- `petbank_learning_daily_review`
- 其他周报 / 错题卡附加键

推荐数据形态：

```json
{
  "2026-07-05": {
```

## 11. 与当前页面的接线关系

| 当前页面 | 新方案中的角色 |
|---|---|
| `今日学习` | 今日学习单入口与当天推荐 |
| `lesson 页` | 打开内容、完成单节学习 |
| `我的进度` | 看累计完成量与累计成长分 |
| `打印讲义` | 打印每日学习单 / 打印讲义 |

结论：

新的每日学习单仍然是 **当前项目里的组织层**，不是新起一个完全独立的网站。

只是当前已落地位置改成了：**学习页做入口，积分页做打勾与收尾。**
