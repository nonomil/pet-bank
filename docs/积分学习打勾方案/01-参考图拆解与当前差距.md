# 01-参考图拆解与当前差距

---

## 1. 参考图核心结构

参考页本质上不是“单项打卡页”，而是 **当天学习总控页**。

从字段上可以拆成 5 块：

### A. 页头信息

- 日期
- 星期
- 孩子状态 / 天气 / 精神情况

### B. 当日任务表

每一行代表一个任务，常见字段包括：

- 学科 / 分类
- 今日计划
- 完成情况
- 预估时长
- 实际时长
- 错题数量

### C. 日内作息与问题汇总

- 起床时间
- 睡觉时间
- 错题合计
- 备注补充

### D. 睡前自评

- 效率
- 正确率
- 书写 / 认真度

### E. 家长反馈

- 家长评语
- 家校沟通

---

## 2. 参考图背后的真实价值

这张纸之所以有用，不是因为它“好看”，而是它把一天的学习行为串成了一条链：

1. 先列计划
2. 再执行任务
3. 记录耗时
4. 标记错题
5. 当天整理
6. 睡前复盘

也就是说，它天然具备 **计划、执行、纠错、反馈** 四层能力。

---

## 2.1 参考图不应直接照搬

这张参考图更像“标准小学作业页”，对幼小衔接来说有两个问题：

1. 字段偏多
2. 家长和孩子每天填写成本偏高

因此本项目不建议直接照搬成：

- 每行都填计划时间
- 每行都填实际时间
- 每行都统计错题
- 每天都写家长评语

更适合幼小衔接的方式是：

- 第一阶段先做超轻量模板
- 第二阶段再补标准模板
- 第三阶段才补错题加强版

---

## 3. 当前项目已有能力

当前学习中心已经具备：

- `今日学习` 推荐入口
- `资料包 / 学习网站 / 打印讲义 / 我的进度`
- lesson 页“读完打勾”
- lesson 奖励与防重复发分
- 同日晨读 + 识字组合奖励
- A4 打印页
- 多孩子学习数据隔离

这说明底座已经不是空白，而是已经完成了“内容打开、完成记录、奖励回写、总进度统计”。

---

## 4. 当前项目与参考图的差距

当前最明显的差距不在“有没有打勾”，而在“有没有当天总表”。

### 当前有

- 一页一页学
- 每节学完点完成
- 能统计学了多少节
- 能累计积分

### 当前缺

- 当天学习单总览
- 预估时间与实际时间记录
- 错题数量与错题整理入口
- 睡前复盘卡
- 家长一眼就能看懂的当天执行情况
- “今日任务全部完成”的日级状态

---

## 5. 当前结构为什么还不够

现有学习中心更像“内容平台 + 进度页”，而参考图更像“学习管理页”。

两者并不冲突，但层级不同：

- 当前 lesson 页负责“学什么”
- 新的每日学习单负责“今天怎么学、学了多久、错在哪里、晚上怎么收口”

如果不补这一层，学习中心虽然能学、能计分，但还不够像一个真正的“每日陪学系统”。

---

## 6. 设计映射建议

参考图里的“学科”列，不建议在当前项目里原样照搬成“语文 / 数学 / 英语”。

更适合当前项目的映射是：

| 参考图字段 | 当前项目推荐映射 |
|---|---|
| 学科 | 学习模块 / 任务类别 |
| 今日计划 | 自动生成的 lesson 标题或自定义补充任务 |
| 完成情况 | 网页打勾状态 |
| 预估时长 | 系统建议分钟数 |
| 实际时长 | 手填或计时器回写 |
| 错题数量 | 该任务产生的错题数 |
| 睡前自评 | 学习单收尾卡 |
| 家长评语 | 家长一句话反馈 |

推荐模块类别：

- 晨读
- 古诗
- 识字
- 学习网站 / 拓展
- 错题整理
- 睡前复盘

---

## 7. 一句话差距结论

当前项目已经有 **“学习内容 + 单页打勾 + 发分 + 总进度”**，
下一步要补的是 **“按日期聚合的每日学习单管理层”**，
但这层应该按年龄和负担感分阶段推进，而不是一次做成完整版。
