# 首页优化设计提示词

> 用途：测试 `.codex/skills/gpt-image-ui-assets` 技能是否能基于当前首页生成可执行的 GPT/Agnes/Cowart 生图输入包。

## 当前首页观察

- 当前首页是 `成长伙伴 · 萌宠冒险岛`，首屏同时包含顶部导航、今日任务、今日概览、栏目轮播、6 个常用入口和宝箱仓库。
- 真实渲染在 1280x720 下无横向溢出，控制台无 warn/error。
- 主要优化目标不是换一张大背景，而是降低信息密度，把首屏压成一个清晰的今日主行动。
- 首页背景继续复用现有 `assets/home-bg.webp` / `assets/home-bg/*` 风格，不单独生成完全不同画风的大背景。

## Prompt 1：完整首页参考图

Create a polished desktop web app homepage redesign reference for a Chinese children's pet reward and learning system named "成长伙伴 · 萌宠冒险岛". Use a 1280x720 browser viewport. The target users are a 5-7 year old child and a parent, so the page must feel warm, clear, playful, and calm, not like a dense admin dashboard.

Keep the product concept: daily tasks, growth points, pet companion, learning center, playground games, exploration adventure, reward shop, and treasure chest. Simplify the first screen around one primary action: today's recommended task. The homepage should have:

1. A compact top navigation bar with logo, current child, growth points, and parent area.
2. A large central "today's main quest" panel with a pet companion standing beside it, one clear primary button, a small progress bar, and star reward feedback.
3. Three large child-friendly entrance cards only: 学习中心, 游乐场, 宠物伙伴. Secondary entries like 探索冒险, 兑换商店, 宝箱仓库 should be smaller chips or lower-priority cards.
4. A small left or bottom summary strip for today's tasks and points, not a long sidebar.
5. A visible pet companion area that feels like the child has a buddy, not just a status number.

Visual style: modern children's web app, soft illustrated adventure island atmosphere, warm green, sky blue, sunny yellow, coral, cream, and a little mint. Reuse the feeling of a gentle forest/adventure background, but keep the center clean and readable. Rounded cards, subtle shadows, soft paper or clay-like UI, large readable Chinese UI labels, strong visual hierarchy, one obvious primary action. Avoid clutter, avoid many small equal cards, avoid dark theme, avoid purple/blue gradient dominance, avoid beige-only palette, avoid heavy retro console shell.

Important layout requirement: The design should show the next section slightly below the fold, but the first viewport must clearly show the main quest, pet companion, and three main entrances. Do not create a marketing landing page. This is an actual usable app dashboard.

## Prompt 2：透明资产爆炸图

Create a transparent background PNG asset explosion sheet for the same homepage redesign of a Chinese children's pet reward and learning system. No readable text, no letters, no numbers, no Chinese characters, no pinyin. Every component must be isolated, complete, non-overlapping, with generous spacing and true alpha transparency. Do not draw a checkerboard background.

Include these separate reusable UI assets:

1. One large rounded "today main quest" blank panel with a cream empty center.
2. One cute pet companion mascot standing and waving, front view, friendly and simple.
3. One small pet speech bubble with a blank cream center.
4. Three large rounded entrance card frames with blank centers and subtle different accent colors.
5. One compact progress bar frame with blank fill area.
6. Three star reward badges in different states: empty star, lit star, sparkling star.
7. One treasure chest chip or small reward capsule.
8. One growth points coin badge without text.
9. One soft navigation capsule frame.
10. A few small decorative leaves, sparkles, and soft clouds that can sit around cards.

Style: modern children's web app UI, soft clay/paper illustration, warm green, sky blue, sunny yellow, coral, cream, mint, rounded shapes, subtle contained shadows, crisp edges, high resolution. Keep every asset text-free because all Chinese UI text will be overlaid later by HTML. Transparent background, true alpha channel, no checkerboard pattern, no grid lines, no cell borders, no white background.

## 给 Codex 的实现约束

- 参考图只用于确定首页视觉层级，不直接切整页背景。
- 爆炸图素材进入项目之前必须裁切、语义命名、写 manifest，并通过透明边缘检查。
- 首页真实 HTML 文案仍由代码输出，不放进 PNG。
- 优先减少首屏等权卡片，保留一个今日主行动。
