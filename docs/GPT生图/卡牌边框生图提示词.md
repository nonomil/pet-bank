# 卡牌边框 GPT 生图规格 + 数值合成方案

> 日期：2026-07-01 | 用途：供 GPT-IMAGE-2 / Agnes 生卡牌背景模板，脚本合成完整卡牌
> 对应方案：[2026-07-01-卡牌系统重构-方案.md](2026-07-01-卡牌系统重构-方案.md) §4 双生成机制

---

## 1. 总体流程

```
GPT 生 4 张稀有度背景模板 (frame-{rarity}.png, 含装饰文字+留白占位)
        ↓
PIL 合成脚本 (compose-card.py)
        ↓ 读 pets.json + assets/pets 立绘
合成 147 张完整卡牌 ({pet_id}.png = 背景 + 立绘 + 名称 + 系列 + 数值)
        ↓
图鉴引用合成卡牌图
```

**核心原则**（用户要求）：
- ✅ GPT 生图**可含装饰文字**（稀有度标签 LEGENDARY、卡牌标题花纹）
- ✅ GPT 生图**不含数值**（HP/ATK/DEF/SPD 留空白占位框）
- ✅ 数值由**脚本对照每宠物写入**（PIL draw）

## 2. 卡牌布局规格（480×640，3:4 竖卡）

```
┌─────────────── 480 ───────────────┐
│  [系列标签区]    [稀有度标签]       │ y: 0~60
│  ●HP(空)              ●SPD(空) ●   │ y: 80~160  ← 数值角标(4个空白圆框)
│                                      │
│           ┌─────────────┐           │
│           │             │           │
│           │  立绘区      │           │ y: 100~440 ← 中央留白(透明)
│           │  (留白透明)  │           │
│           │             │           │
│           └─────────────┘           │
│  ●DEF(空)              ●ATK(空) ●   │ y: 460~540 ← 数值角标
│  ┌─────────── 名称栏 ────────────┐  │ y: 550~620 ← 名称留白
└────────────────────────────────────┘
```

### 分区坐标（供 GPT 生图布局 + 脚本 draw 定位）

| 区域 | 位置 (x, y, w, h) | 说明 |
|------|-------------------|------|
| 稀有度标签 | 顶部中央 (190, 15, 100, 35) | GPT 生图含（如"LEGENDARY"） |
| 系列标签 | 顶部左 (15, 15, 120, 30) | 留白，脚本 draw 系列 |
| HP 角标 | 左上圆心 (55, 130) r=38 | 留空白圆框，脚本 draw ❤+数值 |
| SPD 角标 | 右上圆心 (425, 130) r=38 | 留空白圆框，脚本 draw ✦+数值 |
| DEF 角标 | 左下圆心 (55, 470) r=38 | 留空白圆框，脚本 draw 🛡+数值 |
| ATK 角标 | 右下圆心 (425, 470) r=38 | 留空白圆框，脚本 draw ⚔+数值 |
| 立绘区 | 中央 (80, 110, 320, 320) | **透明/留白**，脚本叠加立绘 PNG |
| 名称栏 | 底部 (40, 555, 400, 55) | 留白，脚本 draw 宠物名 |

## 3. GPT 生图提示词（4 档稀有度背景模板）

### 3.1 通用母版

```
{RARITY_COLOR} ornate game card frame template, vertical 3:4 portrait,
decorative border with intricate {TEXTURE} patterns,
text "{RARITY_LABEL}" at top center in ornate fantasy font,
four empty circular slots at four corners (top-left, top-right, bottom-left, bottom-right) for stats,
large empty transparent area in center for character portrait,
empty name plate bar at bottom,
no character, no creature, no numbers, no stats values,
high quality digital painting, card game asset style like Hearthstone,
transparent background in center area, PNG with alpha
```

### 3.2 四档配置

| 稀有度 | RARITY_COLOR | RARITY_LABEL | TEXTURE | 文件名 |
|--------|--------------|--------------|---------|--------|
| common 普通 | grey silver `#9D9D9D` | COMMON | 钢铁/石头 | `frame-common.png` |
| rare 稀有 | blue `#0070DD` | RARE | 冰晶/水流 | `frame-rare.png` |
| epic 史诗 | purple `#A335EE` | EPIC | 魔法符文/紫晶 | `frame-epic.png` |
| legendary 传说 | orange-gold `#FF8000` | LEGENDARY | 金色龙纹/火焰 | `frame-legendary.png` |

### 3.3 完整提示词示例（legendary）

```
Orange-gold (#FF8000) ornate game card frame template, vertical 3:4 portrait 480x640,
luxurious decorative border with golden dragon patterns and flame motifs,
text "LEGENDARY" at top center in ornate gold fantasy font,
four empty circular slots at four corners for stats (do NOT fill numbers),
large empty transparent area in center for character portrait,
empty name plate bar at bottom,
no character, no creature, no numbers, glowing golden particle effects,
high quality digital painting, Hearthstone legendary card style,
transparent center, PNG with alpha channel
```

### 3.4 关键约束（生图必须遵守）

- ✅ **中心立绘区透明/留白**（脚本叠加宠物立绘）
- ✅ **四角圆形槽留空**（脚本 draw 数值）
- ✅ **名称栏留白**（脚本 draw 名称）
- ✅ **稀有度标签可含文字**（LEGENDARY 等，装饰用）
- ❌ **不含任何数字**（HP/ATK/DEF/SPD 都不填）
- ❌ **不含宠物/角色**（立绘脚本叠加）

## 5. 数值合成脚本（compose-card.py）

> 脚本：`.tmp/compose-card.py` | 读 pets.json + assets/pets 立绘 + frame-*.png → 合成 composed/{pet_id}.png

### 脚本逻辑
1. 读 `data/pets.json`（base_hp/atk/def/spd + rarity + series + name）
2. 读 `assets/pets/poses/{id}_idle.webp`（立绘，已去白底透明）或 emoji 兜底
3. 读 `assets/cards/frame-{rarity}.png`（背景模板）
4. PIL 合成：
   - 背景 frame
   - 立绘贴中央立绘区（缩放到 320×320，居中）
   - draw 名称（名称栏，白色粗体+描边）
   - draw 系列标签（顶部左）
   - draw 4 数值角标（❤HP/✦SPD/🛡DEF/⚔ATK，彩色圆+白数字）
5. 输出 `assets/cards/composed/{pet_id}.png`

### 数值角标 draw 规范
- 圆背景色：HP `#C0392B`红 / ATK `#D35400`橙 / DEF `#2C7BE5`蓝 / SPD `#27AE60`绿
- 数字：白色粗体 + 黑色描边（text-shadow 模拟）
- 字体：思源黑体/SimHei（中文），36px

## 6. 双机制互补

| 机制 | 用途 | 产物 |
|------|------|------|
| **GPT-IMAGE-2**（本规格） | 生 4 档背景模板（含装饰文字） | frame-{rarity}.png × 4 |
| **Agnes** | 生主题变体（节日/系列限定背景） | frame-{rarity}-{theme}.png |
| **PIL 脚本** | 合成完整卡牌（背景+立绘+数值） | composed/{pet_id}.png × 147 |

## 7. 接入图鉴（P2 后）

合成卡牌图产出后，`js/card-collection.js` _renderGrid 改用 `assets/cards/composed/{pet_id}.png` 作为卡牌图（替代当前 CSS 边框+HTML 数值）。
- 有合成图 → 用图
- 无合成图 → 降级当前 CSS 边框 + HTML 数值（P0 方案，不阻塞）
