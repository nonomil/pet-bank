## 关键实现

### 1. 主站接入方式

- 入口页：`index.html`
- 路由 / 页面激活：`js/app.js`
- runtime 归类：`js/runtime-loader.js`
- 样式：`css/playground.css`

原则是“合玩法，不合实现”：

- 主站负责导航、壳层、成长积分、战绩与复盘
- 原型继续负责怪物接近、输入判定、射箭、爆炸和局内节奏

### 2. 积分桥接

子游戏通过 `postMessage` 发两类事件：

- `reward`：每次命中时发送
- `result`：每局结束时发送

主站在 `js/app.js` 中监听：

- 校验来源 `petbank-typing-defense`
- 校验 `iframe.contentWindow`
- 用 `sessionId + seq` 去重
- 命中时调用现有 `addGrowthPoints()`

这样孩子打中的一瞬间，主站积分就会立刻增加，而不是等整局结束。

### 3. 战绩沉淀

新增本地 key：

- `petbank_typing_defense_progress`

记录字段：

- `sessions`
- `wins`
- `totalStars`
- `totalPoints`
- `bestScore`
- `bestCombo`
- `lastMode`
- `modeCounts`

这让 `打字防线` 不再只是一个独立小游戏，而是进入主项目自己的成长记录体系。

### 4. 主项目可见层

已接入：

- 游乐场作战看板
- 每周复盘
- 最近收获
- 跨玩法里程碑

当前摘要面板会显示：

- 最近主练模式
- 各模式局数分布
