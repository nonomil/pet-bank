# 2026-07-05 · 遛弯页户外版宠物小屋收口

## 背景

这一轮的核心目标不是继续给 `遛弯` 页叠新功能，而是把它从“信息很多、像 dashboard 的页面”收口成真正贴近 `宠物小屋` 的户外分身：

- 左边保留一个大场景舞台
- 右边只保留必要的状态、好友与路线信息
- 页面视觉层级更简单，孩子点进去更像“带宠物出去玩”，不是“进入另一个后台”

## 本轮完成

### 1. 页面骨架改成宠物小屋式布局

- `js/app.js`
  - `renderWalkPage()` 改成 `walk-home-shell`
  - 左侧只保留 `walk-scene-stage`
  - 右侧改成三张侧栏卡：
    - `状态互动`
    - `好友遛弯`
    - `路线与记录`
- 原来独立的 `walk-page-hero` 说明层被移除，避免页面像“上面一块、下面一堆”

### 2. 舞台区做减法

- `js/walk.js`
  - 户外主舞台保留：
    - 场景背景
    - 宠物立绘
    - 顶部状态 chip
    - 气泡
    - 路线切换
  - 去掉舞台里的四个大操作按钮
  - 去掉大块说明面板，改成更轻的 `scene tag`

这样左边更接近 `宠物小屋` 的“主场景”，不会再同时出现太多并列层。

### 3. 右侧第一张卡补成真正的状态卡

- `js/app.js`
  - 新增 `walk-home-vitals`
  - 补齐：
    - `HP`
    - `饱食`
    - `快乐`
    - `亲密`
    - `EXP`
  - 同时在低体力、低饱食时给轻提示
- `css/walk.css`
  - 新增对应条形进度样式
  - 配色尽量向 `宠物小屋` 的生命条语义靠拢

这样右侧第一张卡不再只是几个数字，而是真正承担“看状态 + 做互动”的核心职责。

### 4. 好友与路线区继续压紧

- `js/app.js`
  - 好友邀请列表、好友列表增加 `is-compact` 紧凑模式
- `js/walk.js`
  - 路线卡根容器增加 `is-compact`
- `css/walk.css`
  - 邀请列表、好友列表、日志列表改成限定高度的轻量滚动区
  - 路线缩略图高度、路线详情卡高度整体缩短
  - 社交卡按钮和好友卡间距同步减小

目标不是减少功能，而是让右侧更像侧栏工具区，不再把整列撑得过长。

`遛弯` 页已经从“社交 + 说明 + 路线”并排堆叠的页面，收口成了更接近 `宠物小屋` 的户外版：

- 左边看场景和宠物
- 右边看状态、好友和路线
- 页面层级明显更清楚
