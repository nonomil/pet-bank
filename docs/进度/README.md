## 1. 编号规则

- 建议编号格式：`PROGRESS-YYYY-MM-DD-序号`
- 文件命名格式：`YYYY-MM-DD-主题.md`
- 一份进度文档只记录**一个连续阶段**，避免混写过多主题

## 3. 使用建议

- 想看“当前项目总体做到哪了”：
  先读 [../项目现状总览.md](../项目现状总览.md)
- 想看“某轮到底改了什么”：
  从本目录按日期读进度文档
- 想看“方案为何改了”：
  去看 [../方案/README.md](../方案/README.md) 里的 `CR-*` 记录
- 想看“版本发布和文档变更怎么对应”：
  去看 [../changes/README.md](../changes/README.md)
- 想看“下一步准备怎么做”：
  去看 [../plans/README.md](../plans/README.md)
