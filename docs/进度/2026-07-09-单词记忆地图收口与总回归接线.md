# 单词记忆地图收口与总回归接线

> 日期：2026-07-09
> 状态：已完成并待纳入本轮提交

## 这轮做了什么

- 把 `prj/单词记忆射击场原型` 的地图模式再收了一轮，重点补稳 3 个真实回归点：
  - 点第二个中文球后，键盘投掷重新武装
  - 当前地图块内的中文球与英文目标稳定留在视口里，不再被镜头裁到屏幕外
  - 连续两次正确命中后，连击 UI 能在测试观察窗内稳定到达 `2`
- 地图镜头从“持续跟随主角”调整为“地图模式按当前块固定取景”，减少跨块元素和可见性抖动。
- 地图渲染收口到“只渲染当前地图块的中文球和英文目标”，让点击、语音回放与任务条反馈更稳定。
- 锁定目标时的中文球飞行速度提高，命中结算更利落，不再拖慢连击结果展示。
- 新增的 3 个 Playwright 测试正式接入总回归入口：
  - `prj/word_memory_map_bomb_throw.test.mjs`
  - `prj/word_memory_map_rewards_ui.test.mjs`
  - `prj/word_memory_map_voice_playback.test.mjs`
- 学习机原型这边补齐了“汉字语音复用拼音原型共享语音包”的验证闭环：
  - `prj/学习机玩法原型/verify.mjs` 新增共享语音资产、回退策略与专用播报路径断言
  - `prj/学习机玩法原型/README.md` 明确记录本地语音优先、Web Speech 回退
  - `prj/learning_arcade_hanzi_voice_reuse.test.mjs` 与 `prj/shared_prototype_voice_workflow.test.mjs` 一并接入总回归

## 意义

这轮不是再做一个新原型，而是把已经存在的 `word memory` 子玩法推进成：

- 可玩
- 可重复测试
- 能接入总回归
- 共享语音资产的复用关系也有了独立契约，不容易被后续改动悄悄带坏

它离“整个游戏做完整并模拟测试”的目标更近了一步，因为游乐场里这条地图记忆玩法已经开始具备正式收口条件。
