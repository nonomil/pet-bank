## 3. 当前文档体系结构

本轮之后，`docs` 已形成 5 层结构：

| 层级 | 目录 / 文档 | 作用 |
|---|---|---|
| 总览层 | `项目现状总览.md` / `README.md` | 快速搞清项目现在是什么 |
| 规范层 | `规格/`、`设计/`、`路线/` | 查需求、架构、模块、缺口、路线 |
| 方案层 | `方案/` | 查历史方案、change request |
| 计划层 | `plans/` | 查下一阶段设计稿和实施计划 |
| 追溯层 | `进度/`、`changes/`、`CHANGELOG.md` | 查做了什么、为什么改、版本交付什么 |
