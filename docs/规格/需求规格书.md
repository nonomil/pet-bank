# 成长伙伴 · 萌宠冒险岛 需求规格书（PRD）

> **一句话定位**：面向课堂场景的"积分银行 + 宠物养成 + 场景探索"三合一纯前端静态应用，部署于 GitHub Pages，以打卡积分驱动宠物成长，用探索与对战延长留存。
>
> **来源映射**：本规格书由以下原始材料浓缩整合而成——①《2026-0628-宠物积分银行-原始需求.md》（飞书对话，产品方向与功能清单源头）；②《pet-bank-全系统方案v2.md》（设计基准，6 界面 SPA 与参数权威来源）；③《📋 pet-bank 仓库现状评估.md》（落地现状核对，早期为 11 Tab 已铺开但接线待收口）；④《UI和场景参考.md》（首页水彩绘本大地图视觉方向）。详细映射见 [§8 来源映射表](#8-来源映射表)。

## 1. 产品定位与目标

### 1.1 产品定位

| 维度 | 说明 |
|---|---|
| 产品形态 | 纯前端静态 HTML + 模块化 JS，单页应用（SPA）多 Tab 切换 |
| 部署方式 | GitHub Pages 静态托管，零后端、零数据库 |
| 目标用户 | 课堂师生（希沃一体机/浏览器），单人养成为主 |
| 核心循环 | 打卡任务 → 获得积分 → 积分换道具/解锁 → 喂养宠物 → 探索对战 → 更多奖励 |
| 参考基准 | classpet-pro1.0（机制：群组互动/状态量化）、banchong.cn（视觉与养成反馈循环）、精灵乐园（探索/救援/血条交互灵感） |
| 落地现状 | 当前已形成 **13 个页面入口 / 12 个功能模块**（含宠物小屋），主链路可演示；商店家具联动与战斗深化已落地，当前主要缺口转为探索叙事收口与系统性回归验证 |

### 1.2 产品目标

| 层级 | 目标 | 衡量 |
|---|---|---|
| L1 功能广度 | 12 大模块全部可演示，无运行时阻断 | 逐页冒烟可打开、核心链路无报错 |
| L2 闭环稳定 | 任务→积分→宠物→探索→消费 主链路串通 | 单次会话能跑完一次完整循环 |
| L3 体验收口 | 首页水彩绘本大地图、养成反馈、进化动画 | 视觉接近参考图，反馈即时 |

## 2. 功能模块清单

> 共 12 大已落地/规划模块。每模块给出：职责 / 功能点 / 关键参数 / 验收要点。模块间接线缺口见各模块"验收要点"及 [§7 验收标准](#7-验收标准)。

| 职责 | 课堂打卡与成长分的统一账本，是全系统积分的"水源" |
| 功能点 | 6 大成长维度（学习/运动/自控/探索/实践/守护）任务勾选；积分持久化（localStorage）；连续打卡奖励；每日历史记录 |
| 关键参数 | 早起打卡 +20、学习任务 +30、运动 +20、数学PK +25、连签 +50（参考值，以仓库 `DIMENSIONS` 实现为准） |

### 2.2 宠物养成（核心）

| 项 | 内容 |
|---|---|
| 职责 | 全系统的情感锚点与成长反馈中枢 |
| 功能点 | 5 维状态（HP/饱食/快乐/亲密度/经验）；4 互动按钮（喂食/玩耍/洗澡/治疗）；5 阶段进化 + 进化动画；对话气泡；自然衰减 |

### 2.3 场景探索

| 项 | 内容 |
|---|---|
| 职责 | 消耗宠物状态换取道具/经验/战斗机会的中层循环 |
| 功能点 | 场景选择地图（路线地图：探索 tab + 首页双入口复用 `renderSceneGridMap`，卡片散布 + SVG 连线 + 5 章着色）；30 秒沉浸式探索动画；加权随机事件（道具/战斗/稀有/剧情）；积分解锁新场景；**galgame 呈现层**（立绘 + 对话框 + galgame-word 样式） |
| 关键参数 | 单次探索 30 秒；事件权重示例 道具40/战斗30/稀有20/剧情10 |
| 故事单一源 | `data/stories/*.json` 为唯一正式故事源；`exploration-detail.js` 的 `_loadStories` 读 `events`+`ending_text`，无硬编码兜底；加载失败明确提示 |
| 数学场景化 | 12 场景 `math` 事件为固定场景题（森林蘑菇加减 / 瀑布露珠剩余 / 雪山冰柱乘 / 洞穴水晶除 / 太空等差规律 / 星空花园斐波那契等）；`genMathQuestion` 优先 `event.question`，统一 galgame-word 样式 |
| 教育目标 | 5 章顶层 `chapter_skill`（ch1 number_sense / ch2 add_sub_apply / ch3 life_apply / ch4 mul_div_mech / ch5 pattern_logic），目标表见 `data/stories/README.md` |
| 结束叙事 | `ending_text` 并入场景 JSON 顶层（不再有独立 `SCENE_ENDING`） |
| 遭遇去模板 | 12 场景章内专属对手（如 castle 披风守门影 / volcano 火星石怪）+ 36 怪物立绘（Agnes 生图 + 去背景 + emoji fallback）+ 6 场景 choice 即时环境反馈 |
| 验收要点 | 探索→遭遇→结算闭环无报错、掉落 item id 全部命中 `items.json`、锁定场景灰色占位、改 `data/stories/*.json` 运行时直接生效 |

## 3. 角色体系

### 3.1 实际存量与方案对照

> 实际存量统一口径 **147 种**（banchong 91 + classpet 40 + minecraft 8 + original 8），与 `data/pets.json` flat 实际条目数学自洽（91+40+8+8=147）。任务原 **139 种** 口径含历史误差（PVZ8+MC8+banchong91+original8=115≠139 不自洽），本规格书以数据实际 **147 种** 为准。

|---|---|---|---|
| PVZ 植物大战僵尸 | 8 | v2 核心 | 豌豆射手/向日葵/坚果墙/樱桃炸弹/寒冰射手 等 |

### 3.2 与 v2 方案的差异

| 维度 | v2 方案 | 仓库现状 | 差异说明 |
|---|---|---|---|
| 角色数 | 16（8 PVZ + 8 MC） | 147 | 仓库已大幅超 v2，但来源混杂，需统一质量 |
| 进化阶段 | 5 阶段 × 16 = 80 张 | 部分 5 阶段 | v2 的 5 阶段完整图谱尚未对全部角色铺满 |
| 解锁方式 | 积分商店解锁 | 部分接入 | 需确认 147 种的解锁门槛是否分级 |

### 3.3 5 阶段进化机制（权威）

| 阶段 | PVZ 命名 | MC 命名 | 解锁亲密度阈值 |
|---|---|---|---|
| Stage 1 | 🌱 种子 | 🥚 生成蛋 | 0 |
| Stage 2 | 🌿 嫩芽 | 🐣 幼体 | 50 |
| Stage 3 | 🌸 成长 | 🦊 成体 | 150 |
| Stage 4 | 🌳 开花 | 👑 精英 | 300 |

## 5. 关键参数总表

> **权威说明**：以下参数统一以 v2 方案 `js/*.js` 实现值为准，标注「权威」以消除 v2 与仓库偏差。任何模块引用这些值时必须以本表为唯一来源。

### 5.1 宠物 5 维状态

| 维度 | 区间 | 说明 |
|---|---|---|
| HP | 0–100 | 归零触发倒下/救援 |
| 饱食（Hunger） | 0–100 | 归零后 HP 持续下降 |
| 快乐（Happiness） | 0–100 | 影响互动反馈 |
| 亲密度（Intimacy） | 0+ | 决定进化阶段，无上限 |
| 经验（EXP） | 0–expToNext | 满值升级 |

### 5.2 进化阈值（权威）

`STAGE_THRESHOLDS = [0, 50, 150, 300, 600]`（PVZ/MC 通用）

### 5.3 互动与衰减（权威）

| 动作 | 效果 |
|---|---|
| 喂食 | 饱食 +25、积分 -10（或消耗道具）、经验 +10 |
| 自然衰减（每小时） | 饱食 -2、快乐 -1；饱食为 0 时 HP -5 |

### 5.4 探索与战斗（权威）

| 参数 | 值 |
|---|---|
| 单次探索时长 | 30 秒（30000ms） |
| 敌人 HP 区间 | 25–60 |
