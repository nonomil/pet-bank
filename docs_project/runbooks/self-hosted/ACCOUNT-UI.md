# 家长账号入口与菜单层级

## 当前结构

家长端采用“管理大厅 -> 一级业务菜单 -> 设置二级菜单”的结构：

- 顶部一级菜单：管理大厅、设置管理、成长作品、工具箱、进入孩子端。
- 顶部左侧显示当前家庭上下文；右侧“账号与家庭”直接进入家庭账号设置。
- 设置页不再重复显示一套家长区一级菜单，避免入口重复。
- 设置二级菜单按三组组织：账号与家庭、孩子体验、系统。

## 设置级联

```text
设置管理
├─ 账号与家庭
│  ├─ 设置首页
│  ├─ 家庭账号
│  └─ 孩子档案
├─ 孩子体验
│  ├─ 学习与题目
│  └─ 规则模板
└─ 系统
   └─ 高级操作
```

“家庭账号”是家长登录、家庭成员、邀请码和跨设备 SQLite 同步入口；“孩子档案”只管理本机 Profile 与孩子映射，不能把孩子档案误认为登录账号。进入孩子端后，头像菜单只负责切换孩子，不显示家长成员或账号删除操作。

## 参考取舍

班宠乐园采用“选择班级 -> 管理大厅 -> 班级管理二级菜单”；DayDayUp 采用“家庭大厅 -> 顶部业务菜单 -> 设置页”。本项目借鉴两者的上下文先行、管理大厅和设置分层，但保留家庭产品自身的孩子档案、成长作品、学习设置和离线优先模型，不引入班级、小组或教师权限。
