# 家长账号入口与菜单层级

## 当前结构

家长端采用“管理大厅 -> 一级业务菜单 -> 设置二级菜单”的结构：

- 顶部一级菜单：管理大厅、设置管理、成长作品、工具箱、进入孩子端。
- 顶部左侧显示当前家庭上下文；右侧“账号与家庭”直接进入“家庭与孩子”。
- 设置页不再重复显示一套家长区一级菜单，避免入口重复。
- 设置常用区只保留“家庭与孩子”，低频配置收进“更多设置”。

## 设置级联

```text
设置管理
├─ 家庭与孩子
└─ 更多设置
│  ├─ 学习与题目
│  └─ 规则模板
   └─ 高级操作
```

“家庭与孩子”承载最小流程：家长登录或注册 -> 创建或加入家庭 -> 添加孩子。孩子只能挂靠当前家庭，不能从孩子端绕过家长账号创建。邀请另一位家长、切换或加入其他家庭、删除账号默认折叠；进入孩子端后，头像菜单只负责切换孩子。

`/settings/account` 和 `/parent/settings/account` 仍保留静态入口兼容，但统一显示“家庭与孩子”。

## 参考取舍

班宠乐园采用“选择班级 -> 管理大厅 -> 班级管理二级菜单”；DayDayUp 采用“家庭大厅 -> 顶部业务菜单 -> 设置页”。本项目借鉴两者的上下文先行、管理大厅和设置分层，但保留家庭产品自身的孩子档案、成长作品、学习设置和离线优先模型，不引入班级、小组或教师权限。
